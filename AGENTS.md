# AGENTS.md — 参加するAIワーカー共通オンボーディング（無料・ファイル連携）

このリポジトリは、オーナー「てつ」のために複数のAIが協働して
**高岡・氷見・富山を世界（海外読者）に発信する**プロジェクトです。
ChatGPT / Gemini / Claude など、**repoを読み書きできるAIなら誰でも**この作法で参加できます。
連携の実体は git＋ファイルだけなので**追加のAPI課金はありません**（②方式）。

## 0. North Star
**海外の人に記事を読んでもらうこと**（収益・本数より上位）。判断はこれで決める。

## 1. 参加したら最初に読む（順番厳守）
1. `context/STATE.md` … 最新状態・絶対制約(A1〜A7)・決定ログ（**毎回最初に読む**）
2. `docs/role-division.md` / `docs/org.md` … 役割分担（code=主／ワーカー=副）
3. `ops/README.md` … 指示・報告のやり方

## 2. 役割（重要）
- **code（Claude Code）＝主担当・頭脳**：企画・執筆・整理・**台帳/STATEの正本管理**・判断・指示発行。
- **あなた（ワーカー）＝副・実行担当**：自分の得意分野の**実行だけ**を、依頼に応じて行う。
  - 例：cowork＝ブラウザ/note公開/SNS。翻訳ワーカー＝多言語訳。リサーチワーカー＝調査。
- **やってはいけない**：`CMO/_index.md`（台帳）と `context/STATE.md` の編集は**code専任**（あなたは触らない）。企画・方針の独断変更もしない。

## 3. 連携プロトコル（ops/）
- **あなたへの指示** = `ops/inbox/` の `*_*_<you>.yaml`（`python3 ops/process_inbox.py list --to <you>`）
- **あなたの報告** = `ops/outbox/` にファイルで（または `process_inbox.py post --from <you> --to code ...`）
- 着手＝`take`、完了＝`done`（`ops/processed/` へ移動）。
- 1ファイル=1タスク、結果サマリ必須、削除しない。

## 4. 自己紹介（初参加時に1回）
`ops/outbox/` に「自分は誰で、何ができるか」を1ファイル置く：
```
from: <you>   to: code   type: report
title: ワーカー登録（能力申告）
body: 私は <ChatGPT/Gemini等>。できること: [翻訳/調査/画像生成/ブラウザ公開/並列 …]。
      できないこと: [...]。repoアクセス: [push可/コピペのみ]。
```
→ code が能力に合わせてタスクを割り振ります。

## 5. ブランチ・同期
- canonical = `main`。**作業前に `git pull`、作業後すぐ commit & push**（差分を溜めない）。
- 自分のブランチを使う場合は `worker/<name>-*`。最終的に main へ収束（codeが主導）。

## 6. 品質の絶対ルール（A制約・全員必須）
- PII禁止（具体的地区名/町名/住所NG・市まで）／誇張・断定NG／著作権キャラはAI画像生成NG（実物写真のみ）。
- 記事は英語ファースト＋事実検証ノート（基準は `CMO/outputs/2026-06-07_英語travelwriting執筆テンプレ.md`）。

---
要するに：**読む(STATE)→自己紹介(能力)→inboxの自分宛をやる→outboxに報告→pull/push**。これだけで無料で協働できます。
