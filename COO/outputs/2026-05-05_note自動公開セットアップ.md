# note 自動公開セットアップ手順（GitHub Actions 経由）

**警告**：note の利用規約上、ボット自動投稿は**グレー**です。
mac 経由の Plan B より GitHub Actions 経由のほうが BAN 確率が高い（データセンターIP・検出されやすい）。
**承知の上で進めること**。BAN すると note アカウント・既存記事・フォロワーが全消失します。

## 提供される選択肢

| 案 | 自動化レベル | BAN リスク | mac | 運用継続性 |
|---|------------|----------|-----|----------|
| **B. GitHub Actions 自動公開** | 完全自動 | **高** | 不要 | 数週間で Cookie 失効 |
| **C. iPhone 1タップ公開** | 半自動（公開ボタンは人間） | **ゼロ** | 不要 | 高 |

両方実装済。Bを使うかCを使うかは選択可能。**Cを優先利用しBは緊急時のみ**を推奨。

---

## C. iPhone 1タップ公開（推奨・15分セットアップ）

### 設定手順

1. iPhone のショートカットアプリを開く
2. `+` で新規ショートカット作成
3. `~/agent-team/projects/2026-05-05_AI自動収益化引き継ぎ/deploy/iphone_shortcuts.md` の「⑤ note 1タップ公開」のテーブル通りにアクションを追加：
   - 日付フォーマット → URL 構築 → URL の内容取得 → クリップボードコピー → note アプリ起動
4. 「ホーム画面に追加」でアイコン設置

### 毎日の操作

1. iPhone のホーム画面で「note 公開」をタップ（10秒）
2. note アプリで本文欄を長押し → ペースト
3. 公開ボタンを押す
4. 完了

**所要 10秒・BAN リスクゼロ**

## B. GitHub Actions 自動公開（30分セットアップ）

### Step 1: mac で storageState を取得（一度だけ）

```bash
cd ~/agent-team
python3 tools/extract_note_state.py
# ブラウザが開く → note にログイン → ターミナルで Enter
# → note_state.b64 が生成される
```

### Step 2: GitHub Secret に登録

```bash
# クリップボードに base64 をコピー
cat note_state.b64 | pbcopy

# その後ファイルは削除（ローカルに残さない）
rm note_state.json note_state.b64
```

GitHub の Settings → Secrets and variables → Actions：

- New repository secret
  - Name: `NOTE_STORAGE_STATE`
  - Secret: クリップボードからペースト

### Step 3: 有効化

オプション A：常時自動公開（毎朝07:00 JST に投稿）
- Settings → Variables → New variable
  - Name: `ENABLE_NOTE_AUTO_PUBLISH`
  - Value: `true`

オプション B：手動実行のみ
- Actions タブ → "Daily Drafts & Auto-Publish" → Run workflow
- "note を実投稿する" にチェック → Run

### Step 4: テスト（必須）

最初は workflow_dispatch で1度実行 → Discord 通知で結果確認 →
note アプリで実際に公開されているか確認 → 問題なければ自動有効化。

### Step 5: 失効時の対処

数日〜数週間で Cookie が失効する。Discord に「🚨 note 自動公開ジョブ失敗（NOTE_STORAGE_STATE 失効の可能性）」が来たら：

```bash
cd ~/agent-team
python3 tools/extract_note_state.py
cat note_state.b64 | pbcopy
# GitHub Secret NOTE_STORAGE_STATE を更新（古い値を上書き）
rm note_state.json note_state.b64
```

これで再稼働。

## 推奨運用：C を主軸にして B は補助

| 状況 | アクション |
|------|----------|
| 平日（毎朝の習慣化） | **C で1タップ公開**（10秒） |
| 旅行・出張で iPhone も触れない日 | **B が自動で代行**（事前に enable_note=true） |
| storageState が失効した | mac でstate再取得（5分） |
| BAN 兆候を検知（ban_detector） | 即 ENABLE_NOTE_AUTO_PUBLISH=false |

## まとめ

- **C が主・B が補助**で組み合わせるとリスクと利便性のバランスが取れる
- BAN 時の損失（noteアカウント全停止）と自動化の利便のトレードオフを意識する
- 段階的有効化：最初の1週間は workflow_dispatch のみ → 問題なければ自動有効化
