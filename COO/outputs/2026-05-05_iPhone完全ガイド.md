## 2. セットアップ（合計30分・1回だけ）

### Step A: Discord 通知（5分）

1. Discord アプリで自分用サーバーを作成
2. テキストチャンネル → 設定 → 連携サービス → Webhook → 新規作成
3. Webhook URL をコピー
4. iPhone の Safari で github.com にアクセス → リポジトリ → Settings → Secrets → New
   - Name: `DISCORD_WEBHOOK_URL`
   - Secret: コピーしたURL

### Step B: GitHub アプリでドラフト閲覧（1分）

1. GitHub アプリをインストール → ログイン
2. リポジトリ `tyutyutakokaina81-netizen/agent-team` をブックマーク
3. `daily/` ディレクトリをタップして場所を覚える

### Step C: ショートカット⑤「note 1タップ公開」（10分）

iPhone のショートカットアプリで新規作成。以下のアクションを順に追加：

| # | アクション | 設定値 |
|---|----------|------|
| 1 | 日付 | 形式: `yyyy-MM-dd` ・ TZ: `Asia/Tokyo` |
| 2 | テキスト | `https://raw.githubusercontent.com/tyutyutakokaina81-netizen/agent-team/claude/ai-monetization-handover-epkiB/daily/[1]/note_draft.md` |
| 3 | URLの内容を取得 | URL: ステップ2 ・ 方法: `GET` |
| 4 | クリップボードにコピー | 入力: ステップ3 |
| 5 | URLを開く | `https://note.com/notes/new` |
| 6 | 通知を表示 | `本文をコピーしました。長押しでペースト→公開` |

**ホーム画面に追加**：3点メニュー → 「ホーム画面に追加」 → アイコン「📝note公開」

### Step D: ショートカット③「公開を記録」（5分・任意）

`COO/outputs/2026-05-05_iPhone1分運用ガイド.md` の手順 ③ を参照。
記録しなくても KPI 以外への影響は無いので後回しで可。

### Step E: ショートカット④「DRY_RUN 切替（緊急停止）」（5分・任意）

## 7. トラブル対応

| 症状 | 原因と対処 |
|------|----------|
| ショートカット⑤で本文が取れない | URL のブランチ名が間違っている。Step C の表 #2 を確認 |
| Private リポなら Step C #3 で 404 | ヘッダー追加で `Authorization: Bearer <PAT>` |
| Discord 通知が来ない | `DISCORD_WEBHOOK_URL` Secret 名が完全一致しているか |
| Discord に🚨が来た | Actions タブで失敗したジョブ確認・必要なら state 再取得 |
| note アプリが開かない | URL を `note://` にすると直接アプリで開く |

## 11. リンク形式に関する注意

iPhone でリポジトリのファイルを開くには **2種類のURL** がありますが、用途が違います：

| URL 形式 | 用途 | 認証 |
|---------|------|------|
| `https://github.com/<owner>/<repo>/blob/<branch>/<path>` | **人間が読む** | Private なら必須 |
| `https://raw.githubusercontent.com/<owner>/<repo>/<branch>/<path>` | **スクリプトが取得する** | Private なら PAT 必須 |

このガイドの **Step C #2 の URL は raw 形式** です。これは Shortcuts アプリが内部的に取得するためのもので、人間が直接タップして開く用途ではありません。

人間が開く時は必ず `github.com/.../blob/...` の形式で（GitHub アプリ経由が最も確実）。
