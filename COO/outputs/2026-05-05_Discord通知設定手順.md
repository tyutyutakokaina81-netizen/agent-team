# Discord 通知 5分セットアップ

毎朝 07:00 (JST) のドラフト準備完了通知と、ワークフロー失敗時のアラートを iPhone に届けるための設定。
Discord アプリ（無料）が iPhone にあれば即受信可能。

## 手順（5分）

### 1. Discord サーバーを作る（30秒）

- iPhone で Discord アプリを起動
- 左下の `+` → 「自分用」 → 適当な名前（例：ai-auto）
- 既存サーバーがあれば1チャンネル新設でも可

### 2. Webhook URL を作成（1分）

- 作ったサーバーの「テキストチャンネル」（例：`#general`）の歯車アイコン
- 「連携サービス」 → 「ウェブフック」 → 「新しいウェブフック」
- 名前は `ai-auto`
- 「ウェブフックURLをコピー」（`https://discord.com/api/webhooks/.../...` の形式）

### 3. GitHub Secret に登録（2分）

- iPhone のブラウザ or PC で github.com を開く
- リポジトリ `tyutyutakokaina81-netizen/agent-team` の Settings
- Secrets and variables → Actions → New repository secret
- Name: `DISCORD_WEBHOOK_URL`
- Secret: 先ほどコピーした URL
- Save

### 4. テスト実行（1分）

- リポジトリの Actions タブ → "Daily Drafts & Auto-Publish" → Run workflow
- 数分後に Discord に「📝 daily/... にドラフト準備完了」が届けば成功

### 5. 確認（30秒）

毎朝 07:05 頃に通知が届くようになる。
ワークフロー失敗時は 🚨 失敗通知が届く。

## 通知が来ない場合のチェック

| 症状 | 対処 |
|------|------|
| Run workflow したのに通知なし | Webhook URL の Secret 名が `DISCORD_WEBHOOK_URL` 完全一致か |
| Webhook 直叩きでも来ない | Discord サーバー側の Webhook 設定で「ウェブフックURL」が再生成されていないか |
| 失敗通知だけ来ない | ワークフロー履歴で当該ジョブが実際に失敗しているか（Skip 含む） |
