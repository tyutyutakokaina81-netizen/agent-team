> COO管理。オーナー指示「このスキームをしっかり管理して」。
> これは**スキーム全体の唯一の管理台帳**。毎セッション最初に `context/STATE.md` → 本書 → `ops/` の順で見て前進させる。
> 最終更新：2026-06-22。

## 0. スキーム全体図（分業）

```
オーナー（最終GO・写真選定・事実の最終判断）
   ↑ 決裁を上げる            ↓ 「基本はgo」既定で進める
code（私）＝制作・規則自動チェック・[要確認]集約・台帳/STATE正本・cowork発注・結果反映
   ↓ ops/inbox で発注        ↑ ops/outbox で受領
cowork＝外部実行（投稿・ネット事実検証・写真生成）※判断はしない
```

## 3. 受信SOP（cowork報告が outbox に来たら code がやること）

1. `python3 ops/process_inbox.py list --to code` で受領を確認。
2. 報告のIDに応じて：
   - **004（検証）**→ research/15-17・19/29/30 を本文確定（✅外す/⚠️直す/❓「諸説あり」）→ G2再チェック → commit。
   - **005（写真）**→ `thumbnails/freepaper/` を確認 → [写真X] に対応づけ → commit。
   - **003（投稿）**→ URL・反応を本ダッシュボード「実績」へ記録 → CFOリーチ様式/CAO分析へ。
   - **002（公開）**→ 重複/既公開を台帳照合。
3. タスクを `processed/` へ move（`process_inbox.py done`）。
4. **STATE.md と本ダッシュボードを更新 → commit & push**（記録しないと消える＝A7）。
