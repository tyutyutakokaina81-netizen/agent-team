## 担当業務
- プロジェクト横断の進行管理（ロードマップ・タスク・依存関係・期限）
- 役職間の交通整理（誰が何を、いつまでに）と重複の防止
- `ops/` キュー（code↔cowork）の運用監督と滞留の解消
- オーナーの意思決定待ち事項（ブロッカー）の集約と提示
- 実行状況のレビュー（計画 vs 実績）と次アクションの確定

## 担当しないこと
- 企画・執筆・分析そのもの（各役職の領分）。COOは「回す」役。
- `CMO/_index.md`・`context/STATE.md` の正本管理は引き続き code 専任。
