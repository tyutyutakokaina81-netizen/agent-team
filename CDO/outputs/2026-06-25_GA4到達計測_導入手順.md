# 2026-06-25 到達計測（GA4）導入手順 — サイトに唯一欠けていた「計器」

## 要点
- main は多言語SEO・hreflang・OG画像・じゃらん/楽天アフィリ・週次IndexNow・Search Console まで完備。
- **唯一無かったのが「アクセス解析(GA4)」**＝「世界に広まったか／どの国から読まれたか」を数字で見る計器。
- 本変更で `apps/ai-agency-hp/analytics.js` を作り、**全91ページに計測タグを配線**＋pages.yml で配信。GA_ID を1個入れれば全ページ即計測（空なら無害）。

## owner がやること（一度きり・約3分・無料）
1. https://analytics.google.com にログイン →「管理」→「プロパティを作成」。
2. 「データストリーム」→「ウェブ」→ URL：`https://tyutyutakokaina81-netizen.github.io/agent-team/`。
3. 表示される「測定ID」（`G-` で始まる）をコピー → code に渡す（or `analytics.js` の `GA_ID=""` に貼る）→ push。
4. 次のデプロイで全ページの計測開始。

## 計測が始まると見えること
- 訪問者数・**どの国から**読まれたか（North Star＝海外読者の可視化）
- 流入元（Google検索 / Reddit / X / じゃらん経由 等）、人気記事、滞在時間。
- これで初めて「世界に広まったか」を 0/1 でなく **人数・国・流入元** で判断できる。

## 併用：Google Search Console（既に所有権確認済み）
- 検索での表示回数・クリック・検索語・国が無料で見える。owner はログインして「検索パフォーマンス」を見るだけ。
- GA4（サイト内）＋ Search Console（検索での見つかり方）の2点で到達を立体把握。

## 正直な制約
- code は A1 で GA4/Search Console 画面を見られない。数字確認は owner が画面 → code に貼る運用。
