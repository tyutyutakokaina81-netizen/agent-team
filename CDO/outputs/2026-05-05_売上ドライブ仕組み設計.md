# 売上ドライブ仕組み設計（2026-05-05）

**目的**：藤森さんを「売上が立つ行動」に物理的に押し出す仕組みを設計する。
**前提（残酷な事実）**：
- Vol.1は4月から販売中だが**売上未確認**（CFO台帳が空のため）
- Vol.2 .xlsxは完成済だがnote未公開（4日間放置）
- 5/3に1件応募して以降、応募ゼロ
- 既存リマインダー（CHEAT_SHEET / START_HERE / daily_routine）は**何週間も無視されてきた**実績

**Claude側の限界**：藤森さんの行動を強制できない。できるのは「行動への摩擦を下げる」と「行動しないコストを上げる」の2点のみ。

---

## 1. なぜ既存リマインダーが効かなかったか（仮説）

| 仮説 | 根拠 |
|------|------|
| **A. 摩擦が大きい** | Vol.2公開には Drive アップ→Sheets変換→note登録→価格設定の4工程30分。実際の障壁は「30分」というより「複数工程」 |
| **B. 期限が無い** | 「いつでもいい」=「やらなくていい」と脳が処理する |
| **C. 公開コミットがない** | 誰も見ていないのでサボっても無痛 |
| **D. 失敗の可視化がない** | 何日連続でサボっているか自分でも分からない |
| **E. 報酬が遠い** | 売上¥680は「30分後にもらえる¥680」ではなく「公開後N日後に発生するかも」の不確実性 |

→ これら全部に同時に効く仕組みが必要。

---

## 2. 推奨仕組み：3層構造

### Layer 1（摩擦削減）：1コマンド出荷スクリプト（既存資産活用）

すでに作成済の `scripts/publish.py` を「朝のscheduled task」と連結する：

```
8:00 AM scheduled task が起動
  ↓
今日の最大ROI 1行動を判定
  ↓
osascript Mac通知発火（既存）
  ↓
通知本文に「実行コマンド」をクリップボードコピー
  ↓
藤森さんはターミナルでCmd+V→Enterだけ
  ↓
publish.py が必要URLを連続でブラウザに開く
  ↓
clipboard に本文がコピーされた状態
  ↓
あとは Cmd+V → 公開ボタン
```

**摩擦時間**：30分 → **5分以下**。
**実装コスト**：scheduled task のpromptを修正するだけ。新規スクリプト不要。

### Layer 2（期限/コミット）：日次「公開ツイート」生成

scheduled task に以下を追加：

```
今日の1行動を決定後、X（Twitter）公開コミットメント文を生成：
「今日◯◯します（21時までに完了）」
↓
クリップボード待機 → 藤森さんはX起動して投稿するだけ
↓
21時に再度scheduled taskが起動して「達成？未達？」を確認
↓
未達なら「次の打ち手」を提示（その日のうちに対処）
```

**効果**：公開してしまえば、未達は公的な失敗になる。サンクコスト/一貫性原理で行動が出やすくなる（Cialdini）。

### Layer 3（可視化/ストリーク）：dashboard.md にストリーク列追加

`logs/ship_streak.csv` を新設し、毎日の出荷状態を記録：

```
date, action, status, revenue
2026-05-06, vol2_note_publish, completed, 0
2026-05-07, sns_announce, completed, 680
2026-05-08, , skipped, 0   ← 失敗
```

dashboard.md の「収入進捗」直下に追記：

```
## 出荷ストリーク
- 連続出荷日数：N日（最高記録 M日）
- 過去30日中の出荷日：X / 30日
- 直近の失敗：YYYY-MM-DD
```

Update_dashboard.py を最小拡張（10行追加）するだけ。**新規ツール作成なし**。

## 3. 仕組みの限界（誠実な評価）

| 介入 | 期待効果 | 実証された行動科学 | リスク |
|-----|---------|---------------------|-------|
| Layer 1（摩擦削減） | **大** | Fogg Behavior Model: B=MAT (Motivation × Ability × Trigger) のうち Ability を上げる | スクリプトを「使うこと自体」に新たな摩擦 |
| Layer 2（公開コミット） | **中〜大** | Cialdini Commitment & Consistency / 公開コミットの行動変容率は私的コミットの1.5〜3倍 | フォロワーが見ない / 藤森さんがX運用しない場合は無効 |
| Layer 3（ストリーク） | **中** | Don't Break the Chain (Seinfeld method) / 連続日数の可視化はモチベを高める | 数日途切れて諦める可能性 |

**全部やっても効かない場合に必要なもの（藤森さん意思決定領域）**：
- 金銭的コミット装置（stickK等で¥1万預ける／未達なら没収）
- 外部アカウンタビリティ・パートナー（友人・コーチ）
- 環境設計（朝の最初の30分はスマホ・他作業禁止＋出荷専用時間）

これらは**システムでは実装不可**。藤森さんが選ぶか・選ばないか。

## 4. 実装判断（rule v1 厳守）

### 即実装するか？ → **しない**

理由：
- 既存のscheduled task `daily-revenue-action` が稼働開始したばかり
- 2週間の実証データなしに新Layerを積むと、効果分析が不可能になる
- rule 4「戦略膨張禁止」に抵触

### 段階的判断フロー

```
2週間後（5/19）の monthly_review で評価：
  ├─ 通知→実行 変換率 ≥ 30% → 仕組みは効いている → Layer追加不要
  ├─ 変換率 10〜30% → Layer 1（摩擦削減）のみ追加
  ├─ 変換率 < 10% → Layer 1+2（公開コミット）
  └─ 変換率 0% → 仕組みでは解決不可 → 外部アカウンタビリティ要
```

## 5. 今すぐできる「仕組み外」の1手（ROI最大）

仕組みより先に、**Vol.1の現在の売上を確認する**ことが最大ROI：
- note管理画面で4月以降のVol.1売上枚数を確認（5分）
- もし5枚以上売れていたら → Vol.2/3/4を出すモチベが跳ね上がる
- もし0枚なら → 販売ページの問題が判明 → リライトで解決

これは仕組みでも自動化でもなく、**1回5分の確認行動**。
売上データが入ればCFO台帳の運用も意味を持ち始める。
