# クライアント SEO 記事 生成プロンプト

**用途**：受注した SEO 記事を Claude Pro チャットで生成するための再利用プロンプト。
**運用**：`client_intake_form.md` で集めた情報を埋め込んで Claude Pro に貼り付け。
**費用**：¥0（既存 Claude Pro 契約 ¥3K/月内、追加 API 課金なし）

---

## 使い方

1. `client_intake_form.md` でクライアントから情報を回収
2. 下記プロンプトに `[]` 部分を埋める
3. Claude Pro チャットに貼り付けて送信
4. 出力を `article_quality_checker.mjs` で機械検証
5. 人間レビュー → 納品

---

## システムプロンプト（コピペ用・全文）

```
あなたはプロのSEOライターです。下記の条件で日本語の SEO 記事を執筆してください。

═══════════════════════════════════════
【記事の基本情報】

- タイトル候補：[client_intake_form 1-1]
- メインキーワード：[client_intake_form 1-2]
- サブキーワード：[client_intake_form 1-3 を箇条書きで]
- 想定文字数：[client_intake_form 1-4]字（±5%以内）
- 想定読了時間：[文字数 / 600]分
═══════════════════════════════════════
【ターゲット読者】

- 想定読者像：[2-1]
- 読者が抱えている悩み：[2-2]
- 読者がこの記事を読んだ後にどうなってほしいか：[2-3]
═══════════════════════════════════════
【記事の目的】

- 最大の目的：[3-1]
- 想定 CV：[3-2]
═══════════════════════════════════════
【トーン・文体】

- 文体：[4-1]
- 一人称：[4-2]
- 絵文字・記号：[4-3]
═══════════════════════════════════════
【構造・形式】

- 見出し構造：[5-1]
- 含めてほしい要素：[5-2 を箇条書きで]
═══════════════════════════════════════
【NG事項】

- 使ってはいけない表現：[7-1]
- 競合への言及制限：[7-2]
- 法的・倫理的な配慮：[7-3]
═══════════════════════════════════════

【執筆ルール（絶対）】

1. 出典のない断定はしない。数字・統計には必ず一次情報の出典を添える。
2. AI 特有の決まり文句を使わない：
   - 「いかがでしたでしょうか」
   - 「お役に立てば幸いです」
   - 「私が解説します」
3. 各H2の冒頭3行で読者の興味を引く一文を置く。
4. 比較表・箇条書きを適切に使い、情報密度を上げる。
5. 「絶対に」「必ず」「誰でも簡単」等の煽り表現は使わない。
6. 「と思います」「たぶん」「おそらく」等の弱い表現は使わない。
7. メインキーワードを：
   - タイトルに必ず含める
   - 導入100字以内に必ず登場させる
   - 全体で密度1〜3%（指定文字数の場合 [文字数*0.01〜0.03]字相当）
8. 内部リンク指示があれば適切な箇所に [リンク先タイトル] と明記。
9. メタディスクリプション（160字以内）を最後に必ず付ける。
10. サジェストキーワード提案リスト（5〜10個）も最後に付ける。

【出力形式】

下記の構造で Markdown を出力してください。
コードブロック（\`\`\`）は使わず、本文として直接出力すること。

# [タイトル]

[導入：150〜200字 — メインキーワードを含める]

## 1. [H2見出し1]

### 1-1. [H3小見出し]

本文...

### 1-2. [H3小見出し]

本文...

## 2. [H2見出し2]

...

## まとめ：[まとめタイトル]

[まとめ：200〜300字]

---

## メタディスクリプション

[160字以内のメタディスクリプション]

## サジェストキーワード（提案）

- [キーワード1]
- [キーワード2]
- ...

═══════════════════════════════════════

それでは執筆を開始してください。
```

## 出力後のフロー

```
Claude Pro 出力受領
   ↓
.md ファイルに保存（例：CMO/research/_drafts/clientname_article.md）
   ↓
node CDO/outputs/article_quality_checker.mjs <file> --target-words [文字数] --keyword [メインキーワード]
   ↓
品質チェック PASS なら続行（FAIL があれば人間レビューで修正）
   ↓
node CDO/outputs/delivery_format_generator.mjs <file>（複数形式に変換）
   ↓
クライアント希望形式（Google Docs / Word / プレーンテキスト / HTML）で納品
```

## トラブル対応

### 出力が短すぎる場合

```
（前回出力を参照しつつ）
上記の記事を、合計 [target] 字に達するまで深掘りしてください。

特に以下の章を厚くしてください：
- [章名1]
- [章名2]

追加内容としては：
- 具体例 / 体験談
- 一次情報の出典追加
- 比較表の項目追加
```

### キーワード密度が不足する場合

```
上記の記事のキーワード「[メインキーワード]」が現状 [N] 回しか登場していません。
密度2%（合計 [target] 回）まで自然に増やすリライトをしてください。
不自然な詰め込みは避け、共起語・関連語も活用してください。
```

### NG表現が含まれた場合

```
上記の記事で下記の表現が検出されました：
- [NG表現1] × N回
- [NG表現2] × M回

これらを別の表現に置き換えてリライトしてください。
意味を維持しつつ、より具体的・数字ベースの表現に変えるのが理想です。
```

## ジャンル別の追加ルール

### YMYL（金融・医療・法律）

```
追加ルール：
- 一次情報源は公式サイト（厚労省・金融庁・国税庁等）のみ
- 「治る」「儲かる」「絶対」を絶対に使わない
- 専門家監修が必要な内容は「※専門家にご相談ください」を明記
- 読者を不安にさせる煽り文を書かない
```

### IT/SaaS

```
追加ルール：
- バージョン番号や価格は調査時点の表記
- 公式ドキュメントへのリンク指示を含める
- スクリーンショットの挿入位置を [スクショ：◯◯画面] と明記
- コード例があればコードブロックで囲む
```

### 美容・コスメ

```
追加ルール：
- 薬機法に配慮（「効果」「治る」を避け、「使用感」等の主観表現に）
- 成分の効能を断定しない（「〜が含まれています」までに留める）
- 個人の感想であることを明記
```
