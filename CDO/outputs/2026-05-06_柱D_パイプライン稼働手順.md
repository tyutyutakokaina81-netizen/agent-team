## ① 起動前チェック（5 分）

### 環境

```bash
node --version           # v20+ 推奨
python3 --version        # 3.10+ 推奨
which playwright         # 柱 D の実装が Playwright に依存する場合のみ必要
```

### 認証情報

`PIPELINE_TOKEN` を必ず設定する（未設定だと iPhone 操作パネルが認証なしで公開される）：

```bash
export PIPELINE_TOKEN="$(openssl rand -hex 16)"
echo "TOKEN を控える: $PIPELINE_TOKEN"
```

### セッション

`.sessions/` 配下にクラウドワークス・ランサーズのログイン Cookie が存在することを確認：

```bash
ls -la "projects/2026-04-08_月30万自動化/D_エクセル入力スクレイピング/.sessions/" 2>/dev/null
```

無ければ初回のみ `python3 00_session_setup.py` を手動で（**ブラウザでログイン操作が必要**）。

## ② 起動

```bash
PIPELINE_TOKEN=xxxx \
PIPELINE_PORT=3001 \
node pipeline_server.mjs
```

成功時の表示：
```
🤖 パイプラインサーバー起動
   http://localhost:3001
   トークン認証: 有効
```

## ③ 操作（iPhone or PC ブラウザ）

1. `http://[サーバー IP]:3001/?token=xxxx` を開く
2. ステータスバーが `待機中` であることを確認
3. 「**🔍 案件を検索・評価**」をタップ → 確認ダイアログ → 開始
4. ログ欄が緑のドットで流れ始める
5. 完了後「**📋 結果を確認**」で GO/CAUTION/NO-GO 一覧
6. 採用案件のみオーナーが手動で応募（**自動応募はしない**）

---

## ④ 1 日の運用ルール

| 項目 | 上限 | 理由 |
|------|------|------|
| 検索実行 | 1 日 3 回まで | スクレイピング過剰防止 |
| 自動応募 | 0 件（オーナー手動） | ToS 違反・誤応募リスク回避 |
| 同時実行 | 1 ジョブ | サーバー側で 409 弾き |

---

## ⑤ トラブル時

| 症状 | 対処 |
|------|------|
| `exit 1` で止まる | `state.log` を確認 → ログイン切れなら ① セッション再生成 |
| 検索結果ゼロ | キーワード設定（`01_search.py` 内）を見直す |
| `EADDRINUSE :3001` | `lsof -i :3001` で残プロセス kill |
| iPhone から接続不可 | サーバーが `0.0.0.0` で listen しているか確認、ファイアウォール |
