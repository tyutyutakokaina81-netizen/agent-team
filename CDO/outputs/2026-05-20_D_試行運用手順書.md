## 2. 日次手順（所要30分）

### Step 1：セッション確認（5分）
```bash
cd projects/2026-04-08_月30万自動化/D_エクセル入力スクレイピング/pipeline
ls ../.sessions/
```

- `crowdworks_session.json` または `lancers_session.json` が存在することを確認
- どちらもなければ `python 00_session_setup.py` を実行してログイン

### Step 2：検索→評価→応募文生成（15分）
```bash
python run_pipeline.py search
```

- 案件検索→スコアリング→応募文ブラウザ表示まで自動進行
- エラーが出たら **その場で止めて記録**（後述のログ参照）

### Step 3：応募ボタンを手動クリック（5分）
- 自動で開いたブラウザで応募文を確認
- 問題なければ **応募ボタンを手動でクリック**（規約遵守のため必須）
- 案件詳細を outputs/ に保存

### Step 4：ログ追記（5分）
- `D_エクセル入力スクレイピング/outputs/試行ログ_2026-05.md` に追記
- 結果 / エラー有無 / 所要時間を1行で記録

## 4. エラー対応プロトコル

### Case A：ログイン切れ
```bash
python 00_session_setup.py
```
→ 手動ログイン → セッション再保存

### Case B：セレクタが見つからない（DOM 変更）
- スクリプト側の `find_element` のセレクタ調査
- `headless=False` でデバッグスクリーンショット確認
- 修正後 commit（commit message: `fix: [01_search/02_evaluate/etc] セレクタ更新`）

### Case C：応募文の品質が低い
- `03_apply.py` のテンプレートを A のテンプレ集（`CSO/outputs/...応募文テンプレ.md`）と比較
- 修正案を CDO の research に記録 → 翌日反映

### Case D：受注したのに執行できない案件
- 即停止 → CSO にエスカレーション
- 規約・契約条件をオーナーと CFO で確認
