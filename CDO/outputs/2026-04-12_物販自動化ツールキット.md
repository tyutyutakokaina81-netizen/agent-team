# 物販自動化ツールキット — ジャケット販売業務の半自動化プロンプト集

**作成日**: 2026-04-12
**担当**: CDO
**用途**: Claude に物販業務の定型作業を代行させる
**使い方**: 各プロンプトをコピーして Claude（Pro or Code）に貼り付け、[ ] の部分を埋めて実行

---

## 自動化対象と期待効果

| 業務 | 人手 | 自動化後 | 削減時間 |
|---|---|---|---|
| 出品文作成 | 15 分/着 | 2 分/着 | 85% |
| 価格相場リサーチ | 10 分/着 | 3 分/着 | 70% |
| 購入者メッセージ対応 | 5 分/件 | 30 秒/件 | 90% |
| 月次レポート作成 | 60 分/月 | 5 分/月 | 92% |

**→ 10 着/月 運営時：月 約 3〜4 時間の時間節約**

## プロンプト 1：出品文ジェネレーター（最重要）

### 使い方
以下のテンプレの [ ] を埋めて Claude に貼る。出品タイトル・説明文・価格案を一括生成。

### プロンプト本体

```
あなたはメルカリの女性ジャケット販売の専門家です。以下の情報から、売れやすい出品文を作成してください。

【商品情報】
- ブランド：[例 GU / ユニクロ / ZARA / 無印良品 / BEAMS / ノーブランド]
- カテゴリ：[例 ノーカラージャケット / テーラード / デニム / ブレザー / コート]
- サイズ：[例 S / M / L / 38 / 40]
- 色：[例 ベージュ / オフホワイト / 黒 / ネイビー]
- 素材：[例 ポリエステル / コットン / ウール混]
- 季節感：[例 春秋 / 冬 / オールシーズン]
- 状態：[例 美品 / 使用感軽微 / タグ付き新品]
- 実寸：着丈[]cm / 肩幅[]cm / 身幅[]cm / 袖丈[]cm
- 特徴・こだわり：[例 裏地あり / 2WAY / ポケット4つ / 定番人気]
- 汚れ・傷：[例 なし / 襟に小さなシミ]
- 自宅環境：[例 ペットなし・非喫煙者]

【求める出力】
1. タイトル案（40字以内、3パターン）
2. 説明文（メルカリ風、400〜600字、絵文字なし、正直な表記）
3. 推奨価格（¥1,500〜¥10,000 のレンジで根拠付き）
4. 下限価格（値下げ交渉の底値）
5. 推奨カテゴリ階層
6. 推奨発送方法（ネコポス/宅急便コンパクト/宅急便）

【出力ルール】
- 絵文字は使わない
- 誇大表現禁止（「激安」「超美品」等）
- 実寸は必ず「平置き・素人採寸」と明記
- 嘘はつかない（汚れがあれば必ず記述）
- 販売価格は同ブランドの相場を考慮
```

### 使用例

```
【商品情報】
- ブランド：ユニクロ
- カテゴリ：ノーカラージャケット
- サイズ：M
- 色：ベージュ
- 素材：ポリエステル100%
- 季節感：春秋
- 状態：美品（2回着用）
- 実寸：着丈58cm / 肩幅38cm / 身幅45cm / 袖丈57cm
- 特徴：軽量・シワになりにくい・洗濯可
- 汚れ・傷：なし
- 自宅環境：ペットなし・非喫煙者
```

→ Claude に貼り付けると、そのまま出品できる完成文が返ってきます。

## プロンプト 2：価格相場リサーチャー

### 使い方
メルカリで同じ商品を検索する前の事前判断に。Claude の知識で大まかな相場観を掴む。

### プロンプト本体

```
あなたはメルカリの中古品相場に詳しい専門家です。
以下の商品の相場を推定し、出品戦略を提案してください。

【商品】
- ブランド：[ブランド名]
- カテゴリ：[アイテム名]
- サイズ：[サイズ]
- 色：[色]
- 状態：[状態ランク]
- 定価（推定）：[定価または不明]

【回答してほしいこと】
1. メルカリでの予想売却価格帯（最安〜最高）
2. 売れやすい時期・季節（今出すべきか、待つべきか）
3. 同ブランド同カテゴリで人気の色・サイズ
4. 売れないリスク要因（もしあれば）
5. 出品時の推奨初期価格（成約率重視 / 利益重視 の2パターン）
6. 競合が多い場合の差別化ポイント

【前提】
- 回答は 2026 年春時点のメルカリ市場を想定
- 確度の低い推測は「推測」と明記する
- 数字は「〜くらい」「〜前後」で柔軟に
```

## プロンプト 3：購入者メッセージ自動分類＋返信案

### 使い方
コメントや取引メッセージが来たら、そのまま Claude に貼って適切な返信案をもらう。

### プロンプト本体

```
あなたはメルカリのベテラン出品者です。
以下の購入者メッセージを分類し、最適な返信案を作成してください。

【購入者メッセージ】
"[メッセージ本文をここに貼る]"

【商品コンテキスト】
- 商品：[商品名]
- 出品価格：[価格]
- 自分の下限価格：[下限]
- 現在のステータス：[出品中 / 購入後 / 発送済 / 受取待ち]

【出力してほしいこと】
1. メッセージの分類（値下げ交渉 / 質問 / 購入意思 / トラブル / その他）
2. 緊急度（高 / 中 / 低）
3. 推奨アクション（受諾 / 交渉 / 情報提供 / 断る / 事務局連絡）
4. 返信文案（丁寧な日本語、150〜300字、絵文字なし）
5. 注意点（トラブルの芽があれば警告）

【返信ルール】
- 敬語を使う
- 謝罪が必要な場面では素直に
- トラブルの芽があれば毅然と
- 個人情報は絶対に聞かない
- 取引外の連絡先（LINE等）への誘導は拒否
```

## プロンプト 4：月次収支レポート生成

### 使い方
月末に `CFO/outputs/2026-04-12_物販収支管理シート.md` の取引記録を貼って、月次サマリを自動作成。

### プロンプト本体

```
あなたは物販事業の経理担当です。
以下の月次取引データから、月次レポートを作成してください。

【月次取引データ】
[物販収支管理シート.md の取引記録テーブルをここに貼る]

【出力形式】
# 2026-XX月 物販レポート

## 1. サマリー
- 出品数：X 着
- 成約数：X 着
- 成約率：X%
- 売上総額：¥X
- 純利益：¥X
- 固定費：¥3,000
- 月次最終収支：¥X

## 2. ベストセラー（上位 3 着）
1. [商品名] — 純利益 ¥X
2. ...

## 3. 売れなかった要因分析
- 価格面の課題
- 写真・説明文の課題
- カテゴリ選定の課題

## 4. 来月の改善提案
- 仕入れ方針
- 価格戦略
- 出品タイミング

## 5. 固定費回収状況
- 月次固定費 ¥3,000 に対して、物販純利益 ¥X
- 回収率：X%

## 6. トラブル記録
- [あれば箇条書き]

## 7. 次月のアクション（3 つ）
1. ...
2. ...
3. ...
```

## プロンプト 5：仕入れ判定（Week 2 以降）

### 使い方
古着屋・リサイクルショップで商品を手に取った瞬間に、Claude に判断を仰ぐ。スマホで使える。

### プロンプト本体

```
あなたはメルカリせどりの熟練者です。
以下の仕入候補の「買うべきか」を判定してください。

【候補商品】
- ブランド：[ブランド]
- カテゴリ：[アイテム]
- サイズ：[サイズ]
- 色：[色]
- 状態：[見た感じの状態]
- 仕入値：¥[金額]
- 店舗名：[店名]

【判定項目】
1. メルカリ想定売却価格（最低〜最高）
2. 想定純利益（手数料・送料込み）
3. 仕入値に対する利益率
4. 売れるまでの想定日数
5. リスク要因（サイズ不人気・色不人気・季節外れ等）
6. 総合判定：**買う / 買わない / 保留**

【判定基準】
- 利益率 30% 以下 → 買わない
- 売却想定 14 日以上 → 慎重に
- 状態に不安 → 買わない
- ブランド力あり＋定番色＋人気サイズ → 買う
```

## 自動化の運用ルール

### 1 日の流れ

```
朝（5 分）
├─ メルカリアプリでいいね・コメント確認
├─ コメントあれば プロンプト3 で返信案生成
└─ 通知設定を確認

昼（10 分）
├─ 新規出品 1〜2 着
└─ プロンプト1 で出品文生成

夜（10 分）
├─ 値下げ要求の対応（プロンプト3）
├─ 売れた商品の梱包・発送準備
└─ CFO 収支管理シートに記録
```

### 1 週間の流れ

```
月〜金：日次運用（上記）
土：プロンプト2 で売れない商品の再評価
日：仕入れ（Week 2 以降、プロンプト5 使用）
月末：プロンプト4 で月次レポート生成
```

---

## Claude へのログイン不要化（オプション）

Claude Pro を使っている場合、Claude アプリに以下をブックマーク：

1. プロンプト 1（出品文生成）のテンプレを定型文として登録
2. プロンプト 3（メッセージ返信）のテンプレを定型文として登録
3. 1 タップで呼び出せる状態にしておく

**→ 1 着出品あたりの作業時間が 15 分 → 3 分に短縮**

---

## 注意事項

- Claude は実物を見られないので、写真判定はできない
- 相場は大まかな目安、最終判断は人間
- 「絶対に売れる」などの保証はない
- ブランド・素材の誤情報は Claude でも発生し得るので、タグ確認必須

---

## 次のステップ

1. 今日、プロンプト 1 を使って **1 着目の出品文を生成**
2. メルカリで出品
3. 取引が発生したらプロンプト 3 で返信
4. 月末にプロンプト 4 で月次レポート

これで物販運営の「頭脳労働」の 80% が Claude に移管されます。オーナーは **写真撮影・梱包・発送** という物理作業に集中できます。
