## 2. OAuthブロッカーの選択肢

### 選択肢A：Google OAuth セッション保存（推奨・本命）

```
Playwright の storageState を活用：
1. 初回はオーナーが手動でログイン（Playwright の headed モード）
2. login_state.json に Cookie/LocalStorage を永続化
3. 以降の自動実行は storageState=login_state.json でロード
4. セッション切れ（30〜90日）時のみ再ログイン
```

**実装工数**：2時間（既存 pipeline コードに storageState の保存・ロード処理を追加）。
**リスク**：Google 側のセキュリティ強化で Bot 検出される可能性 → 手動ログインを定期化することで回避。

### 選択肢B：CrowdWorks API 公式利用

CrowdWorks はパートナー向け API を提供している可能性あり（要確認）。
- メリット：認証が安定・規約準拠
- デメリット：申請・審査が必要、個人開発者には開放されない可能性大

### 選択肢C：手動運用（柱A集中・柱D凍結）

柱D は実験柱（月収目標 ¥0）。柱A の手動応募で月¥150K が達成すれば柱Dは不要。
**推奨**：柱A が ¥50K 安定するまで柱D は凍結。
