## 1. D柱：応募文生成（APIキー無しフォールバック・即稼働）

### 手順
```bash
cd /home/user/agent-team/projects/2026-04-08_月30万自動化/D_エクセル入力スクレイピング/pipeline/
unset ANTHROPIC_API_KEY
python3 01_search.py     # data/jobs.json 生成
python3 02_evaluate.py   # ルールベース評価で絞込
python3 03_apply.py      # status:"template" で応募文一括生成
```

### 出力
- 各案件ごとに応募文がJSON形式で生成される
- クリップボード経由でクラウドソーシング画面に貼付して **人間が送信**

### 所要時間
- 1案件あたり 約30秒（生成5秒＋人手レビュー&貼付25秒）
- 48時間中 実働4h×2日 → **80〜120件応募可能**

### 罠と回避策
1. `.env` に `ANTHROPIC_API_KEY` が残っていると raise → 必ず `unset` または `env -i` で実行
2. テンプレ文は category 判定に依存 → 新カテゴリは `スクレイピング` / `データ入力` に寄せる

### 規約遵守
- **絶対：自動送信しない**。応募文は人間がコピペして送信
- プロフィールに「AI補助＋人間確認」明記済（templates/profile_bio.md）

## 2. C柱：Vol.2/Vol.3 PDF化（5分ワークフロー）

### 手順
1. 元原稿：
   - Vol.2：`projects/2026-04-08_月30万自動化/C_テンプレ販売/vol2_sns_calendar.md`
   - Vol.3 #1〜#3：上記ファイル内
   - Vol.3 #4〜#20：`CPO/outputs/2026-05-06_Vol3_プロンプト4-20.md`
   - Vol.2マニュアル：`CPO/outputs/2026-05-06_Vol2_PDFマニュアル.md`
2. **Googleドキュメント新規作成** → Markdown本文を貼付
3. 貼付前に **書式なしで貼付**（Cmd+Shift+V / Ctrl+Shift+V）→ 見出しはH2/H3を手動指定
4. ヘッダーにタイトル＋連絡先1行、フッターにページ番号を挿入
5. **ファイル → ダウンロード → PDF (.pdf)**
6. ファイル名規則：`tpl_vol{N}_{slug}_v{YYYYMMDD}.pdf`
   - 例：`tpl_vol2_sns_calendar_v20260509.pdf`

### 保存先
```
/home/user/agent-team/projects/2026-04-08_月30万自動化/C_テンプレ販売/outputs_pdf/
（無ければ mkdir -p）
```

### 所要時間：1本15分（貼付5＋整形7＋PDF出力3）→ Vol.2/Vol.3を土曜午前で完了可

### 罠と回避策
1. Markdownの表が崩れる → **必ず書式なし貼付**してから見出しを再指定
2. 絵文字フォント未埋込 → PDF化後にプレビューで2ページ目までの表示確認

## 3. note/BOOTH 販売リンク発行チェックリスト

### note（所要3分）
1. 新規記事を作成
2. **「有料エリア」設定**で本文の有料／無料境界を指定
3. PDFをファイル添付（有料エリア内に配置）
4. 価格入力（Vol.2=¥680、Vol.3=¥1,980）
5. 公開

### BOOTH（所要4分）
1. 商品登録 → デジタル商品を選択
2. PDFをアップロード
3. サムネ：Canvaで1280×720固定流用（Vol.1デザインを色違いで使い回す）
4. 公開

### URL記録
- 発行URLを `projects/2026-04-08_月30万自動化/C_テンプレ販売/_sales_links.md` に追記
- CMOへ「URL確定」連絡（メモ追記でも可）

### 罠と回避策
1. BOOTHはサムネ未設定だと検索表示で埋もれる → **必ずサムネ設定**
2. noteは有料エリア設定後に本文編集すると添付PDFが外れる場合あり → **公開前にプレビューでDLテスト**

### 所要時間：1商品7〜8分。Vol.2/Vol.3 計2商品 → **15分以内で公開完了**
