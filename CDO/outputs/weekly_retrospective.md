# 週次レトロスペクティブ テンプレート

**用途**：週末（金 or 日）に1週間を振り返り、翌週の方針を決める。
**運用**：30〜45分の集中時間を確保する。
**保存先**：`CDO/research/retrospectives/YYYY-WNN_retro.md`（W = 週番号）
**費用**：¥0（既存ツールのみ）

# 週次レトロのフォーマット（KPT＋数字）

## 1. 今週の数字（Numbers First）

事実から始める。気持ちは後。

```markdown
| 指標 | 今週 | 先週 | 推移 |
|------|------|------|------|
| コミット数 | 14 | 10 | +40% |
| 追加行数 | 3,200 | 2,100 | +52% |
| note PV合計 | 0 | 0 | ±0 |
| note 公開数 | 0 | 0 | ±0 |
| クラウドワークス応募数 | 0 | 0 | ±0 |
| 受注数 | 0 | 0 | ±0 |
| 売上 | ¥0 | ¥0 | ±0 |
```

→ 数字が動いていない指標が3つ以上あれば、原因を 2 で深掘り。

## 2. KPT（Keep / Problem / Try）

### Keep（続けたいこと）

今週うまくいったことを3つまで。

例：
- 集客記事のフォーマット（3つの習慣）を3本量産できた
- CDO スキャフォルドで Vol.5 の制作工程を50%短縮できた
- 費用ゼロ運用を1週間維持

### Problem（問題・摩擦）

具体的に。「○○できなかった」より「○○のせいで △△が発生した」。

例：
- note 公開を後回しにしている → 「Canva のアイキャッチを作るのに毎回30分かかる」が原因
- クラウドワークス登録が止まっている → 「本人確認書類の写真撮影が面倒」
- 開業届の期限が近いのに準備していない → 「e-Tax の利用者識別番号取得が必要」

### Try（来週試すこと）

最大3つ。Problem に対する小さな対策。

例：
- アイキャッチをテンプレート化（5分で量産できる Canva テンプレを Vol.1 で作成）
- 本人確認書類は「明日の朝一」に撮影（時間枠を予約する）
- e-Tax のセットアップだけを月曜の最初の30分で完了

## 3. 戦略レビュー：3つの問い

### Q1. 「やめる勇気」を持つべきタスクはあるか？

例：
- 業種別サンプル記事の追加生成 → Vol.1 の集客記事が動いていない段階で増やす意味なし → **やめる**

### Q2. 「優先順位の上下」を入れ替えるべきか？

例：
- 来週は「準備物作成」より「公開・応募の実行」を最優先 → スタンドアップで実行ギャップが3日連続🟡なら戦略変更

### Q3. 「次のマイルストーン」は何か？

直近の達成可能な目標を1つだけ。

例：
- 来週末までに：note 記事を1本でも公開、クラウドワークス登録完了、初応募1件
- 達成基準：応募実績トラッカーに1行記録される

## 4. 改善ログ（improvement_log）の集計

過去7日の `CDO/research/improvements/` を見て、頻出する摩擦・改善案を抽出。

```markdown
**頻出する摩擦 TOP3**
1. （例）アイキャッチ作成に時間がかかる（4日で言及）
2. （例）応募の初動が遅い（3日で言及）
3. （例）_index.md 更新を忘れる（2日で言及）

**実行可能な改善 TOP3**
1. アイキャッチ Canva テンプレ化
2. 朝のルーティンに「応募1件」を組み込む
3. スクリプトで _index.md を半自動更新
```

## 5. 来週の Top3 アクション

最大3つに絞る。各アクションは「具体・期限・成功条件」を明記。

```markdown
## 来週の Top3

| # | アクション | 期限 | 成功条件 |
|---|-----------|------|---------|
| 1 | note Vol.1 集客記事を公開 | 月曜中 | 公開URL取得＆XでURL投稿1本 |
| 2 | クラウドワークス登録＆プロフィール完成 | 火曜中 | プロフィール完成度100% |
| 3 | 1日5件の応募ルーティン開始 | 水曜から | 1週間で20件以上応募 |
```

→ Top3以外のアイデアは「来週は意図的にやらない」リストに移動。

---

## 6. 来週は意図的にやらないこと

集中力を奪うものを明文化する。

例：
- 業種別サンプル記事の追加（Vol.1 サンプル1本で十分）
- Vol.6 以降の設計（Vol.5 が公開されてから判断）
- SNS運用代行関連（柱Bは方針除外済み）

# 振り返りを「儀式」として続けるコツ

- 毎週金曜 17:00〜17:45 など、固定枠で予約
- 完璧でなくてもいい、「やった」事実を作る

- AI（このCDO）にレビューしてもらう習慣を作る

## ③ 数字から始める
- 感情から書き始めない
- 数字が出ない週でも「動いていない事実」を直視
