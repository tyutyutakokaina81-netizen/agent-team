# 入金督促・契約更新アラートツール

担当：CDO（実装）× CFO（運用）× CSO（更新交渉実行）
作成日：2026-05-19
スクリプト：`CDO/outputs/scripts/payment_alerts.mjs`

---

## 概要

毎日 or 毎週、A受注CSVと取引先マスターCSVを読み込み、以下を抽出する：

1. **入金予定日超過の未入金リスト**（A受注CSV）
2. **契約更新45日以内のクライアント**（取引先マスター）
3. **インボイス対応が必要な取引先**（適格請求書発行義務ありなのに番号未登録）

`monthly_report.mjs` と同じ設計思想：依存ゼロ・Slack通知・cron運用。

---

## 使い方

```bash
# 当日の状況をmarkdownで表示
node CDO/outputs/scripts/payment_alerts.mjs

# 任意の基準日で実行
node CDO/outputs/scripts/payment_alerts.mjs --today=2026-08-01

# Slack 通知のみ（cron用）
SLACK_WEBHOOK_URL=https://hooks.slack.com/... \
  node CDO/outputs/scripts/payment_alerts.mjs --slack --quiet

# 実データCSVを指定
node CDO/outputs/scripts/payment_alerts.mjs \
  --order-csv=CSO/outputs/2026-05_受注実績.csv \
  --client-csv=CFO/outputs/2026-05-19_取引先マスター.csv
```

### 引数

| オプション | 内容 |
|----------|------|
| `--today=YYYY-MM-DD` | 基準日（デフォルト：今日） |
| `--slack` | Slack Webhook 通知 |
| `--quiet` | 標準出力抑制 |
| `--order-csv=` | A受注CSVのパス |
| `--client-csv=` | 取引先マスターCSVのパス |

### 終了コード
- `0`：要対応なし
- `10`：未入金 or インボイス対応必要あり
- `1`：引数エラー
- `2`：Slack URL 未設定
- `3`：Slack 通知失敗

→ cronで `|| send_alert` のような条件分岐が可能。

## 入金督促エスカレーション

| 経過日数 | アクション | 担当 |
|---------|----------|------|
| 1〜5日 | 1回目リマインドメール（柔らかく） | CFO |
| 6〜14日 | 2回目督促メール（具体的期日提示） | CFO |
| 15〜30日 | 電話・契約再確認 | CSO |
| 31日以上 | 内容証明・法的手続き検討 | CSO・CFO |

### 1回目リマインドメール（経過1〜5日）

```
件名：【お振込のご確認のお願い】請求書 [請求書No]

[クライアント名] [ご担当者名] 様

お世話になっております。
YYYY年MM月DD日付でご請求いたしました
請求書 [請求書No] につきまして、本日時点で
入金の確認が取れておりません。

行き違いの場合は何卒ご容赦ください。
お振込予定について、お手数ですがご確認いただけますと幸いです。

請求書を再送いたしますのでご確認ください。

よろしくお願いいたします。
```

### 2回目督促メール（経過6〜14日）

```
件名：【再度のご確認】請求書 [請求書No] お振込のお願い

[クライアント名] [ご担当者名] 様

お世話になっております。

先日もご連絡いたしましたが、請求書 [請求書No]
（請求額：[金額]）につきまして、現在も入金の
確認が取れておりません。

お振込予定日について、明確なご回答を
[今週金曜日／○月○日まで] にいただけますと幸いです。

何かご事情がございましたらお知らせください。
事情に応じた対応のご相談も可能です。

よろしくお願いいたします。
```

### 3回目（電話含む・経過15〜30日）
- 必ず電話で直接連絡
- 連絡取れない場合は契約書記載の連絡先全てを試す
- 結果をtransactions.csvと取引先マスターに記録

## 契約更新フロー

| 残日数 | アクション | 担当 |
|--------|----------|------|
| 45日前 | 更新意向ヒアリング送信 | CSO |
| 30日前 | 更新確定の意思表示要請 | CSO |
| 14日前 | 自動更新確定 or 解約手続き | CSO・CFO |
| 7日前 | 更新内容の最終確認・新契約書送付 | CFO |

### 更新意向ヒアリングメール（残45日前後）

```
件名：【ご相談】次期ご契約の更新について

[クライアント名] [ご担当者名] 様

お世話になっております。

ご契約期間が [年月日] までとなっており、
あと約45日でご契約満了となります。

つきましては、次期ご契約の更新意向について
ご相談させていただきたく、ご連絡いたしました。

▼ 次期ご契約のご提案
・契約期間：[YYYY-MM-DD〜YYYY-MM-DD]
・月額：[金額]（変更なし）
・自動更新：あり／なし

▼ ご相談事項
1. 現プランの継続でよろしいか
2. プラン変更（アップグレード／ダウングレード）のご希望
3. 解約のご検討

[YYYY-MM-DD]までにご返信いただけますと幸いです。

よろしくお願いいたします。
```

## cron 運用例

```cron
# 毎日朝9時に当日の状況をSlack通知
0 9 * * * cd /path/to/agent-team && \
  SLACK_WEBHOOK_URL=$SLACK_WEBHOOK_URL \
  node CDO/outputs/scripts/payment_alerts.mjs --slack --quiet

# 毎週月曜にmd形式でログ保存
0 9 * * 1 cd /path/to/agent-team && \
  node CDO/outputs/scripts/payment_alerts.mjs > \
  CFO/outputs/alerts/$(date +\%Y-\%m-\%d)_alert.md
```
