# 月次レポート生成ツール

担当：CDO
作成日：2026-05-19
スクリプト：`CDO/outputs/scripts/monthly_report.mjs`

---

## 概要

A受注トラッカー / B営業パイプライン / 経費管理 の3つのCSVを読み込み、
月次サマリを Markdown で生成する依存ゼロのCLIツール。

毎月初に1回実行して `CFO/outputs/YYYY-MM_月次サマリ.md` を生成し、
CFO月次運用フローの起点として使う。

---

## 使い方

```bash
# 標準出力に表示
node CDO/outputs/scripts/monthly_report.mjs 2026-05

# ファイル出力
node CDO/outputs/scripts/monthly_report.mjs 2026-05 \
  --out=CFO/outputs/2026-05_月次サマリ.md

# C テンプレ販売CSVを含めて集計
node CDO/outputs/scripts/monthly_report.mjs 2026-05 \
  --c-csv=CFO/outputs/2026-05_note_sales.csv

# ファイル出力 ＋ Slack通知（環境変数 SLACK_WEBHOOK_URL 必須）
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/XXX/YYY/ZZZ \
  node CDO/outputs/scripts/monthly_report.mjs 2026-05 \
  --out=CFO/outputs/2026-05_月次サマリ.md \
  --c-csv=CFO/outputs/2026-05_note_sales.csv \
  --slack --quiet
```

### 引数

| 位置 | 必須 | 内容 |
|------|------|------|
| 第1引数 | 必須 | 対象月（YYYY-MM 形式） |
| `--out=` | 任意 | 出力ファイルパス。未指定なら標準出力 |
| `--c-csv=` | 任意 | C テンプレ販売CSVのパス（noteエクスポート等） |
| `--slack` | 任意 | `SLACK_WEBHOOK_URL` 環境変数の Webhook に送信 |
| `--quiet` | 任意 | 標準出力への本文を抑制（cron／CI 用途） |

## 読み込むCSV

| 種別 | パス（既定） | 期待される列 |
|------|------------|-------------|
| A受注 | `projects/2026-04-08_月30万自動化/A_ライティング/order_tracker_template.csv` | 受注日 / プラン / 金額 / ステータス / 入金確認 |
| B営業 | `projects/2026-04-08_月30万自動化/B_SNS運用代行/sales_pipeline_template.csv` | 業種 / 初回送信日 / 返信日 / 商談日 / 契約開始日 / 受注金額 / ステータス / 失注理由 |
| 経費 | `CFO/templates/2026-05-19_経費管理シート.csv` | 日付 / 科目 / 金額 / 事業按分 |
| Cテンプレ | `--c-csv=` で指定（既定なし） | 販売日 / 商品名 / 価格 / プラットフォーム / 数量 / 合計金額 |

### Cテンプレ販売CSV の作り方

note ダッシュボードの「売上履歴」をエクスポートして上記列に整形するか、
手動で記録する。サンプル：`CDO/outputs/scripts/sample_c_sales.csv` 参照。

> 各CSVが存在しなければそのセクションをスキップして残りを生成する。
> 実運用では templates の代わりに `CFO/outputs/` 配下の実データCSVに差し替えて使う。
> 既定パスを変更したい場合はスクリプト末尾の `sources` 定義を編集する。

## 出力内容

1. A：ライティング受注（件数・売上・入金状況・プラン別・ステータス別）
2. B：SNS運用代行 営業パイプライン（送信数・返信率・商談化率・成約率・業種別・失注理由）
3. C：テンプレ販売（商品別売上・プラットフォーム別・販売数）
4. 経費（科目別内訳・按分後合計）
5. 損益（簡易）（A売上＋B売上＋C売上 − 経費）

## Slack 通知

`--slack` を付けると、サマリ（A/B/C/経費/売上合計/利益）を
Slack Webhook 経由で投稿する。block_kit 形式で見やすく整形済み。

### Slack Webhook の作り方

1. Slack ワークスペースで「App ディレクトリ」→ Incoming Webhooks を有効化
2. 投稿先チャンネルを選択し Webhook URL を取得
3. `SLACK_WEBHOOK_URL` 環境変数に設定
4. 月次レポート実行時に `--slack` を付ける

### cron 設定例

```cron
# 毎月1日の朝9時に前月分のサマリを Slack 通知
0 9 1 * * cd /path/to/agent-team && SLACK_WEBHOOK_URL=$SLACK_WEBHOOK_URL \
  node CDO/outputs/scripts/monthly_report.mjs $(date -d "1 day ago" +\%Y-\%m) \
  --out=CFO/outputs/$(date -d "1 day ago" +\%Y-\%m)_月次サマリ.md \
  --slack --quiet
```

## CSV仕様の注意点

- 日付は `YYYY-MM-DD` 形式（前方一致で年月を判定）
- 金額は数値のみ（カンマや¥は除去される）
- 事業按分は `100%` `50%` のように % 表記（前後空白OK）
- 入金確認は `TRUE` `true` `1` のいずれか
- 列名は完全一致（半角全角・スペースに注意）

## メンテナンス

- スクリプトは依存ゼロ（Node 標準モジュールのみ）
- Node.js 18+ で動作
- 出力フォーマットを変えたい場合は `buildOrderSection` / `buildSalesSection` / `buildExpenseSection` を編集
