# サムネ取りこぼし監視 運用手順（2026-07-01・CDO）

## 要点
- **目的**：サムネ未設定の note 記事を定期的に検出し、`note-thumbnails.yml` の再実行で自己修復させる。
- **道具**：`check_thumbnail_coverage.py`（読み取り専用・依存ゼロ・ネット不要の集計スクリプト）。
- **定例**：週1回、または記事を追加/公開した直後にチェック → 未取得があれば workflow を手動再実行。

---

## 1. チェックスクリプトの使い方

場所：`CDO/outputs/note_publisher/check_thumbnail_coverage.py`

```bash
# 全記事の充足率＋未取得一覧
python3 CDO/outputs/note_publisher/check_thumbnail_coverage.py

# 日付などで絞る（部分一致）
python3 CDO/outputs/note_publisher/check_thumbnail_coverage.py --filter 2026-06

# 要約のみ（未取得一覧を省く）
python3 CDO/outputs/note_publisher/check_thumbnail_coverage.py --quiet

# 機械可読（監視ログ等に使う）
python3 CDO/outputs/note_publisher/check_thumbnail_coverage.py --json
```

出力例：

```
=== サムネ充足率チェック（読み取り専用・集計のみ） ===
記事数        : 144
サムネ取得済み: 104
未取得        : 40（ファイル無し 40 / 素性不明 0）
充足率        : 72.2%
```

**判定ロジック**（`fetch_thumbnails_wikimedia.py` と同一基準）：
- 対象記事 = `CMO/outputs/*note記事*.md`（`サムネ生成プロンプト` を含む補助 md は除外）。
- 取得済み = `thumbnails/{stem}.jpg` が存在し、かつ `_provenance.json` の backend が
  **GOOD_BACKENDS**（`wikimedia` / `pexels` / `openai` / `gemini` / `pollinations`）。
- 未取得 = 上記を満たさない記事。内訳を「ファイル無し」と「素性不明（jpg はあるが provenance 不明）」に分ける。
- **終了コード**：未取得0で `0`、1件以上で `1`（cron/CI の判定に流用可）。

> ¥0・ネット/キー不要・破壊的操作なし（集計だけ）。既存スクリプトには一切触れない新規追加。

---

## 2. 定例タイミング

| タイミング | やること |
|---|---|
| **週1回（例：月曜）** | `check_thumbnail_coverage.py` を実行し充足率を確認 |
| **記事を追加/公開した直後** | 同上。新規記事はサムネ未取得なので必ず出る |
| **`note-thumbnails.yml` を回した後** | 再実行して充足率が上がったか（自己修復できたか）を確認 |

補足：`note-thumbnails.yml` は `CMO/outputs/**` への push で自動起動し不足分のみ取得する。
通常は push で自動修復されるが、**取りこぼし（ネットワーク失敗・候補画像なし等）が残る**ため、
この監視で定期的に検出して手動再実行で埋める。

---

## 3. 未取得が出たときの自己修復手順

未取得（充足率100%未満）が出たら、`note-thumbnails.yml` を手動で `workflow_dispatch` 再実行する。
不足分のみ取得する増分設計なので、既存サムネは再取得されず ¥0・安全。

### GitHub Web UI から（推奨）
1. リポジトリ → **Actions** タブ。
2. 左メニューの **"Generate note thumbnails"** を選択。
3. 右上 **"Run workflow"** をクリック。
4. `filter` 入力：
   - 空欄 → 全記事の不足分をまとめて再取得。
   - 日付（例 `2026-06-09`）→ その日の記事だけ再取得（対象を絞りたいとき）。
5. **"Run workflow"** で実行。完了後、bot が `thumbnails/*.jpg` と `_provenance.json` をコミットする。

### 実行後の確認
```bash
git pull                 # bot のコミットを取り込む
python3 CDO/outputs/note_publisher/check_thumbnail_coverage.py --quiet
```
充足率が上がっていれば自己修復成功。まだ残る記事は、Wikimedia に適切な候補が無い等の理由。
その場合は次のいずれかで対応：
- 検索語を `fetch_note_thumbnails.py` の `query_for()` 対応表に追記して再実行（CDO 作業）。
- note 見出しは **みんなのフォトギャラリー（無料・実写）** をオーナーが手動採用。

---

## 4. 注意・制約
- スクリプトは**読み取り専用**。サムネ生成・取得は行わない（それは `note-thumbnails.yml` の役目）。
- **AI 生成は既定 OFF**（owner 方針＝画質が荒いため）。実写は Wikimedia（キー不要）→ Pexels（無料キー）の順。
- code（Claude Code）はネット遮断のため、サムネ取得の実行はクラウド（Actions）かオーナー側で行う。
- 日本語ファイル名は Python の glob で直接読むため `git config core.quotepath` 等の設定は不要。
