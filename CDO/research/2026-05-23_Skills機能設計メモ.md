# Claude Code Skills 機能設計メモ

---

## 1. 背景

CPOメソッド（YouTube 出典）において、Claude Codeの「Skills」機能を用いて以下の専門業務を自動化する構想がある。

- **事業評価スキル**: 6軸での採点（Go/No-Go判定）
- **中身の設計スキル**: カリキュラム・プログラムの構造化出力

CDO として、これを当社のリポジトリ運用ルールに沿って実装する方針を整理する。

---

## 2. Skills とは（CPOメソッド文脈での定義）

- Claude Code で呼び出すだけで特定の専門業務を遂行する仕組み
- 「鬼コンサル人格」など独自ペルソナを設定し、厳しく評価させる用途に有効
- 過去の上司の思考・著名経営者の音声データ・PDFを読み込ませ「AI社員」として育成可能
- **ノートブックLMではなくローカル環境推奨**（データ秘匿性・カスタマイズ性のため）

---

## 3. 当社への実装方針（案）

### 3.1 配置場所（実装済）

```
CDO/outputs/skills/             ← Git管理（canonical）
├── business/
│   └── SKILL.md                ← /business スラッシュコマンド本体
├── curriculum/
│   └── SKILL.md                ← /curriculum スラッシュコマンド本体
└── install.sh                  ← .claude/commands/ への展開スクリプト

.claude/commands/                ← gitignore対象（ローカル実行版）
├── business.md                 ← install.sh で展開
└── curriculum.md               ← install.sh で展開
```

**運用フロー**：
1. canonical版を `CDO/outputs/skills/<name>/SKILL.md` で編集・Git管理
2. `./CDO/outputs/skills/install.sh` を実行して `.claude/commands/` に展開
3. Claude Code を再起動するとスラッシュコマンドが認識される

**この設計の意図**：
- `.claude/` 全体は「ローカル専用」として `.gitignore` で除外（既存ルール）
- 当社のIPであるSkills本体は `CDO/outputs/` で永続化・共有可能
- エフェメラル環境でもinstall.shを実行すれば即復元できる

### 3.2 事業評価スキル（business-eval）の6軸（仮）

実際の動画で言及された具体的6軸は要再確認。仮置きで以下を想定：

1. 市場規模・成長性
2. 競合優位性・差別化
3. 収益性（粗利・LTV）
4. オペレーション実行可能性
5. 必要資金・回収期間
6. リスク（法務・依存・撤退コスト）

> ※ オーナー承認後、CPOガイド本体に正式な6軸を反映する。

### 3.3 ペルソナ設定

- ベース人格：「ゴールドマン・サックス、BCG、マッキンゼーのトップコンサル集結体」
- トーン：忖度なし／論理重視／数値根拠を要求

### 3.4 カスタムコマンド設計（CPO完全版より反映）

CPOガイド完全版で言及された `/business` 形式のカスタムコマンドを当社流に整備する：

| コマンド案 | 用途 | 内部呼び出しスキル |
|---------|------|----------------|
| `/business` | 新規事業の5ステップ評価を一気通貫 | business-eval |
| `/business price` | ステップ2のプライシング単独実行 | business-eval（部分） |
| `/business compare` | ステップ4の競合比較（5分版） | business-eval（部分） |
| `/curriculum` | カリキュラム・教材設計 | curriculum-design |

実装は `.claude/commands/` 配下のMarkdown定義として整備予定。

### 3.5 一次情報の入力規約（CPO完全版より反映）

スキル実行時に読み込ませる一次情報の標準フォーマット：

- **顧客アンケート**: CSV（id, 業種, 質問1〜N, 自由記述）
- **面談文字起こし**: Markdown（日付・相手・要約・全文）
- **会食メモ**: `context/diary/` または `context/references/` 配下

> CSO に収集テンプレ整備を依頼予定。

---

## 4. ガードレール（CDO 観点）

- **ハルシネーション対策**: 数値出力には必ず「出典が不明な場合は『要確認』と明記せよ」を組み込む
- **データ取扱**: 顧客PII・契約情報をスキルに渡さない（CFO 領域）
- **再現性**: スキルのバージョンを `SKILL.md` 冒頭に明記し、変更履歴を残す
