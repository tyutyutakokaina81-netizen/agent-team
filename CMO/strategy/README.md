# CMO マルチメディア運用システム

> 出典戦略: `context/references/2026-05-30_ClaudeCode-CMO自動化戦略_動画メモ.md`
> 設計ドラフト: `../research/2026-05-30_マルチメディア戦略_当社CMO運用への落とし込み.md`
> 実装日: 2026-05-30

少人数で10人規模のマーケチームに匹敵する成果を出すための「仕組み」。
**週ワンテーマ → メイン媒体（note）→ 派生媒体（X／LINE／ブログ）** の一方向フローで回す。

## フォルダの役割
```
CMO/
├── strategy/   ← この運用システムの中枢（台帳・スキル・文体規定・メタスキル）
├── note/       ← メイン媒体。完成稿（英語要約付き）の置き場
├── X/          ← noteからの派生：連投スレッド／単発ポスト案
├── LINE/       ← noteからの派生：配信文・ステップ配信案
├── outputs/    ← 確定版の最終アーカイブ（従来どおり）
└── research/   ← 下書き・調査
```
※ 既存 `outputs/` の過去記事はそのまま残置。新規分から媒体別に振り分ける。

## 回し方（毎週）
1. **週テーマを1つ決め切る**（オーナー＝一次情報）→ `週テーマ台帳.md` に記入
2. **メイン媒体でnote記事を1本**書く（`媒体別スキル.md` のnote/SEO規定に従う）
3. **メタスキルで派生展開**（`メタスキル_note派生展開.md`）→ X連投・LINE・ブログ構成を一括生成
4. オーナーが最終チェック（事実・トーン・公開可否）→ 公開
5. `_index.md`・`STATE.md` を更新

## 中枢ドキュメント
| ファイル | 役割 |
|---|---|
| `週テーマ台帳.md` | 週ごとのワンテーマと展開状況を管理 |
| `媒体別スキル.md` | 各媒体のアルゴリズム最適化ルール（焼き込みスキル） |
| `文体規定.md` | 人間らしさ・AI定型句禁止・2:8分担・裏取りルール |
| `メタスキル_note派生展開.md` | note完成稿 → 全派生媒体を一括生成する上位フロー |
