# メタスキル：note完成稿 → 全派生媒体を一括生成

> 上位の業務フロー。**完成したnote記事1本**を入力に、X連投・LINE・ブログ構成を**ワンパッケージで一括生成**する。
> 一方向フロー固定：**note（核）→ 派生**。逆流させない（インプット源をnoteに固定し、毎回の書き直しを防ぐ）。

---

## 入力
- メイン媒体の**完成note記事**（`CMO/note/` または `CMO/outputs/` のファイル）
- 週テーマ（`週テーマ台帳.md`）

## 適用ルール
- 各媒体の最適化は `媒体別スキル.md` に従う。
- 文体は `文体規定.md` に従う（AI定型句禁止・一次情報・裏取り）。
- 派生は**note本文の主張をズラさない**。媒体ごとに「翻訳・最適化」するだけ。

## 出力（4点を一括生成）

### ① X 連投スレッド → `CMO/X/YYYY-MM-DD_テーマ.md`
- 1投目＝フック（意外な結論／数字／問い）
- 本文5〜7投（各投が単体でも成立）
- 最終投＝CTA（note誘導）
- ハッシュタグ1〜2個

### ② X 単発ポスト（予備）→ 同ファイル内に併記
- 140字前後で記事の一番おいしい所を1投に凝縮

### ③ LINE 配信文 → `CMO/LINE/YYYY-MM-DD_テーマ.md`
- 1通完結版（開封1行＋価値＋CTA1つ）
- ＋ステップ配信3通版（①価値→②深掘り→③オファー）

### ④ ブログ／SEO構成 → `CMO/research/YYYY-MM-DD_テーマ_SEO構成.md`
- 想定検索意図／主要KW
- タイトル案3つ
- H2/H3見出し構成＋各セクションの要点
- 内部リンク・CTA案

## 実行手順（CMOが回す）
1. 対象noteを読み込む
2. 週テーマ・媒体別スキル・文体規定を読み込む
3. ①〜④を上記パスに生成
4. `CMO/_index.md` の成果物ログに4点を追記
5. `週テーマ台帳.md` の該当週の状態を更新
6. オーナーへ最終チェック依頼（公開はオーナー手動）

## 制約（当社環境）
- 公開API・GA4/サーチコンソール連携は当コンテナ不可（外部403）。
  分析データはオーナー端末で取得→貼り付け→CMOが分析、の運用。
- note自動公開はオーナーMacの `CDO/outputs/note_publisher/` 経由（Playwright・text-only実証済み）。
