- 種別: 運用設計ドラフト（オーナーレビュー待ち）
- 元ネタ: `context/references/2026-05-30_ClaudeCode-CMO自動化戦略_動画メモ.md`
- 目的: 動画の「Claude Code = CMO」戦略を、当社の `月30万自動化` プロジェクトと既存 `CMO/` 構造に接続する。

---

## 要点（先に結論）
1. **週イチ・ワンテーマ運用**に切り替える。現状の「実行して＝今日のnote記事1本」を、**週テーマ → 媒体展開**の一方向フローに格上げする。
2. **メイン媒体を確定**する（候補：note／YouTube）。一次情報を最も出しやすい媒体を核に固定。
3. **媒体別フォルダ**を `CMO/` 配下に新設し、台本・派生投稿・運用ガイドを構造化する。
4. **2:8分担を明文化**：オーナー＝テーマ＋一次情報＋最終チェック（2割）／CMO（AI）＝展開・最適化（8割）。

## 1. 現状とのギャップ
| 観点 | 動画の推奨 | 当社の現状 | 反映アクション |
|------|-----------|-----------|--------------|

## 2. 提案：CMO配下の媒体別フォルダ構成
```
CMO/
├── _index.md            ← 週テーマ固定欄＋成果物ログ
├── strategy/            ← 全体設計・週テーマ台帳・展開フロー図
├── YouTube/             ← （将来）台本・サムネ・概要欄
├── note/                ← note記事（現outputsの実体をここへ寄せる検討）
├── X/                   ← note派生のポスト案（連投・単発）
├── LINE/                ← 配信文・ステップ配信案
└── outputs/             ← 確定版の最終置き場（現状維持）
```
※ いきなり全部は作らない。**まず `strategy/` と `note/` `X/` の3つ**から始めるのが現実的。

## 3. 一方向の展開フロー（固定）
```
週テーマ決定（オーナー＝一次情報）
        ↓
メイン媒体: note記事（完成稿・英語要約付き）   ← 既存の強み
        ↓ 派生（AIが自動翻訳・最適化）
   ├─ X: 連投スレッド案（フック1行＋本文5〜7投）
   ├─ LINE: 配信文（1通・CTA付き）
   └─ ブログ/SEO: 見出し構成案（H2/H3＋検索意図）
```
逆流（Xから書き始める等）はやらない。インプット源を毎回noteに固定する。

## 4. 役割分担（2:8）— 当社版
- **オーナー（2割・人間にしか出せない部分）**
  - 週テーマの決定
  - 一次情報の提供（高岡の現地体験、月30万プロジェクトの実数・失敗談）
  - 最終チェック（事実関係・トーン・公開可否）
- **CMO＝AI（8割）**
  - note完成稿の執筆 → X/LINE/ブログ案への展開
  - 形式最適化（文字数・フック・CTA）
  - `_index.md`／`STATE.md` の記録更新

## 5. 「人間らしさ」ルール（当社の文体規定に追加）
- AI定型句（「いかがでしたでしょうか」等）を**禁止語**に。
- 高岡シリーズで実証済みの「実体験・固有名詞・脱線」を意図的に残す。
- 数字・事例は**裏取り必須**（公的統計／公式リリース）。AI単独生成の無裏付け数値は出さない。

## 6. 環境制約による現実解（重要）
- **GA4/サーチコンソール連携は当コンテナでは不可**（外部API全403）。
  → データはオーナー端末で取得し、CSV/数値をチャットに貼り付け → CMOが分析、の運用にする。
- WordPress自動化等も同様に直接連携は不可。オーナー端末側の `note_publisher` 方式（Playwright）を踏襲する。
