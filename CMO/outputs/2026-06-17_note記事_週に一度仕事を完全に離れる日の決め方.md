# note記事：週に一度、仕事を完全に離れる日を決めた。

- **日付**: 2026-06-17（暮らし／ひとり起業）

## タイトル
```
週に一度、仕事を完全に離れる日を決めた。
```

## 本文

```
ひとりで仕事をしていると、「休み」の輪郭がない。

締め切りがなければ書けるし、クライアントから連絡が来れば返せる。
土曜も日曜も、物理的には仕事できる。
気づくと、「今週ちゃんと休んだか」がわからなくなっていた。

それで去年、決めた。
「日曜日は、仕事のことを考えない日」と。

---

■ 決めることで初めて守れる

「なんとなく休む」は、休めない。
メールを開けば仕事になる。タスク管理アプリを触れば、次のToDoが目に入る。

だから、日を決めた。
日曜日は、PCを開かない。Slackも見ない。ノートにタスクを書かない。

守れない日もある。でも「決めた」という事実があると、守れなかったとき気づける。
守れた日は、翌月曜の仕事の入り方が、明らかに違う。

---

■ 完全に離れる、のハードル

最初は「何もしないこと」に落ち着かなかった。

仕事をしていないと、進んでいない気がする。
アウトプットを出さないと、不安になる。

でも、この感覚自体が疲れていたのかもしれない、と後から思う。
常に「何かしなければ」と動いていると、充電がゼロのまま走り続けることになる。

週に一日、意図的に何もしない時間を取ることで、
残りの六日間の仕事の密度が変わった。少なくとも、私の場合は。

---

■ 高岡の日曜日

高岡の日曜は、商店街が静かになる。
午前中に市場を歩いて、昼は家で食べて、午後は本を読んでいると、
あっという間に夕方になる。

この「のんびり感」が、当初は物足りなかった。
でも今は、これくらいがちょうどいい。

仕事は月曜から再開できる。
その月曜に「また始められる」という感覚があるのは、日曜に離れているからだと思う。

---

休みを決めるのは、さぼることじゃない。
翌週の燃料を確保する、仕事の設計の話だ。

---

🌏 For English readers

Working alone means there's no clear line between work days and rest days.
Meetings don't force a rhythm. Deadlines can stretch. So the default is to
never fully stop.

I made a rule: Sundays are off. No laptop. No task lists. No work messaging.
Not because I had to — because I chose to.

The first few Sundays felt restless. No output felt like no progress.
But that discomfort turned out to be the sign I needed rest most.

The effect showed up on Mondays. The same desk, same tasks — but entered
differently. Quieter Sundays made for sharper weeks.

In Takaoka, Sunday mornings have a particular stillness. Markets open early,
then the town quiets down by afternoon. I've come to rely on that rhythm as
much as any schedule I built myself.
```
