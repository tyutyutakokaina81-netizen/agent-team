# note記事：月曜朝に、問いを一つ立てることにした。

- **日付**: 2026-06-19（ひとり起業／仕事術）

## タイトル
```
月曜朝に、問いを一つ立てることにした。
```

## 本文

```
月曜朝の習慣について、また書く。

以前は「最初にやるタスクを決めておく」という話を書いた。
それは今も続けている。ただ、もう一つ加えたものがある。

「今週、一つだけ考えたいこと」を決めることだ。

---

■ タスクじゃなくて、問い

タスクは「やること」だ。終わればチェックが入る。

問いは違う。答えが出るかどうかわからない。
でも「考え続けている」ということが、週の途中で判断するときの軸になる。

今週であれば、「SNSの発信頻度と収益化のタイミング、どう考えるか」。

これを月曜朝に決めておくと、
水曜に何かのコンテンツを書くとき、木曜に見積もりを出すとき、
問いの角度からそれを見ることができる。

---

■ タスクをこなすだけの週とは違う

問いを持たない週は、タスクをこなすだけで終わる。
こなしてはいるが、どこへ向かっているのかが見えない。

問いがあると、同じタスクをこなしながらも「これが答えの一部になるかもしれない」
という感覚で仕事ができる。

---

■ 答えが出なくてもいい

週末に答えが出なくても、問いを持っていたこと自体に意味がある。

「これ、まだよくわかっていないな」という状態で次の週に進むことで、
情報収集やコンテンツ制作の解像度が変わってくる。

問いは積み重なる。答えはあとからついてくる、くらいの感覚で続けている。

---

■ 月曜朝のノートに一行書く

やり方はシンプル。
月曜朝にノートを開いて、「今週の問い：」と書いて、一行書く。

それだけ。

高岡の月曜朝は、天気がよければ窓から山が見える日もある。
問いを書いて、コーヒーを飲んで、始める。

---

🌏 For English readers

I already wrote about choosing one task to start Monday with. This is the
other thing I added: choosing one question to carry through the week.

Not a task — a question. Something I want to think about, not necessarily
answer. This week's: what's the relationship between posting frequency and
monetization timing?

Having a question changes how the same tasks feel. Writing content, reviewing
proposals, reading something — all of it becomes data for the open question.

The week doesn't feel like checking boxes as much. It feels like moving
toward something, even when the destination is still unclear.

The question doesn't have to be answered by Friday. Sometimes the value is
in arriving at Monday with a sharper version of the same question.
```
