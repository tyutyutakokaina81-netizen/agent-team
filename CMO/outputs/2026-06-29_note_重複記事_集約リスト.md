## owner / cowork の実行手順
1. A表の「非公開化推奨」側が**公開済みか**を note で確認。
2. 公開済みなら、古い重複を**非公開**にする（または正へ統合）。URLが分かれば 6/23棚卸しの4グループと突き合わせ。
3. B表は維持。可能なら2本を相互リンク（回遊）。
4. 完了後、どれを非公開化したかを code に共有 → registry/マガジンから除外し台帳を更新。

> **正直な注記**：リポジトリの md が2本でも、note 上は1本しか公開されていない可能性もある（片方は下書きのまま）。
> その場合は「非公開化」不要＝ note 側が正。**最終判断は note の公開状態を見られる owner/cowork**。
