# 受託パイプライン（クラウドソーシング案件の受注→制作→納品）

> 2026-06-23 新設。クラウドソーシング（ランサーズ/クラウドワークス/ココナラ等）の受託案件を、
> **受注メモ → 制作 → 品質チェック → 納品** の順で機械的に流す仕組み。
> 役割：**応募・受注・客対応・納品アップロードは人（オーナー/cowork）**／**制作・品質チェックは code（私・エージェント）**。
> A1（ネット不可）でも回る。支出ゼロ。

## フォルダ
```
受託/
├── inbox/     ← 受注した案件メモ（人が new で作る or 貼る）
├── wip/       ← 制作中（code が start で移動・ここに成果物を書く）
└── delivered/ ← 納品済（deliver で移動・客に渡す現物）
```

## 標準フロー
1. **受注したら**：`python3 tools/jobs.py new --type 記事 --title "..." --budget 5000 --deadline 2026-06-30`
   - `受託/inbox/` に案件カードが出来る。案件の詳細（発注者の要件）を本文に追記。
2. **制作開始**：`python3 tools/jobs.py start <id>` → `wip/` へ。
   - code（私）が `jutaku-delivery` スキルの手順で制作 → `受託/wip/<id>_成果物.md` に完成稿。
   - **必ず** `python3 tools/quality_check.py 受託/wip/<id>_*.md` で規則チェック（HARD=0を確認）。
3. **納品準備**：`python3 tools/jobs.py deliver <id> --file 受託/wip/<id>_成果物.md` → `delivered/` へ。
   - 人が現物を発注者に納品（コピペ/ファイル添付）。
4. **入金確認**：`python3 tools/jobs.py done <id> --paid 5000` → 売上を記録。

## 黒字の見える化
- `python3 tools/jobs.py list` で 受注中/制作中/納品済/入金額 を一覧。
- 月の入金合計が固定費¥5,800を超えたら黒字。CLIが累計を出す。
