# CPO 事業壁打ちシステム

> 出典戦略: `context/references/2026-05-30_ClaudeCode-CPO事業壁打ち戦略_動画メモ.md`
> 実装日: 2026-05-30

CPO＝会社が**「何を売るか」を決める責任者**。経営者が一人で抱えがちな事業判断を、
Claude Code に「壁打ち相手」として任せ、**通常1日の作業を30分**で終わらせるための仕組み。

## フォルダ構成
```
CPO/
├── strategy/   ← この壁打ちシステムの中枢
│   ├── README.md              ← 全体（このファイル）
│   ├── 事業壁打ちフロー.md      ← 30分・5ステップの実行手順
│   ├── スキル_GoNoGo判定.md     ← 厳格コンサル人格・6軸採点（カスタムコマンド）
│   └── 壁打ち案件台帳.md        ← 評価した事業アイデアのログ
├── outputs/    ← 確定した壁打ち結果・カリキュラム等
└── research/   ← 下書き・調査
```

## 使い方（1案件あたり30分）
1. **4つの入力要素**を渡す（業種／規模感／やりたいこと／ゴール）＋一次情報があれば添付
2. `事業壁打ちフロー.md` の5ステップを順に実行
3. S5で `スキル_GoNoGo判定.md` を適用し、厳格に採点
4. 結果を `outputs/YYYY-MM-DD_案件名_壁打ち.md` に保存
5. `壁打ち案件台帳.md` と `_index.md` に記録
6. **最終意思決定は必ず人間（社長）が行う**（AIのA評価＝GOではない）

## 鉄則
- **一次情報を読ませる**（顧客アンケート・会食メモ・面談文字起こし）。一般論回答を避ける。
  - 一次情報は当社の `context/`／`CSO/` の顧客データを活用。NotebookLMではなくClaude Codeのローカルで蓄積（「育つ」運用）。
- **ファクトチェック必須**：数字・競合情報は人間が最終確認。
- **忖度させない**：S5は「非常に厳しく評価して」と明示。
