**作成日**: 2026-06-22
**役割**: お試し1本を受注した瞬間に、**最短5営業日で高品質なSEO記事**を納品するための実物。
**狙い**: スピード×品質で「お試し→継続」に変える。継続が月¥30万の正体。
**連携**: リモートAI（私）は、テーマが決まれば構成案と本文ドラフトを即作る。下記は人＝オーナー/Coworkが回す枠組み。

## 制作フロー（受注→納品、5営業日）

```
Day1  テーマ確定 → キーワード選定 → 構成案を作成 → 客に送って承認をもらう
Day2-3 本文執筆（2,000〜3,000字）＋ 事実確認
Day4  品質チェックリストで自己点検 → 納品フォーマットに整形
Day5  納品 → 修正1回まで対応 → 「継続どうですか？」を必ず添える
```

> リモートAIの使い方：テーマと業種をボードに書けば、私が「構成案→本文ドラフト」を即返す。
> オーナー/Coworkは事実確認（地元情報・店固有情報）と最終調整に集中すればよい。

## 1. キーワード選定（10分でやる簡易版）

```
① 「地域 × 悩み/用途」で考える
   例）整体院 → 「肩こり 高岡」「腰痛 整体 富山」
       飲食店 → 「富山 ランチ 個室」「高岡 ◯◯料理」
② Google検索で、そのキーワードを打って下までスクロール
   → 「関連する検索キーワード」「他の人はこちらも検索」をメモ（=実需要）
③ 検索結果の上位3記事をざっと見て、「何を書けば勝てるか」を1行でメモ
   （上位が薄ければチャンス。地元視点で深く書けば上に行ける）
④ 主キーワード1つ＋関連キーワード2〜3つを決定 → 構成に散らす
```

## 2. SEO記事 構成テンプレ（この型に流し込む）

```
■ タイトル（32文字前後・主キーワードを前方に・具体性）
   例）肩こりが高岡で取れないあなたへ｜自宅ストレッチと整体の使い分け

■ リード文（150字・「この記事で解決すること」を明示）
   - 読者の悩みを言い当てる → この記事を読むとどうなるか

■ 見出しh2 ①（悩みの背景・なぜ起きるか）
■ 見出しh2 ②（自分でできる対処・具体的な手順）
■ 見出しh2 ③（プロ／店に頼むべきケース・選び方）← ここに自然に自店への導線
■ 見出しh2 ④（よくある質問 Q&A 2〜3個）← 検索の取りこぼし回収

■ まとめ（要点3つ・次の一歩）
■ CTA（来店/予約/問い合わせへの誘導・電話/フォームリンク）

※各h2に関連キーワードを1つずつ自然に入れる（詰め込みすぎない）
※文字数 2,000〜3,000字目安
```

## 3. 品質チェックリスト（納品前に必ず）

```
内容
□ 検索意図に答えているか（読者が知りたいことが冒頭で分かる）
□ 主キーワードがタイトル・リード・h2に自然に入っているか
□ 具体例・数字・手順があるか（抽象論で終わっていない）
□ 自店への導線が「押し売り」でなく「自然な選択肢」になっているか

正確性・安全（CLAUDE.md準拠）
□ 事実確認済み（営業時間・価格・固有情報は客に確認）
□ 誇張・断定NG（「世界一」「必ず治る」等は使わない／医療系は特に慎重に）
□ PII・具体的な住所/個人名を不用意に書いていない
□ 薬機法・景表法に触れる表現がないか（美容・治療・健康系は要注意）

体裁
□ 1文が長すぎない／段落が適度に分かれている
□ 誤字脱字チェック済み
```

## 4. 納品フォーマット

```
形式：Googleドキュメント（共有リンク）or Word
含めるもの：
  - 本文（見出し構造つき）
  - タイトル案 2〜3パターン
  - メタディスクリプション案（120字・検索結果に出る説明文）
  - 使用キーワード（主＋関連）
  - 画像の指示（「ここに〇〇の写真」程度・撮影/選定は客側）

納品メールに必ず添える一文：
「今回いかがでしたか。月2〜4本の継続なら、テーマ設計から一括でお引き受けできます。
 来月分のテーマ案もすぐお出しできますが、ご興味ありますか？」
→ お試しを“その場で”継続提案につなぐ。これを毎回やる。
```
