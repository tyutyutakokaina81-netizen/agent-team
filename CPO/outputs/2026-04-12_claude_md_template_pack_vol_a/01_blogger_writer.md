# CLAUDE.md — 個人ブロガー・Webライター向け

このファイルは Claude Code に対するプロジェクト指示書です。ブログ記事・SEO記事の執筆を効率化するための設定です。

## プロジェクトの目的

個人ブロガー・Webライターとして、SEOに強い読者価値の高い記事を高速で執筆する。

## 担当領域

- キーワードリサーチ支援
- 記事構成（H2/H3）の設計
- 本文の下書き生成
- 校正・リライト
- メタディスクリプション作成
- 画像キャプション・alt テキスト案

## やらないこと

- 医療・法律・金融分野の断定的な記述（YMYL領域は必ず人間が最終確認）
- 引用元不明の統計・データの創作
- 他記事の無断転載・リライト
- ステルスマーケティング・誇大広告

## 記事執筆のワークフロー

1. **キーワードを受け取る** → 検索意図を3パターン推測
2. **競合記事を並べる**（URL を貼る） → 共通構成と差別化ポイントを抽出
3. **H2/H3 構成を提案** → 読者の悩み順に並べる
4. **本文を生成** → 1H2 = 500〜800字、具体例を必ず入れる
5. **メタディスクリプション** → 120字以内、キーワードを自然に含める
6. **校正** → 重複表現・冗長表現・敬体統一チェック

## 禁止事項（記事内）

- 「〜と言われています」「〜という噂があります」等の出典不明表現
- 「絶対」「必ず」等の断定（医療・効能系）
- 感嘆符の連続使用（！！！等）
- 同一段落内での同じ語尾の3回以上連続

## 推奨フォーマット

```markdown
## [H2見出し：読者の疑問形が望ましい]

[導入：1〜2文で共感 or 問題提起]

### [H3見出し]

[本文：箇条書きと段落のバランス、各段落 3〜5 文]

[必要なら具体例 or 引用]
```

## フォルダ構造

```
project/
├── CLAUDE.md          ← このファイル
├── drafts/            ← 下書き
├── published/         ← 公開済み
├── keywords/          ← キーワードリサーチ結果
└── references/        ← 参考資料・競合URL
```

## 成果物の命名規則

- 下書き：`drafts/YYYY-MM-DD_キーワード.md`
- 公開済：`published/YYYY-MM-DD_タイトル.md`
