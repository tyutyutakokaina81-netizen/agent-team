## 2. ビジネスモデルの構造

### 2.1 収益フロー

```
AI agent (Claude/GPT/etc.) asks user question
  ↓
Agent calls MCP server /search endpoint
  ↓
Server returns HTTP 402 + payment descriptor
  ↓
Agent signs USDC transaction on Base network
  ↓
x402 facilitator verifies payment (~2 seconds)
  ↓
Server returns the data
  ↓
USDC lands directly in owner's self-custody wallet
```

## 8. 撤退基準（事前コミット）

以下の条件に該当したら、その月末で**速やかに撤退**する：

1. **2026-06 末時点で累計収益 < ¥3,000**（3 ヶ月成果ゼロ）
2. **2026-09 末時点で累計収益 < ¥20,000**（6 ヶ月で目標の 50% 未達）
3. **月次バーンレートが runway を 3 ヶ月以下に圧縮**
4. **オーナーの心理的継続意欲が顕著に低下**（これは数字ではないが最重要）

**撤退時の対処**：
- MCP は無料モードに切替（burn address に戻す or 価格ゼロ化）
- autonomous loop を dry-run モードに固定
- Claude Pro 解約検討
- リポジトリは GitHub に portfolio として保存
- 再起動は 5 分で可能な状態を維持

## 10. 最終結論

### このビジネスは「賭け」である

数字は正直です。現状条件での期待値は：
- **悲観ケース 35%**：¥-55,000〜¥-70,000（6 ヶ月損失）
- **中央ケース 45%**：¥-14,000〜¥+5,000（ほぼトントン）
- **強気ケース 15%**：¥+35,000〜¥+100,000
- **超強気 5%**：¥+300,000+（バズ発生）

### 期待値計算

$$
EV_{6ヶ月} = 0.35 \times (-60K) + 0.45 \times 0K + 0.15 \times 70K + 0.05 \times 300K
$$
$$
= -21K + 0 + 10.5K + 15K = \mathbf{+¥4,500}
$$

**6 ヶ月 EV = +¥4,500**。限界的にプラス。

### リスク調整後の判断

- 限界的プラスの EV
- 下振れリスクは耐えられる範囲（最悪 ¥70K）
- 上振れは人生を変える可能性（¥300K+、narrative 価値）
- **Claude Pro を ¥10K/月に削るなら、継続は合理的**
- **現状維持（月 ¥30K）だと数字が合わない、削減必須**

### CFO の最終提言

> **"Claude Pro を ¥10K 以下に削って、MCP ポートフォリオ戦略を 6 ヶ月走らせる。
>  撤退基準を事前コミットし、感情ではなく数字で判断する。"**

これが、**数字が支持する唯一の合理的な道** です。
