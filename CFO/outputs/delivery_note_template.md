# 納品書／納品確認書テンプレート

> 納品時に成果物と一緒に送付。後日の「納品済み証明」として機能する。
> 受領サインを返してもらうと、未払いトラブル防止に最強。

---

## 納品物の標準セット

### A柱（ライティング）の場合
```
deliverables/
├── [案件名]_v1_final.docx          # 本文（Word/Googleドキュメント）
├── [案件名]_meta.md                # メタディスクリプション・タイトル
├── [案件名]_outline.md             # 構成・見出し設計
└── README.md                        # 使い方・修正依頼方法
```

### B柱（SNS運用代行・月次納品）の場合
```
deliverables/YYYY-MM/
├── posts/                           # 月分の投稿文
│   ├── 2026-05-01.md
│   ├── 2026-05-02.md
│   └── ...
├── hashtags.md                      # ハッシュタグセット
├── monthly_report.pdf               # 月次レポート
└── README.md
```

### C柱（テンプレ販売）の場合
```
直接 note/BOOTH ページから配布
購入者には Google Drive リンクを自動送信
```

### D柱（業務自動化）の場合
```
deliverables/
├── src/                             # ソースコード
│   ├── main.py
│   └── ...
├── docs/
│   ├── README.md                    # セットアップ手順
│   ├── USAGE.md                     # 使い方
│   └── TROUBLESHOOTING.md           # トラブル対応
├── tests/                           # テストデータ
└── deployment_video.mp4             # 操作デモ動画（任意）
```

## 運用ルール

### 番号採番
- フォーマット：`DLV-YYYY-NNNN`
- 見積 `QUO-YYYY-NNNN` → 納品 `DLV-YYYY-NNNN` → 請求 `INV-YYYY-NNNN` で同連番に揃えると管理が楽

### 検収期限
- 納品から **7日以内** に検収依頼を出す
- 7日経過しても返答がない → 「みなし検収」のメールを送付（後日トラブル予防）

```
件名：【みなし検収のご確認】[案件名]｜DLV-[番号]

納品から7日が経過しましたが、検収のご連絡を頂戴しておりません。
契約書に基づき、本日をもって検収完了とみなさせていただきます。

修正のご希望があれば、48時間以内にご連絡ください。
```

### 保存
- 納品書・確認書を `CFO/outputs/deliveries/YYYY-MM/` に保存
- 納品物自体は `クライアント別フォルダ` で管理（**gitにコミット禁止**）
