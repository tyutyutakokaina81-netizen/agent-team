# フリーランス時給・案件採算カリキュレーター

依存ゼロの単一ファイル Web ツール。収入・経費・作業時間を入れると「実質時給」「案件別の採算」「目標月収に必要な単価」の**目安**を計算します。外部CDN・API・サーバー送信なし。オフラインでも動きます。

- ファイル：`index.html`（HTML+CSS+JS 全部入り）
- 保存：入力内容はブラウザの `localStorage` にのみ保存（外部送信しない）
- グラフ：`<canvas>` で自前描画（ライブラリ不使用）
- 対応：スマホ／PC、日本語UI

---

## 使い方

1. `index.html` をブラウザで開く（ダブルクリックでOK）。
2. **1. ひと月の収支と稼働**：月の売上・経費・請求対象時間・請求できない時間を入力。
3. **2. 実質時給**：自動で次の目安が出ます。
   - 実質時給（請求時間ベース）＝（売上−経費）÷ 請求対象時間
   - 実質時給（総稼働ベース）＝（売上−経費）÷（請求時間＋請求できない時間）
   - 手取りに近い額（税・社会保険料の**前**）／稼働率
4. **3. 目標月収**：目標売上を入れると、いまの経費・稼働のままで必要な実質時給と現状との差が出ます。
5. **4. 案件ごとの採算**：案件を追加し、報酬と時間を入力。時給と判定（良／並／注意）と棒グラフが出ます。判定基準は「実質時給（請求時間ベース）」。基準が無いときは仮に 2,000 円/時。

入力は自動保存され、次回開いたときに復元されます。「入力をリセット」で消去できます。

> ⚠️ 金額はすべて**税・社会保険料を引く前の目安**です。税額の断定はしません。正確な計算は確定申告の手引きや税理士をご確認ください。

## GitHub Pages での公開手順

このツールは静的 HTML 1枚なので、Pages にそのまま置けます。

### 方法A：リポジトリ直下に置く場合
1. `index.html` をリポジトリの公開対象パスに置く。
2. GitHub → リポジトリ → **Settings → Pages**。
3. **Build and deployment** → Source を **Deploy from a branch**。
4. Branch を `main`（公開したいフォルダがある場合は `/docs` など）に設定して Save。
5. 数十秒〜数分で `https://<ユーザー名>.github.io/<リポジトリ名>/` に公開。

### 方法B：このサブフォルダごと公開する場合
本ファイルは `apps/freelance-rate-calculator/index.html` にあります。サブパスで公開したいなら、上記の Pages 設定後に
`https://<ユーザー名>.github.io/<リポジトリ名>/apps/freelance-rate-calculator/`
でアクセスできます（Pages がリポジトリ全体を配信する設定の場合）。

> 既存の `pages.yml` は `tavern.html` のみをデプロイする構成です。この計算機を一緒に公開したい場合は、ワークフローの `_site` コピー対象にこのフォルダを追加するか、別リポジトリ／別 Pages として切り出してください（運用判断はオーナー）。

## 収益化メモ（控えめ・正直に）

ダークパターンは使いません。**無料ツール単体で完結**します。

- **(a) 商品導線**：末尾の `【商品リンク】` プレースホルダを、当社「フリーランス収支管理スプレッドシート（Vol.1）／テンプレ集」の販売URLに差し替える。本ツールが扱う「案件別時給」を、Vol.1 は**月次推移・経費の科目分け・年間サマリー（確定申告の下準備）**まで自動化する上位互換 → 自然な相互送客になります。
- **(b) 広告枠**：HTML内に `<!-- ad slot -->` を1箇所だけ用意。将来 広告を入れる場合のみここに差し込む。デフォルトは非表示で、UXを損ないません。

### 差し替え箇所（index.html 内）
- 商品リンク：`href="【商品リンク】"` を実URLに。
- 広告枠：`<!-- ad slot -->` の位置に必要時のみ広告タグを追加。

## 制約への適合

- A2/A3：画像生成なし（CSS/canvasのみ・サムネ不要）。
- A4：案件名は自由入力だが「個人名・住所などの個人情報は入れないでください」と明示。ツール内に PII を持たない。
- A5：誇張表現なし。税の断定をせず「目安」と明記。

## 動作確認（自己テスト）

主要な純粋関数（`compute` / `projRate` / `verdict` / `benchmark`）は `window.__frlc` に公開しており、ブラウザのコンソールで検証できます。検算例：

- 売上40万・経費6万・請求120h・その他40h・目標50万
  - 経費後 = 340,000 円／実質時給(請求) = 2,833 円/時／実質時給(総稼働) = 2,125 円/時／稼働率 = 75.0%／必要実質時給 = 3,667 円/時／差 = +833 円/時
- ゼロ除算（時間0）は `null` を返し表示は「—」。クラッシュしません。
