  owner の最小労力（管理画面の数字をコピペ）で測る。**数値が見えないと全施策が"仮説の在庫"のまま**（A1：code は閲覧数を見られない）。

## ステップ0：いつ測るか
- 公開（または編集）から **1〜2週間後**。note の閲覧は数日〜数週でじわじわ積むため、即日は見ない。

## ステップ1：どの数字を拾うか（記事ごと・管理画面の「ダッシュボード」から）
各記事で次の4つだけ。スキ率は計算してもしなくてもOK（code側で出せます）。
- **ビュー（閲覧数）** / **スキ数** / **コメント数** / （あれば）**流入元**（検索・おすすめ・フォロー等の内訳）

## ステップ3：判定の目安（code が分析します）
- **スキ率が上がった** → 写真＋締めが効いている。仕事系で効いてなければ写真を外す。
- **検索流入が増えた／ビューが伸びた** → タイトルKWが効いている。残り記事も同型に寄せる。
- **変化なし／下がった** → その施策は撤回 or 別案。**効かないものは正直にやめる**（A5の精神）。

1. 上の **比較A・B のどちらか1つ**でいいので、対象記事の数字を1〜2週後にこの表へ貼る（または code にコピペで渡す）。
