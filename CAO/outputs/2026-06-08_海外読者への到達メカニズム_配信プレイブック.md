種別: 戦略（outputs）／担当: CAO主導・CMO/cowork連携／North Star直結（海外の人に読んでもらう）

> 班1の継続。既存の `2026-06-07_高岡ドラえもん_海外発見経路の深掘りと戦略.md` は「**なぜ／どの角度で**刺さるか（需要源・検索意図・記事角度）」を解明した。
> 本書はその次＝「**良い記事が、物理的にどうやって海外の読者に届くか**」という**到達の仕組み**を定義する。事実と仮説を分けて記す。

## 0. 問題の核（なぜ「到達」を独立テーマにするか）

【事実】成果物は ~106本あるが公開済は **3本のみ**（STATE.md）。仮に全部公開しても —
【事実】**note.com は海外向けの自然検索（Google英語）に弱い**：日本語ドメイン優位・英語被リンク薄・記事個別の英語SEO制御が限定的（メタ/URL/hreflang を細かく設定できない）。
【考察】つまり「英語で良い記事を書いて note に公開する」だけでは、海外読者の検索結果にはほぼ出ない。**到達は"待ち（SEO頼み）"では成立しない＝能動的に押し出す配信設計が必須**。
【結論】到達メカニズム＝**Push（こちらから差し込む）→ Pull（検索で見つかる）への橋渡し**を、チャネル別に仕組み化する。

## 1. 到達ファネル（全体像）

```
[制作: code/CMO]         [配信: cowork]              [着地: note]            [回収: CAO]
英語ファースト記事  →  ①検索面に置く(Pull種まき) →  note記事(ハブ)    →  海外PV/国別/流入元
  + 入口語/フック        ②コミュニティに差す(Push) →  CTA(じゃらん/有料) →  勝ち角度を二重化
  + 写真スポット         ③SNS映えで拡散(Viral)
```

到達は3経路の合算で作る（単一チャネル依存にしない）:

| 経路 | 性質 | 主戦場 | 立ち上がり | 持続性 |
|---|---|---|---|---|
| **A. Pull（検索）** | 中長期・複利 | Google英語 / Pinterest検索 / YouTube検索 | 遅い（数週〜） | 高い（資産化） |
| **B. Push（コミュニティ）** | 即効・スパイク | Reddit / ファン圏(越/印/中/西) / FBグループ | 速い（即日） | 低い（流れる） |
| **C. Viral（映え）** | 不定・爆発 | Instagram / TikTok / Pinterest | 不定 | 中（当たれば資産） |

## 2. チャネル別・到達メカニズム（具体）

### A. Pull＝検索面に「note以外の入口」を置く（最重要・note弱点の補完）

note単体が検索で弱いなら、**検索に強い面に"入口ページ"を置き、そこからnoteへ送る**（ハブ&スポークの逆流防止）。

- **A-1. Pinterest（最優先・低コスト高効果）**【仮説・要検証】
  - Pinterestは**画像×英語検索エンジン**として機能し、旅行系の海外女性層に強い。ピン1枚=1記事の縦長画像＋英語タイトル＋note URL。
  - "Doraemon hometown Japan" "hidden Japan day trip from Kanazawa" 等の入口語をピンのタイトル/説明に。**写真風サムネがそのまま検索資産になる**＝サムネ必須ルールと直結。
  - 実行: cowork がボード作成→記事ごとにピン投入（サムネ＋URL＋英語説明）。
- **A-2. YouTube/Shorts の検索面**【仮説】
  - "Doraemon Tram Takaoka" 等は動画検索需要あり。静止画スライド+字幕の短尺でも検索に乗りうる。素材=オーナー写真＋英語キャプション。**当面は優先度中**（制作コスト高）。
- **A-3. note記事自体の英語SEO最低限**【事実+実行】
  - できる範囲: 英語タイトル/英語サブタイトル/英語ハッシュタグ統一/1行目に入口語/URLスラッグは変更不可でも見出しに入口語。
  - 個別の被リンクを Reddit/Pinterest から張ることで、note記事の英語評価を底上げ（B/CがAを育てる相互作用）。

### B. Push＝ファン圏・コミュニティに「自然に」差す（即効・最重要の立ち上げ）

【事実】ドラえもん需要の主役は**ベトナム・インド・中国・スペイン語圏**（発見経路ドキュメント）。ここに**英語＋現地語の入口**で差すのが最速。

- **B-1. Reddit**【仮説・要マナー厳守】
  - r/JapanTravel・r/JapanTravelTips（"hidden gem / Kanazawa day trip"文脈）、r/Doraemon（巡礼文脈）。
  - **スパム厳禁**：宣伝投稿でなく「住人の一次観察」を共有する体裁（"I live where Doraemon's creator grew up — foreigners line up to photograph the statues"）。本文で価値提供→noteは出典として末尾に。
  - 実行: cowork。1コミュニティ1回・即連投しない。コメントで質問に答える運用。
- **B-2. ファンコミュニティ（越/印/中/西）**【仮説】
  - ベトナムのドラえもんファンはSNS/ファンクラブで組織的（発見経路ドキュメント）。**現地語の一言＋英語記事リンク**でseed。
  - "巡礼"議論は薄い＝**先行者になれる空席**。だが外部者の投下は警戒される→**現地語フックは慎重に・コミュニティ規範を尊重**（cowork判断、不確実なら保留）。
- **B-3. Facebookグループ / 旅行Q&A**【仮説】
  - "Japan Travel Planning" 系大型グループ、Kanazawa/北陸の旅行相談スレに「金沢2日目の代替」として回答の形で差す。

### C. Viral＝SNS映えを「最初から記事に仕込む」（制作と一体）

【事実・一次観察】おとぎの森公園のドラえもん像に**外国人が撮影の順番待ち**＝行動の核は写真・SNS。

- 記事制作時点で **(1)ベストアングル明記 (2)5キャラが揃う広場という"映え"の言語化 (3)Instagram/TikTokハッシュタグ候補** をセットで入れる（CMO担当）。
- → 読者自身が撮ってSNSに上げる＝**読者が次の到達経路を生む**（UGCループ）。
- 実行: cowork が Instagram/TikTok にサムネ＋短文＋入口タグで点火。

## 3. ハブ&スポーク（到達の基本構造）

```
        [Reddit]   [Pinterest]   [ファン圏]   [IG/TikTok]   ← スポーク(Push/Pull種)
            \          |             |            /
             \         |             |           /
                ┌───────────────────────────┐
                │   note記事（ハブ＝本文の家）  │  ← 価値の本体・CTA・アフィリ
                └───────────────────────────┘
                              │
                    [じゃらん / 有料note / Klook等]  ← 収益(Phase2)
```

- **ハブ=note記事**（価値の本体・回遊・収益CTA）。
- **スポーク=各SNS/コミュニティ**（到達の入口・noteへ送客）。
- 鉄則: **スポークでは価値を出し切らない**（フックと一次観察まで）→続きと実用情報をnoteで。ただし**露骨な誘導は嫌われる**ので、スポーク単体でも満足できる情報量を保ちつつ「もっと詳しくは」でnoteへ。

## 4. 1記事あたりの到達オペレーション（テンプレ化＝再利用資産）

公開する各英語記事に、以下を**セットで回す**（cowork実行・code発行）:

| # | 工程 | 担当 | 中身 |
|---|---|---|---|
| 1 | note公開＋サムネ | cowork | 写真風サムネ必須。英語タイトル/サブ/タグ |
| 2 | Pinterest ピン | cowork | 縦長サムネ＋英語タイトル＋入口語＋note URL |
| 3 | Reddit 1投稿 | cowork | 一次観察ベース・価値提供・出典でnote |
| 4 | ファン圏 seed | cowork | 該当記事のみ・現地語フック慎重に |
| 5 | IG/TikTok | cowork | 映えサムネ＋入口タグ |
| 6 | 計測登録 | CAO | 記事を計測テンプレに追加（KPI欄） |

- **code側の事前納品物**（各記事に同梱）: 英語本文／Pinterest説明文／Redditタイトル&本文案／IGキャプション&ハッシュタグ／入口語リスト。**＝配信パッケージ**。これを `ops/inbox` で cowork に渡す。
- 1日全記事にフルセットは過負荷→**リード記事（Skip Shirakawa-go等）から優先**して厚く回す。

## 5. シーケンス（立ち上げ2週間・現実的な順序）

【前提】公開が3本で止まっている＝まず「ハブを建てる」が先。

- **Week 1（ハブ建設）**: ドラえもん英語ファースト5本を公開（サムネ付）。各記事に配信パッケージを同梱。Pinterestボード開設＋5ピン。
- **Week 1後半（Push点火）**: リード記事を Reddit に1本（一次観察フック）。反応を見て2本目。
- **Week 2（拡張）**: 反応が出た角度のファン圏seed＋IG/TikTok。Pinterestピンを既存英語記事へ横展開（backfill 31本ぶん）。
- **常時**: code は英語backfill（食・観光）を進め、ハブの数を増やす（スポークが効く面積を広げる）。

## 6. KPI＝到達が起きているかの判定（CAO・公開2週後から）

既存 `CAO/outputs/2026-06-07_海外読者KPI集計テンプレ.md` に**流入元（リファラ）軸**を追加して回す。

| 指標 | 何を見るか | 合格ライン（初期仮）|
|---|---|---|
| 海外PV比率 | note解析の国別 | 海外 >20% で「届き始め」 |
| 流入元内訳 | Pinterest / Reddit / 検索 / SNS | 単一依存でない（2経路以上が機能）|
| 入口語検索 | "Doraemon" 系検索流入 | 記事別に発生していること |
| 国別 | 越/印/中/西の出現 | 想定需要源と一致するか |

## 7. リスク・禁止事項（到達を壊さないため）

- ❌ **スパム投稿**（同一リンク連投・コミュニティ規範無視）→ BAN＝到達経路を失う。1コミュニティ1回・価値提供優先。
- ❌ **現地語の不自然な投下**（機械翻訳丸出し・外部者の宣伝臭）→ 反感。不確実なら保留。
- ❌ **誇張**（日本語A5は継続）。英語は travel writing 新方針内の定型フレームのみ可（"This is the Japan most tourists never see" 等）。
- ❌ **PII**（地区名・住所）はスポーク文面にも入れない。
