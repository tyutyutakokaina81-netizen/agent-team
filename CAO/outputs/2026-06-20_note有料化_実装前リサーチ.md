## リサーチ③ : オーナーのテンプレ販売データ（Mac で確認）

**目的**: 「テンプレ ¥980 が実際に何部売れたか」から、記事有料化の成功率を推測

### チェック対象

- [ ] **note の「売上」ページ** を開く (https://note.com)
  - [ ] テンプレ Vol.1（フリーランス収支管理スプレッドシート ¥980）の売上数を確認
    - 実売: ___ 部
    - 月間: 約 ___ 部 / 月
    - 累計売上: 約 ¥___
  - [ ] BOOTH 販売分もあれば合算

- [ ] **何が売れたのかの分析**:
  - [ ] テンプレは「見た人の何%が買ったか」（CVR）を推定
    - note の「テンプレ記事」の PV を推定（公開日・コメント数から）
    - 売上数 ÷ PV = CVR（変換率）
    - 例: 1000PV で 30 部売れた → 3% CVR

### 記録

```
テンプレ Vol.1 売上:
- 累計: ___ 部 × ¥980 = ¥___
- 月間: 約 ___ 部 / 月 → 月 ¥___
- 推定 CVR: ___ ％

判断: 「テンプレで _% CVR が実証されたので、有料記事も同程度の CVR が期待できる」
```

### 推論

```
仮説: 有料記事 ¥300-500 の場合も、テンプレと同等の「買いやすさ」であれば
      テンプレと同等の CVR を期待できる。

例: 
  - 有料記事 1 本の PV: 500-1000 想定
  - CVR: 3-5% 想定
  - 売上: 500PV × 3% × ¥400 = ¥6K / 記事
  - 10 本有料化: ¥60K / 月
```

## リサーチ⑤ : 価格設定の妥当性（リサーチ② の事例から）

**目的**: 「¥300-500 の単品有料記事」「月 ¥500 のメンバーシップ」「¥1980 マガジン」が適正価格か判定

### チェック項目

- [ ] **競合事例から見た価格相場**:
  - 有料記事の平均価格: ¥___ （安い ¥100-300 / 標準 ¥300-500 / 高い ¥500-1000）
  - メンバーシップの月額: ¥___ / 月 （相場: ¥300-1000）
  - マガジンの初回: ¥___ （相場: ¥500-3000）

- [ ] **「オーナーの記事クオリティ」との比較**:
  - オーナーのコンテンツは「相場と比べて」[上]か[中]か[下]か
  - → 相場より安くするか、相場並みにするか判定

### 判定フロー

```
if オーナー記事クオリティ == "上" and 読者エンゲージ == "高い":
  → 価格を「相場より+¥100」に設定（¥400→¥500 など）
elif オーナー記事クオリティ == "中":
  → 相場並みに設定（¥300-500）
else:
  → 最初は低目（¥300）で、反応見てから値上げ
```
