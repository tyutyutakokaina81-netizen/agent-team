## シート 1「ハイライト」（オーナー向け一画面）

### レイアウト（A4 横サイズ・印刷可能）

```
[行 1〜2] 当月の KPI 5 つ（巨大フォント）
   月収        ¥XXX,XXX  目標 ¥300K （X%）
   純利益       ¥XXX,XXX  目標 ¥250K （X%）
   受注率        X%       目標 5%
   LTV          ¥XX,XXX  目標 ¥30K
   1 投稿あたりインプ X    目標 1,000

[行 4〜5] 月次推移（Vol.1〜M3 のグラフ）
   棒グラフ：月別売上（4 ヶ月間）
   折れ線：純利益率

[行 7〜8] 柱別の進捗バー
   柱 A: ████░░░░░░ 40%
   柱 B: ████████░░ 80%
   柱 C: ██░░░░░░░░ 20%
   柱 D: ░░░░░░░░░░ 0%

[行 10] 今週のアラート（赤背景・該当時のみ）
```

### 主要数式
```excel
月収          = SUMIFS(日次!売上, 日次!日付, ">="&月初, 日次!日付, "<="&月末)
純利益        = 月収 - 5800 - SUMIFS(日次!手数料, 日次!日付, ...)
受注率        = COUNTIFS(...受注) / COUNTIFS(...提案)
LTV           = SUMIF(日次!売上, 日次!顧客ID=対象顧客) ／ COUNT(対象顧客)
進捗率（柱A） = 柱A月累計 / 300000 (300K)
```

## シート 2「日次データ」（生データ入力）

### 列定義

| 列 | データ型 | 内容 | 入力方法 |
|----|---------|------|---------|
| A | 日付 | 取引日 | 手動 |
| B | 顧客 ID | 識別 ID（個人情報除外） | 手動 |
| C | 柱 | A / B / C / D | ドロップダウン |
| D | プラットフォーム | ココナラ / CW / note / 直接 | ドロップダウン |
| E | 売上 | 円 | 手動 |
| F | 手数料率 | % | プラットフォームから VLOOKUP |
| G | 手数料額 | =E*F | 自動 |
| H | 純売上 | =E-G | 自動 |
| I | 入金日 | | 手動 |
| J | ステータス | 請求済 / 入金済 | ドロップダウン |
| K | 案件名（識別用） | 短縮文字列 | 手動 |
| L | 関連動画 / SNS | 投稿 ID | 手動（任意） |

### 行数
- 1 行目：ヘッダー
- 2 行目以降：1 取引 = 1 行
- 想定：月 50〜200 行（半年で 1,000 行程度）

### データ検証
- C 列（柱）：ドロップダウン「A, B, C, D」
- D 列（プラットフォーム）：ドロップダウン「ココナラ, クラウドワークス, note, BOOTH, 直接」
- J 列（ステータス）：ドロップダウン「未請求, 請求済, 入金済, 督促中」

## シート 5「アラート」

### 自動判定ロジック

```excel
=IF(月収累計<月予算*0.5*月進捗率, "🔴 月収進捗遅延", "")
=IF(受注率<0.02, "🔴 受注率異常", "")
=IF(平均インプ<500, "🟡 SNS リーチ低下", "")
=IF(連続売上ゼロ週>=2, "🔴 売上停滞", "")
=IF(純利益率<0.6, "🟡 手数料負担過多", "")
```

### 表示
- 🔴 = 即時アクション
- 🟡 = 要観察
- 🟢 = 健全
