## 2. V2の構造（1本道）

```
[集客] 英語SEO（code自動）＋ note毎日5本（既存ワーカー）＋ IndexNow（自動）
   ↓
[受け] 英語サイト50ページ（Plan your trip 導線・受け皿実装済み）
   ↓
[収益] 宿アフィリ（Booking等・成果報酬・¥0）… affiliates.js の LINKS.stay にIDを入れるだけで
       49ページ同時に有効化。以後の最適化（配置/文言/ページ追加）は code が単独で回す
```

- **単価**: 宿予約1件 ≒ 数百〜千円強（予約額の約4%）。分岐点¥5,800 ≒ **月5〜15予約**。
- **正直な時間軸**: 英語SEOの複利は3〜6ヶ月かけて立ち上がる。初予約は「数週間〜数ヶ月内のどこか」。早くはないが、**owner工数ゼロで単調増加**する唯一の線。

## 3. ownerにお願いする「月20分」の中身（今月分・1回だけ）

**Travelpayouts（Booking.com等を扱う無料アフィリネットワーク）に登録し、アフィリIDを code に渡す。**
- 登録無料・審査ゆるめ・¥0。所要15〜20分。
- IDの渡し方：この会話に貼る（数字/短い文字列。секретではないが、リポジトリには code が適切な形で埋める）。
- それを受けて code が `affiliates.js` の `LINKS.stay` を埋めて main へ反映 → **その日から49ページ全部が収益ページ化**。以後の改善は全部 code がやる。

（余力があればA8.netも登録→日本語note用。無ければ不要。）

## 4. code が自動で回し続けるもの（owner関与ゼロ）

1. **世界ウケ英語ページの増設**（パイプライン15題材→en-*.html化・sitemap・IndexNow）＝検索面積の拡大
2. **CVR改善**：宿導線の配置・文言のA/B（2週間サイクルで白黒）
3. **note→サイト送客の強化**：新規記事の英語要約からenページへのリンクを標準化
4. 週次の数字レビュー（Pages のトラフィックは Search Console 登録があれば見える→無ければ予約数=Travelpayouts管理画面が唯一の真実。CFO様式で記録）
5. 日次self-check（outbox監視・止まったら通知）

## 7. リスクと正直な注記

- **最大リスク＝今回も蛇口が開かないこと**。V2がowner に求めるのは過去のような毎日の作業ではなく、**1回20分だけ**。これで開かなければ、¥0制約下で収益化する経路は事実上残らない（広告出稿等は¥0制約に反する）。
- SEO流入は保証できない（検索順位は外部要因）。ただしコストゼロなので**下振れしても損失ゼロ**＝赤字にならない設計は維持。
- 有料note・CTA等の note 側収益は「ワーカーが動いた日」に自動で乗る上積みとして残す（捨てない・追わない）。
