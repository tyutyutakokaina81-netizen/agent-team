# 赤字にならない収益化プラン（¥0固定費・成果報酬のみ）

- **作成日**: 2026-07-01 ／ **担当**: CAO（code）／関与: CFO(損益)・CMO(チャネル)
- **依頼**: owner「赤字にならないように収益化して」
- **前提**: A1（code は note 投稿/閲覧数取得は不可＝実装は稼働ワーカー/owner）／North Star＝海外読者に高岡・氷見・富山。

---

## 0. 結論（要点）

**「赤字にならない」は設計で保証する。** 使うのは**初期費用ゼロ・月額固定費ゼロ・成果報酬（売れた/予約された分だけ）**のチャネルだけ。
広告出稿・有料ツール・在庫仕入れは**一切使わない**。→ **収益化そのものからは赤字が構造的に発生しえない。**

- 唯一の固定費は既存の運営費 **月¥5,800**（既発生）。**損益分岐＝月の収益が¥5,800を超えた時点で黒字**。
- 既に資産は揃っている（公開334本＋新規13本＋英語SEO50本＋伸ばす仕込み全部）。**足すのは「リンク」だけ＝コストゼロ。**

---

## 1. 大原則（これを守れば絶対に赤字にならない）

1. **固定費を増やさない**：月額課金の有料API・ツール・広告は禁止（¥0厳守＝A制約と同じ）。
2. **成果報酬型だけ**：登録無料・売れた時だけ手数料が引かれる仕組みのみ（アフィリ／note決済／BOOTH）。
3. **在庫を持たない**：物販は自分で仕入れない（アフィリでメーカー/店の在庫を紹介するだけ）。
4. **読者体験を壊さない**：押し売り・記事と無関係な広告はしない（A5）。北陸の記事に、北陸に関係するものだけを、記事の流れで自然に。剥がしてもOK。

> この4つを守る限り、**収益がゼロでも損失はゼロ**（既存¥5,800以外に新規支出がない）。だから「赤字」になりようがない。

---

## 2. 損益分岐（CFO・月¥5,800をどう超えるか）

運営費 **月¥5,800** を「これだけ稼げば赤字ゼロ」の基準にする。ゼロ設定費チャネルの組み合わせで到達する。

| チャネル | 1件あたり目安 | ¥5,800到達に必要な月間成果（単独の場合） | 設定費 |
|---|---|---|---|
| 旅行アフィリ（宿予約・楽天トラベル/じゃらん/Booking等） | 予約額の1〜4% ≒ ¥100〜400/件 | 15〜40予約 | ¥0 |
| 物販アフィリ（Amazon/楽天/もしも＝富山の食お取り寄せ） | 販売額の2〜4% ≒ ¥30〜200/件 | 30〜150件 | ¥0 |
| note有料記事/マガジン | ¥100〜1,000/本 | 6〜60本 | ¥0（noteが決済） |
| テンプレ販売（既存Vol.1・BOOTH/note） | ¥500〜10,000 | 1〜12本 | ¥0 |
| note メンバーシップ（月額サブスク） | ¥300〜500/人 | 12〜20人 | ¥0 |

> **単独で到達させる必要はない。** 複数を薄く重ねれば、閲覧が伸びるほど自動で分岐点を越える。**baseline PV が出れば、必要CVR（0.x%）を逆算して現実性を判定**（下記4）。

## 3. 実行順（ROI × リスクゼロ順）

### フェーズ1：即・実装できる ¥0リスク（＝今すぐでも赤字にならない・"販売"でなく"リンク付与"）
「あと5,000ビュー伸ばしてから販売」方針と両立する。**これは能動的な販売プッシュではなく、既存記事に関連リンクを1つ添えるだけ**なので、閲覧増を待たずに置ける（置いても赤字にならず、読者体験も損なわない範囲で）。

1. **旅行アフィリを地域記事の末尾へ**（最有力）。高岡・氷見・富山の宿/体験を1リンク。
   - 海外読者の英語ページ（`apps/toyama-guide/en-*.html`）→ **Booking.com / Agoda**（海外OTA・海外予約に強い）。既に「Plan your trip」動線あり。
   - 日本語 note の食・場所記事 → **楽天トラベル / じゃらん（A8）**。
2. **物販アフィリを"食"記事の末尾へ**。氷見の干物・ますの寿司・白えびせんべい・氷見牛・地酒 等は**お取り寄せ需要**が明確 → Amazon/楽天/もしもで「お取り寄せするなら」を1リンク。
3. **ステマ規制対応のPR表記を全アフィリ記事に付ける**（下記5・必須）。

### フェーズ2：+5,000ビュー到達後に本格化（能動的"販売"）
4. **note有料マガジン**：無料334本で信頼→「富山旅行 完全ガイド（保存版）」等を¥300〜500の有料に。設定¥0。
5. **メンバーシップ**：月額でローカル情報/限定記事。ファン化した層に。
6. **テンプレVol.2**：Vol.1実績を踏まえ拡充。

## 4. 次の一手（baseline 依存・A1でcodeは取得不可）

- **必要データ＝現状の月間PV**（稼働ワーカーが note API 取得中）。
- 取得できたら：`必要CV数 ÷ 月間PV = 必要CVR` を出し、**現実的（一般にアフィリCVR 0.1〜1%）か判定**。例）月間5,000PV・記事末リンククリック3%・予約CVR2% → 予約 3件/月 ≒ ¥600〜1,200。**まだ¥5,800に届かない → 閲覧増（+5,000）が効く**という関係が数字で見える。
- **だから「収益化」と「+5,000ビュー」は矛盾しない**：リンクは今ゼロリスクで置き、**分岐点超えは閲覧増が押し上げる**。

---

## 5. 法令・規約（赤字回避と同じくらい重要＝アカウント凍結は最大の損失）

- **ステマ規制（2023.10〜景表法）**：アフィリ・PRは**必ず明示**（下記CMOアセットのPR表記を全該当記事に）。
- **各ASPの規約遵守**：Amazonアソシエイトの価格記載ルール等。**虚偽・誇張НГ（A5と一致）**。
- **note規約**：外部リンク/アフィリの可否は最新規約を確認（一般に本文リンクは可、露骨な誘導は避ける）。
- **景品表示法・薬機法**：食品で効能を断定しない（「体にいい」等の断定回避）。

## 付記：なぜ「絶対に赤字にならない」と言い切れるか
新規に発生する支出が**ゼロ**（登録無料・固定費なし・在庫なし・広告なし）だから。収益がどれだけ小さくても、増えるのは収入側だけ。**最悪ケース＝収益¥0でも、損失は既存¥5,800のまま増えない。** これが「赤字にならない収益化」の実体。
