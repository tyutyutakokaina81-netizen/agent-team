# note閲覧数を伸ばす 実行プラン（CAO・2026-06-20）

> 依頼：「閲覧数のばしたい」。実データ（note公開395本／ユニーク321／重複74／英語併記の旅記事100本超）を踏まえ、
> **ROIの高い順**に並べた。各レバーに「なぜ効くか」「具体アクション（オーナーのMac）」を付す。
> 大前提：記事は **もう十分多い（321本）**。問題は量ではなく **発見されやすさ（CTR）・回遊・外部流入・重複による評価分散**。

---

## 要点（結論）

伸ばす順番はこれ：
1. **重複74本を下書きに戻す**（=評価の再集中）… いちばん速い回復。掃除＝成長。
2. **サムネを付ける**（特にkeep側321本の無サムネ）… note内CTRの最大レバー。
3. **Xクロスポスト再開**… 外部流入の最大源。1記事=本文1段落＋リンク。
4. **マガジン化**（321本をテーマ別に束ねる）… 回遊（PV/セッション）＋SEO＋共有ハブ。
5. **英語記事の海外導線**（既存100本超を活かす）… 競合の薄い海外検索を取りに行く。
6. **公開再開**（掃除後・Mac版安全ゲート）… 新規初速で底上げ。

## 1. 重複74本を下書きに戻す（最優先・最速）

【事実】同一タイトルが最大5本公開（かぶら寿司×5、おとぎの森×4、AIと働く1日×4 等）。74本が重複。
【考察】重複URLは **検索・おすすめの評価を分散**させ、1本あたりのPVを削る。スパム的に見えてアカウント評価も下げうる。
　→ 74本を1本に集約するだけで、残った正規記事へ評価が再集中し **PVが戻る**。掃除は成長施策そのもの。
【アクション（Mac）】
```
python3 CDO/outputs/note_publisher/unpublish_notes.py                 # まず対象確認（dry-run）
python3 CDO/outputs/note_publisher/unpublish_notes.py --execute --limit 1   # 1本で動作確認
python3 CDO/outputs/note_publisher/unpublish_notes.py --execute             # 残り全部
```
※削除ではなく「下書きに戻す」＝復元可能。どれを残すかは dedup_unpublish_list.tsv の keep 行。

## 2. サムネを付ける（note内CTRの最大レバー）

【事実】多くの公開記事がサムネ未設定（6/10公開分など）。
【考察】タイムライン／おすすめは **サムネのクリック率**で表示回数あたりのPVが決まる。文字なし写真1枚が最強（既存戦略記事）。
【アクション（Mac）】`generate_thumbnails.py`（キー不要フォールバックあり）で不足分を生成 → note編集画面で設定。
　優先順＝「直近公開＋keepした正規記事」から。実写が用意できるものは実写優先。

## 3. Xクロスポスト再開（外部流入の最大源）

【事実】X連携は停止中。クロスポスト素材は生成済みだが投稿されていない。
【考察】noteは外部流入（特にX）で初速が変わる。「note更新しました」ではなく **本文の刺さる1段落＋「続きはnoteに」＋リンク**。
【アクション】keepした強い記事から毎日1〜2本、JP＋ENで投下。素材が無い分は CDO/outputs/cross_post/ で生成可。
　※公開URLには note のユーザー名が必要（note.com/＜ユーザー名＞/n/＜id＞）。

## 4. マガジン化（回遊＋SEO＋共有ハブ）

【事実】321本がバラバラに存在。テーマの塊（グルメ／ドラえもん聖地／フリーランス×AI／英語旅）が大きい。
【考察】note マガジンに束ねると **1人が複数記事を読む（PV/セッション増）**、検索のテーマ性が強まる、SNSで「まとめ」を共有できる。
【提案マガジン（既存記事で今すぐ作れる）】
- 「富山・高岡グルメ」：白えび丼／昆布じめ／富山ブラック／鱒寿司／ノドグロ／かぶら寿司 …（食20本超）
- 「ドラえもんの故郷・高岡」：おとぎの森／藤子F不二雄／万葉線トラム／氷見潮風ギャラリー …
- 「住んで書く高岡」：雨晴/瑞龍寺/高岡大仏/金屋町/古城公園 …
- 「ひとり×AIで月30万」：AI5役職／AdAIと働く1日／失敗ログ／有料化 …
- 「Takaoka, Japan (English)」：英語併記の旅記事100本超 → 海外向けハブ
【アクション（Mac）】note管理画面でマガジン作成→各記事を追加（手動）。割り当て表は希望あればCAOが作る。

## 5. 英語記事の海外導線（既存資産の回収）

【事実】note のみ233本の大半が **英語併記の旅記事**（Amaharashi/Zuiryu-ji/Doraemon 等）。既に大量に存在。
【考察】英語で「日本の地方×具体地名」を検索する層は競合が薄い。**作る必要はもう無い。届ければいい。**
【アクション】Reddit（r/JapanTravel 等）／英語ハッシュタグ／Medium転載。EN/ の既存素材を順次投下。

## 6. 公開再開（掃除後・底上げ）

【事実】6/13以降 新規公開ゼロ。
【アクション】1〜5が回り始め、重複掃除＋台帳同期が済んだら、Mac版（安全ゲート）で日次再開。新規の初速がベースを押し上げる。
