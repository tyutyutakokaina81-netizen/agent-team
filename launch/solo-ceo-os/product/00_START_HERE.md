# Solo CEO OS — Start Here

Welcome. You just got the operating system for running a one-person business
like a real company — with a team of AI executives.

This is **not** a pile of prompts. It's a system: roles, memory, hand-offs,
and rules that make six AI officers behave like one company.

## Set up in 3 steps (about 30 minutes)

1. **Create the structure.** In your AI workspace (a folder on your computer,
   a Notion space, or a Claude/ChatGPT project), recreate the folders in
   `03_folder_structure.md`. This gives every officer its own memory.

2. **Install the rulebook.** Copy `01_company_operating_system.md` into the
   root. This is the constitution every officer follows.

3. **Activate an officer.** Open `02_officers/` and paste the role you need
   (e.g. CMO) into your AI. It now thinks and works as that executive.

That's it. Give it a goal — "plan this month's launch" — and the right officer
takes over, does the work, and logs the output.

## What's in this package

| File | What it is |
|------|-----------|
| `01_company_operating_system.md` | The governance rules (reporting, hand-offs, decisions) |
| `02_officers/` | The six ready-to-paste executive prompts |
| `03_folder_structure.md` | The drop-in folder + memory template |
| `04_document_team_prompt.md` | Plan → Draft → Summarize → Review, in one pass |
| `05_grow_your_company.md` | The rule that auto-spawns new departments as you scale |
| `06_quickstart_examples.md` | Real example tasks to run on day one |

## How to actually get value

- Don't activate all six at once. Start with the **one** officer whose work is
  your current bottleneck. Add others as you grow.
- Let each officer **write to its own log** after every task. The memory is
  what makes tomorrow's output better than today's.
- When a task needs two officers, that's a **project** — see the operating
  system file for how they hand off.

You're now the CEO. Let the company run.
