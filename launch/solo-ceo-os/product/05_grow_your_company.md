# Growing Your Company — The Auto-Spawn Rule

Your company should grow new departments by itself as you scale. Here's the
rule that lets the CDO create a new officer without asking you every time.

## When to create a new officer

Create one only when **all** of these are true:

1. A task keeps coming up that fits **none** of the existing six officers.
2. It will clearly **repeat** in the future (not a one-off).
3. You can define the role's name and responsibility in one sentence.
4. When in doubt, **don't** — handle it with an existing officer. The default
   is fewer roles, not more.

## How to create one

1. Pick a name in the `C?O` pattern (e.g. CHO = People, CLO = Legal,
   CXO = Customer Experience).
2. Create its folder with the same structure as the others:
   ```
   NEW_ROLE/
   ├── _log.md
   ├── prompt.md
   ├── research/
   └── outputs/
   ```
3. Write its role prompt using the same shape as the six included officers:
   how it thinks, its workflow, its boundaries, its output style.
4. Add it to your company's org chart and tell yourself it exists.

## Examples of natural next roles

- **CHO (People)** — once you hire contractors or VAs.
- **CLO (Legal)** — once contracts and terms get serious.
- **CXO (Customer Experience)** — once support volume grows.

The point: the system scales with you. You're not buying six roles — you're
buying a company that can grow its own departments on demand.
