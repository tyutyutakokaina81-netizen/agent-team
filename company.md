# 会社共通ルール（Company Rules）

## 1. 報告ルール

- 作業完了時は必ず **要点 → 詳細 → 次アクション** の形式で報告する。
- 事実と推測・意見は明確に区別して記載すること（下記「判断基準」参照）。
- 重要な決定・変更は必ずオーナーに事前確認を取ること。
- 外部への送信（メール、SNS投稿、請求書発行など）は事前承認必須。

## 2. 成果物の保存先

| 役割 | 調査・リサーチ | 最終成果物 |
|------|--------------|-----------|
| CMO（マーケティング） | `CMO/research/` | `CMO/outputs/` |
| CPO（プロダクト） | `CPO/research/` | `CPO/outputs/` |
| CFO（財務・事務） | `CFO/research/` | `CFO/outputs/` |
| CSO（営業） | `CSO/research/` | `CSO/outputs/` |
| CDO（技術） | `CDO/research/` | `CDO/outputs/` |
| 一次情報・インプット | `context/` | — |

- ファイル名には日付をプレフィックスとして付ける（例：`2026-04-08_企画案.md`）。
- 下書きは `research/`、確定版は `outputs/` に保存する。

## 3. 事実と意見を分ける判断基準

### 事実として記載できるもの
- 数値データ（売上、アクセス数、顧客数など）
- 発生した出来事・行動の記録
- 契約・合意の内容
- 引用元が明示されている情報

### 意見・推測として明示するもの
- 分析・解釈（「〜と考えられる」「〜の可能性がある」）
- 提案・アイデア（「〜が効果的ではないか」）
- 予測・見通し（「〜になりそうだ」）

**記載例：**
```
【事実】先月のYouTube登録者数は1,200人増加した。
【考察】コンテンツ投稿頻度を週2回に増やした影響と考えられる。
【提案】来月も同頻度を維持し、効果を検証することを推奨する。
```

## 4. 各役割の担当領域

- **CMO**：YouTube台本、SNS投稿、LP（ランディングページ）作成、マーケティング戦略
- **CPO**：教材スライド作成、セミナー構成、プロダクト改善
- **CFO**：請求書発行、契約書管理、経費管理、事務全般
- **CSO**：顧客との対話記録、営業提案、商談管理
- **CDO**：プロンプト管理、Claude Code活用、技術検証、ツール整備

## 5. contextフォルダの使い方

`context/` フォルダはオーナーの一次情報を格納する場所です。

- 日記・メモ・考え事
- インタビュー記録・音声の文字起こし
- 参考資料・リサーチ素材

各AI社員はタスク実行前に `context/` を参照し、オーナーの意図や背景を把握してから作業を開始すること。
