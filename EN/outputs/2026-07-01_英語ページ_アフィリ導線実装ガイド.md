## 2. そのまま貼れる英語アフィリブロック（Booking / Agoda）

### ルートA（推奨・最短）— `affiliates.js` の `LINKS` を埋める（1回で全ページ）
静的HTMLの追記は不要。`apps/toyama-guide/affiliates.js` の下記だけ置換する。

```js
  var LINKS = {
    stay:  "【LINK:Booking.com(Travelpayouts経由)】", // 宿・ホテル
    tours: "【LINK:Klook/GetYourGuide】",             // 現地ツアー（任意）
    rail:  "【LINK:12Go/JapanRailPass】",             // 鉄道（任意）
    esim:  "【LINK:Airalo eSIM】"                     // eSIM（任意・en-sim-wifi向け）
  };
```

- `stay` だけ埋めれば「宿ボタン」が全英語ページの意図に応じて表示される。tours/rail/esim は空のままなら非表示（無害）。
- rel・disclosure・注入位置は `affiliates.js` が自動処理済み（追加作業不要）。
- **Agoda を JS ルートで併記したい場合**は `affiliates.js` に `stay2` スロットを足す小改修が必要（code が対応可・別途依頼）。当面は Agoda を静的ブロック（ルートB）で優先Aページにだけ足すのが確実。

### ルートB（Agoda 明示・優先Aページに静的追記）
`affiliates.js` の宿ボタン（Booking系）と**併用**し、Agoda を1本足す形。**優先Aの宿ページのみ**（1ページ計1〜2リンクの上限を守る）。
既存の国内aff（じゃらん）が既にあるページでは、**国内aff＋海外affで合計2本まで**に抑える（超える場合は国内affを1本に整理）。

差し込み位置＝**本文中の「宿の話をしている段落の直後」**（既存の `class="aff"` 静的リンクがあるページはその隣）。無ければ本文末（最後の `<p class="tag">` の直前）。

```html
<!-- 海外OTA（Agoda）宿導線 / overseas readers — PR -->
<a class="aff" href="【LINK:Agoda】" target="_blank" rel="sponsored nofollow noopener">Compare hotel prices in Toyama on Agoda (PR) →</a>
```

ページ別に文言を最適化する場合（推奨・A5の"テンプレ連発NG"回避）:

```html
<!-- en-himi.html 用 -->
<a class="aff" href="【LINK:Agoda】" target="_blank" rel="sponsored nofollow noopener">Find seaside inns in Himi on Agoda (PR) →</a>

<!-- en-takaoka.html 用 -->
<a class="aff" href="【LINK:Agoda】" target="_blank" rel="sponsored nofollow noopener">Browse stays in Takaoka on Agoda (PR) →</a>

<!-- en-toyama-city.html 用 -->
<a class="aff" href="【LINK:Agoda】" target="_blank" rel="sponsored nofollow noopener">Compare hotels near Toyama Station on Agoda (PR) →</a>

<!-- en-where-to-stay.html 用（Booking と Agoda を並べて比較させる） -->
<a class="aff" href="【LINK:Booking】" target="_blank" rel="sponsored nofollow noopener">Search Toyama hotels on Booking.com (PR) →</a>
<a class="aff" href="【LINK:Agoda】" target="_blank" rel="sponsored nofollow noopener">Compare the same dates on Agoda (PR) →</a>
```

### Disclosure（PR表記・ステマ規制対応・**必須**）
- **ルートA**: `affiliates.js` が注入する一文でカバー済み（追加不要）。
- **ルートB**（静的ブロックを足したページ）: そのページに disclosure が無ければ、宿ブロックの直後に下記を1つ足す。既存の `<p class="pr">` があるページは重複させない（1ページ1つで十分）。

```html
<p class="pr">Disclosure: this page contains affiliate links (PR). I only suggest places I'd genuinely recommend to a friend, and booking through a link helps keep this guide free at no extra cost to you. Prices and availability change — please check each site for the latest.</p>
```

> `class="pr"` / `class="aff"` のCSSは既存ページ（例 `en-budget.html`）の `<style>` に定義済み。優先Aページは全て既存の英語ガイドテンプレートなので追加CSSは不要。

## 4. 実装手順（owner/ワーカー・A1でcodeは不可）

1. **無料登録（¥0・審査あり）**
   - **Booking.com（＋Klook/12Go/Airalo）**: [Travelpayouts](https://www.travelpayouts.com/) に1社登録で一括発行（海外送金OK・英語）。
   - **Agoda**: Agoda Partners（Agoda Affiliate Program）に登録。または Travelpayouts / ASP経由でAgoda案件があればそれを利用。
   - いずれも**登録・審査は無料**。承認後にリンク発行。
2. **リンク発行 → プレースホルダ置換**
   - ルートA: `apps/toyama-guide/affiliates.js` の `LINKS.stay`（と任意で tours/rail/esim）に発行URLを貼る。**これで全英語ページの海外OTA導線が一斉有効化**。
   - ルートB: §2/§3 の優先Aページの `【LINK:Agoda】` `【LINK:Booking】` を発行URLへ置換。1ページ2本まで厳守。
3. **導線ゼロの2ページを整える**
   - `en-is-toyama-safe.html`: `</body>` 直前に `<script defer src="/agent-team/toyama/affiliates.js"></script>` を追加（他ページと同じ相対）。記事性質上リンクは控えめ（宿ボタンのみ）。
   - `en-sim-wifi.html`: `affiliates.js` は読込済 → `LINKS.esim`（Airalo等）を埋めればeSIMボタンが出る。宿ボタンは不要。
4. **PR表記の確認**
   - ルートAはJSが自動付与。ルートBで静的ブロックを足したページは §2 の disclosure が1つあるか確認（既存 `<p class="pr">` があれば足さない）。
5. **公開・報告**
   - commit & push（`apps/toyama-guide/` 配下）。GitHub Pages（`pages.yml`）で反映。
   - `ops/outbox/` に「どのページに何を付けたか（Booking/Agoda別・本数）」を報告 → code が `CAO/outputs/2026-06-29_note施策_効果検証シート.md` へ効果を追記。

## 5. 注意（赤字回避＝規約順守が最優先）

1. **1ページのアフィリリンクは1〜2本まで。** 既存の国内affと合算して数える（3本以上にしない）。
2. **PR表記なしのアフィリ設置をしない**（ステマ規制違反＝凍結リスク＝最大の損失）。ルートBで静的追加したページは disclosure の有無を必ず確認。
3. **虚偽・誇張・断定NG（A5）**：「最安」「世界唯一」等は書かない。価格・空室は変動する旨（"Prices and availability change"）を残す。
4. **PII禁止（A4）**：具体的な地区名・町名・住所・宿名は書かない（市レベルまで）。リンク先ページはOTA側の検索結果に委ねる。
5. **記事価値が主役。** 導線は本文の役に立つ位置にだけ置く。`en-is-toyama-safe.html`（安全情報）等は押し売りしない。
6. **実アフィリID/リンクはこのMD・PRに書かない**（プレースホルダのみ）。owner管理下でリポジトリの該当HTML/JSにのみ差し込む。
7. **Agoda を JSルートに統合したい場合**は `affiliates.js` の小改修（`stay2` スロット追加）が要る。必要なら code へ依頼（本ガイド外・別タスク）。
