# 案件評価スコアリングツール（2026-04-13）

「この案件、応募すべき？」を**30秒で判定**する採点表。

迷って時間を溶かす代わりに、数字で即判断する。

---

## 🎯 スコアリングの目的

### よくある失敗
- 「この案件やろうかな…でも単価が…」と30分悩む
- 安すぎる案件に応募してしまい時給300円
- 高単価だけを狙って0件応募続き

### スコアリングの効果
- 30秒で「応募すべきか」判定
- 感情を排除した数値判断
- 月収の最大化

---

## 📊 案件評価スコアリング表（10点満点）

各項目を採点して合計。**7点以上なら即応募**。

### 評価項目

| 項目 | 配点 | 採点基準 |
|------|------|---------|
| **1. 時給換算** | 3点 | 計算して判定 |
| **2. 作業内容の明確さ** | 2点 | 明確2 / 普通1 / 不明瞭0 |
| **3. クライアント評価** | 2点 | 5点2 / 4点1 / 3点以下0 |
| **4. 継続性** | 1点 | 継続案件1 / 単発0 |
| **5. 自分のスキル適合** | 1点 | 得意1 / 普通0.5 / 不得意0 |
| **6. 競合の少なさ** | 1点 | 応募5件以下1 / 10件以下0.5 / 多数0 |
| **合計** | **10点** | 7点以上：応募 / 4-6：検討 / 3以下：スキップ |

---

## 🧮 時給換算の計算方法（配点3点）

### 計算式
```
時給 = 報酬 ÷ 想定作業時間
```

### 採点基準
```
時給3,000円以上 → 3点
時給2,000〜2,999円 → 2点
時給1,000〜1,999円 → 1点
時給1,000円未満 → 0点（応募しない）
```

### 業務別の想定時間目安

| 業務 | 想定時間目安 |
|------|------------|
| データ入力 1,000件 | 5〜10時間 |
| リスト作成 100件 | 3〜5時間 |
| SEO記事 3,000字 | 2〜4時間（AI活用で） |
| SEO記事 5,000字 | 4〜6時間 |
| SNS投稿 1回 | 30分〜1時間 |
| コピペ作業 100件 | 1〜3時間 |
| 文字起こし 1分動画 | 10〜15分 |

### 時給UPのコツ
- AI活用前提で時間を短縮（AIで50%時短可能）
- 得意分野に絞る（慣れれば時短）
- 複雑な案件は時間見積もりを多めに

## 📝 採点実例

### 例1：良い案件
```
案件名：継続的なブログ記事執筆（月4本）
報酬：月20,000円（1記事5,000円）
文字数：3,000字/記事
想定時間：4時間/記事
クライアント評価：5.0（依頼10件以上）

【採点】
時給換算：5,000円÷4時間=1,250円 → 1点
作業内容：明確 → 2点
クライアント評価：5.0 → 2点
継続性：継続 → 1点
スキル適合：得意 → 1点
競合：応募3件 → 1点

合計：8点 → ✅ 応募すべき
```

### 例2：微妙な案件
```
案件名：データ入力
報酬：3,000円
件数：500件
クライアント評価：3.5

【採点】
時給換算：3,000円÷5時間=600円 → 0点
作業内容：明確 → 2点
クライアント評価：3.5 → 0点
継続性：単発 → 0点
スキル適合：得意 → 1点
競合：応募20件 → 0点

合計：3点 → ❌ スキップ
```

### 例3：悩む案件
```
案件名：コピペ作業200件
報酬：5,000円
クライアント評価：4.2
想定時間：3時間

【採点】
時給換算：5,000円÷3時間=1,666円 → 1点
作業内容：明確 → 2点
クライアント評価：4.2 → 1点
継続性：継続可能性あり → 0.5点
スキル適合：得意 → 1点
競合：応募8件 → 0.5点

合計：6点 → 🤔 検討（時間に余裕あれば応募）
```

## 🚨 レッドフラグ（即スキップ）

以下のいずれかに該当する案件は、スコアに関係なく**スキップ**。

### ❌ 即スキップ案件
- LINEへの誘導がある
- 「初期費用」「登録料」を要求される
- 「簡単に月100万」など非現実的な収入を謳う
- 作業内容が極めて曖昧
- クライアントが過去の評価で批判されている
- 納期が異常に短い（1時間で3,000字など）
- 修正回数が無制限
- 著作権を「完全譲渡」を要求（単価安い場合）

### 🚩 要注意案件
- クライアント評価が3.5以下
- 応募者が50件以上（競争激しすぎ）
- 報酬が相場の半額以下
- 「テスト的に無料で」と要求される
- 連絡方法が指定されていない

## ⚡ 30秒判定フロー

```
1. 案件ページを開く
2. タイトル・報酬・作業内容を確認（10秒）
3. レッドフラグチェック（5秒）
   → あれば即スキップ
4. 時給換算を計算（10秒）
   → 1,000円未満なら即スキップ
5. 他の評価項目をざっくり採点（5秒）
6. 7点以上なら応募文をコピペ送信

合計：30〜60秒で判定
```

## 📊 応募数と受注数のシミュレーション

### 毎日5件スコアリング → 2〜3件応募する場合

| 日 | スコアリング | 応募数 | 受注（想定受注率10%） |
|----|-----------|--------|--------------------|
| 1 | 5件 | 2 | 0 |
| 7 | 35件 | 14 | 1 |
| 14 | 70件 | 28 | 3 |
| 30 | 150件 | 60 | 6 |

**月に60件応募できれば、6件受注。平均単価5,000円で月3万円の初受注月。**

## 💡 スコアリングを続けるコツ

### 記録を残す
```
スプレッドシートに以下を記録：
- 日付
- 案件名
- スコア
- 応募したか
- 結果（受注/失注/無反応）
```

**→ 1ヶ月続けると「自分にとって受注しやすい案件の特徴」が見える。**

### 週次で振り返る
- スコア8点以上で応募→受注率が実際は何％か
- スコア5-6点で応募した案件の結果
- 傾向を分析して採点基準を微調整

### 採点の厳しさを時期で変える

**最初の1ヶ月：** 採点ゆるく（6点以上で応募）→実績作り優先
**2ヶ月目以降：** 採点厳しく（7点以上で応募）→時給UP
**3ヶ月目以降：** 超厳しく（8点以上で応募）→プレミアム案件のみ

## 🧠 心理的ハック

### 「1分迷ったらスキップ」ルール

応募するか悩む時間 > 応募文を書く時間 になる案件は、時間の無駄。

**1分以上迷ったら、スコアが微妙な証拠。スキップ。**

### 「10件応募ルール」

「応募しようか迷う案件があったら、まず他に条件が明確な9件を応募してから戻る」

**→ 迷う案件は後回しでよい。**

## 💰 スコアリングで変わる月収

### スコアリングなしの場合
- 感情で判断→安い案件ばかり取る
- 月60件応募→月受注6件
- 平均単価2,000円
- **月収：12,000円**

### スコアリングありの場合
- 数値で判断→効率的な案件だけ取る
- 月40件応募（厳選）→月受注6件
- 平均単価5,000円
- **月収：30,000円**

**同じ受注数でも、月収が2.5倍に。**
