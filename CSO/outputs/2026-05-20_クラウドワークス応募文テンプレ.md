# クラウドワークス応募文テンプレ（5パターン）

- 日付：2026-05-20
- 担当：CSO
- 用途：クラウドワークス公募案件への応募提案文

---

## 共通ルール

1. **400字前後**を厳守（長すぎると読まれない／短すぎると熱意が伝わらない）
2. **冒頭2行で「読む価値あり」と判断させる**（案件名固有要素を必ず1つ入れる）
3. **AI言及は中盤以降に控えめに**（案件説明文に「AI使用可」と明記されている場合のみ冒頭でも可）
4. **末尾は必ず質問1個 or 構成案提示の申し出**（返信ハードルを下げる）
5. 案件説明に書かれたキーワードを **2〜3個自然に応募文に織り込む**（読まれた証拠を残す）

## テンプレ①：SEO記事系（汎用）

```
ご担当者様

「[案件タイトルの主要キーワード]」の記事執筆案件、興味深く拝見しました。
SEOライティングを中心に活動しております、○○と申します。

【ご提案できること】
・キーワード分析〜競合上位記事の構成把握〜本文執筆まで一貫対応
・執筆着手前に必ず構成案（H2/H3）をご提示し、方向性合意後に本文へ進みます
・初回ご返信は24時間以内、修正は2回まで無料で承ります

【ご提示単価への対応】
ご提示の文字単価で承ります。初回は実績作りも兼ねて、
通常より丁寧にレビューを行います。

【ご質問】
1点だけ確認させてください。
ターゲット読者は「[案件文面から想定した読者像]」で合っていますでしょうか？
合っていれば、本日中に構成案ドラフトをお送りすることも可能です。

ご検討よろしくお願いいたします。
○○
```

## テンプレ②：継続案件・複数本まとめ発注系

```
ご担当者様

[案件文面の業種/メディア名]の継続執筆案件、拝見しました。
SEOライティングの○○と申します。

継続案件は **トーン・構成パターンを早期に揃える** ことが重要と考えます。
そのため、初回1〜2本は私側の検証も兼ねて以下の体制でお受けします。

【ご提案】
・初回1本目：構成案を3パターンご提示 → ご選択いただいてから執筆
・2本目以降：1本目で確定したフォーマットに沿って効率納品
・週次定例（15分・任意）でテーマ・優先度をすり合わせ

【納品ペース】
・週3本まで安定対応可能
・繁忙期は事前ご相談いただければ週5本まで対応

【ご質問】
過去納品いただいている記事のうち「特に評価が高かった1本」のURL等を
共有いただけますと、トーンを正確に揃えられます。

ご検討よろしくお願いいたします。
```

## テンプレ③：初心者歓迎・実績不問案件向け

```
ご担当者様

「初心者歓迎」とのことで、応募させていただきました。
SEOライティングを始めて間もない○○と申します。

クラウドワークスでの実績はこれからですが、以下の点でご期待にお応えできます。

【お約束できること】
・初回ご返信は24時間以内
・執筆前に必ず構成案をご提示し、方向性合意後に本文着手
・修正は2回まで無料
・納期は厳守（万一遅れる場合は前日までに必ずご連絡）

【サンプル記事】
2,000字程度のSEO記事サンプルを3本ご用意しております。
ご希望のジャンルをお知らせいただければ、リンクをお送りします。

【単価】
ご提示の条件で承ります。
丁寧な仕事で評価をいただき、長くお付き合いしたいと考えております。

【スタートアッププラン（5件限定）のご案内】
ご評価をいただくため、最初の5名様限定で
2,000字の記事を 4,000円（通常 8,000円）でお受けしております。
納品後にご評価コメントをお願いできれば幸いです。
（残り枠数は応募時にご案内します）

ご質問・ご要望ありましたらお気軽にお知らせください。
```

## テンプレ⑤：AI使用可・AI推奨と明記された案件向け

```
ご担当者様

「AI執筆可」とのこと、ご応募させていただきます。
執筆支援ツールを活用しつつ、必ず人間がレビュー・編集を行う方針で
ライティングを行っております、○○と申します。

【ワークフロー】
1. ヒアリング（キーワード・読者像・トーン）
2. 構成案ご提示・合意
3. 執筆支援ツールで初稿生成 → 事実確認・文章調整・固有名詞チェック
4. メタディスクリプション含めて納品

【品質保証】
・コピペチェックツール通過済みを納品基準とします
・事実関係は公的情報・公式サイトを参照しソースを併記可
・修正は2回まで無料

【単価・納期】
ご提示の条件で承ります。

【ご質問】
ツール出力をベースにする旨、記事末尾やメタデータに表記が必要でしょうか？
（不要な場合もそのまま品質基準は維持します）

ご検討よろしくお願いいたします。
```

## 運用上の注意

- テンプレは「枠組み」として使い、**[]部分は必ず案件ごとにカスタム**する
- 同じ文面を連投すると規約違反通報リスクがある
- 1案件1応募。同一発注者の複数案件に同じ文面を出さない
- 返信が来た案件は `CSO/research/対話ログ/` に日付つきで保存
