# 非コンテンツ型 収益モデル 3案 — 構造から違う設計

作成: 2026-06-23 ／ 担当: CEO直轄 新規事業 ／ 場所: `projects/2026-06-23_別モデル/`

## この文書の位置づけ

これまでの当社の収益発想は **「AIでコンテンツ（記事・電子書籍・テンプレ）を作って、消費者に直接売る」** が中心だった
（Kindle量産・Gumroadデジタル商品・有料マガジン・テンプレ販売）。
本文書は、その **B2C・コンテンツ物販型とは“構造から”違う** 収益モデルを3つ設計する。

3モデルの構造的な分類：

| | 誰が払うか | 何に払うか | 当社が売る相手 |
|---|---|---|---|
| **A メディア/アテンション型** | 広告主・スポンサー・アフィリ提携先 | 集まった「視聴者の注意」 | 視聴者には**無料**。お金は第三者から |
| **B B2B卸/ライセンス型** | 事業者（B2B） | コンテンツ/発信/自動化の**素材・実行・利用権** | 消費者ではなく**事業者** |
| **C 継続課金/コミュニティ型** | 少数の濃いファン（B2C） | **過程・更新・関与**（モノでなく継続） | 消費者だが**単発でなくストック** |

> 共通制約（A1〜A7・全案厳守）：ゼロ予算／外部送信はオーナー・coworkが実行（code=私は外部到達不可A1）／
> 画像生成不可（A2）／PII禁止＝具体地区名・町名・住所NG（A4）／誇張・断定・成果保証NG（A5）。
> 「借りた集客があるか」を各案で明示する（自前でフォロワーをゼロ構築する案は North Star 以外では避ける）。

# モデルA — メディア/アテンション型（視聴者には何も売らない）

**一言で**：高岡・氷見・富山の発信資産（127本＋英語力）を素材に、faceless動画とSEOページで
**「読者・視聴者には1円も請求せず」**、お金は広告ネットワーク・スポンサー・アフィリエイト提携先という
**第三者**から受け取る。読者無料＝North Star（海外読者最大化）と完全一致するのが最大の強み。

## ① 仕組み図

```
                        ┌─ 既存資産：高岡/氷見/富山 127記事 + 英語版 ─┐
                        │                                              │

   AI台本（英語/日本語） ─→ faceless動画（顔出しなし）          SEOページ群（英語）
       │                     │  画像=A2制約で生成しない               │
       │                     │  → 既存写真/オーナー撮影/             │ 既存記事を
       │                     │     無料ストック（cowork取得）         │ 英語Web向けに再編集
       ▼                     ▼                                       ▼
   無料ツールで音声          YouTube / TikTok / (Pinterest)        自前ドメイン or 無料ホスティング
   （TTS）＋自動字幕          ＝プラットフォームの発見エンジンに乗る   （GitHub Pages 既存・tavern.html枠）
       │                     │                                       │
       └──────→ [cowork/オーナーが投稿・アップロード] ←──────────────┘

              ┌───── 視聴者（無料で見るだけ）─────┐
```

## ② なぜ今までと構造が違うか

- これまで＝**読者＝顧客**（記事や本を読者に直接売る）。本案＝**読者≠顧客**。読者は無料で消費し、
  お金は「読者の注意を買いたい第三者（広告主・スポンサー・提携先）」が払う。**支払者と消費者が分離**している。
- だから **A4/A5（PII・誇張NG）と North Star（読者無料で最大到達）と利益が同じ方向を向く**：
  読者に売る圧力がないので、無理な煽り・誇張をする動機が構造的に消える。発信の質を上げるほど到達が増え、
  到達が増えるほど第三者が払う価値が上がる。**「良い発信＝そのまま収益基盤」**になる。

## ③ ゼロ予算で可能か

可能。すべて無料枠で成立する。
- 台本：code（私）が既存記事から制作（コストゼロ）。
- 音声：無料TTS（ブラウザ内蔵read-aloud／無料枠の音声合成）。
- 字幕：プラットフォーム（YouTube/TikTok）の自動字幕、または無料字幕ツール。
- 画像：**A2で生成しない**。オーナー撮影写真・既存サムネ・商用可の無料ストック（coworkが取得）。動画は
  「写真＋テロップ＋ナレーション」のスライド型で成立（顔出し・撮影機材不要）。
- ホスティング：SEOページは既存の GitHub Pages 枠（`tavern.html`/Pages 既設）を流用。

> 唯一の注意：YouTubeの広告分配は登録者・再生時間の基準到達が前提（時間がかかる）。
> **着金が遠い**のは正直に認識する（⑤参照）。アフィリは基準なしで即開始できるのが救い。

## ④ オーナー/coworkの最小操作

- code（私）：台本・字幕テキスト・概要欄・ハッシュタグ・アフィリ文面を**全部**用意。
- cowork：無料ストック/写真の取得（A3でcode不可）、動画の書き出し・アップロード、SEOページの公開。
- オーナー：チャンネル/アカウント初回作成1回。アフィリ提携の申請（本人名義が必要なため）。あとは
  「スポンサー問い合わせが来たら拾う」だけ（営業はしない＝B代行の営業エンジンに合流）。

## ⑤ 収益構造と黒字までの距離

- 固定費はゼロ（既存¥5,800/月のインフラ内に収まる）。**赤字化しない構造**。
- 黒字＝月¥5,800以上の着金。最も近い経路は **アフィリエイト**（プラットフォーム基準なしで今すぐ開始可）。
  動画広告分配は基準到達まで遠く、**数値の約束はできない（A5）**。
- 【事実】読者無料モデルは着金が後ろにずれる（注意を貯めてから換金する順序）。
  【考察】当社は既に発信を続けており、**追加の制作コストがほぼゼロ**なので、貯まるまで待てる体質と相性が良い。
  【提案】「広告分配で稼ぐ」を主目的にせず、**アフィリ＋スポンサー（来たら拾う）を主、広告分配は副**と置く。

## ⑥ 正直なリスク

- **着金が遠い**（広告分配は基準到達前提・保証なし）。短期の現金にはならない。
- プラットフォーム依存（アルゴリズム変更・アカウント停止リスク）。借りた集客の宿命。
- 著作権キャラのAI画像はNG（A3/governance）。藤子作品関連は**実物写真のみ**、無断のキャラ生成は使わない。
- faceless動画は量が出ると埋もれやすい。**「富山・北陸の英語発信」という具体ニッチ**で差別化する前提。

## ⑦ 最初の一歩

code が **既存記事2本から英語ナレーション台本2本**を起こす（下にサンプル2本を添付済み）。
cowork が無料ストック/オーナー写真でスライド動画化 → YouTube/TikTok にテスト投稿。
同時にオーナーが宿・体験予約・物産系のアフィリ提携を1社申請。**まず1本出して反応を見る**。

## 「借りた集客があるか」 → **YES（強い）**

YouTube/TikTok/Pinterest の**発見アルゴリズム**と、SEOページの**検索流入**に丸ごと相乗りする。
自前フォロワーのゼロ構築に依存しない。これは North Star（海外読者最大化）の配信網そのもの。

# モデルB — B2B卸/ライセンス型（消費者に売らず、事業者に卸す）

**一言で**：消費者に1本ずつ売るのをやめ、当社の「英語発信・コンテンツ・自動化テンプレ」を
**事業者（B2B）に“まとめて卸す／月額で使わせる”**。当社は既に **B2B英語発信代行の営業エンジン11点**
（`projects/2026-06-23_B2B英語発信代行_営業エンジン/`）を持っており、その自然な拡張。

## ① 仕組み図

```
   [当社の在庫]                          [卸す/貸す相手＝事業者]
   ・英語発信ノウハウ/テンプレ      ┌──→ ① 地域の宿・飲食・物産事業者
   ・記事/SNS制作の型               │      「英語発信を毎月◯本」を月額で代行・テンプレ提供
   ・自動化スクリプト               │
     （publish系/jobs.py/          ├──→ ② 同業の個人事業/小規模制作者
      quality_check等）            │      「英語note/SEOの量産テンプレ＋自動化キット」を利用権で卸す
   ・多言語化の型（中/韓/英）       │
                                   └──→ ③ 観光/地域メディア・自治体周辺の制作会社
   [code=私が整備]                        「富山・北陸の英語素材」をライセンスで提供（記事/写真台本/翻訳）
        │
        ▼ パッケージ化（消費者向けでなく“納品物＋利用条件”）
   ┌─────────────────────────────────────────┐
   │ 卸メニュー（3レイヤー）                  │
   │  L1 ワンショット卸：記事/SNS素材を束で納品（買い切り）        │
   │  L2 月額ライセンス：テンプレ＋自動化キットの利用権（継続）    │
   │  L3 月額代行：制作まで当社（cowork実行）が回す（高単価・継続）│
   └─────────────────────────────────────────┘
        │
   [オーナー/cowork] 提案送付・契約・納品（外部送信）→ 事業者が支払い → 着金
```

## ② なぜ今までと構造が違うか

- B2C物販＝**1人に1本、薄利・都度ゼロから集客**。本案＝**1社に束で、または継続課金**。
  1件の単価が桁違いに大きく（消費者の数百円→事業者の数万円）、**1〜2件で固定費を回収**できる。
- 「コンテンツそのもの」ではなく **「コンテンツを作る型・実行・利用権」を売る**。在庫（127記事・英語力・
  自動化スクリプト）を**何度でも転用できる資産**として貸し出す＝1回の制作が複数社に効く。

## ③ ゼロ予算で可能か

可能。新規制作はほぼ不要で、既存の営業エンジン11点・記事資産・自動化スクリプトの**再パッケージ**が中心。
決済はオーナーの既存手段（国内は銀行振込/請求書、海外はGumroad等のクレカ/PayPal）。SaaS課金なし。

## ④ オーナー/coworkの最小操作

- code（私）：卸メニュー表・利用規約・納品テンプレ・提案書の事業者別差し替え・FAQを整備（既存エンジン拡張）。
- cowork：提案文の送付、ライセンスキット（テンプレ＋スクリプト）のZIP納品、月額分の制作実行。
- オーナー：契約締結（押印/合意）、請求・入金確認。営業先の最終選定。

## ⑤ 収益構造と黒字までの距離

- L1ワンショット：1件数万円（買い切り）。**着金が速い**。
- L2月額ライセンス：1社 月額（小〜中額）の**ストック収入**。解約まで積み上がる。
- L3月額代行：1社 月¥30K〜50K想定（既存CFO試算の利益エンジン）。**黒字までの距離が最短**（1社で固定費の数倍）。
- 【事実】既に営業エンジン（見込み先・提案書・契約・実績見本・反論FAQ）が揃っている＝**着手障壁が最も低い**。
- 【提案】まず L1（買い切り卸）で実績を作り、L3（月額代行）へアップセル。L2は「自分で回したい事業者」向けの取りこぼし回収。

## ⑥ 正直なリスク

- B2Bは**信頼と実績がないと初契約が遠い**（最初の1社が一番重い）。実績見本で埋める前提。
- 成果保証はしない（A5）。「英語発信を代行する／素材を卸す」までで、集客成果の約束は契約に書かない。
- 当社の制作実行は cowork 依存（外部実行）。受注が増えると実行キャパが律速になる→ L2（テンプレ卸）で逃がす。
- 営業送付はオーナー/cowork。code（私）は外部送信不可（A1）なので、**送る人がいないと止まる**。

## ⑦ 最初の一歩

既存の `B2B英語発信代行_営業エンジン/` を**3レイヤー卸メニュー**に再構成し（code）、
「L1買い切り卸」の見本パッケージ（記事束＋英語SNS素材）を1つ完成 → cowork/オーナーが見込み先1社に提案送付。
**1社の反応で価格と刺さりを確かめる**。

## 「借りた集客があるか」 → **間接的（弱め・要営業）**

B2Bは検索/発見エンジンに乗りにくく、**営業（送付）依存**。ただし、事業者が当社に発注する動機は
「自分では届かない海外読者に当社の発信網（モデルA・North Star）で届く」点＝**当社の借りた集客を事業者に貸す**構造。
当社のメディア（A）が育つほど、Bの提案力が上がる。**AとBは相互補完**。

# モデルC — 継続課金/コミュニティ型（過程と更新に課金・ストック収入）

**一言で**：完成品を1個ずつ売るのではなく、**「高岡・氷見・富山を世界に発信していく“過程”と、その更新」**に
少額・継続で課金する。メンバーシップ／有料ニュースレターで、**単発販売でなくストック（積み上がる定期収入）**を作る。
既存資産（127記事・舞台裏・英語化の現場）が、そのまま会員向けの「受け皿」になる。

## ① 仕組み図

```
   [既存資産＝受け皿の材料]
   ・127本の記事（無料公開のものも含む）
   ・英語化/発信の“過程”（編集中・ボツ・反応分析CAOの知見）
   ・地域の食/場所の継続観察（住人視点）
        │
        ▼ code が「会員向けの層」を切り出す（新規執筆を最小化）
   ┌──────────────── 3層の受け皿 ────────────────┐
   │ 無料層（North Star）：記事・動画は無料で全公開＝最大到達      │
   │   ↓ ここで“知る”                                            │
   │ 入口（無料ニュースレター）：週1ダイジェスト＝見込みを貯める   │
   │   ↓ 一部が“もっと近くで見たい”に転換                          │
   │ 有料層（月額・少額）：①更新の優先/限定ノート ②舞台裏・制作過程 │
   │                      ③会員だけの質問/リクエスト窓口          │
   └──────────────────────────────────────────────┘
        │
   [プラットフォーム：note メンバーシップ / Substack有料 / Buy Me a Coffee 等]
        │  （決済・会員管理はプラットフォーム任せ＝当社は構築不要）
        ▼
   [cowork/オーナー] 月数回の更新を投稿 → 会員が月額課金 → 着金（ストック）
```

## ② なぜ今までと構造が違うか

- これまで＝**フロー収入**（1本売れて終わり、また次をゼロから売る）。本案＝**ストック収入**
  （1人が会員でいる限り毎月入る）。売上が**積み上がり、予測可能**になる。
- 売るのは「完成したモノ」ではなく **「続いていく過程と関係」**。だから A5（誇張）と無縁：
  約束するのは“更新を続けること”だけで、誇大な効能を売らない。**「続けられて黒字」**という本命条件に最も近い形。
- 既存127記事は「在庫」ではなく**会員に見せる“蓄積の証拠”**として再評価される（書き直し不要）。

## ③ ゼロ予算で可能か

可能。note メンバーシップ／Substack有料／Buy Me a Coffee はいずれも**初期費用ゼロ・手数料は売上発生時のみ**。
会員管理・決済はプラットフォーム任せ。新規制作は「既存の過程を会員向けに切り出す」中心で、code が担当。

## ④ オーナー/coworkの最小操作

- code（私）：会員向け層の設計、月次更新の原稿、無料→有料の導線文、入口ニュースレターのダイジェストを用意。
- cowork：月数回の投稿（コピペ）、ニュースレター配信。
- オーナー：プラットフォーム初回設定1回（メンバーシップ開設）、価格決定、入金確認。

## ⑤ 収益構造と黒字までの距離

- 例：月額少額 × 会員数。**少人数でも固定費（¥5,800/月）は超えやすい**（例：月¥500×12人＝¥6,000で分岐）。
  ※具体的会員数は**約束しない（A5）**。あくまで損益分岐の構造説明。
- 【事実】ストックなので、一度分岐を超えると**毎月継続して固定費を上回りやすい**（解約がなければ）。
- 【考察】当社は既に無料層（記事・将来の動画）で到達を作っているので、**入口→有料の転換だけ**を設計すればよい。
  ゼロから集客しなくていいのが効く。
- 【提案】読者課金は North Star（無料最大到達）と衝突しうるため、**無料層は絶対に削らない**。有料は
  「過程・舞台裏・優先・関与」に限定し、**記事本体は無料のまま**（読者を排除しない設計）。

## ⑥ 正直なリスク

- 会員獲得は時間がかかる（信頼の蓄積待ち）。**着金は中速**（Aより速くBより遅い目安）。
- 「過程に毎月お金を払う」濃いファンは数が少ない。**規模の上限は低め**（固定費は超えても大きくは伸びにくい）。
- 更新が止まると解約に直結＝**継続義務**が発生（続けられないと逆効果）。当社の自動制作体質が前提。
- 読者課金は North Star と緊張関係。**有料化のさじ加減を誤ると無料読者を失う**。無料層死守が条件。

## ⑦ 最初の一歩

code が **無料ニュースレター（入口）の創刊号＋第2号**を既存EN記事から組む → cowork が無料配信で立ち上げ
（North Star のチャネルとして先行）。購読が貯まってから、**有料メンバーシップ層**（舞台裏＋月次更新）を後付けで開設。
**まず無料入口を回し、有料は転換が見えてから**。

## 「借りた集客があるか」 → **YES（入口は借り、課金は自前関係）**

入口の無料ニュースレター（Substack/Beehiiv）は **recommendation・network 機能で他媒体から購読者が流入**する＝借りた集客。
note メンバーシップも note の検索・発見に乗る。一方で**有料転換は当社と読者の関係（自前）**に依存する二段構え。
