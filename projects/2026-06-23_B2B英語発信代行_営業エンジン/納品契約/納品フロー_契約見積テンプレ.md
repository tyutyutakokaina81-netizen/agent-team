# 英語発信代行 — 納品フロー＋簡易契約・見積テンプレ一式

> 作成：CSO 兼 CFO（営業・事務責任者）。受注後に「やる」と言われてから困らないための実務一式。
> 位置づけ：営業エンジン（提案書／営業文／お試しサンプル）で受注が見えた後の **運用・契約・請求** を担う部品。
> 提案書のプラン体系（ライト¥30,000／スタンダード¥40,000／フル¥50,000・いずれも税別・月額）と整合させている。

---

## ⚠️ 重要な前提（必ず最初に読む）

- **A4（PII禁止）**：本ファイルおよび契約・見積の雛形には、実在の店名・地区名・町名・住所・人名を一切書かない。穴埋め欄（〔　〕）はオーナーが実物に記入する。**記入後の契約書・顧客情報は Git にコミットしない**（CFO/CSO の outputs はそもそも .gitignore 対象）。
- **A5（成果保証NG）**：契約・見積・運用のどの文面でも「来客増・予約増・売上増を約束する」表現を使わない。約束するのは **「質の高い英語発信を計画的に制作・提供し続けること」** だけ。
- **法務・税務の免責**：本ファイルは雛形であり、**法的助言・税務助言ではない**。実際の契約締結・インボイス／消費税の扱い・源泉徴収の要否などは、**必ず専門家（弁護士・税理士・行政書士）および国税庁等の最新の公式情報で確認**してから確定すること。雛形中の〔要確認〕は確定前に潰す。

## ① オンボーディング手順（受注直後〜運用開始まで）

「やる」と返事をもらってから初回納品までを、ヒアリング → 素材受領 → 初回設定 → 初回納品の順で進める。

### 1-1. 初回ヒアリング（無料・30〜60分／対面・電話・メールいずれも可）

聞く項目のチェックリスト。聞き漏らすと後で手戻りになる順に並べてある。

| # | ヒアリング項目 | 確認する理由 |
|---|---|---|
| 1 | 業種・提供しているもの（料理／宿／体験など） | 発信の切り口・原資にする記事在庫を決める |
| 2 | 海外客の今の状況（来ている／これから増やしたい） | 期待値の調整。来客保証はしない前提を共有 |
| 3 | 発信したい媒体（どのSNS・自社サイト・予約サイト等） | 投稿主体と運用範囲を決める |
| 4 | **投稿は誰が行うか**（先方が投稿／こちらが下書き納品のみ／アカウント権限を預かる） | 後述の運用フロー・契約の「業務内容」に直結 |
| 5 | 使える写真・動画素材の有無と量 | 撮影は行わないため素材の確保が前提 |
| 6 | 日本語の元原稿・既存の紹介文の有無 | 英語化のベースになる |
| 7 | 出してほしくない情報（住所詳細・個人名・非公開メニュー等） | A4遵守・トラブル防止 |
| 8 | 希望プラン・予算感・開始希望時期 | 見積・契約に進む |
| 9 | 月次確認の窓口担当者・連絡手段（メール／チャット等） | 確認サイクルを回す相手を固定する |

### 1-2. 必要素材の受領（契約前後に確保）

- **写真・動画素材**：先方の既存素材を受領（撮影は含まない）。利用してよい範囲（SNS可・記事可・期間）を口頭でなく書面/チャットで確認。
- **日本語原稿**：メニュー名・料理の説明・店/宿の由来・こだわりなど。英語化の精度はここで決まる。
- **NG情報リスト**：出してはいけない情報（住所詳細・個人情報・価格を出したくない等）を先に固定。
- **媒体アクセス**：こちらが投稿まで行う契約の場合のみ、アカウント権限の受け渡し方法を取り決め（パスワード共有は避け、可能なら編集者権限付与等。方法は媒体仕様により〔要確認〕）。

> 素材が集まらないと納品できないため、**「素材が揃った日」を起点に初回スケジュールを引く**。

### 1-3. 初回スケジュール（標準モデル・初月）

| 日程の目安 | 工程 | 主担当 |
|---|---|---|
| Day 0 | ヒアリング実施・プラン仮決め | CSO（オーナー） |
| Day 1〜2 | 無料お試しサンプル 1〜2本 試作・提示 | 制作（code支援） |
| Day 3〜5 | 契約・見積の取り交わし（プラン確定） | CSO/CFO |
| Day 5〜7 | 素材受領・トンマナ（語り口・世界観）設計・月次発信カレンダー初版 | 制作 |
| Day 8〜 | 初回コンテンツ制作 → 確認 → 納品（初月分スタート） | 制作 |

> 初月は「設計＋初回納品」で手数がかかるため、初期設定費を取るか初月キャンペーンで吸収するかを見積段階で明示する（後述④）。

## ② 月次運用フロー（毎月くりかえす標準サイクル）

制作 → 確認 → 納品 → 公開 の4ステップを月内で回す。本数とサイクル・修正回数の上限を最初に握っておく。

```
[1] 月初：発信カレンダー確定（今月いつ・何を出すか）
        ↓
[2] 制作：プランの本数分の英語コンテンツを制作（下書き）
        ↓
[3] 確認：先方が内容チェック（事実・固有名詞・NG情報の確認）
        ↓  ←― 修正は1コンテンツあたり規定回数まで（下表）
[4] 納品：確定稿を納品（テキスト＋ハッシュタグ等を所定フォーマットで）
        ↓
[5] 公開：契約で決めた投稿主体が公開（先方 or こちら）
        ↓
[6] 月末：簡単な振り返り（フルプランは月次レポート）
```

### 本数とサイクル（プラン別・提案書と整合）

| プラン | 月額(税別) | 月次の本数 | 月次レポート |
|---|---|---|---|
| ライト | ¥30,000 | 英語SNS投稿 月4本＋月次カレンダー | なし（簡単な振り返りのみ） |
| スタンダード | ¥40,000 | 英語SNS投稿 月6本＋英語紹介記事 月1本＋返信テンプレ | 簡易 |
| フル | ¥50,000 | 英語SNS投稿 月8本＋英語紹介記事 月2本＋季節企画 | あり |

### 修正回数の上限（トラブル防止の肝）

- **各コンテンツの修正は「2回まで」を標準**とする（事実誤り・NG情報の訂正は回数に含めない＝何度でも直す）。
- 3回目以降の大幅な作り直し・コンセプト変更は、別途追加費用の相談対象とする（見積に明記）。
- 確認の返答期限：納品下書きの確認は **〔5〕営業日以内**に。期限内に返答がない場合は内容了承とみなし次工程へ進める（カレンダー遅延防止）。回数・期限は契約で確定。

**英語発信代行 業務委託契約書（簡易版）**

委託者　〔発注者（屋号・代表者名）〕（以下「甲」という。）と
受託者　〔受託者（屋号・氏名）〕（以下「乙」という。）は、英語発信の制作・運用業務に関し、以下のとおり契約を締結する。

**第1条（業務内容）**
乙は甲に対し、次の業務を行う。具体的な本数・媒体・投稿主体は別紙「申込内容（プラン）」による。
1. 英語によるSNS投稿文・ハッシュタグ・紹介文の制作
2. 英語による紹介記事の制作（プランに含む場合）
3. 日本語原稿の英語化（編集翻訳）
4. 月次発信カレンダーの作成
〈注〉撮影、予約サイト/OTAの直接運用、決済代行、接客の英語化は**業務に含まない**ことを明記する。投稿操作を乙が行うか甲が行うかは別紙で定める。

本業務の委託料は **月額〔　〕円（税別）** とする。消費税の取り扱いは第〔　〕条による。
〈注〉初期設定費を取る場合は「初月に限り別途〔20,000〕円（税別）」等を追記。初月無料キャンペーンで吸収する場合はその旨を記載。

**第4条（委託料の支払い）**
甲は、毎月末日締めで当月分の委託料を、翌月〔末日〕までに乙の指定する口座に振り込む方法により支払う。振込手数料は甲の負担とする。

**第5条（解約）**
甲または乙は、〔30〕日前までに相手方に書面（電子メール等を含む）で通知することにより、本契約を解約できる。解約時点までに発生済みの委託料は支払うものとし、前払い分がある場合の精算方法は甲乙協議のうえ定める。
〈注〉「いつでも30日前通知で解約可」は地方小規模事業者が契約しやすい条件。最低契約期間内の中途解約の扱い（残期間の支払い要否）は〔要確認〕で双方納得の形に。

**第6条（成果物の著作権・利用）**
乙が制作した成果物（英語投稿文・記事等）の著作権は、当該成果物に係る委託料の支払い完了時に乙から甲へ譲渡する。ただし乙は、自己の制作実績の紹介のため、**実在の店舗名・所在地・個人を特定しない範囲**で当該成果物を参考事例として提示できる。
〈注〉「支払い完了で著作権が甲に移る」は発注者が安心する条件。一方で乙（こちら）が実績紹介に使える余地を残す。**実績提示はA4を必ず守り、特定できる情報は出さない。**

**第7条（守秘義務）**
甲乙は、本業務を通じて知り得た相手方の非公開情報を、相手方の事前の同意なく第三者に開示・漏えいしてはならない。本条は契約終了後も〔2〕年間有効とする。
〈注〉甲から受領した非公開メニュー・素材・顧客情報の取り扱いを縛る。乙側の運用（PII不記載）とも整合。

**第8条（免責・成果非保証）**
乙が約束するのは、**質の高い英語発信を計画的に制作・提供すること**である。乙は、本業務による甲の来客数・予約数・売上等の増加を**保証しない**。発信内容は甲の事前確認を経て公開するものとし、甲が確認・承認した内容に起因する結果について乙は責任を負わない。乙の損害賠償責任は、故意または重過失による場合を除き、**当該月の委託料を上限**とする。
〈注〉A5の核。成果保証をしない旨をここで明文化。賠償上限を設けて青天井のリスクを避ける。

## ⑤ 請求サイクル（一般的な例・要確認）

> 以下は**一般的な商習慣の例**であり、税務・会計の助言ではない。実運用は税理士・先方の経理と擦り合わせて確定する。

- **締め日**：月末締め（当月1日〜末日分を当月末で締める）。
- **請求書発行**：締め後すみやかに（例：翌月第〔3〕営業日までに発行）。
- **支払期日**：翌月末払い（例：5月分を6月末までに入金）。地域・先方の慣行により「翌々月10日払い」等もある〔要確認〕。
- **支払方法**：銀行振込（振込手数料は先方負担を基本に契約で明記）。
- **インボイス**：適格請求書発行事業者か否かで請求書様式が変わる。登録番号の記載要否・免税事業者の対応は〔要確認〕。
- **領収書**：振込控えを領収書代わりにするか、別途発行するかを取り決める。

**月次の請求オペレーション（標準）**
1. 月末：当月の納品実績を確認（本数・追加作業の有無）。
2. 翌月初：請求書を作成・送付（プラン月額＋初月のみ初期設定費＋追加作業分）。
3. 入金確認：支払期日後に入金チェック。未入金は丁寧にリマインド（⑥参照）。
4. 記録：請求・入金を帳簿に記録（CFO管理・gitignored）。

## ⑥ トラブル時の対応指針

起きやすい順に、初動と落としどころを決めておく。いずれも**成果保証はしない／品質と継続は守る**のトーンで一貫させる。

| 想定トラブル | 初動 | 落としどころ |
|---|---|---|
| **「効果が出ていない（客が増えない）」** | 契約・提案で「来客保証はしない／中長期で効く取り組み」と説明済みである旨を冷静に再確認。これまでの発信実績（本数・内容）を提示。 | プラン継続の意義を再説明。納得が得られなければ解約条項（30日前通知）に沿って円満終了。 |
| **修正が無限に続く／要求が膨らむ** | 契約の「修正は1コンテンツ2回まで（事実訂正は回数外）」を提示。 | 上限超過分は追加費用の見積を提示。コンセプト変更は別案件として再見積。 |
| **確認の返答が来ず納品が滞る** | 「確認は5営業日以内、無返答は了承とみなす」条項を案内し、リマインド。 | カレンダーどおり進行。遅延の責任は乙にない旨を記録に残す。 |
| **支払い遅延・未入金** | 支払期日後にまず事務連絡として丁寧にリマインド（行き違いを想定）。 | 改善なければ当月以降の制作を一時停止する旨を通知（契約に停止条件を入れておくとよい〔要確認〕）。 |
| **事実誤り・NG情報を公開してしまった** | ただちに該当箇所の差し替え稿を無償で用意（事実訂正は修正回数外）。原因（確認漏れか制作ミスか）を切り分け。 | 再発防止策を共有。先方確認後の公開だった場合も、誠実に訂正対応する。 |
| **素材が集まらず制作できない** | 必要素材リストを再提示し、揃った日を起点にスケジュールを引き直す。 | 素材未提供による遅延は乙の責に帰さない旨を契約・連絡記録で明確化。 |
| **アカウント権限・パスワード関連の不安** | パスワード直接共有を避け、編集者権限付与等の安全な方法を提案。 | 投稿主体を「乙は下書き納品のみ」に変更し、公開は先方が行う形に切り替える選択肢も提示。 |
| **クレーム・関係悪化** | 感情的に応酬せず、契約条項と事実記録に基づき対応。やり取りは書面/チャットで残す。 | 修復不能なら解約条項に沿って清算・円満終了。実績紹介は当該先のNG意向を尊重。 |

**通底する原則**
- やり取りは口頭だけで済ませず、**メール/チャットで記録を残す**（言った言わないを防ぐ）。
- 約束していないこと（成果・集客）を後から求められたら、契約・提案の該当文を落ち着いて指し示す。
- 困ったら無理に抱え込まず、解約条項（30日前通知）で円満に終わらせる道を常に確保しておく。
