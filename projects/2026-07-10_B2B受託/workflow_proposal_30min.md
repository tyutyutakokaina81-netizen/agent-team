# ワークフロー：商談議事録 → 提案書draft を30分で（worker運用手順）

- 目的：B2B受託の中核フック「**商談30分後に提案書が届くと顧客が驚く**」を worker（Mac上の `claude -p`）で回せる手順に落とす。
- 正本テンプレ：`projects/2026-07-10_B2B受託/template_proposal.md`
- 生成スクリプト：`projects/2026-07-10_B2B受託/gen_proposal.py`（依存ゼロ・標準ライブラリのみ）
- 出典：`CAO/research/2026-07-10_かいち10本_全文精読の完全抽出.md`（動画5：議事録＋見積PDF→前段/中段/後段でWord生成、デモ45ページ）
- 役割分担：**設計・台帳・判断＝code（Claude Code）／実行・変換・納品＝worker**。金が動く操作・公開・削除は人間がボタンを押す。

---

## 0. 大原則（曖昧inputを禁じる）

かいちCHRO教訓＝「ペルソナ・要件が曖昧なまま生成すると刺さらない」。
**下の必須5要素が埋まるまで生成を始めない。**不足があれば worker は生成せず「不足項目リスト」を返す（＝CLAUDE.md「不明点は先に質問してからスタート」）。

### 入力の必須5要素（これが揃って初めて着手）

| # | 要素 | 例 | 埋まらない時 |
|---|---|---|---|
| 1 | 業種 | 士業／小売／製造 等 | 議事録から拾えなければ客に確認 |
| 2 | 規模 | 従業員数・店舗数・月間取引件数 等 | 同上 |
| 3 | 課題 | 「未登録取引の仕訳に毎月20時間」 | 議事録の一次情報から抽出 |
| 4 | ゴール数字 | 「仕訳工数を月20h→2hに」 | 数字が無いゴールは却下（"数字と期限"原則） |
| 5 | 予算感 | 「月5万まで」「初期20万まで」 | 分からなければ3プランの幅で吸収 |

---

## 1. 手順（合計 約30分）

### STEP 1. 入力を集める（5分）
- `minutes.txt`：商談の議事録テキスト（文字起こしそのままでよい）
- `req.txt`：要件メモ（下記フォーマット）。**必須5要素を先に埋める。**
- **一次情報を必ず差し込む**：議事録にある顧客の実発言・実数字・失敗談を、テンプレの現状整理／課題に直接反映する（一般論のWeb要約に堕とさない）。

`req.txt` のフォーマット（`key: value`、1行1項目、`#`はコメント、値内の `\n` は改行になる）：
```
client: サンプル商事株式会社   # ダミー名（A4：実企業名・実PIIは書かない）
industry: 税理士事務所
scale: 顧問先60社・スタッフ4名
issue: freee未登録取引の仕訳に毎月20時間かかっている
goal: 仕訳工数を月20時間→2時間（90%削減）
budget: 月5万円以内・初期費用20万円まで
...（テンプレの他プレースホルダも埋める。埋め残しは【未入力】として出力される）
```

### STEP 2. draft を生成（3分）
```bash
python3 projects/2026-07-10_B2B受託/gen_proposal.py \
  --minutes minutes.txt \
  --req req.txt \
  --out proposal_draft.md
```
- スクリプトはテンプレの `{{...}}` を `req.txt` の値で差し替える。
- **埋まらなかったプレースホルダは `【未入力: key】` として残る**（＝壊れず、抜けが一目で分かる）。
- 生成物末尾に「未入力一覧」と「⚑要ファクトチェック件数」が自動追記される。

### STEP 3. 人間チェック（15分）★省略不可
- **⚑要ファクトチェックの数値をすべて一次情報で裏取り**（競合価格・削減額・削減率・市場規模・相場）。AIが出した数字をそのまま納品しない（かいち落とし穴①／A5）。
- **A4：実在の企業名・個人名・住所などPIIが無いか**（ダミー→実名の置換漏れも点検）。
- `【未入力】` と `{{...}}` の取り残しをゼロにする。
- **A評価でも即納品しない**：実行体力・資金・スケジュールを一度重ねる（かいち落とし穴②）。
- 議事録の認識が本当に合っているか（推測を事実として書いていないか）。

### STEP 4. 納品形式に変換して納品（5分）
- 形式：**Word / PDF / Googleドキュメント**のいずれか（顧客の希望に合わせる。デモはWordで十分・デザインは出力そのままでOK）。
- 変換は worker（Mac）側で実施。**送信・公開・支払いにかかわる操作は人間が最終ボタンを押す**。

---

## 2. 安全ルール（金が動く/公開/数字は人間）

1. **AIが出した数字は、人間が最終チェックしてから納品**（`⚑要ファクトチェック` を全消し込みするまで納品禁止）。
2. **原本（`minutes.txt` / `req.txt`）は上書き・削除しない**（ABC方式のA＝インプットは触らない）。生成物は別ファイルに出す。
3. **PII禁止（A4）**：サンプルはダミー名で作る。実データは worker のローカルに置き、リポジトリにコミットしない。
4. **削除は「対象1点＋"それ以外は消すな"」の2文**で指示する（例：「proposal_draft.md だけ消して。他は絶対に消さないで」）。
5. 送信・公開は下書きまでを AI、**送信ボタンは人間**。

