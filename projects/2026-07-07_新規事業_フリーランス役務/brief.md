## 3. 受注→納品フロー（owner最小・code全自動）
1. クライアント発注 → owner が要件（テーマ/字数/キーワード/納期）を ops/inbox 経由で code へ。
2. **code が24時間以内に納品原稿**（本文＋タイトル3案＋事実検証ノート）を生成 → `CMO/outputs/受託/` へ。
3. owner が検収 → 納品ボタン → 請求（CFO様式に金額記録）。
4. 評価が付いたら gig 単価を上げる（Fiverrはレビュー数がSEO）。

## 4. 立ち上げ手順（owner一度きり）
1. Fiverr（世界）と Upwork に無料登録（本人確認）。プロフィール文＝上記。
2. gig を上表から3つ作成（まずSEO記事EN・翻訳・inbound英語ページ）。
3. Upworkは最初の提案を5件だけ送る（応募文＝既存 `CSO_受託応募キット.md` を英語化して流用）。
4. 以降：受注は code が全自動生産、owner は検収と評価対応のみ。

## 5. 規律（スモールスタート）
- **最初の1件（型の獲得）を最優先**。単価より受注実績。
- 応募/gigは絞る（乱造しない）。反応ゼロなら文面/gig画像を1回改訂。
- 禁止：実績誇張・AI利用を隠す・赤字受注の連発。
- 報告：応募数/返信数/受注数を週次レビューのB柱に記録。
