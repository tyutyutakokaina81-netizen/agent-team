# CFO 事務テンプレ一式（見積書・契約書・請求書）

**作成日**: 2026-06-21
**用途**: SEO記事代行・SNS運用代行の初成約時に「慌てない」ための事務一式
**保存場所の理由**: CFO/outputs は .gitignore 管理（実データ保護）のため、
                    **空テンプレ**は本プロジェクトフォルダに置き、実発行物だけCFO/outputsへ
**注意**: 実際の金額・顧客名を入れた発行物は **Gitにコミットしない**（CFO/outputs配下で管理）

## 2. 業務委託契約書テンプレ（簡易版・個人〜中小向け）

> ※ 高額・長期案件は専門家（行政書士・弁護士）の確認を推奨。これは小規模取引の出発点。

```
業務委託契約書

[委託者：顧客名]（以下「甲」）と [受託者：自分の屋号/氏名]（以下「乙」）は、
以下の通り業務委託契約を締結する。

第1条（業務内容）
  乙は甲に対し、以下の業務を提供する。
  ・[SEO記事制作 / SNS運用代行 等]
  ・詳細は別紙（仕様・本数・対応範囲）による。

第2条（委託料・支払）
  1. 委託料は [月額◯◯円 / 1件◯◯円]（消費税別）とする。
  2. 支払は [月末締め翌月末日までに] 乙指定口座へ振込む。
     振込手数料は甲の負担とする。

第3条（契約期間）
  本契約は [YYYY年MM月DD日] から [1ヶ月間] とし、
  期間満了の [14日前] までに双方申し出なき場合、同条件で更新する。

第4条（成果物の取扱い・著作権）
  納品物の著作権は、委託料の完済をもって甲に移転する。
  乙は実績として[匿名で/許諾を得て]公開できるものとする。

第5条（修正対応）
  納品物の修正は [◯回まで] 委託料に含む。
  これを超える修正は別途見積りとする。

第6条（成果の非保証）
  乙は誠実に業務を遂行するが、検索順位・フォロワー増等の
  具体的成果を保証するものではない。

第7条（秘密保持）
  双方は、本業務で知り得た相手方の情報を第三者に開示しない。

第8条（解約）
  双方は [30日前] の書面通知により本契約を解約できる。
  既に発生した委託料の支払義務は存続する。

第9条（協議）
  本契約に定めなき事項は、双方誠実に協議して定める。

  YYYY年MM月DD日
  甲：[顧客名・住所・代表者]          印
  乙：[屋号/氏名・住所]              印
```

> **第6条（成果の非保証）が最重要**。これがないと「順位が上がらない＝返金」の
> トラブルになる。初期の自衛として必ず入れる。

## 3. 請求書テンプレ

> ※ 自動生成は `CMO/outputs/有料/2026-06-21_有料03_請求書を3分で作るプロンプト完全版.md` のプロンプト参照

```
─────────────────────────────────
                 請　求　書
─────────────────────────────────
                          請求番号：INV-YYYYMM-NN
                          発行日：YYYY年MM月DD日
                          お支払期限：YYYY年MM月DD日

[顧客会社名] 御中

下記の通りご請求申し上げます。

  ご請求金額   ¥ 56,100 （税込）

─────────────────────────────────
  品目                 単価        数量      金額
  ──────────────────────────────
  [SEO記事制作 6月分]  ¥18,000      3      ¥54,000
  ──────────────────────────────
                       小計              ¥54,000
                       消費税(10%)        ¥5,400
                       合計              ¥59,400
─────────────────────────────────

【お振込先】
  [銀行名] [支店名] [普通] [口座番号]
  [口座名義]

  ※ 振込手数料はご負担をお願いいたします。

                    [屋号 / 氏名]
                    [連絡先 / 登録番号(インボイス制度対応時)]
─────────────────────────────────
```

> **インボイス制度**：課税事業者なら「登録番号(T+13桁)」を請求書に記載。
> 免税事業者の場合の取引条件は、事前に顧客と確認しておく。

## 4. 入金管理（最小台帳・CSV）

```csv
請求番号,請求日,顧客,品目,請求額,支払期限,入金日,状態
INV-202606-01,2026-06-30,A社,SEO記事6月分,59400,2026-07-31,,未入金
```

→ 「状態」が未入金のまま支払期限を過ぎたら、丁寧な督促を1通。
→ 入金実績は `CAO/outputs/kpi_dashboard.py --record` のactual_revenueに転記。

**事実検証ノート**
- ⚠️ 契約書ひな形は小規模取引の出発点。高額・長期は専門家確認を推奨
- ⚠️ インボイス/消費税の扱いは事業者区分により異なる。税理士確認を前提に
- ✅ 第6条（成果非保証）はSEO/SNS代行の自衛上、必須条項
