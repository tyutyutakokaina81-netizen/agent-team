## 1. 全体フロー（手動・半自動の区分）

```
【手動】① 登録 ─────────────  初日のみ
   ↓
【手動】② プロフィール整備 ──  初日のみ（テンプレ流用）
   ↓
【手動 5分】③ 案件検索 ────  毎日
   ↓
【自動】④ 案件評価 ──────────  02_evaluate.py
   ↓
【自動→手動】⑤ 応募文生成・送信 ──  03_apply.py → 人手コピペ送信
   ↓
【自動】⑥ 作業実行 ──────────  04_execute.py
   ↓
【自動】⑦ 品質念査 ──────────  05_review.py
   ↓
【手動 5分】⑧ 納品 ──────────  06_deliver.py で文面生成 → 人手送信
```

**応募・送信・納品はすべて人手** ← プラットフォーム規約遵守の絶対要件

## 3. 規約遵守の絶対ルール

| ルール | 違反時 |
|--------|--------|
| 応募送信は **人手で行う**（自動応募ボット禁止） | アカウント即停止・損害賠償リスク |
| AI使用を **プロフィールと応募文に明示** する | 規約違反・評価低下 |
| プラットフォーム自体のスクレイピング禁止 | アカウント即停止 |
| 個人情報（クライアント名・連絡先）の SNS／外部公開禁止 | 規約違反・損害賠償 |

### 応募文の AI 明示例

```
本案件はAI補助＋人間確認のフローで対応します：
1. Claudeで初稿作成・効率化
2. 人間がレビュー・品質確認
3. 必要に応じて手作業で調整
ご懸念がある場合は事前にご相談ください。
```

## 4. 日次運用フロー（1日 30分）

### 朝（10分）
- [ ] クラウドワークス・ランサーズで「データ入力」「スクレイピング」「SEOライティング」を検索
- [ ] 良さそうな案件を 5〜10 件、`02_evaluate.py` に貼り付けて評価

- [ ] GO 案件に応募文を生成（`03_apply.py` または `A_ライティング/application_templates.md` から流用）

- [ ] 手動で応募送信

### 夜（10分）
- [ ] 受注した案件があれば作業着手
- [ ] 完了案件の納品（`06_deliver.py` で文面生成 → 手動送信）
- [ ] 評価コメントへの返信

## 5. 受注後の対応フロー

### 5-1. 契約締結
- [ ] プラットフォームの「契約」機能で正式契約
- [ ] **高単価案件（¥30K 以上）**：契約書テンプレを別途送付推奨  
   → `CLO/outputs/2026-05-21_業務委託契約書_汎用テンプレ.md`
- [ ] **継続案件（月額契約）**：契約書 + NDA を締結  
   → `CLO/outputs/2026-05-21_NDA秘密保持契約書テンプレ.md`

### 5-2. 要件確認（必須質問）
クライアントに次を確認する：
- データ取得元のサイト・対象範囲
- 取得項目・出力形式（Excel／CSV／JSON）
- 件数・納期
- **個人情報の有無**（あれば利用目的を確認）
- 検収方法・修正回数

### 5-3. 作業実行
- [ ] `04_execute.py` で作業実行
- [ ] `05_review.py` で念査
- [ ] 人間が最終チェック

### 5-4. 納品
- [ ] `06_deliver.py` で納品文を生成
- [ ] ファイルを添付して手動送信
- [ ] 検収依頼

### 5-5. 評価対応
- [ ] クライアントからの評価コメントに返信
- [ ] 5星評価ならその旨に感謝
- [ ] 低評価なら誠実に対応（プロフィールにも影響）

## 6. 規制業種・NG 案件チェック

以下の依頼は **受注しない**：

| カテゴリ | 理由 |
|---------|------|
| ログイン突破・認証回避 | 不正アクセス禁止法違反 |
| 食べログ・ぐるなび等の DB サイト全件取得 | データベース著作権侵害リスク |
| 個人情報の名簿販売・営業リスト作成 | 個人情報保護法違反 |
| 競合誹謗中傷・嫌がらせ目的 | 共犯リスク |
| 詐欺・副業勧誘・受け取り送金系 | 詐欺加担リスク |

`02_evaluate.py` で自動的に「辞退」「保留」フラグが立つようになっている。

## 7. KPI と目標

### 月次目標（Month 1〜3）

| 項目 | Month 1 | Month 2 | Month 3 |
|------|---------|---------|---------|
| 応募数 | 50件 | 50件 | 50件 |
| 受注数（D） | 5件 | 10件 | 15件 |
| 受注数（A） | 5件 | 12件 | 20件 |
| 売上 | ¥75K | ¥255K | ¥530K |

### KPI ウォッチポイント
- 応募 → 受注の転換率（最低 10%）
- 1案件あたりの工数（目標：1時間/¥10K）
- クライアント評価（4.5星以上をキープ）

## 8. トラブル時の対応

### アカウント停止
- 規約違反内容を確認
- CLO に相談、必要なら弁護士相談
- 別プラットフォームへの分散を検討

### クレーム・低評価
- 24時間以内に返信（誠実な対応）
- 原因を分析・改善
- 必要なら追加対応・返金検討

### 支払遅延
- 期日後 3 日：催促メール
- 期日後 7 日：プラットフォーム経由で督促
- 期日後 14 日：CLO 経由で正式督促文書
