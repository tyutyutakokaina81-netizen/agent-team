応募数 0 → 月 30〜200 件規模に立ち上げるための運用手順書。

---

## 1. セッション設定（初回のみ・人手）

```bash
cd pipeline
python3 00_session_setup.py
```

ブラウザが開くので **クラウドワークス** と **ランサーズ** にログイン後、
ターミナルで Enter → セッションが `.sessions/*.json` に保存される。

セッションは数週間〜数ヶ月有効。期限切れになったら再実行する。

---

## 2. 日次運用（毎朝 5〜10 分）

### 2-1. 検索 → 評価 → 応募文生成

```bash
python3 run_pipeline.py search
```

自動で以下が走る：
1. 14 キーワードでクラウドワークス・ランサーズを検索（最大 30 件/プラットフォーム）
2. 4 軸スコアリング（技術／法的／採算／明確性）で GO / CAUTION / NO-GO 判定
3. GO + CAUTION 案件に対して 4 要素応募文を生成
4. 上位 3 件のページをブラウザで自動オープン

### 2-2. 応募（人手・1 件あたり 30 秒）

ブラウザで開いた案件ページに：
1. 生成された応募文をコピペ
2. 「応募する」ボタンを押す

### 2-3. 受注したら作業 → 納品

```bash
python3 run_pipeline.py deliver
```

メニューから受注した案件を選ぶ → 自動で実行・念査・納品文生成。
ブラウザに納品ページが開くのでファイル添付して送信。

## 4. トラブルシュート

| 症状 | 原因 | 対応 |
|------|------|------|
| `[SKIP] セッション未設定` | セッション切れ | `00_session_setup.py` 再実行 |
| 検索結果 0 件 | サイト構造変更 / セッション期限切れ | デバッグ用 `outputs/*_debug.png` を確認 |
| `ANTHROPIC_API_KEY が設定されていません` | `04_execute.py` の Claude スクリプト生成は API 必須 | キー設定するか、04 を手動実装で代替 |
| 全件 NO-GO | キーワードに対し市場が薄い／詐欺案件多い | 別キーワードを `01_search.py` に追加 |

### 既知の制約
- `03_apply.py` 単体実行時、`recommend` フィールドで絞り込むがこのフィールドは
  `02_evaluate.py` の出力に存在しない。`run_pipeline.py` 経由なら問題なし。
- `04_execute.py` のスクレイピング自動実行はサンドボックス環境で `subprocess`
  を使うため、本番環境では Claude が生成したコードを必ず目視確認すること。

## 5. 安全運用ルール

- 各プラットフォームの規約変更を月1回チェック（自動応募 NG なら半自動運用に切替）
- セッションファイル（`.sessions/`）は git 除外済み（`.gitignore` 参照）
- 納品物（`outputs/`）も git 除外済み（顧客情報保護）
- 応募文は **必ず人間が最終確認** してから送信する
