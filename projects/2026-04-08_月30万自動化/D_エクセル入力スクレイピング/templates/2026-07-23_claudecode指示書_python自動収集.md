> 柱D（エクセル入力・Webスクレイピング）で受注した案件を Claude Code に実装させるための

## 0. 使い方（3ステップ）

1. **要件を確定する** — 対象URL・取得項目・出力形式・件数・ページネーション有無を案件から拾う。
2. **§2 の枠をコピー** し、`[ ]` の箇所を埋める（迷ったら §4 のチェックリストで確認）。
3. Claude Code に貼る → 出力コードを **少件数で試走** → 問題なければ本番件数で実行。

> ⚠️ **貼る前に §3「コンプライアンス」を必ず確認**。規約NG・個人情報を含む案件はそのまま実装させない。

## 1. どのテンプレを使うか

| 案件のタイプ | 使うテンプレ |
|---|---|
| 静的HTML（JSなしで中身が見える） | §2-A（requests + BeautifulSoup / 軽量・高速） |
| JS描画・ログイン・「次へ」で動的読み込み | §2-B（Selenium / 重いが確実） |
| どちらか判断できない | §2-B を使う（Selenium は静的サイトにも使える） |

判定のコツ: 対象ページで「ページのソースを表示」して欲しい項目の文字列が見えれば静的（§2-A）、
見えず開発者ツールの Elements にしか無ければ動的（§2-B）。

## 2-A. 【静的サイト用】requests + BeautifulSoup

```
以下の要件に基づき、PythonでWebサイトからデータを収集し一覧リストを作成するスクリプトを作成してください。

# 目的
[例：指定した店舗一覧ページから店舗情報を取得し、Excelリストを作る]

# 使用技術
- Python 3.x
- requests + BeautifulSoup4（HTMLが静的なため軽量構成）
- pandas, openpyxl（整形・Excel出力）

# 処理フロー
1. 対象URL: [取得元URLを入力]
2. 取得したいデータ項目（CSSセレクタは実際のHTMLを見て決めてよい）:
   - [例：店舗名]
   - [例：所在地]
   - [例：電話番号]
   - [例：URL]
3. ページネーション: [なし / 「次へ」リンクを辿る / ?page=N を1〜Nまで]
4. 出力: 全件を pandas.DataFrame にまとめ、[output_list.xlsx / output_list.csv] で保存

# 例外処理・負荷対策
- 目的の要素が無い場合はエラーで止めず、その項目を空欄（None）にして次へ進む
- リクエストの間に time.sleep(2) 程度の待機を入れる（サーバー負荷軽減）
- User-Agent を通常ブラウザ相当に設定する
- HTTPエラー（4xx/5xx）は該当ページをスキップし、ログに残して続行する

# 出力してほしいもの
- そのまま実行できる完全なPythonコード
- 冒頭に必要な `pip install` コマンド
- どのCSSセレクタを何の項目に対応させたかのコメント
```

## 2-B. 【動的サイト用】Selenium（ヘッドレス）

```
以下の要件に基づき、PythonとSeleniumを用いたデータ収集・リスト作成の自動化スクリプトを作成してください。

# 目的
手作業のデータ入力を効率化するため、指定サイトから情報を自動取得し Excel（.xlsx）で一覧リストを出力する。

# 開発環境・指定ライブラリ
- Python 3.x
- Selenium（webdriver-manager でドライバ管理を自動化）
- pandas, openpyxl（整形・Excel出力）

# 処理フロー
1. ブラウザ起動: Selenium で Chrome をヘッドレスモードで起動
2. アクセス & 取得: 対象URLから以下を CSSセレクタ / XPath で取得
   対象URL: [取得元URLを入力]
   - [例：会社名 / 組織名]
   - [例：所在地（住所）]
   - [例：電話番号]
   - [例：WebサイトのURL]
3. ページネーション: 「次へ」ボタンがあれば自動クリックし、複数ページを取得し続ける
   （最大 [N] ページで打ち切る / 「次へ」が無くなるまで）
4. エクスポート: 全データを pandas.DataFrame に格納し [output_list.xlsx] で保存

# 例外処理・負荷対策
- 目的の要素が存在しない場合はエラーで止めず「取得不可」または空欄として次へ進む
- ページ遷移・クリックの間は WebDriverWait（明示的待機）を基本とし、必要に応じ time.sleep(2〜3) を併用
- 要素待ちがタイムアウトしたページはスキップし、ログに残して続行する

# 出力してほしいもの
- そのまま実行できる完全なPythonコード
- 冒頭に必要な `pip install` コマンド
- 主要なセレクタ・待機処理にコメント
```

## 3. コンプライアンス（貼る前チェック・必須）

柱Dの受注方針（`brief.md` / `templates/profile_bio.md`）と全社制約に沿うこと。

- **利用規約・robots.txt を先に確認**。スクレイピング禁止のサイトは実装しない → 「半自動（人手で収集）」に切替。
- **個人情報（PII）の収集案件は受けない**。氏名・個人の連絡先・住所などを対象とする案件は辞退。
  - 収集してよいのは「公開されている事業者情報（店名・会社名・代表電話・公開URL 等）」まで。
  - 会社CLAUDE.md 制約A4に準拠：具体的な地区名・町名・番地レベルの個人住所は扱わない。
- **アクセス負荷をかけない**。`sleep` を必ず入れ、短時間の大量並列アクセスをしない。
- **AI補助の明示**。納品時に「AI補助を使用」と透明性をもって伝える（プロフィール文と整合）。
- 迷ったら実装前にオーナーへ確認（自動応募・規約グレーは特に）。

## 7. 記入例（完成サンプル・そのまま貼れる形）

```

# 目的
```

**埋め方の対応表**（この例で何をどう決めたか）:

| `[ ]` の箇所 | この例での埋め方 | 決め方 |
|---|---|---|
| 目的 | 加盟店一覧→事業者情報のExcel化 | 案件文の1行要約 |
