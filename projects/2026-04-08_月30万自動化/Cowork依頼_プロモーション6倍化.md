# Cowork 依頼：プロモーションを6倍化する（多チャネル展開）

**作成日**: 2026-06-21  
**依頼元**: オーナー → Cowork（オーナーMac・ネット/ログイン有効）  
**準備**: リモートAI（データをもらえば分析・素材作成する）  
**背景**: 売上ゼロの主因は商品ではなく「到達（プロモーション）が弱すぎる」。今1チャネル(note)を実働5〜6チャネルにする。

---

## ⚠️ 大原則：宣伝の前に「何が読まれてるか」を測る

闇雲に6倍宣伝すると「空振りを6倍」する。
**まず実データを取り、伸びてる記事・テーマを増幅する。** 宣伝とニーズ測定を同時にやる。

---

## STEP 0（最初にやる・素材より前）：媒体の棚卸し

宣伝素材は用意済みだが、**そもそも投稿する器（アカウント）があるか未確認**。
6倍チャネルの前に、今の媒体の生死をはっきりさせる。

```
【生死を確認する】
□ note    … 稼働中（確認のみ）
□ Gumroad … 稼働中（商品Solo CEO OSあり・確認のみ）
□ X(Twitter) … アカウントは生きているか？ フォロワー数は？（休眠なら再開 or 作り直し）
□ BOOTH   … 登録済みか？ 未登録なら開設するか判断
□ Mailchimp … 設定完了しているか？ 使える状態か？

【新規開設（富山コンテンツと相性最良なのにゼロ）】
□ Instagram … 富山・高岡の食/風景アカウントとして新規開設
   プロフィールにnote/Gumroadリンク

【記録】
□ 上記の結果（媒体名・状態・フォロワー/登録数）を context/STATE.md 末尾に書く
   → リモートAIはこれを見て「実際に使えるチャネル」だけで戦略を組み直す
```

**ここで判明した「生きている媒体」にだけ、用意済みの素材
（`CMO/outputs/プロモ素材_富山定番5本_IG_X_SEO.md`）を投入する。**

---

## STEP 1：noteの実数値を取得

Coworkはnoteにログインできる。ダッシュボードから**記事別のビュー数・スキ数**を取得する。

```
□ note ダッシュボード → 各記事のビュー/スキ を上位20本ぶん書き出す
□ 形式：記事タイトル / ビュー / スキ / カテゴリ(食・場所・暮らし・ひとり起業・AI)
□ 書き出し先：context/STATE.md の末尾に「note実績トップ20」として貼る（or リモートAIに渡す）
```

**これが手に入った瞬間、初めて「推測」から抜けられる。**
- 食が伸びてる → 富山食コンテンツを増幅
- ひとり起業が伸びてる → ビジネス層に寄せる
- 1本だけ突出 → その路線に全振り

リモートAIはこの数値を見て、増幅すべきテーマと商品を確定する。

---

## STEP 2：未使用チャネルを立ち上げる（STEP1のデータで優先順位を確定）

**デフォルト推奨：Google SEO × Instagram（プル型＝探してる人に届く）**

### A. Google検索（SEO）
```
□ 伸びてる食・観光記事のタイトルを検索意図に合わせて調整
   （例「富山のアジ」→「富山湾のアジを刺身で｜地元の食べ方と旬」）
□ 見出し(h2)に「富山 ◯◯」など検索語を入れる
□ note記事のハッシュタグ・関連記事リンクを整える
※ 既存記事が資産。「富山 グルメ/観光」は実検索需要あり
```

### B. Instagram（視覚が最強の食・風景）
```
□ アカウント開設（未開設なら）：富山・高岡の食/暮らし発信として
□ 既存の食・場所記事を 画像1枚＋短文キャプション で投稿（週5本目安）
□ ハッシュタグ：#富山 #富山グルメ #高岡 #北陸 #富山観光 等
□ プロフィールにnote/商品リンク
```

### C. X（既存・連投せず1日1本）
```
□ 伸びてる記事を1日1本、ローカルフックで投稿
```

### D. 海外クロスポスト（仕組みあり）
```
□ gen_all.py で Reddit/EN 素材を生成して展開（既存ツール活用）
```

---

## STEP 3：1〜2週間、チャネル別に計測

```
□ チャネルごとの流入(クリック/フォロワー増/ビュー)を記録
□ どのチャネルが一番「人を連れてきたか」を比較
□ 勝ちチャネルに集中、負けは撤退
```

---

## ⚠️ 安全ガード（2026-06-12 大量公開インシデント再発防止）

- note記事の**大量自動公開はしない**（自動公開スクリプト不使用）
- **PII禁止**：具体的な地区名・町名・住所を投稿・キャプションに書かない（市レベルまで）
- 各投稿は**目視確認してから**公開
- 1日の新規投稿は常識的な本数に抑える（連投・スパム化しない）

## リモートAI（私）の役割

- STEP1のnote数値をもらったら → 増幅すべきテーマ・商品を確定（推測でなくデータで）
- 各チャネルの**素材テンプレ**（SEO記事リライト案・IGキャプション・Xスレッド）を作成
- STEP3の計測結果から次の手を出す

**まずSTEP1の数値を取ってください。それが全ての起点です。**
