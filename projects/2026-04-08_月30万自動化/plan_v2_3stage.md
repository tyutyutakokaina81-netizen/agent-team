## 1. v1からの方針転換

### v1（4本柱）がなぜ失敗したか

| 問題 | 症状 |
|---|---|
| プロダクトアウト | 「自分が作れるもの」起点、顧客不在 |
| 設計ループ | 4日間で設計書量産、実売上ゼロ |
| automation-before-validation | 手作業1件もせずにパイプラインを書いた |
| 実績ゼロで高単価 | ¥15K記事を何の実績もなく売ろうとした |
| 既存資産の無視 | オーナーのClaude Code経験を武器にしていない |

### v2の核心原則

**「AIチームは既に持っているものを増幅するだけ」** — オーナーが既にClaude Code / CLAUDE.md / 多役職フレームワークを構築した4日間の経験そのものを売る。
