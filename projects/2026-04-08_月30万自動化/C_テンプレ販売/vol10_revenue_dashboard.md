## ダッシュボード構成

### シート1：今月のサマリ（メイン画面）
| セル | 項目 | 数式 |
|------|------|------|
| B2 | 対象月 | =TEXT(TODAY(),"YYYY-MM") |
| B3 | 当月目標 | （手入力 例：300000） |
| B4 | 当月実績 | =SUMIFS(取引!E:E, 取引!A:A, ">="&月初, 取引!A:A, "<="&月末) |
| B5 | 達成率 | =B4/B3 |
| B6 | 残目標額 | =MAX(0,B3-B4) |
| B7 | 経過日数 | =DAY(TODAY()) |
| B8 | 残日数 | =EOMONTH(TODAY(),0)-TODAY() |
| B9 | 必要日割 | =B6/B8 |
| B10 | 当月予測着地 | =B4/(B7/DAY(EOMONTH(TODAY(),0))) |
| B11 | ステータス | =IF(B5>=B7/DAY(EOMONTH(TODAY(),0)),"順調","遅れ") |

### シート2：取引（生データ）
| 列 | 項目 | 形式 |
|----|------|------|
| A | 取引日 | YYYY-MM-DD |
| B | 柱 | プルダウン A/B/C/D |
| C | クライアント | テキスト |
| D | 案件名 | テキスト |
| E | 金額（税抜） | 数値 |
| F | 入金状態 | プルダウン 請求済/入金済 |
| G | 入金日 | 日付 |
| H | 備考 | テキスト |

### シート3：柱別 月次推移
| 列 | 項目 |
|----|------|
| A | 年月 |
| B | A柱（SEOライティング） |
| C | B柱（SNS運用代行） |
| D | C柱（テンプレ販売） |
| E | D柱（データ入力代行） |
| F | 月合計 = SUM(B:E) |
| G | 目標 |
| H | 達成率 = F/G |

### シート4：年間累計
| 列 | 項目 |
|----|------|
| A | 月（1〜12） |
| B | 売上（実績） |
| C | 累計売上 |
| D | 月別目標 |
| E | 累計目標 |
| F | 差分 = C-E |
| G | 進捗率 = C/年間目標 |

## 計算ロジック詳細

### ペース判定
```
順調：達成率 >= 経過日数率
遅れ：達成率 < 経過日数率

経過日数率 = 経過日数 / 当月総日数
```

### 来月必要案件数（逆算）
```
A柱：=CEILING(B6/15000, 1)  ← 1記事¥15Kで割る
B柱：=CEILING(B6/50000, 1)  ← 1社¥50Kで割る
C柱：=CEILING(B6/980, 1)    ← 1テンプレ¥980で割る
D柱：=CEILING(B6/15000, 1)  ← 1案件¥15Kで割る
```

### 入金タイミング考慮
```
請求月 ≠ 入金月 のため、キャッシュフローは別管理
シート2の F列「入金状態」と G列「入金日」で追跡
```

## 運用マニュアル

### 日次（30秒）
1. 受注・納品があったらシート2に1行追加
2. 自動でシート1のサマリが更新される

### 週次（5分）
1. シート1で今月の進捗を確認
2. ステータスが「遅れ」なら来週の動きを調整

### 月初（10分）
1. 当月目標をシート1のB3に入力
2. シート3の前月実績を月別推移に転記
3. 各柱の目標配分を見直し

### 月末（15分）
1. 全請求が入金済みかシート2で確認
2. 未入金は督促メール送信
3. シート4の年間累計を確認
