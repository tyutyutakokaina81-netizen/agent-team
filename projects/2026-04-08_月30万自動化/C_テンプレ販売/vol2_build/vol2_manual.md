# SNS投稿カレンダー｜使い方マニュアル

このたびはご購入ありがとうございます。
このマニュアルは5分で読み終わるように作りました。
順番に進めれば、今日から使えます。

---

## 0. このテンプレでできること

- 1ヶ月分のSNS投稿予定を1枚のシートで管理
- Twitter/X・Instagram・TikTok・YouTube Shorts・Threads の5媒体に対応
- 投稿のステータス（未投稿→投稿済→要修正）を色で可視化
- 月末に「投稿達成率・媒体別件数・月別件数」が自動集計される
- 投稿テーマ50選が付属（ネタ切れ防止）

---

## 1. テンプレを自分用にコピーする（所要：1分）

1. 購入時にお渡しした **Googleスプレッドシートのリンク** を開く
2. 画面上部「ファイル」→「コピーを作成」をクリック
3. 保存先：自分のGoogleドライブ
4. ファイル名：そのままでもOK（例：「SNS投稿カレンダー_2026年4月」など月単位の方が管理しやすい）
5. 「コピーを作成」を押すと、自分用のシートが開きます

> 元のテンプレに直接書き込む必要はありません。必ず「コピーを作成」してから使ってください。

---

## 2. 各シートの役割

### シート1：カレンダー（メイン作業画面）

ここに日々の投稿予定を書き込みます。
1日1行が基本です。

| 列 | 入れるもの | 補足 |
|---|---|---|
| A 日付 | 投稿予定日 | 1日入れれば下方向にドラッグでコピー可 |
| B 曜日 | 自動表示 | 触らなくてOK |
| C プラットフォーム | ドロップダウンから選ぶ | 複数媒体に出すなら行を分ける |
| D 投稿テーマ | 投稿の主旨を1行 | シート2から拝借するのが楽 |
| E 本文ドラフト | 投稿の本文 | ここで完成させてから貼る |
| F ハッシュタグ | #を含めた一覧 | 「#マーケ #個人事業」のようにスペース区切り |
| G ステータス | ドロップダウン | 未投稿→投稿済→（差し戻しがあれば要修正） |
| H 投稿URL | 公開後の投稿URL | 後から振り返る時に便利 |

### シート2：テーマ50選（ネタ帳）

何を書くか迷ったらここを開きます。
50個のテーマがA〜Eの5カテゴリに分かれています。

- A. 信頼構築（自己紹介・実績）
- B. 価値提供（学び・ノウハウ）
- C. 共感（失敗談・ビフォーアフター）
- D. 話題便乗（時事・季節ネタ）
- E. エンゲージ（問いかけ・対話）

カテゴリ列のフィルタで絞り込めます。
「狙う感情」と「サンプル切り口」も書いてあるので、そのまま参考にしてください。

### シート3：分析（自動集計）

ここは触らなくて大丈夫。カレンダーシートの内容が自動で集計されます。

- 投稿予定数 / 投稿済 / 未投稿 / 要修正 / 投稿達成率（%）
- プラットフォーム別の件数
- 月別 × ステータス別のクロス集計（QUERY関数）

月末に開いて「今月どうだったか」を眺めるためのシートです。

## 3. 5分運用フロー（毎日続けるコツ）

### 月初（1回・30分）

1. シート2「テーマ50選」を開く
2. 今月使うテーマを15〜20本ピックアップ（A→B→C→D→Eの順でローテーション）
3. シート1「カレンダー」のD列に投稿日に合わせて貼り付ける
4. プラットフォーム（C列）も同時に決めておく
5. G列「ステータス」は全部「未投稿」のままでOK

### 毎日（5分）

1. カレンダーシートを開く
2. 今日の行（薄青でハイライトされている）を見る
3. E列「本文ドラフト」を書く
4. F列「ハッシュタグ」を書く
5. SNSへ投稿
6. 公開後はG列を「投稿済」に変更してH列にURLを貼る
7. 反応イマイチで書き直したいときはG列を「要修正」に変える

### 月末（1回・5分）

1. 分析シートを開く
2. 投稿達成率・媒体別件数・月別ステータスを確認
3. 反応がよかった投稿テーマを別途メモしておく
4. 翌月の月初フローへ

## 4. 自分の業種に合わせてカスタマイズする

### プラットフォームを変える場合

C列のドロップダウンを編集します。

1. C2:C200 を選択
2. データ → データの入力規則
3. 値を書き換える（例：LINE公式,Pinterest,note を追加）
4. 保存

> 分析シートの「プラットフォーム別」のCOUNTIF式も同じ名前に書き換えてください。

### テーマを業種特化に置き換える

シート2のB列「テーマ」を、自分の業種で言い換えます。

例：飲食店なら「3秒で覚えられる豆知識」→「お店のこだわり食材を1つ紹介」
例：講師業なら「お客様の感想（許可済）」→「受講生の変化エピソード」

> カテゴリ（A〜E）は変えない方が、A→Bの順番ローテが崩れず使いやすいです。

### ステータスを増やす

「校正待ち」「上司確認」など独自のステータスを足したい場合：

1. G列のデータ入力規則に項目を追加
2. 必要なら条件付き書式（カスタム数式 `=$G2="校正待ち"`）も追加して色分け

## 5. トラブルシューティング（よくある3つ）

### Q1. 曜日が「Mon」「Tue」になってしまう

A. B2のセルを選択し、数式バーで `=TEXT(A2,"ddd")` の **A2** が日付セルを指しているか確認してください。
それでも英語表記になる場合：

1. ファイル → 設定 → ロケール → 「日本」に変更
2. シートを再読み込み
3. それでも直らない場合は `=TEXT(A2,"aaa")` に書き換えると確実に「月・火・水…」になります（aaaは日本語ロケール用）

### Q2. ステータスを変えても色が変わらない

A. 条件付き書式が正しく設定されていない可能性があります。

1. G2:G201 を選択
2. 表示形式 → 条件付き書式
3. ルール一覧に「セルの値が完全一致 → 投稿済 → 緑」「未投稿 → 黄」「要修正 → 赤」の3つが入っているか確認
4. 値の文字がドロップダウンの項目と完全一致しているか確認（半角・全角の混在に注意）

### Q3. 分析シートのQUERYがエラー（#REF! / #VALUE!）になる

A. 主に2つの原因があります。

- **原因1：シングルクォートが全角になっている**
  → QUERYの中の `where A is not null` の部分や、項目名のクォートが全角になっていないか確認
- **原因2：カレンダーシートのデータ範囲が空**
  → 1行も投稿予定が入っていないとエラーになります。試しに1行だけダミーデータを入れてみてください（動作確認後に削除）
