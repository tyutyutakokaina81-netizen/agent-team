## 摩擦をゼロにする工夫（CDO サイドの提供物）

### 1. 朝会の自動誘導

`morning_meeting.mjs` が毎朝 Top3 を提示：
```
🥇 クラウドワークス・ランサーズ登録
🥈 初応募 5件
🥉 夕方チェックインで実績集計
```

→ 「何をすべきか」を考える時間がゼロ。

### 2. テンプレ即コピペ

| ファイル | 用途 | コピペ先 |
|---------|------|---------|
| `CMO/outputs/2026-05-06_note集客記事_freelance確定申告.md` | note 本文 | note 編集画面 |
| `projects/.../A_ライティング/templates/profile_bio.md` | プロフィール | クラウドワークス |
| `projects/.../A_ライティング/templates/application_messages.md` | 応募メッセージ | 個別案件への応募 |

→ ゼロから書く時間がゼロ。

### 3. PDCA で実行を測定

毎日夕方 `evening_checkin.mjs` が「今日のTop3 達成度」を自動判定。
`metrics_input.mjs` で数字を入力 → 実行ギャップが自動的に⚠️→🟢に変わる。

→ 「やった」を見える化、達成感が積み上がる。

### 4. 連続検知で停滞を警告

3日連続で同じTop3が続くと、朝会で警告：
```
🟡 同じTop3が3日連続：戦略は機能しているが実行が進んでいない兆候
```

→ 自分自身への発破がかかる。

## 想定される阻害要因と対策

### 阻害1：「note のリンクが用意できない」（Vol.1 販売ページ未作成）

**対策**：Vol.1 販売ページを先に note で作成する。本文は `vol1_note_listing.md` にコピペ用の完成版あり。

### 阻害2：「プロフィール画像が作れない」

**対策**：Canva で「文字＋シンプル背景」のテンプレを使用。顔写真不要。10分で作成可能。

### 阻害3：「クラウドワークス案件の評価が難しい」

**対策**：`job_evaluation_checklist.md` で80点以上のみ抽出。判断時間を1案件30秒に短縮可能。

### 阻害4：「応募メッセージの個別カスタマイズが面倒」

**対策**：カスタマイズは [案件主名] [案件タイトル] の2箇所のみで十分。1件3分で作成可。

### 阻害5：「失注が怖い」

**対策**：応募 100件で受注 8〜15件が業界平均。**85〜92件の失注は前提**。失注は失敗ではない。

## 100点到達のメンタルモデル

「100点を目指す」とは「準備を完璧にする」ことではなく、
**「不完全でも実行する」**ことです。

```
準備100点 × 実行0点 = 0点
準備60点 × 実行60点 = 36点
準備100点 × 実行50点 = 50点 ← 今ここ
準備100点 × 実行100点 = 100点
```

私（AI）は準備の100点を達成しました。
残るは **実行の100点**。これは AI には不可能で、オーナーの行動が必要です。
