## シート1：営業先リスト（プロスペクト管理）

### 列構成（A〜N列）

| 列 | 項目 | 入力方式 | 備考 |
|----|------|---------|------|
| A | ID | 自動採番 | =ROW()-1 |
| B | 登録日 | 手入力 | YYYY/MM/DD |
| C | 業種 | プルダウン | 飲食店/士業/コーチ/美容院/整体院/歯科/ジム/教育/EC/不動産 |
| D | 会社/店舗名 | 手入力 | 必須 |
| E | 担当者名 | 手入力 | 不明なら空欄可 |
| F | 連絡先（チャネル） | プルダウン | メール/HP問合せ/Instagram DM/X DM/LinkedIn/電話 |
| G | 連絡先（URL/アドレス） | 手入力 | 実際の連絡先 |
| H | InstagramURL | 手入力 | 任意 |
| I | フォロワー数 | 手入力（数値） | 投稿頻度の参考 |
| J | 投稿頻度 | プルダウン | 毎日/週3-5回/週1-2回/月数回/ほぼ停止 |
| K | 推定地域 | 手入力 | 東京都港区など |
| L | 優先度 | プルダウン | A（高）/B（中）/C（低） |
| M | ステータス | プルダウン | 未送付/送付済/返信あり/商談中/契約/失注 |
| N | 備考 | 手入力 | 特記事項 |

### 主要な数式

#### A列：ID自動採番
```
A2: =IF(D2="","",ROW()-1)
```

#### M列のステータス用 条件付き書式
- 範囲：`M2:M500`
- 「契約」→ 背景緑（#d4edda）
- 「失注」→ 背景グレー（#e9ecef）
- 「商談中」→ 背景黄（#fff3cd）
- 「返信あり」→ 背景青（#cfe2ff）

### プルダウン設定

#### C列：業種
- 範囲：`C2:C500`
- 値：`飲食店,士業,コーチ,美容院,整体院,歯科,ジム,教育,EC,不動産`

#### F列：連絡先チャネル
- 値：`メール,HP問合せ,Instagram DM,X DM,LinkedIn,電話`

#### J列：投稿頻度
- 値：`毎日,週3-5回,週1-2回,月数回,ほぼ停止`

#### L列：優先度
- 値：`A,B,C`

#### M列：ステータス
- 値：`未送付,送付済,返信あり,商談中,契約,失注`

### 優先度の判定基準

| 優先度 | 条件（J列・I列の組合せ） |
|-------|----------------------|
| A（高） | 投稿頻度「月数回」or「ほぼ停止」+ フォロワー1,000以上 |
| B（中） | 投稿頻度「週1-2回」 or フォロワー500-1,000 |
| C（低） | 投稿頻度「毎日」（既に運用が回っている）or フォロワー500未満 |

### フィルター活用例
- 「優先度A・ステータス未送付」→ 今週送るべき先
- 「ステータス送付済・送付日3日以上前」→ フォローアップ対象

## シート2：アプローチ履歴（送付・返信ログ）

### 列構成（A〜J列）

| 列 | 項目 | 入力方式 | 備考 |
|----|------|---------|------|
| A | ID | 自動採番 | =ROW()-1 |
| B | 日付 | 手入力 | YYYY/MM/DD |
| C | 営業先ID | 手入力（数値） | シート1のID参照 |
| D | 営業先名 | VLOOKUP自動 | シート1から自動取得 |
| E | アクション種別 | プルダウン | 初回送付/フォロー1/フォロー2/返信受信/商談実施/見積送付/契約締結/失注 |
| F | チャネル | プルダウン | メール/Instagram DM/電話/Zoom/対面 |
| G | 文面/メモ | 手入力（長文） | 送った内容・受けた返信の要約 |
| H | 結果 | プルダウン | 返信なし/興味あり/興味なし/即決/検討中 |
| I | 次のアクション | 手入力 | 次に何をすべきか |
| J | 次のアクション期日 | 手入力（日付） | カレンダーに連動 |

### 主要な数式

#### D列：営業先名 自動取得
```
D2: =IFERROR(VLOOKUP(C2,営業先リスト!A:D,4,FALSE),"")
```
→ C列にID入力すると、自動で営業先名が表示

### プルダウン設定

#### E列：アクション種別
- 値：`初回送付,フォロー1,フォロー2,返信受信,商談実施,見積送付,契約締結,失注`

#### F列：チャネル
- 値：`メール,Instagram DM,X DM,電話,Zoom,対面`

#### H列：結果
- 値：`返信なし,興味あり,興味なし,即決,検討中`

### 期日が近いタスクをハイライト
- J列の条件付き書式：
  - `=AND(J2<>"",J2<=TODAY()+1)` → 背景赤（明日まで）
  - `=AND(J2<>"",J2<=TODAY()+3)` → 背景黄（3日以内）

## シート3：契約管理（成約後の運用）

### 列構成（A〜M列）

| 列 | 項目 | 入力方式 | 備考 |
|----|------|---------|------|
| A | 契約ID | 自動採番 | =ROW()-1 |
| B | 契約日 | 手入力 | YYYY/MM/DD |
| C | クライアント名 | 手入力 | |
| D | 業種 | プルダウン | （シート1と同じ） |
| E | プラン | プルダウン | スモール/スタンダード/フル/カスタム |
| F | 月額（税抜） | 手入力（数値） | |
| G | 月額（税込） | 自動計算 | =F2*1.1 |
| H | 契約開始月 | 手入力 | YYYY/MM |
| I | 契約終了月 | 手入力（任意） | 未定なら空欄 |
| J | 投稿数/月 | 手入力（数値） | 8/15/30など |
| K | 担当媒体 | 手入力 | Instagram, X など |
| L | 振込先 | プルダウン | 銀行振込/PayPal/その他 |
| M | 備考 | 手入力 | 特記事項 |

### 主要な数式

#### G列：税込金額
```
G2: =IF(F2="","",F2*1.1)
```

### 月次累計売上の集計（シート上部に追加）
```
セルO1: =SUM(F2:F100)  → 月額売上合計（税抜）
セルO2: =SUM(G2:G100)  → 月額売上合計（税込）
セルO3: =COUNTA(C2:C100)  → 契約社数
```

## 運用フロー（日次・週次）

### 毎日（朝5分）
1. シート2「アプローチ履歴」のJ列で「期日が今日以前」のタスクを確認
2. フォローアップが必要な先に対応
3. KPIダッシュボードでファネル全体を眺める

### 週次（月曜10分）
1. シート1「営業先リスト」で「優先度A・未送付」をフィルタ
2. 今週送付する10〜15件をピックアップ
3. シート5から該当業種の営業文をコピー → カスタマイズ → 送付
4. シート2に送付ログを記録

### 月次（月末30分）
1. シート4「KPIダッシュボード」で月次振り返り
2. 業種別の反応率を見て、来月の重点業種を決定
3. 契約獲得社の月額売上を CFO の売上記録シートに連携
4. 失注先の理由をシート1のN列に記録（改善材料）

## プライバシー・セキュリティ

### 絶対にコミットしないもの
- 営業先の実名・連絡先
- スプレッドシートのURL
- 契約金額の実数値

### 保存先
| データ | 保存先 |
|--------|--------|
| スプレッドシート本体 | オーナーの個人GoogleDrive（プライベート） |
| スプレッドシートURL | `CSO/outputs/sales_sheet_url.md` (.gitignore保護下) |
| 契約済クライアント情報 | `CSO/outputs/clients/`（.gitignore保護下） |
| 本仕様書 | プロジェクト配下（公開可・実データなし） |

### Googleスプレッドシート側のセキュリティ
- 共有設定：「リンクを知っている人」「閲覧者」にしない
- 二段階認証を有効化（Googleアカウント）
- 不要な閲覧履歴は定期的に確認
