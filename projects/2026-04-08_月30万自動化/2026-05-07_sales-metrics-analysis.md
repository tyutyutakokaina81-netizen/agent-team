## 2. 考察：なぜ売上は依然として 0 か

### 売上の分解式

```
売上 = 応募数 × 採用率 × 平均受注単価 × (1 + 継続契約獲得率 × 継続案件数) × (1 + 単価アップ成功率)
```

- 採用率・評価・継続・単価アップはすべて **「応募が走った後」** に効く後段指標。
- 応募数（活動量）が 0 のままだと、後段をいくら強化しても掛け算で 0 になる。
- ただし採用率の改善だけは「同じ応募数でより多く受注できる」効率改善でもあるため、応募ボリューム拡大とセットで効く。

### 制約条件の特定

| ファネル段 | 状態 | ボトルネック度 |
|-----------|------|--------------|
| ① 応募対象案件の発見 | キーワード5種・上限20件/プラットフォーム | ★★★ 主因 |
| ② 応募文の質（→採用率） | テンプレ汎用文1種類 | ★★ 副因 |
| ③ 作業実行 | パイプライン整備済 | — |
| ④ 納品・評価 | 整備済 | — |
| ⑤ 継続提案・単価アップ | 仕組みなし | ★★（後段最適化、まず①②） |

→ **打ち手の優先順位は ① 案件発見 → ② 応募文** の順。

## 3. 打ち手（提案）

### A. 案件発見の強化（`01_search.py`）

**狙い**：応募候補母数を増やし、かつ「採算の良い案件」「継続前提の案件」が拾えるようキーワードを多面化する。

- 既存キーワード（汎用5種）を維持しつつ、以下を追加：
  - 形式系：リスト作成 / 転記 / 名簿作成 / 一覧化
  - 高単価系：Excel自動化 / VBA / マクロ / API連携 / 定期業務
  - 継続系：定期 / 月次 / 週次（継続契約に直結しやすい）
- 取得上限を 1キーワード10件 → 15件、プラットフォーム上限 20 → 30 に引き上げ。
- ただしクローリング負荷を抑えるため、詳細ページ取得は引き続き上位 20 件。

### B. 応募文の改善（`03_apply.py`）

**狙い**：採用率 8% → 12% を実装側から後押しする。あわせて継続・単価アップの布石を初回応募に仕込む。

- カテゴリ別テンプレを精緻化（Excel入力 / スクレイピング / データ収集の3系統）。
- 応募文に以下4要素を必ず含める：
  1. **案件タイトルへの具体的応答**（汎用文ではなく要件への理解を見せる）
  2. **AI支援＋人間ダブルチェックの明示**（品質シグナル）
  3. **納期確認の能動姿勢**（信頼シグナル）
  4. **継続・追加対応の余白を残す一文**（継続契約・単価アップの布石）
- 文字数 200〜300字を厳守し、押しつけがましさを排除。
