# 受注後の納品プロセス完全ドキュメント（柱A SEOライティング）

**用途**：クラウドワークス・ランサーズで受注 → 納品 → 入金 までの全工程を、抜け漏れなく標準化する。
**前提**：本ドキュメント通りに動けば、初受注でも品質・スピード・継続率を最大化できる。
**費用**：¥0（既存ツール内で完結）

---

# 全体フロー（タイムライン）

```
受注確定 ──→ Day 0 ：受注御礼＋確認（30分以内）
            ↓
            Day 1 ：構成案提示（任意）
            ↓
            Day 2-3：執筆（AI補助＋人間レビュー）
            ↓
            Day 3-4：自己チェック（30分）
            ↓
            Day 4 ：納品＋次回打診
            ↓
            Day 5-30：修正対応（あれば）＋評価獲得
            ↓
            Day 15-30：プラットフォーム手数料引き入金
```

# Phase 1：受注確定後（Day 0・30分以内）

## アクション

### ① 御礼メッセージ送信（5分）

下記をテンプレ化して即送信。

```
[案件主名] 様

このたびは○○の案件にてご縁をいただき、ありがとうございます。
SEOライティングを承りました○○です。

【現時点の認識共有】
・キーワード：[キーワード]
・想定文字数：[文字数]字
・納期：[納期]
・参考記事URL：[あれば記載]

執筆前に簡易な構成案を別途お送りしてもよろしいでしょうか？
方向性のすり合わせを早めにできれば、修正回数を減らせます。

何卒よろしくお願いいたします。
```

→ 24時間以内に返信があればフェーズ2へ。
→ 返信がない場合：48時間後に1回だけリマインド。

### ② Notion または応募実績スプレッドシートに転記（5分）

- ステータスを「受注済」に更新
- 受注日・納期・単価・案件URLを記録
- Vol.5 案件管理 Notion テンプレを使うとスムーズ

### ③ 自己進行カレンダーへ予定登録（5分）

- 「Day 1：構成案作成」
- 「Day 2-3：執筆」
- 「Day 4：納品」
- Google カレンダー / Outlook に登録（既存契約内）

### ④ 不明点の整理（10分）

下記の点で**疑問が1つでもあれば、Phase 2 の構成案で確認**：

- ペルソナ・ターゲット読者
- トーン・文体（です/ます調 or である調）
- 内部リンク・外部リンクの指定
- 画像挿入の指示
- 公開先（自社ブログ / 外部 / 寄稿）
- 著者名（ゴーストライター可否）

# Phase 2：構成案の提示（Day 1・任意）

## なぜ任意か
案件主によっては「構成不要・本文のみで」と指示があることが多い。指示がない場合のみ、こちらから提案する形がベター。

## 提案する場合のテンプレ

```
[案件主名] 様

執筆前のすり合わせとして、構成案をご共有いたします。

━━━━━━━━━━━━━━━━━━━━
■ 記事タイトル（仮）
[タイトル]

■ 想定キーワード
[キーワード]

■ メタディスクリプション（仮）
[160字以内のメタ]

■ 見出し構成
H2: [見出し1]
　└ H3: [小見出し1-1]
　└ H3: [小見出し1-2]
H2: [見出し2]
　└ H3: [小見出し2-1]
　...

■ 文字数配分
- 導入：100〜200字
- 各H2：500〜700字
- まとめ：200字

合計：[文字数]字
━━━━━━━━━━━━━━━━━━━━

ご確認いただき、修正・追加のご要望があればご教示ください。
ご了承いただけましたら、本文執筆に進みます。

[氏名]
```

→ 構成案承認 → 執筆開始
→ 構成案に修正要望 → 反映後再提示（最大2回まで）

# Phase 3：執筆（Day 2-3）

## ワークフロー（AI補助 × 人間レビュー）

### ステップ1：Claude Pro でドラフト生成（30分）

下記のプロンプトを Claude Pro に貼って初稿を生成：

```
あなたはSEOライターです。以下の条件で記事を作成してください。

【記事情報】
- タイトル：[タイトル]
- ターゲット：[ペルソナ]
- 想定キーワード：[キーワード]
- 文字数：[文字数]字
- トーン：[です/ます調 or である調]

【見出し構成】
[Phase 2 で承認された構成]

【絶対ルール】
1. 出典のない断定をしない（数字・統計には必ず一次情報を引用）
2. AI 特有の決まり文句を使わない（「いかがでしたでしょうか」等）
3. 各H2の冒頭3行で読者の興味を引く
4. 比較表・箇条書きを適切に使用
5. メタディスクリプション（160字以内）を最後に付ける

【出力形式】
Markdown 形式で、コードブロックの中に本文全体を入れてください。
```

→ Claude Pro 内で完結（既存契約 ¥3K/月内・追加費用なし）

### ステップ2：事実確認（30分）

- 数字・統計の出典確認
- 競合記事との重複チェック（無料：Google Docs Reverse search 等）
- リンク先の生存確認

### ステップ3：トーン調整・リライト（60分）

- 案件主のサイトを再確認し、トーンを合わせる
- 同じ言い回しの連続を排除
- 読み上げて違和感のある箇所を修正

### ステップ4：SEO最終チェック（30分）

```
□ タイトルにキーワードが含まれている
□ メタディスクリプションがある（160字以内）
□ H2 にキーワード or 共起語が含まれている
□ 導入100字以内にキーワードが出る
□ 内部リンク1〜3本（指示があれば）
□ 画像 alt テキスト指示（指示があれば）
□ サジェストキーワード3つ以上含む
```

合計：**1記事あたり 2.5〜3時間**（4,000字想定）

# Phase 4：自己チェック（Day 3-4・30分）

## 必須チェックリスト

```
■ 内容
□ 文字数が指定範囲内（±5%以内）
□ 見出し構成が承認版と一致
□ 誤字脱字なし（読み上げチェック）
□ 同じ表現の連続なし
□ 一次情報の引用が正確

■ SEO
□ タイトル：32字以内
□ メタディスクリプション：120〜160字
□ H1〜H3 階層が正しい
□ キーワード密度 1〜3%
□ 内部リンク・外部リンク指示通り

■ ブランド
□ 案件主のトーンに合致
□ NG ワード使用なし
□ 競合製品への言及が指示通り
□ 著者表記（必要なら）

■ 法務
□ 著作権侵害なし（無断転載なし）
□ 薬機法・景表法違反なし
□ 個人情報・固有名詞の取り扱い適切
```

# Phase 6：修正対応（Day 5-30）

## 修正範囲

- **無料修正の範囲**：1回のみ・大幅な構成変更を含まない範囲
- **追加修正の対応**：原則として有料（事前合意必要）

## 修正対応フロー

### 修正依頼受信時
```
[案件主名] 様

修正のご依頼ありがとうございます。
ご指摘内容を確認し、24時間以内にご返信いたします。

[氏名]
```

### 修正対応中（24時間以内）
- 指摘内容を整理
- 不明点があれば確認質問
- 対応可能範囲を明示
- 納期を提示（通常 1〜2日）

### 修正納品時
```
[案件主名] 様

ご指摘いただいた点を反映いたしました。
修正版を以下のURLよりご確認ください。

[Google Docs URL]

【修正内容】
1. [指摘1] → [対応内容]
2. [指摘2] → [対応内容]

引き続きご確認のほど、よろしくお願い申し上げます。

[氏名]
```

# Phase 7：評価獲得＋次回打診（Day 5-7）

## 検収完了の確認

検収が完了し評価依頼ができる状態になったら、下記を送る：

```
[案件主名] 様

このたびは検収完了をいただき、ありがとうございました。

ご好評いただけましたら、ぜひ評価のご入力をお願いいたします。
今後の励みになります。

また、継続でのご依頼をご検討いただける場合は、
２回目以降は受注ペース・単価のご相談も承ります。
お気軽にお声がけください。

今後ともよろしくお願い申し上げます。

[氏名]
```

## 評価獲得のコツ

1. **検収後 24 時間以内** に評価依頼を送る（記憶が鮮明なうち）
2. **強要しない** ：「ご評価いただけますと幸いです」のトーン
3. **悪い評価だった場合**：即座に追加対応 → 評価更新を依頼

# Phase 9：入金確認（Day 15-30）

## クラウドワークス・ランサーズの場合

- 検収完了 → プラットフォーム手数料（5〜20%）控除 → 月末締め翌月末払い
- 入金確認したら **応募実績スプレッドシートの「実入金日」を更新**

## 直接契約の場合

- 請求書発行（CFO の `invoice_templates.md` 参照）
- 入金期限：請求書発行から7〜14日
- 期限超過の場合の督促フロー：

```
振込期限経過後の対応：
- 期限+1日：穏やかなリマインド
- 期限+7日：再請求
- 期限+14日：強い督促 + 振込先確認
- 期限+30日：法務相談（弁護士無料相談 等）
```

# 失敗パターンと対策

## 失敗1：受注後にトーンの認識違い発覚

**原因**：受注前にサンプル記事を提示しなかった
**対策**：応募時に必ずポートフォリオURLを添付

## 失敗2：納期遅延

**原因**：受注本数が処理能力を超えた
**対策**：週5本までに上限設定 / 受注時に必ず「他案件状況」を社内チェック

## 失敗3：低評価（★3以下）獲得

**原因**：構成・トーンの認識違い・誤字脱字
**対策**：
- Phase 2 の構成案承認を厳守
- Phase 4 の自己チェックを30分以上確保
- 即対応で評価更新を依頼

## 失敗4：継続が途切れる

**原因**：納品後の連絡が機械的
**対策**：
- 納品時に「次回のテーマ案」を1〜2個提案
- 月1回、軽い近況報告（業界トレンド・関連記事のシェア）

# まとめ：このドキュメントの使い方

1. **初受注時に印刷 or タブで開いておく**（行き当たりばったりを避ける）
2. **失敗があれば「失敗パターンと対策」を更新**（このドキュメント自体が学習する）
3. **月次レビューで Phase 7-8 の継続率を確認**

このドキュメント通りに動けば、**初受注からの継続契約獲得率は60%以上** が現実的な目標です。
