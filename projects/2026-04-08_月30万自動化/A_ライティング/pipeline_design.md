# 柱A 受注ワークフロー パイプライン設計

担当：CDO  
方針：柱D（エクセル/スクレイピング）のパイプラインを再利用しつつ、ライティング固有の要件を加える。
有料 API は使用しない（ルールベース・テンプレートで完結）。

---

## 1. 全体フロー

```
[初回] オーナーがクラウドワークス・ランサーズにライター用プロフィールを整備
       ↓
[毎朝]  python3 run_pipeline.py search   # 柱A モード
   │
   ① 01_search.py    ライティング系キーワード16種で検索
   ② 02_evaluate.py  ライティング向け4軸採点
   ③ 03_apply.py     カテゴリ別応募文生成（4要素＋執筆サンプル提示）
   │                  → ブラウザ自動オープン → 人手で応募
   │
[受注後]
   ④ 04_outline.py   キーワード・参考URLから見出し構成を生成
   ⑤ 05_draft.py     見出しごとに本文ドラフト（テンプレ）
   ⑥ 06_review.py    ライティング念査（文字数・キーワード密度・重複文・冗長表現）
   ⑦ 07_deliver.py   納品文生成（評価★4獲得構成）
```

柱D との共通モジュール：`00_session_setup.py`、`run_pipeline.py`、検索基盤  
柱A 固有モジュール：04-07（執筆系）

---

## 2. 柱D との差分

### 検索キーワード（柱A 16 種）

| カテゴリ | キーワード |
|---------|----------|
| 形式系 | SEO記事、ブログ記事、コラム執筆、リライト |
| 業種系 | 美容、不動産、医療、法律、IT、士業 |
| 高単価系 | YMYL、専門記事、取材記事、監修記事 |
| 継続系 | 定期、月次、複数本セット |

### 評価軸の調整

柱D 4 軸を柱A 用にリマップ：

| 軸 | 柱D | 柱A |
|----|-----|-----|
| 軸1 | 技術実現性 | 専門性適合（取材不要・公知範囲） |
| 軸2 | 法的・規約リスク | YMYL 該当度・薬機法/景表法リスク |
| 軸3 | 採算性 | 文字単価 ¥1.5/字以上か |
| 軸4 | 要件明確性 | キーワード・文字数・トーンが明記されているか |

GO 基準は同じく 70 点以上。

### 応募文（4 要素 + 1）

柱D の 4 要素に **執筆サンプル提示** を追加：
1. 案件タイトルへの具体的応答
2. 品質シグナル（AI 補助＋人間レビュー）
3. 納期確認の能動姿勢
4. 継続・追加対応の余白
5. **同ジャンルの執筆サンプルへのリンクを 1 件提示**（採用率底上げ）

## 3. 各モジュール仕様

### 04_outline.py（見出し生成）
- 入力：メインキーワード、文字数、ペルソナ、参考 URL（任意）
- 出力：H2 見出し 4-6 個、各 H2 配下 H3 を 2-3 個
- 実装：ルールベース（キーワード共起・SEO 構造テンプレ）
- 注：API は使わない。Claude Code（ローカル）でカスタマイズしたいときは、生成された見出し案を手で調整する想定

### 05_draft.py（本文ドラフト）
- 入力：04 が出した見出し構成、ターゲット文字数、トーン
- 出力：見出しごとの本文ドラフト（Markdown）
- 実装：「リード文 → 結論 → 根拠 3 → まとめ」のフレームに沿ったテンプレ展開
- 文字数調整：H2 ごとに目標字数を割り当て、ドラフト時点で ±10% 内に収める

### 06_review.py（念査）
ルールベースで以下を機械判定：
- 文字数（指定 ±10% 内）
- メインキーワード出現密度（2〜5%）
- 同一文の重複・近似文（編集距離で判定）
- 冗長表現（「〜することができます」「〜ということ」等）
- 禁止表現（断言・誇大・薬機法 NG ワード辞書）

### 07_deliver.py（納品文）
柱D の 06_deliver.py のテンプレを流用。プレースホルダだけ差し替え：
- 「作業件数」→「納品文字数」
- 「データ件数」→「執筆記事本数」

## 4. 共通基盤の再利用

| モジュール | 柱A への流用方法 |
|----------|--------------|
| `00_session_setup.py` | そのまま流用（プラットフォーム同一） |
| `01_search.py` | キーワードリストを差し替え（KEYWORDS 定数を柱A 用に） |
| `02_evaluate.py` | 軸定義を上書きするフックを追加（pillar 引数） |
| `03_apply.py` | テンプレ関数を pillar ごとに分岐 |

→ 既存ファイルを編集せず、`pillar="A"` を渡す薄いラッパー方式で衝突を避ける。

```

└── pipeline/                ← 実装は次タスクで追加
```
