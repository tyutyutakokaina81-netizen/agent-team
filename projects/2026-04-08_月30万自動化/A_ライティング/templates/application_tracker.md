# SEOライティング：応募実績トラッカー

**用途**：応募・受注の実績を記録し、月次で改善するためのスプレッドシート構造。
**実装**：Google スプレッドシート（無料）に下記のシート構造を作る。
**運用**：1応募あたり30秒で記録できるシンプルな構造を維持する。
**費用**：¥0（既存 Google アカウント・無料スプレッドシート）

---

# シート1：応募ログ（Applications）

毎日の応募を1行ずつ記録する。

## 列構成

| 列 | 項目 | 入力例 | 入力方法 |
|----|------|--------|---------|
| A | 応募日 | 2026-05-07 | 手入力（=TODAY()でも可） |
| B | プラットフォーム | クラウドワークス | プルダウン |
| C | 案件タイトル | 美容コラム執筆募集 | 手入力 |
| D | ジャンル | 美容 | プルダウン |
| E | 文字単価（円） | 1.2 | 数値 |
| F | 想定文字数 | 4000 | 数値 |
| G | 想定報酬（円） | 4800 | =E*F（自動計算） |
| H | 評価スコア | 85 | 数値（job_evaluation_checklist.md で算出） |
| I | 案件URL | https://crowdworks.jp/... | リンク |
| J | 使用テンプレ | 美容 | プルダウン |
| K | 応募ステータス | 応募済 | プルダウン |
| L | 案件主返信 | あり/なし/24h以内 | プルダウン |
| M | 結果 | 受注/不採用/未確定 | プルダウン |
| N | 受注金額（円） | 4800 | 数値（受注時のみ） |
| O | メモ | 構成指示あり、急ぎ案件 | フリーテキスト |

## プルダウン候補

```
B（プラットフォーム）：クラウドワークス / ランサーズ / その他

D（ジャンル）：美容 / 飲食 / 不動産 / 金融 / 医療 / IT / フリーランス / その他

J（使用テンプレ）：美容 / 飲食 / 不動産 / 金融 / 医療 / IT / カスタム

K（応募ステータス）：応募済 / 質問中 / 提案待ち / 取り下げ

L（案件主返信）：24h以内 / 24〜72h / 72h以上 / なし

M（結果）：受注 / 不採用 / 連絡なし / 取り下げ / 未確定
```

# シート2：週次サマリー（Weekly Summary）

週次で振り返るための集計。式で自動計算。

## 列構成

| 列 | 項目 | 計算式（例） |
|----|------|-----------|
| A | 週 | 2026-W19 |
| B | 応募数 | =COUNTIFS(Applications!A:A, ">="&開始日, Applications!A:A, "<="&終了日) |
| C | 受注数 | =COUNTIFS(Applications!M:M, "受注", Applications!A:A, ">="&開始日, ...) |
| D | 受注率 | =C/B |
| E | 受注金額合計 | =SUMIFS(Applications!N:N, Applications!M:M, "受注", ...) |
| F | 平均単価 | =E/C |
| G | 1応募あたり期待金額 | =E/B |

## 目標値

| 指標 | Week 1 目標 | Week 4 目標 |
|------|-----------|-----------|
| 応募数 | 25 | 50 |
| 受注率 | 8% | 15% |
| 受注数 | 2 | 7〜8 |
| 月間売上 | ¥30,000 | ¥120,000 |

# 運用ルール

## 入力タイミング
- 応募直後：応募ログにA〜L列を記入（30秒）
- 結果通知時：M〜N列を記入
- 毎週金曜：週次サマリーを目視で確認
- 毎月初：月次サマリーを CFO に連携

## 改善のループ
```
週次：受注率の低いジャンルを特定
  ↓
受注率の低いテンプレを見直す（言い回し・フォーマット）
  ↓
翌週の応募で改善版テンプレを試す
  ↓
2週間後、改善前後を比較してテンプレ確定
```

## CFO 連携
- 月次サマリー（受注金額・手数料）を月初に CFO へ送付
- CFO は本データを基に確定申告・経費計算に使用

# Google スプレッドシートのテンプレ作成手順

1. Google ドライブで新規スプレッドシート作成
2. ファイル名：「2026_SEOライティング応募実績」
3. 5シートを作成（Applications / Weekly / Monthly / Genre / Clients）
4. 列見出しを上記の通り入力
5. プルダウンは「データ → データの入力規則」で設定
6. 共有設定：自分のみ閲覧可（センシティブ情報のため）
7. 月初にバックアップを `CFO/research/` 配下に CSV エクスポート（gitignore対象）
