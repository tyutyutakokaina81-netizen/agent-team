# A：SEOライティング 受注運用ワークフロー

担当：CDO（自動化設計）× CSO（受注対応）× CMO（品質）
作成日：2026-05-19

## 目的

A：SEOライティング案件を「問い合わせ → 納品 → 入金」まで滞りなく回す運用を定義する。
1人運営でも月20本（スタンダード中心）を回せる前提で設計する。

---

## 全体フロー（7ステップ）

```
[1] 案件発見・応募
       ↓
[2] 受注確定・キックオフ
       ↓
[3] 構成案作成・承認
       ↓
[4] 本文生成（Claude）
       ↓
[5] レビュー・編集
       ↓
[6] 納品・修正対応
       ↓
[7] 請求・入金確認
```

各ステップに想定所要時間とテンプレ／成果物を紐づける。

---

## Step 1：案件発見・応募

### 発見ルート（優先順）
1. クラウドワークス（「ライティング」「SEO記事」で日次検索）
2. ランサーズ（同上）
3. SNS経由のDM相談（CMO投稿経由）

### 応募基準（受ける／受けない）
| 条件 | 判定 |
|------|------|
| 単価が文字単価2円以下 | 受けない |
| AI使用禁止が明記 | 受けない |
| 文字数1,500字未満 | 受けない（時間効率悪） |
| 専門資格必要（医療・士業） | 受けない |
| 上記以外 | 応募する |

### 応募文テンプレ

```
[クライアント名] 様

はじめまして。SEOライティングを承っております○○と申します。
募集要項を拝見し、ぜひお力になれればと思いご応募いたしました。

【提案ポイント】
・AI執筆支援を活用した短納期対応（4,000字 / 3営業日）
・H2/H3構成案を事前にご確認いただいた上で執筆
・納品後1回までの修正対応無料

【関連実績】
・[実績①ジャンル：例 美容系オウンドメディア]
・[実績②ジャンル：例 BtoB SaaS紹介記事]

ご検討のほどよろしくお願いいたします。
```

## Step 2：受注確定・キックオフ

### キックオフでクライアントから取得する情報
- ターゲットキーワード（メイン + サブ）
- 想定読者ペルソナ
- 競合参考URL（最大3本）
- トーン・口調（です／ます、だ／である、フランク 等）
- 文字数（最低・上限）
- 納品形式（Googleドキュメント／Word／テキスト）
- 納期
- NGワード・禁止事項
- 内部リンク要否

### キックオフ確認テンプレ

```
ご依頼ありがとうございます。
着手前に以下を確認させてください。

▼ 案件詳細
・メインKW：
・サブKW：
・想定読者：
・参考URL：
・トーン：
・文字数：
・納品形式：
・希望納期：
・NGワード：

ご返信いただき次第、構成案を作成しまずご確認いただきます。
```

## Step 3：構成案作成

### Claudeプロンプト（構成案生成）

```
以下の条件でSEO記事の構成案を作成してください。

【メインKW】
[KW]

【サブKW】
[KW]

【想定読者】
[ペルソナ]

【参考競合URL】
[URL]

【目標文字数】
[字数]

【出力形式】
- H1（タイトル案：3パターン）
- リード文（150字程度）
- H2見出し（5〜6本）
  - 各H2のH3見出し（2〜3本）
  - 各H2の想定文字数
- まとめ
- メタディスクリプション（120字）

【条件】
- 検索意図を最優先に網羅性重視
- 競合の見出しを参考にしつつ独自視点を1つ加える
- 想定読者の悩みに対する具体的解決策を見出しに含める
```

### クライアント確認テンプレ

```
構成案を作成しました。ご確認ください。

▼ 構成案
[構成案を貼り付け]

▼ ご確認ポイント
1. タイトル案のうちご希望のもの
2. 見出しの過不足
3. 各H2の想定文字数のバランス

問題なければ「OK」とご返信いただければ本文執筆に着手します。
修正点があれば箇条書きでお知らせください。
```

## Step 4：本文生成（Claude）

### Claudeプロンプト（本文生成）

```
以下の構成案に従ってSEO記事の本文を執筆してください。

【構成案】
[Step3で承認された構成案]

【トーン】
[です・ます／だ・である／フランク]

【文字数】
[目標字数] ±10%

【ルール】
- 各H2の冒頭で読者の疑問を一言提示
- 具体例・数字・体験談を最低1つ含める
- 一文を80字以内にして読みやすく
- 専門用語は初出時にカッコ書きで補足
- 「である調／ですます調」を混在させない
- 文末バリエーション（「〜です」連続を避ける）

【禁止】
- 出典不明の統計引用
- 「絶対」「100％」など断定的訴求
- 競合他社批判
- [クライアント指定NGワード]
```

## Step 5：レビュー・編集（30分）

### チェックリスト

| カテゴリ | 項目 | 確認 |
|---------|------|------|
| 事実 | 数字・固有名詞・日付 | □ |
| 事実 | 引用元の正確性 | □ |
| 構成 | 見出し階層の正しさ | □ |
| 構成 | 想定文字数±10%以内 | □ |
| 文体 | トーン統一 | □ |
| 文体 | 一文80字以内 | □ |
| 文体 | 文末バリエーション | □ |
| SEO | メインKW出現箇所（H1・リード・H2の1つ） | □ |
| SEO | サブKWの自然な配置 | □ |
| 品質 | 重複表現の削除 | □ |
| 品質 | 誤字脱字 | □ |
| クライアント | NGワード不使用 | □ |
| クライアント | 指定トーン | □ |

### 編集の判断基準
- 1段落で意味が通らない場合：書き直し
- 事実に疑義がある場合：別ソース確認 or 削除
- 抽象的すぎる箇所：具体例を追加
- 「AIっぽさ」が残る箇所：体験談ベースの表現に置き換え

## Step 6：納品・修正対応

### 納品メールテンプレ

```
ご依頼の記事を納品いたします。

▼ 納品物
[Googleドキュメント or 添付ファイル]

▼ 仕様確認
・タイトル：[最終タイトル]
・文字数：[字数]字
・メインKW：[KW]（出現回数：[回]）
・メタディスクリプション：[内容]

▼ 修正について
・納品から7日以内：1回まで無料
・大幅な構成変更：別途お見積り

ご確認のほどよろしくお願いいたします。
```

### 修正受付ルール
- 7日以内・構成内修正：無料（即対応）
- 7日経過 or 構成変更：別途見積り提示
- 修正回数3回以上：追加費用提示

---

## Step 7：請求・入金確認

### クラウドワークス案件
1. 「納品」ボタン → 仮払い解除依頼
2. クライアント検収後、自動入金（15〜30日後）

### 直接契約案件
1. 月末締め → 翌月5日までに請求書送付（CFO担当）
2. 振込期限：翌月末
3. 未入金時：翌月5日にリマインドメール

### CFO連携
- 入金確認後、CFO `_index.md` に売上記録（金額は記載しない、ファイル名のみ）
- 確定申告用に取引データを月次でまとめる

## 品質保証ルール

1. **AI使用明示**：クライアントへの提案文・サービスページで「AI執筆支援」を明示する
2. **人間レビュー必須**：Step 5を省略しない
3. **修正対応の基準明確化**：修正範囲を契約段階で合意する
4. **クライアントトーン優先**：自分のトーンより指定トーンを優先する
5. **NGワード遵守**：Step 2で取得したNGワードリストを編集時に必ず確認する
