# インバウンド応答プレイブック（柱A SEOライティング）

**用途**：受注前後に案件主・クライアントから来る質問・要望への即答テンプレ集。
**目的**：返信スピード（24時間以内 → 1時間以内）と質を両立させ、**受注率と継続率を向上**させる。
**前提**：応募メッセージ（`application_messages.md`）はアウトバウンド用。本書は受け取り側。
**費用**：¥0（既存メール・チャットツールで完結）

---

## 返信速度の業界標準

| 返信時間 | 採用率への影響 |
|---------|------------|
| 1時間以内 | 高（クライアントの記憶が鮮明・他候補への流出を防ぐ） |
| 24時間以内 | 中（業界標準・最低ライン） |
| 24〜48時間 | 低（候補から外される可能性） |
| 48時間超 | 採用率激減 |

→ **すべての返信は 1時間以内** を目標とし、内容に時間がかかる場合は「24時間以内に再返信します」の一報を即送る。

# Section 1：応募後の質問への対応（採用前）

## 1-1. 「サンプル記事を見せてもらえますか？」

```

ご質問ありがとうございます。
サンプル記事を Google Docs にて共有いたします。

▼ サンプル1：フリーランス確定申告解説（4,200字）
[Google Docs URL]

▼ サンプル2：プロジェクト管理ツール5選比較（4,300字）
[Google Docs URL]

ご希望のジャンル・トーンに近い既存サンプルがあれば
追加でお送りすることも可能です。

ご検討よろしくお願いいたします。

[氏名]
```

## 1-2. 「単価交渉できますか？」（初回案件）

```
[案件主名] 様

ご返信ありがとうございます。

初回案件につきましては、現状の応募内容（1文字 [X] 円）でお願いさせていただいております。
品質・納期面でご評価いただければ、2件目以降のご依頼時に
継続契約や条件のすり合わせをさせていただければ幸いです。

なお、もしお急ぎ案件・大量発注などの特殊条件がございましたら、
別途相談させていただけますと幸いです。

何卒よろしくお願いいたします。
```

## 1-3. 「テスト記事（無料）を書いてもらえますか？」

```
[案件主名] 様

ご検討ありがとうございます。

無償でのテスト執筆につきましては、誠に恐縮ですが
ご対応いたしかねます。

代わりに、すでに執筆済みのサンプル記事を共有させていただきます。
以下の Google Docs より、私の品質・トーンをご確認いただければ幸いです。

[Google Docs URL]

または、初回お試しとして「短尺・低価格」での1記事もご相談可能です。
（例：2,000字 ¥3,000 など）

ご検討のほど、よろしくお願いいたします。
```

> **設計意図**：丸呑みすると搾取される業界悪習。代替案を提示しつつきっぱり断る。

## 1-4. 「他のライターと比べて何が強みですか？」

```
[案件主名] 様

ご質問ありがとうございます。

私の強みは下記の3点です。

【1. AI補助×人間品質の両立】

そのうえで必ず人間が事実確認・トーン調整・リライトを行うため、
品質も維持されます。

【2. 取材・一次情報への姿勢】
公式サイト・公的機関・専門書からの引用を必ず明示し、
読者の信頼を得る記事構成を心がけています。

【3. 構成提案からの一貫対応】
本文だけでなく、構成案・メタディスクリプション・サジェスト対応まで
一貫してお引き受けできます。

サンプル記事もぜひご確認ください：[Google Docs URL]

ご検討、よろしくお願いいたします。
```

# Section 2：受注確定後の質問への対応

## 2-1. 「構成案を先に共有してください」

```
[案件主名] 様

承知いたしました。

可能な範囲でご教示いただけますと幸いです。

【ターゲット読者】
（年齢・職業・抱えている悩み等）

【記事のゴール】
（情報提供／商品比較／登録誘導 等）

（御社サイト or 他社サイトのURL）

```

## 2-4. 「修正依頼」が来た場合

```
[案件主名] 様

ご指摘いただきありがとうございます。
24時間以内に修正対応のうえ、修正版をお送りいたします。

念のため確認させてください：
1. [指摘内容1] — このように修正でよろしいでしょうか？
2. [指摘内容2] — 〇〇〇〇という認識でよろしいでしょうか？

不明な点があれば、修正前にご教示いただけますと幸いです。

[氏名]
```

> **設計意図**：修正対応の前に「認識違い」を潰す。質問せずに修正すると別の方向に行きかねない。

# Section 3：継続交渉・アップセル

## 3-1. 「次回もお願いしてもいいですか？」

```
[案件主名] 様

ありがとうございます。次回もぜひお手伝いさせてください。

【継続のご相談】
今回ご好評いただけましたので、
よろしければ「月◯本の継続契約」をご検討いただけませんか？

【メリット（御社）】
✅ 月◯本まで優先対応（即時着手可）
✅ 単価 ¥◯◯（現状から ◯%減）
✅ 月次レポート（成果KPI）作成
✅ 急ぎ案件への対応（24時間納品可）

【条件（私側）】
- 月初に本数の見通しを共有いただく
- 振込：月末締め翌月末払い
- 解約：前月20日までにご連絡

ご相談のほど、よろしくお願いいたします。
```

## 3-2. 「単価を上げてもいいか相談したい」

```
[案件主名] 様

ご検討ありがとうございます。

これまで●本のご依頼をいただき、品質・納期にご満足いただけているようで嬉しく存じます。
つきましては、次の依頼から下記の単価でお願いできれば幸いです。

【現状】
1文字 ¥[X]、4,000字 ¥[X*4000]

【ご提案】
1文字 ¥[X*1.2]、4,000字 ¥[X*4000*1.2]
※20%上昇

【ご提案の理由】
・継続的にご評価いただいており、次のステージに進みたい
・他社からのご依頼との兼ね合い
・現状の単価では月の本数が制限される

ご無理を言って恐縮ですが、ご検討いただければ幸いです。
```

> **設計意図**：単価交渉は「お願い」の体裁ではなく「事実ベース」で。交渉決裂のリスクは織り込み済み。

## 3-3. 「他のライターを紹介してもらえますか？」

```
[案件主名] 様

ありがとうございます。

現状、私個人で対応しておりますので、
他のライターのご紹介はお受けできません。

ただし、私の処理能力を超える本数をご希望でしたら、
ご相談いただければ可能な範囲を率直にお伝えします。

代替案として、私自身の処理ペースを上げる（急ぎ対応）
というご提案もさせていただけます。

ご検討のほどお願いいたします。
```

# Section 4：トラブル・難案件対応

## 4-1. 「期限内に届かなかった」場合の謝罪

```
[案件主名] 様

このたびは納期に遅延が発生し、誠に申し訳ございません。

【現状】
本日中に納品させていただきます。
具体的には[時刻]までにお届け予定です。

【遅延の理由】
（具体的かつ簡潔に。言い訳にならないよう）

【再発防止】
今後、進捗が予定より遅れる兆候が見えた段階で、
事前にご連絡を入れる運用に変更いたします。

なお、お詫びとして次回ご依頼時の単価から[X]%お値引きさせていただきます。

ご迷惑をおかけしましたこと、重ねてお詫び申し上げます。

[氏名]
```

> **設計意図**：謝罪 → 状況 → 理由 → 再発防止 → 償い の順。順番を間違えると逆効果。

## 4-2. 「品質に不満」を伝えられた場合

```
[案件主名] 様

ご指摘ありがとうございます。
ご期待に添えず申し訳ございません。

【ご指摘の理解】
（具体的なポイントを箇条書きで復唱）

【ご対応案】
・全面的な書き直し（無償）：◯日以内
・指摘箇所のみ修正（無償）：◯日以内
・別ライター紹介：私からは現状不可

ご希望の対応をご教示いただければ、即時着手いたします。

このたびはご期待に添えず、申し訳ございませんでした。
今後の参考にさせていただきますので、
具体的な改善ポイントをいただけると幸いです。

[氏名]
```

## 4-3. 「契約外の要望」が来た場合

```
[案件主名] 様

ご相談ありがとうございます。

ご要望の[作業内容]につきましては、当初の契約範囲外となるため
別途お見積もりとさせていただけますと幸いです。

【お見積もり】
・[作業内容]：¥[X]
・納期：[X]日以内

別途のご発注として進めるか、ご検討いただければ幸いです。
当初契約分につきましては、引き続き予定通り対応いたします。

ご検討のほどお願いいたします。
```

> **設計意図**：請けるか断るかの二択にせず、「別途見積もり」で値段交渉に持ち込む。タダ働きを防ぐ。

# Section 5：オブジェクション対応集

応募メッセージや事前面談で発生しやすい反論への返答。

## オブジェクション1：「AI を使ってる人には頼めない」

```
ご懸念ごもっともです。

AI 執筆支援を活用していることをお伝えしているのは、
**透明性とご納得をいただくため** です。

実際のフローは：
1. AI で初稿生成（時間の30%）
2. 人間レビューでの事実確認・リライト（時間の70%）

最終的に **人間が品質を保証** する記事を納品いたします。

「AI 使用なし」の体裁で執筆することも可能ですが、
おすすめはしません。AI 補助を使う/使わないにかかわらず、
真の差別化要因は **人間レビューの深さ** です。

サンプル記事をご確認いただければ、品質をご判断いただけます。
```

## オブジェクション2：「単価が高すぎる」

```
ご検討ありがとうございます。

ご予算とお見合わせのうえ、ご判断いただければ幸いです。
私の単価は、下記を含めた価値設定としております：

・構成案 + 本文 + メタディスクリプション 一括対応
・納品後1回までの修正は無償
・サジェストキーワード対応

もし「本文のみ・修正対応なし」のような simplified 版でしたら、
別途お見積もりも可能です。お気軽にご相談ください。
```

## オブジェクション3：「実績がないので信用できない」

```
ご懸念は尤もです。

実績の証明として下記をご用意しております：

【サンプル記事】
[Google Docs URL] — 4,200字
[Google Docs URL] — 4,300字（業種違い）

【取り組み姿勢】
納品後の評価（★4以上）獲得を最優先しております。
万が一ご満足いただけない場合は、無償リライトまたは
返金対応をさせていただきます。

「初回お試し」での短尺・低価格案件もご相談可能です。
```

## オブジェクション4：「他のライターより速いことを示せるか」

```
速度については下記が目安です：

・2,000字：2日（一般的な相場：3〜5日）
・4,000字：3日（一般的な相場：5〜7日）
・8,000字：5日（一般的な相場：7〜10日）

急ぎ案件であれば 24時間納品の実績もございます。
（その場合は単価+20%）

具体的な納期をご共有いただければ、調整可能か即時お返事いたします。
```

# Section 6：返信運用ルール

## 即返信すべきもの（1時間以内）

- 受注の確定通知
- 納期確認
- 修正依頼
- 単価交渉の打診
- 「他のライターと比較中」のニュアンスがあるメッセージ

## 24時間以内でよいもの

- 構成案の質問（中身を考える時間が必要）
- 詳細仕様の確認
- アップセル・継続契約の提案

## 即返信できないとき（業務中など）

```
[案件主名] 様

ご連絡ありがとうございます。
本日 [X] 時までにご返信いたします。

[氏名]
```

→ 1行でいいので **必ず一報を入れる**。沈黙が最大の機会損失。

# Section 7：効果測定

返信運用を開始したら、下記を月次で記録（CSO/_index.md）：

| 指標 | 算出方法 |
|------|--------|
| 平均返信時間 | 質問受信〜初回返信までの中央値 |
| 1時間以内返信率 | 全質問の何% |
| オブジェクション克服率 | 反論を受けた案件のうち成約した割合 |
| 継続契約獲得率 | 単発案件のうち2件目以降に至った割合 |
| 単価アップ成功率 | 単価交渉を打診したうち成功した割合 |

→ 改善ログに記録し、テンプレを継続的にアップデート。
