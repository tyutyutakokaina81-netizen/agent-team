## 4. 運用ルール（スモールスタートの規律）

- **応募は5件まで**。反応ゼロなら応募文を1回改訂してもう5件。それ以上は撒かない。
- 受注したら：code が下書き→検証→納品原稿まで工場で生産（owner は検収と納品ボタンのみ）。
- 納期は「受注から3営業日」で受ける（工場的には1日で作れるが、バッファを持つ）。
- 禁止：実績の誇張・「AI使用」を隠すこと（聞かれたら「AI活用＋人の検証」と正直に答える）・赤字受注の連発（トライアルは各クライアント1本まで）。
- 報告：応募数・返信数・受注数を週次レビューのB柱に記録。

## 5. 初受注が来たときの流れ

1. 案件内容を ops/inbox 経由で code へ（テーマ・字数・キーワード・納期）
2. code が24時間以内に納品原稿（本文＋タイトル3案＋検証メモ）を CMO/outputs/受託/ に生成
3. owner が検収→納品→請求（CFO様式に金額記録・請求書はCFOテンプレ）
