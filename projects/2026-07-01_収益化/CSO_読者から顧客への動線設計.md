# CSO：読者から顧客への動線（ファネル）設計

- **作成日**: 2026-07-01 ／ **担当**: CSO（code）／関与: CMO(文言)・CAO(KPI/計測)・EN(海外受け皿)
- **依頼**: note読者を"顧客/ファン"に変える動線（ファネル）設計。A1でcodeは実装不可＝**設計と手順まで**。
- **方針**: 「あと5,000ビュー→販売本格化」なので今は**"仕込み"**。押し売り・営業プッシュはしない。**¥0厳守**（固定費を増やす受け皿＝有料メルマガ/CRM等は使わない）。
- **親資料**: `CAO/outputs/2026-07-01_赤字にならない収益化プラン.md` ／ `CMO/outputs/2026-07-01_収益化アセット_アフィリブロック＋PR表記.md` ／ `CMO/outputs/2026-07-01_配信キット_X投稿＋分散ルーティン.md`
- **制約**: A4（PII/実名/地区・住所を書かない・市レベルまで）・A5（テンプレ/誇張NG）。顧客の実名・個人情報は本設計に一切載せない（**設計のみ**）。
- **本体の置き場所メモ**: `CSO/outputs` は .gitignore 対象（顧客PII保護）のため、本設計は PII を含まない設計物として `projects/` に置く。

## 0. 結論（要点3行）

1. **今の勝ち筋は"売る"より"逃さない"**：読者は既に来ている（334本＋英語50本）。取りこぼしている**フォロー化・回遊・再訪**の穴を、文言整備だけ（¥0）で今すぐ塞ぐ。
2. **海外読者の受け皿がゼロ**：英語ページ（`apps/toyama-guide/en-*.html`）には現状**フォロー先も連絡先も無い**（アフィリと免責のみ）。ここが最大の漏れ＝「読んだ→もう戻れない」。低コストの受け皿を1つ用意する。
3. **メンバーシップ/有料は+5,000後**：今はスイッチを「入れる準備」（入口文言・マガジン編成）まで。本格オンは閲覧が押し上がってから（CAOのCVR逆算に連動）。

## 2. 各段の"取りこぼし"を減らす施策（文例つき・そのまま貼れる）

### 取りこぼし① 認知→回遊（「1記事で離脱」を防ぐ）

- **記事末に「次に読む」3本**を必ず置く（テーマの近い記事へ）。CMOのマガジン編成（`2026-06-29_note_マガジン編成リスト.md`）の同カテゴリから選ぶ。
- 文例（記事末・日本語）:
  ```
  ▼ この続きに、たぶん好きなやつ
  ・（同テーマ記事タイトル）→ URL
  ・（同テーマ記事タイトル）→ URL
  ・（同テーマ記事タイトル）→ URL
  ```
- **押し売りにならない理由**：これは商品でなく「次の読み物」の提案。読者の滞在を伸ばすだけ。

### 取りこぼし② 回遊→フォロー（記事末フォローCTA）

- 記事末の**最後**（アフィリブロックより下でも上でもよいが、"読み終わった満足"の直後）に、短いフォローCTAを1つ。**長文・煽りNG（A5）**。
- 文例A（地元目線・素直）:
  ```
  高岡・氷見・富山のこういう話を、住んでる人間の目線で週に何本か書いています。
  よかったらフォローしてください。次の記事が届きます。→ （noteフォロー）
  ```
- 文例B（食記事向け）:
  ```
  富山の「食べる/見る」をこのペースで淡々と更新しています。
  次の一本が要るときにフォローを。→ （noteフォロー）
  ```
- 文例C（X併記・海外/拡散狙いの記事）:
  ```
  X でも、写真1枚＋一言で富山の景色を流しています。→ （Xフォロー）
  ```
- **1記事1CTA**。複数並べない（うるさい＝A5）。フォローは「押し売り」ではない（無料・剥がしても損しない）＝方針と両立。

### マガジン誘導（回遊の束ね＝クリエイターページ回遊を増やす）

- 各マガジンの説明文を"入口"として整える。文例:
  ```
  ▷ このマガジン「歩ける高岡・富山」に、同じテーマの記事をまとめています。
    順に読むと、地図なしでも街のイメージがつながります。→ （マガジンURL）
  ```
- 記事本文の途中（関連する話題が出た所）にマガジンリンクを**1回だけ**差す＝自然な回遊。

### 取りこぼし③ フォロー→リピート（休眠させない）

- **フォロワーへの"次の理由"は新規更新そのもの**。配信キット（`2026-07-01_配信キット_*.md`）の1日5分ルーティンを継続＝更新が続く＝通知が届く。
- 季節フック：投稿カレンダー上、季節ネタ（春=雪の大谷/ホタルイカ、冬=寒ブリ/カニ）を先んじて出す→検索とリピートの両方に効く（CAOと連携）。

### note メンバーシップの入口設計（**設計のみ・オンは+5,000後**）

- **入口の考え方**：無料334本で信頼済みの層に、月額¥300〜500で「地元の続き」を出す。無料記事を有料化するのではなく、**"無料の延長線上の少しだけ濃い話"**（この施策自体はフェーズ2）。
- 今やる仕込み（¥0・オフのまま）:
  1. メンバーシップの**名前と一言説明の下書き**を用意（例：「富山のローカル便り」＝地元目線の月数本＋質問に答える回）。※実開設・課金設定は+5,000後・owner/ワーカー。
  2. 記事末フォローCTAの**"予告"文言は今は入れない**（まだ売らない方針）。開設後に1行足すだけで済むよう、CTAブロックの構造だけ揃えておく。
- **禁止**：今の段階で「メンバー募集」を本文に出すこと（押し売り＝方針違反）。準備は裏で、露出は+5,000後。

## 3. 海外読者の受け皿（英語ページからの導線）

**現状の事実**：`apps/toyama-guide/en-*.html` には、アフィリリンクとPR/免責はあるが、**フォロー先も連絡先も一切ない**（`grep` で mailto/social/note リンクなし＝確認済み）。＝海外読者は「読んだら終わり」で戻る手段がゼロ。ここが最大の漏れ。

### 受け皿の選択肢（¥0・固定費を増やさない）

| 受け皿 | コスト | 向き | 備考 |
|---|---|---|---|
| **既存の分散チャネルへ送客**（Reddit/Pinterest の既存アカ・X） | ¥0 | フォロー・再訪 | 一番安全。既に `EN/outputs/` に素材あり |
| **ガイド内の相互リンク強化**（"次に読む"英語版） | ¥0 | 回遊・滞在 | 実装＝HTML1行追加。owner/ワーカー |
| **軽量な問い合わせ**（無料フォーム or 既存Gmail 1本） | ¥0 | 声を拾う | メール直書きはA4/スパムに注意→フォーム推奨。**新規有料ツール禁止** |

> ⚠️ **メール直書き（mailto）は原則避ける**：迷惑メール/収集対象になりやすく、owner個人メールの露出はA4的にも避けたい。**無料フォーム（¥0枠）** か **既存のX/Redditへの誘導**を優先。実装判断は owner。

### 英語CTA文例（`en-*.html` の末尾・"Explore the guide" 付近／PR免責とは別ブロック）

- **フォロー/再訪（分散チャネルへ）**:
  ```
  Liked this? I write these honest, local guides to Toyama, Takaoka & Himi
  one at a time. Follow along on X for a photo and a line most days,
  or save the guide — the seasons here are worth timing. → [X] · [Save this guide]
  ```
- **回遊（次に読む・英語版の取りこぼし①対策）**:
  ```
  Read next: ・(related en-page title) → ・(related en-page title) →
  ```
- **声を拾う（軽い・押し付けない）**:
  ```
  Planning a trip and stuck on something? There's no shop here — just a local.
  If a quick question would help, [ask here] (a simple form, no sign-up).
  ```
- **原則**：英語CTAも**煽らない・売らない**。「there's no shop here — just a local」で押し売りでないことを明示＝ブランドと方針に一致。

### 実装分担（A1でcode不可）

- code（本書）：文言・設置位置・受け皿の選択肢まで。
- owner/ワーカー：受け皿の実体（無料フォーム or 既存アカ）を用意 → 英語ページ末尾に CTA ブロックを追加 → `ops/outbox/` へ完了報告（どのページに何を付けたか）。

## 4. 顧客の声を拾う仕組み（コメント/スキ/Xリプ → FAQ化）

**狙い**：反応を"次の記事ネタ"と"FAQ"に変える。これが将来のプロダクト（有料ガイド/メンバーシップ）の素になる。**PIIは残さない**（誰が言ったかは書かず、"問い"だけ抽象化して保存）。

| 拾う場所 | 拾い方（owner/ワーカー・1日5分内） | 行き先 |
|---|---|---|
| note コメント | 質問・要望を1行に抽象化（実名/地区は落とす＝A4） | `CAO` 反響ログ → 頻出は FAQ 候補 |
| note スキ | どの記事に集中したかを傾向として記録（個人は特定しない） | 効果検証シート（`CAO/outputs/2026-06-29_note施策_効果検証シート.md`） |
| X リプ/引用 | よくある疑問・褒め・訂正を種別だけメモ | 同上 → 英語 `en-faq.html` の追補候補 |
| 英語フォーム（設置後） | 旅行の"詰まりどころ"の質問を匿名で集約 | 英語FAQ拡充・記事ネタ |

- **FAQ化の入口**：同じ問いが**2回以上**来たら FAQ 候補。日本語は新規記事 or 既存追補、英語は `en-faq.html` に1問追加（既にFAQPage schema あり＝追加が効く）。
- **禁止**：コメント者の名前・地区・個人情報を台帳/記事に転記すること（A4）。拾うのは"問いの型"だけ。

## 5. KPI（フォロワー転換率・回遊率・リピート率）

**注意**：A1で code は数値取得不可。**baseline はワーカー/owner が取得**。ここでは**測る項目の定義**と目標の置き方だけ設計する（数値は baseline 後にCAOと確定）。

| 指標 | 定義（分子/分母） | 取得元 | baseline後に置く目標の考え方 |
|---|---|---|---|
| **フォロワー転換率** | 期間の新規フォロー ÷ 期間ビュー | note ダッシュボード | まず現状を測り、記事末CTA前後で比較（改善幅で判断） |
| **回遊率** | 1訪問あたり閲覧記事数（or 記事末リンクのクリック率） | note / 英語=analytics.js | "次に読む"設置記事とそうでない記事を比較 |
| **リピート率** | 再訪した読者割合 / フォロワーの再閲覧 | note ダッシュボード | 季節ネタ投下週の再訪増を見る |
| **英語受け皿転換** | 英語ページ→受け皿（X/フォーム）遷移数 | analytics.js（設置後） | 現状ゼロ→"1"を作ることが第一目標 |
| **CVR（収益・参考）** | アフィリ成果 ÷ クリック | 各ASP管理画面 | CAOのCVR逆算（親資料§4）に接続 |

- **baseline取得後に測る項目**（ワーカー依頼）：①月間ビュー、②現フォロワー数、③記事末リンクのクリック有無が分かる範囲、④英語ページの外部遷移。
- 取れたら CAO が `必要CV数 ÷ ビュー = 必要CVR` に接続（親資料§4）＝「+5,000がなぜ効くか」を数字で見せる。
- **A/Bは軽く**：全記事一斉に変えない。数本で"次に読む"あり/なしを比べ、効いた型だけ横展開（CMO/CAOと連携）。

## 付記：なぜ"仕込み"だけで意味があるか（方針との整合）

売らないでも、**フォロー・回遊・声の蓄積**は資産として積み上がる。+5,000到達時に「フォロワーが増えている／回遊導線が張れている／FAQの種が溜まっている」状態なら、**その瞬間に本格化のスイッチを入れるだけ**で収益段が立ち上がる。今はスイッチの配線工事＝押し売りゼロ・¥0で全部できる。
