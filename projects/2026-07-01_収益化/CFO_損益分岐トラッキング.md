# CFO 損益分岐トラッキング様式（赤字にならない収益化）

- **作成日**: 2026-07-01 ／ **担当**: CFO（数字の正確性・損益）
- **参照**: `CAO/outputs/2026-07-01_赤字にならない収益化プラン.md`（§2 損益分岐・チャネル別目安、§4 baseline 逆算）
- **位置づけ**: 本ファイルは**様式と計算式のみ**。実額（機密数値）は書かない。実数トラッキングは `CFO/outputs/`（gitignore）で管理する。

---

## 0. 前提（構造的に赤字不能）

| 項目 | 内容 |
|---|---|
| 固定費 | **月 ¥5,800 のみ**（既発生の運営費）。これ以外の固定費なし。 |
| 新規支出 | **ゼロ**（ASP登録＝無料／有料ツール・広告・在庫・仕入れ＝一切なし／成果報酬型のみ） |
| 収益化の構造 | 増えるのは**収入側だけ**。売れた/予約された分だけ手数料が入る。売れなくても支出は増えない。 |
| 結論 | **収益がゼロでも損失は既存 ¥5,800 のまま**。増えないので**構造的に赤字が発生しえない**。 |
| 損益分岐点 | **月間 収益合計 > ¥5,800** を超えた月＝黒字。それ未満＝「未達（但し赤字ではない＝固定費内）」。 |

> 用語注意：本様式で「差分」がマイナス＝「分岐点未達」であり、**支出超過（赤字）ではない**。固定費 ¥5,800 は収益ゼロでも一定。

## 1. 週次トラッキング表（様式）

集計単位：週次で積み上げ、月内累計で ¥5,800 到達を追う。金額欄は `CFO/outputs/`（gitignore）側に実数を記入し、本表はヘッダ様式のみ。

|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| W1 | | | | | | | | | | | | | | | | ☐/◯ |
| W2 | | | | | | | | | | | | | | | | ☐/◯ |
| W3 | | | | | | | | | | | | | | | | ☐/◯ |
| W4 | | | | | | | | | | | | | | | | ☐/◯ |
| **月計** | | | | | | | | | | | | | | | | ☐/◯ |

**列の定義**
- `月間PV(累計)`：その週末時点の当月累計ページビュー（baseline 起点／§3）。
- `<チャネル>:クリック`：記事末アフィリリンクのクリック数（旅行・物販のみ計測対象）。
- `<チャネル>:成果`：予約確定・購入確定・有料購入・テンプレ販売・メンバー加入の**確定件数/人数**。
- `<チャネル>:収益`：確定成果 × 単価（下記§2 の単価目安を使用。実額は gitignore 側）。
- `収益合計(月累計)`：全チャネル収益の当月累計。
- `対¥5,800差分`：`収益合計(月累計) − 5,800`（プラス＝黒字達成／マイナス＝分岐点まで残り）。
- `黒字か`：`収益合計(月累計) > 5,800` なら ◯、未満は ☐。

## 2. 計算式（分岐点の逆算）

チャネル単価は CAO プラン §2 の目安を採用（実額確定は成果発生後）。

### 2-1. 必要成果件数（そのチャネル単独で ¥5,800 を賄う場合）

```
必要CV数 = 5,800 ÷ 1件あたり単価
```

| チャネル | 1件あたり単価(目安) | 必要CV数 = 5,800 ÷ 単価 |
|---|---|---|
| 旅行アフィリ | ¥100〜400/件 | 約 15〜58 件/月 |
| 物販アフィリ | ¥30〜200/件 | 約 29〜194 件/月 |
| note有料 | ¥100〜1,000/本 | 約 6〜58 本/月 |
| テンプレ販売 | ¥500〜10,000/本 | 約 1〜12 本/月 |
| メンバーシップ | ¥300〜500/人 | 約 12〜20 人 |

> 単独到達は前提にしない。**複数チャネルを薄く重ねる**：`Σ(各チャネル収益) > 5,800`。

### 2-2. 必要CVR（PV から逆算・アフィリ系）

```
必要CV数  = 5,800 ÷ 単価
必要CVR   = 必要CV数 ÷ 月間PV
実効経路  = 月間PV × リンククリック率 × 成果CVR = 成果件数
月間収益  = 成果件数 × 単価
```

- 判定：`必要CVR` が一般的なアフィリ実力値（クリック率 数%・成果CVR 0.1〜1%）で**到達可能か**を見る。
- 未達なら打ち手は2つ：**単価の高いチャネルを足す**（テンプレ/note有料）か、**月間PVを増やす**（+5,000ビュー方針が効く）。

### 2-3. 差分・達成率

```
対¥5,800差分 = 収益合計(月累計) − 5,800
達成率(%)    = 収益合計(月累計) ÷ 5,800 × 100
黒字判定     = (収益合計(月累計) > 5,800) ? 黒字 : 分岐点未達（赤字ではない）
```

## 4. 黒字化までのマイルストーン

段階的に小さな黒字点を刻む。各段の「トリガー」を満たしたら次段へ。

| 段 | 月間収益ライン | 位置づけ | トリガー（次段へ進む条件） |
|---|---|---|---|
| M0 | ¥0（現状） | 支出のみ ¥5,800・収益ゼロ＝**赤字ではない**（固定費内） | 記事末アフィリリンクを設置し「成果1件」を計上 |
| M1 | 月 ¥1,000 | 成果が発生する仕組みが回り始めた状態 | 2チャネル以上で成果が継続的に発生（単発でない） |
| M2 | 月 ¥3,000 | ¥5,800 の半分超・射程内 | 月間PVが baseline 比で増加 or 単価高チャネル(note有料/テンプレ)が加算 |
| M3 | 月 ¥5,800 超 | **損益分岐点突破＝黒字化** | 単月で `収益合計 > 5,800` を達成 |
| M4 | 黒字の定着 | 連続黒字 | 直近3か月連続で `> 5,800` を維持 |

> 各段の実額推移・達成日は `CFO/outputs/`（gitignore）で記録。本ファイルには機密数値を書かない。
