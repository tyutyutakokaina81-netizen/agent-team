# ひとり会社 実務テンプレ集

> フリーランス・個人事業・副業・小規模事業の「日々の事務」を、すぐ使える書式でまとめた実務テンプレ集です。
> 月次の収支管理から、請求書・見積書・案件管理・週次レビュー・議事録・経費の早見表まで、8種類を収録しています。

---

## はじめに（使い方・免責）

### この商品でできること
- 月ごとの収支とキャッシュ（手元資金）の見通しを、シンプルな表で把握する
- 請求書・見積書を、毎回ゼロから作らずに使い回す
- 案件ごとの「実質時給」を出して、割に合う仕事かどうかを見える化する
- 週に一度の振り返りで、滞っている仕事を早めに見つける
- 打ち合わせメモ・経費区分を、決まった型で残す

### 同梱物
1. 本ファイル（`template_kit.md`）— 8テンプレの構成定義・使い方・記入例・スプレッドシート化の手順
2. `csv/` フォルダ — 表形式が明確なテンプレのヘッダ行＋サンプル行（CSV）
   - `gesshu_shushi.csv`（月次収支管理シート）
   - `cash_mitooshi.csv`（キャッシュ見通しシート）
   - `anken_kanri.csv`（案件管理シート）
   - `shuji_review.csv`（週次レビューシート）
   - `keihi_kubun_hayami.csv`（経費区分の早見リスト）

### 使い方の流れ
1. お使いの表計算ソフト（Googleスプレッドシート／Excel／Numbers など）を開く
2. 本ファイルの「構成定義」を見ながら列（ヘッダ）を並べる、または `csv/` のCSVを取り込む
3. サンプル行を参考に、自分のデータに置き換える
4. 数式が示されている列は、説明のとおりに数式を入れる

CSVの取り込みは、多くのソフトで「ファイル → インポート（取り込み）」から行えます。文字化けする場合は「UTF-8」を選んでください。

### 免責（必ずお読みください）
- 本商品は**書式・整理の道具**を提供するものであり、税務・法務・会計の助言ではありません。
- 税率・インボイス（適格請求書）制度・勘定科目・経費の判断・控除などの取り扱いは、改正や個別事情によって変わります。本文中の数値や区分は**一般的な目安**であり、最終的な判断は**税理士・行政書士・社会保険労務士などの専門家、および国税庁・各省庁の最新の公式情報**でご確認ください。
- 本商品の利用によって生じたいかなる結果についても、制作者は責任を負いかねます。
- 記載の会社名・氏名・金額・番号などはすべて記入例（架空）です。

### 表記ルール
- 数式は Googleスプレッドシート／Excel で共通して使える書き方を基本にしています（一部は読み替えが必要な場合があります）。
- セル位置（例：`F2`）は「1行目をヘッダ、2行目を最初のデータ行」とした場合の例です。

## テンプレ1：月次収支管理シート

Vol.1（収支スプレッドシート）が「年間の集計・確定申告の下準備」に寄せているのに対し、本テンプレは**「月ごとのサマリーと、手元資金がいつ細るかの見通し」**に寄せています。日々の入出金を1枚に貯め、もう1枚で月別サマリーとキャッシュの見通しを見ます。

### 1-1. 入出金台帳（メインの入力シート）

| 列 | 列名 | 意味 | 入力 / 自動 | 入力規則・補足 |
|---|---|---|---|---|
| A | 日付 | 取引が発生した日 | 入力 | 日付形式 |
| B | 区分 | 収入か支出か | 入力（プルダウン） | 収入／支出 |
| C | 科目 | 売上・通信費 などの分類 | 入力（プルダウン） | テンプレ8の早見リストから選ぶ |
| D | 相手先 | 取引先・支払先 | 入力 | 任意 |
| E | 内容 | 何の取引か | 入力 | 例「記事執筆5本」 |
| F | 収入額 | 入ってきた金額 | 入力 | 区分が収入のとき |
| G | 支出額 | 出ていった金額 | 入力 | 区分が支出のとき |
| H | 支払/入金方法 | 振込・現金・クレジット等 | 入力（プルダウン） | 銀行振込／現金／クレジット／口座振替 |
| I | 入金/支払予定日 | お金が動く予定日 | 入力 | キャッシュ見通しに使う |
| J | 精算済 | 入金・支払が完了したか | 入力（チェック） | TRUE／FALSE |
| K | 備考 | 請求書番号など | 入力 | 任意 |

**数式の考え方**
- 月の収入合計（例：6月）：`=SUMIFS(F:F, A:A, ">="&DATE(2026,6,1), A:A, "<="&DATE(2026,6,30))`
- 月の支出合計（例：6月）：`=SUMIFS(G:G, A:A, ">="&DATE(2026,6,1), A:A, "<="&DATE(2026,6,30))`
- 科目別の支出合計：`=SUMIF(C:C, "通信費", G:G)`（科目名を変えて並べる）

**記入例（CSV：`csv/gesshu_shushi.csv` と同じ）**

| 日付 | 区分 | 科目 | 相手先 | 内容 | 収入額 | 支出額 | 方法 | 予定日 | 精算済 | 備考 |
|---|---|---|---|---|---|---|---|---|---|---|
| 2026-06-01 | 収入 | 売上 | A社 | 記事執筆5本 | 165000 | | 銀行振込 | 2026-06-30 | FALSE | 請求書No.2026-001 |
| 2026-06-03 | 支出 | 通信費 | Bモバイル | 携帯・回線 | | 7800 | 口座振替 | 2026-06-25 | FALSE | |
| 2026-06-05 | 支出 | ソフトウェア | Cサービス | 月額サブスク | | 2980 | クレジット | 2026-06-27 | FALSE | |

### 1-2. キャッシュ見通しシート（差別化の中心）

「黒字なのにお金が足りない」を防ぐための、ごく簡単な資金見通し表です。月初の残高に、その月の入金予定と支出予定を足し引きして、月末の見込み残高を出します。

| 列 | 列名 | 意味 | 入力 / 自動 | 補足 |
|---|---|---|---|---|
| A | 月 | 対象月 | 入力 | 例「2026-07」 |
| B | 期首残高 | 月初の手元資金 | 入力（初月のみ）/ 自動（翌月以降） | 翌月以降は前月のFを引用 |
| C | 入金予定 | その月に入る見込み額 | 入力 | 台帳の予定日から拾う |
| D | 支出予定 | その月に出る見込み額 | 入力 | 固定費＋変動費 |
| E | 純増減 | 入金−支出 | 自動 | `=C2-D2` |
| F | 期末見込残高 | 月末の手元資金見込み | 自動 | `=B2+E2` |
| G | メモ | 注意点 | 入力 | 「入金は月末」など |

**翌月の期首残高（B3）**：`=F2`（前月の期末見込みをそのまま引き継ぐ）

**記入例（CSV：`csv/cash_mitooshi.csv`）**

| 月 | 期首残高 | 入金予定 | 支出予定 | 純増減 | 期末見込残高 | メモ |
|---|---|---|---|---|---|---|
| 2026-06 | 300000 | 215000 | 180000 | 35000 | 335000 | A社入金は月末 |
| 2026-07 | 335000 | 50000 | 160000 | -110000 | 225000 | D社入金は中旬 |

> 見方：7月は入金が少なく残高が減る見込み。早めに次の案件や請求を動かす、といった判断に使います。

## テンプレ2：請求書テンプレ

繰り返し使う請求書の項目をまとめた書式です。表計算でもワープロでも作れます。

### 記載項目（一般的な構成）

| 区分 | 項目 | 補足 |
|---|---|---|
| 表題 | 「請求書」 | |
| 宛先 | 相手先名（御中／様） | 会社名・担当者名 |
| 発行者 | 自分の氏名・屋号 | 住所・連絡先・（あれば）登録番号欄 |
| 番号・日付 | 請求書番号／発行日／支払期限 | 番号は連番で管理すると探しやすい |
| 明細 | 品目／数量／単価／金額 | 行ごとに記載 |
| 金額 | 小計／消費税／合計 | 税率は取引内容で異なる場合あり |
| 振込先 | 銀行名・支店・口座種別・口座番号・名義 | 振込手数料の負担も明記すると親切 |
| 備考 | 支払条件・補足 | |

### 明細部の構成（表計算で作る場合）

| 列 | 列名 | 入力 / 自動 | 数式の例 |
|---|---|---|---|
| A | 品目 | 入力 | |
| B | 数量 | 入力 | |
| C | 単価 | 入力 | |
| D | 金額 | 自動 | `=B2*C2` |

- 小計：`=SUM(D2:D10)`
- 消費税（税率は要確認）：`=小計セル*0.1`
- 合計：`=小計セル+消費税セル`

### インボイス番号欄についての注記（重要）
- 適格請求書（インボイス）制度に対応する場合、**登録番号**などの記載欄を設けることがあります。
- ただし、**登録の要否・記載のしかた・必要事項は、事業者の状況や制度改正によって変わります。** 本テンプレに「登録番号」欄を置くかどうか、何を書くかは、**国税庁の最新の公式情報を確認するか、税理士にご相談ください。** 本商品は記載欄の置き場所を示すのみで、制度上の判断は行いません。

### 記入例
```
請求書

A社 御中                                    請求書番号：2026-001
                                            発行日：2026-06-20
                                            支払期限：2026-06-30

下記のとおりご請求申し上げます。

  品目              数量   単価        金額
  コラム記事執筆     5     33,000     165,000
  ─────────────────────────────────
                          小計        165,000
                          消費税(*)    （税率は取引内容で要確認）
                          合計        （小計＋消費税）

お振込先：○○銀行 △△支店 普通 0000000 ｶﾅ ｼﾒｲ
備考：振込手数料は貴社にてご負担ください。

発行者：（氏名／屋号・住所・連絡先）
（※インボイス対応時の登録番号欄：必要かどうかは最新の公式情報を確認）

## テンプレ3：見積書テンプレ

請求書とほぼ同じ構成で、表題と一部の項目が変わります。受注前に金額・範囲・条件を合意するための書式です。

### 記載項目

| 区分 | 項目 | 補足 |
|---|---|---|
| 表題 | 「見積書」 | |
| 宛先 | 相手先名 | |
| 発行者 | 自分の氏名・屋号・連絡先 | |
| 番号・日付 | 見積番号／発行日／**有効期限** | 有効期限を入れるのが請求書との違い |
| 件名 | 何の見積か | 例「LP制作一式」 |
| 明細 | 品目／数量／単価／金額 | 請求書と同じ構成 |
| 金額 | 小計／消費税／合計 | 税率は要確認 |
| 条件 | 納期／支払条件／作業範囲・前提 | **「含まれない作業」も書くと後の認識ズレを防げる** |
| 備考 | 補足 | |

### 明細部の数式
- 金額：`=数量*単価`
- 小計：`=SUM(金額の範囲)` ／ 合計：`=小計+消費税`

### 記入例
```
見積書

E社 御中                                    見積番号：Q2026-010
                                            発行日：2026-06-21
                                            有効期限：2026-07-21
件名：LP（ランディングページ）構成案一式

  品目                数量   単価        金額
  構成案作成           1     50,000     50,000
  ワイヤーフレーム      1     30,000     30,000
  ─────────────────────────────────
                            小計        80,000
                            消費税(*)    （税率は要確認）
                            合計        （小計＋消費税）

納期：発注後10営業日
作業範囲に含まれないもの：デザイン制作、コーディング、画像素材の購入費
支払条件：納品後翌月末払い

発行者：（氏名／屋号・連絡先）
```

| 5 | 修正・追加対応 | 修正は何回まで／追加費用の扱い | ☐ |
| 6 | 納期とスケジュール | 中間締切・遅延時の扱い | ☐ |
| 7 | 知的財産・著作権 | 成果物の権利は誰に帰属するか | ☐ |
| 8 | 秘密保持 | 機密情報の取り扱い | ☐ |
| 9 | 再委託の可否 | 第三者に外注してよいか | ☐ |
| 10 | 契約解除・中途終了 | やめる場合の条件・精算 | ☐ |
| 11 | 損害・責任の範囲 | トラブル時の責任の上限など | ☐ |
| 12 | 経費・実費の負担 | 交通費・素材費などの扱い | ☐ |

## テンプレ5：案件管理シート

案件ごとの状態・単価・締切・実働時間を1枚で管理し、**案件別の「実質時給」**を出します。

| 列 | 列名 | 意味 | 入力 / 自動 | 入力規則・補足 |
|---|---|---|---|---|
| A | 案件ID | 通し番号 | 入力 | 例「2026-001」 |
| B | 案件名 | 仕事の名前 | 入力 | |
| C | 相手先 | クライアント | 入力 | |
| D | 状態 | 進捗 | 入力（プルダウン） | 見積中／進行中／完了／請求済／入金済 |
| E | 契約形態 | 料金の決め方 | 入力（プルダウン） | 月額／記事単価／一式／スポット |
| F | 単価 | 受注金額 | 入力 | |
| G | 想定実働時間 | 見込み工数（時間） | 入力 | |
| H | 実働時間 | 実際にかかった時間 | 入力 | |
| I | 実質時給 | 単価÷実働時間 | 自動 | `=IF(H2="","",F2/H2)` |
| J | 着手日 | 開始日 | 入力 | |
| K | 締切 | 納品期限 | 入力 | |
| L | 納品日 | 実際の納品日 | 入力 | |
| M | 請求状況 | 請求の進み | 入力（プルダウン） | 未請求／請求済／入金済 |
| N | 備考 | メモ | 入力 | |

**実質時給の出し方・読み方**
- 数式：`実質時給 = 単価 ÷ 実働時間`。実働時間が空のうちは表示しない（上のIF式）。
- 「想定実働時間」と「実働時間」を比べると、見積もりのズレ（=値付けの精度）が見えます。
- 実質時給が極端に低い案件は、値上げ交渉・効率化・受注見送りなどの判断材料になります。

**記入例（CSV：`csv/anken_kanri.csv`）**

| 案件ID | 案件名 | 相手先 | 状態 | 形態 | 単価 | 想定 | 実働 | 実質時給 | 締切 | 請求状況 |
|---|---|---|---|---|---|---|---|---|---|---|
| 2026-001 | コラム記事5本 | A社 | 進行中 | 記事単価 | 165000 | 15 | 12 | 13750 | 2026-06-20 | 未請求 |
| 2026-002 | SNS運用代行6月 | D社 | 完了 | 月額 | 50000 | 20 | 22 | 2273 | 2026-06-30 | 請求済 |

## テンプレ6：週次レビュー＆タスク棚卸しシート

週に一度、進んだ仕事・滞った仕事・来週の最優先を書き出すための1枚です。案件管理（テンプレ5）の案件IDと紐づけると、滞留が見つけやすくなります。

| 列 | 列名 | 意味 | 入力 / 自動 | 補足 |
|---|---|---|---|---|
| A | 週 | 週の通し番号 | 入力 | 例「2026-W25」 |
| B | 対象期間 | 月日の範囲 | 入力 | 例「06/16-06/22」 |
| C | 今週やったこと | 主な実績 | 入力 | |
| D | 進んだ案件 | 案件ID | 入力 | テンプレ5と紐づけ |
| E | 滞った案件 | 案件ID | 入力 | 早期発見が目的 |
| F | 稼いだ目安額 | その週の売上目安 | 入力 | |
| G | 来週の最優先3つ | 重要タスク | 入力 | 3つに絞るのがコツ |
| H | 棚卸しメモ | 抱えている懸案 | 入力 | やめる・分割する判断にも |
| I | 所感 | ひとことの振り返り | 入力 | |

**使い方のコツ**
- 「滞った案件」に同じIDが2週続けて並んだら、進め方を変える合図です（分割する／相談する／断る）。
- 「来週の最優先3つ」は、多くしすぎないことが続けるコツです。

**記入例（CSV：`csv/shuji_review.csv`）**

| 週 | 期間 | 今週やったこと | 進んだ | 滞った | 稼いだ目安 | 来週の最優先3つ |
|---|---|---|---|---|---|---|
| 2026-W25 | 06/16-06/22 | A社記事3本納品 | 2026-001 | 2026-003 | 90000 | 2026-003見積提出／請求書発行／週次の仕組み化 |

## テンプレ7：議事録／打ち合わせメモ テンプレ

打ち合わせの内容と「決まったこと・次にやること」を取りこぼさないための型です。表計算でも文書でも使えます。

### 構成（見出し）

| 項目 | 書く内容 |
|---|---|
| 日時 | いつ |
| 相手先・出席者 | 誰と |
| 件名・目的 | 何のための打ち合わせか |
| 議題 | 話す予定だったこと |
| 決まったこと | 合意事項（後で見返す最重要欄） |
| 宿題（ToDo） | 誰が・何を・いつまでに |
| 保留・持ち帰り | 決まらなかったこと |
| 次回 | 次の予定 |
| メモ | その他の気づき |

**ToDo部分は表にすると追いやすい**

| 担当 | やること | 期限 | 状態 |
|---|---|---|---|
| 自分 | 見積書を送る | 06/24 | ☐ |
| 相手 | 原稿の素材を共有 | 06/26 | ☐ |

### 記入例
```
■ 打ち合わせメモ
日時：2026-06-23 14:00-14:40（オンライン）
相手先・出席者：E社（担当者）／自分
件名・目的：LP制作の進め方すり合わせ

【議題】

## スプレッドシート化の手順（共通）

### 方法A：CSVを取り込む（表形式のテンプレ向け）
1. Googleスプレッドシート／Excel／Numbers などを新規作成する
2. 「ファイル → インポート（取り込み）」から `csv/` 内のCSVを選ぶ
3. 区切り文字は「カンマ」、文字コードは「UTF-8」を選ぶ
4. 取り込んだら、サンプル行を自分のデータに置き換える
5. 本ファイルの「数式の例」を見て、自動計算したい列に数式を入れる

### 方法B：手で列を並べる（書式系のテンプレ向け）
1. 各テンプレの「構成定義」の列名を1行目に入力する
2. プルダウンを設定する列は、入力規則（データの入力規則 → リスト）で選択肢を設定する
   - 例（区分）：`収入,支出` ／ 例（状態）：`見積中,進行中,完了,請求済,入金済`
3. チェックボックスは「挿入 → チェックボックス」で設定する
4. 数式は「数式の例」をコピーし、2行目に入れてから下方向にコピーする

### よくあるつまずき
- **CSVが文字化けする** → インポート時に「UTF-8」を選ぶ
- **数式がそのまま文字で表示される** → 先頭が半角の `=` になっているか確認する
- **日付の集計が合わない** → 日付列が「日付形式」になっているか確認する（文字列だと集計できない）
