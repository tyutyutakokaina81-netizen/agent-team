## まえがき — この本が約束すること、約束しないこと

この本は、すでに仕事を持っている会社員・事務職の方が、毎日の作業時間を減らすために書きました。営業事務、総務、経理補助、人事、カスタマーサポート、企画、アシスタント職。職種は問いません。共通しているのは「業務そのものを変えることはできないが、その業務にかかる時間は減らしたい」という状況です。

最初に、この本が約束しないことをはっきり書いておきます。この本は「AIを使えば残業がゼロになる」とも「誰でも評価が上がる」とも約束しません。そういう保証はできませんし、する気もありません。AIは魔法ではなく、優秀だけれど指示が必要な道具です。使い方を間違えれば、かえって時間が増えることもあります。間違った内容を堂々と出力することもあります。

では、この本が約束することは何か。「これまであなたが手で全部やっていた作業の、最初の七割をAIに肩代わりさせる具体的な手順をお渡しすること」です。メールの下書き、資料のたたき台、表の整理、長い文章の要約、議事録の清書、英文メールの草案。これらの「ゼロから一を作る部分」と「面倒な整形作業」は、AIがかなり得意とするところです。残りの三割、つまり「事実かどうかの確認」と「最終的に出すと決める判断」は、あなたがやる。この線引きが本書の背骨です。

私がこの本で繰り返し言うのは、「AIに下書きをさせて、人が仕上げる」という一点です。逆ではありません。AIに最終版を作らせて人がそれをそのまま出す、という使い方は危険です。AIは事実を間違えますし、あなたの会社の事情も、上司の好みも、取引先との過去のやり取りも知りません。だから、AIが作るのはあくまで「七割の下書き」。そこから先はあなたの仕事です。

この線引きさえ守れば、AIは安全で強力な相棒になります。毎朝のメール返信に30分かかっていたのが10分になる。会議のあと、議事録を清書するのに1時間かかっていたのが15分になる。英文メールに辞書を引きながら格闘していたのが、草案チェックだけで済むようになる。劇的に十倍速くなるわけではありませんが、一日のあちこちで15分、20分と削れていく。それが積み重なると、定時で帰れる日が増えます。

本書の構成は、まず「考え方」から入ります。何をAIに任せ、何を任せないか。ここを最初に押さえないと、便利な道具を危険に使ってしまうからです。そのあとは、メール、資料、表計算、調べ物、議事録、翻訳と、実際の業務の順に進みます。各章には、そのまま使える手順、ビフォーアフターの具体例、コピペできるプロンプト(AIへの指示文)、そしてチェックリストを入れました。第8章はプロンプトの型を集めた逆引き集、第9章は情報漏えいとAIの間違いへの対策です。この第9章だけは、便利さに浮かれる前に必ず読んでください。

読み物として眺めるのではなく、横にAIの画面を一つ開いて、一緒に手を動かしながら読んでもらえると一番効果が出ます。それでは始めましょう。大げさな成功談はありません。代わりに、明日の朝から使える手順をお渡しします。

## 第1章 AIを仕事に入れる考え方 — 「任せる」と「任せない」の線引き

### AIは「速い新人」だと思う

AIの性格を一言で言うと、「とにかく速いが、思い込みが激しく、確認をしない新人」です。指示を出せば数秒で文章を返してきます。徹夜でも文句を言いません。けれど、知らないことを知らないと言わず、それらしい嘘を堂々と書きます。あなたの会社のルールも、過去の経緯も知りません。

この新人をどう使うか。優秀な上司なら、こう考えるはずです。「定型的で、間違っても取り返しがつく作業を任せ、出てきたものは必ず自分で確認する」。これがそのままAIの使い方になります。任せていいのは、下書き・整形・要約・たたき台づくり。任せてはいけないのは、最終判断・事実の保証・責任を負う部分です。

### 任せていい作業、任せてはいけない作業

判断の基準は二つです。「間違っていたときに、自分で気づけるか」と「間違っていたときに、取り返しがつくか」。この両方がイエスなら任せていい。どちらかがノーなら、人がやるか、人が必ず確認します。

**任せていい作業の例**

- メールや文章の「下書き」(最終的に自分が読んで送る)
- 長い資料・記事の要約(元の資料は手元にある)
- 表やリストの整形・体裁直し(元データは自分が持っている)
- 議事録の清書(録音やメモが手元にある)
- 英文メールの草案(意味は自分で確認できる)
- アイデア出し・たたき台(採用するかは自分が決める)
- 文章の言い換え・トーン調整

**任せてはいけない、または必ず確認する作業の例**

- 数字・固有名詞・日付・金額(AIは平気で間違える)
- 社外にそのまま出す最終文章(必ず人が読んでから)
- 法律・税務・契約・医療など専門的な判断(専門家に確認)
- 社内の機密情報・個人情報を含む処理(第9章で詳述)
- 「これで合っていますか」と聞いて鵜呑みにすること(AIは自信満々に間違える)

### 「七割」を狙う

完璧を狙わないことが、AI活用のコツです。AIに「完成品」を求めると、確認の手間が増えて、かえって時間がかかります。狙うのは七割。ゼロから書くのが一番つらいので、その「ゼロから七割」の坂をAIに登らせて、残りの三割を自分で仕上げる。これが一番速くて安全です。

たとえばお詫びメールなら、AIに状況を伝えて下書きをもらい、そこから「うちの会社はこういう言い回しは使わない」「この件は前回も似たことがあったから一言添える」といった、人間しか知らない情報を足して仕上げます。AIが七割、あなたが三割。この配分を意識してください。

### 道具をどう用意するか

本書はどのAIチャットツールでも使えるように書いています。会社で利用が認められているAIサービスがあれば、まずそれを使ってください。個人で試す場合も、無料で使えるAIチャットサービスが複数あります。重要なのはツールの銘柄ではなく、使い方です。本書のプロンプトは、特定のサービスに依存しない書き方にしてあります。

ただし一点だけ、最初に確認してほしいことがあります。**あなたの会社で、AIツールの業務利用が許可されているか、入力してよい情報の範囲はどこまでか。** これは第9章で詳しく扱いますが、考え方の段階で頭に入れておいてください。許可されていないツールに会社のデータを入れるのは、便利さ以前の問題です。

### この章のチェックリスト

- [ ] 「間違いに気づけるか」「取り返しがつくか」でAIに任せる作業を選んでいる
- [ ] 数字・固有名詞・日付は必ず自分で確認すると決めた
- [ ] AIには「完成品」でなく「七割の下書き」を求めると決めた
- [ ] 社外に出す文章は必ず人が最終確認すると決めた
- [ ] 会社のAI利用ルール(許可されているか、入力可能な情報範囲)を確認した

## 第2章 メール・ビジネス文章の時短

### なぜメールに時間がかかるのか

メール一通を書くのに時間がかかるのは、書く内容が難しいからではありません。「どう切り出すか」「失礼にならない言い回しは何か」「過不足ない情報量はどれくらいか」を毎回ゼロから考えているからです。ここがAIの一番得意なところです。要点さえ渡せば、形式の整った下書きを数秒で返してきます。

### 基本の使い方 — 箇条書きを渡して文章にしてもらう

メールでAIを使う一番簡単な方法は、「伝えたいことを箇条書きで渡し、ビジネスメールの形にしてもらう」ことです。きれいな文章を書く必要はありません。メモ書きで十分です。

**コピペプロンプト①(基本のメール作成)**

```
次の要点を、社外向けの丁寧なビジネスメールにしてください。

【宛先】取引先の担当者(面識あり)
【目的】来週の打ち合わせの日程変更をお願いしたい
【要点】
・当初は来週火曜の14時で予定していた
・社内の都合で動かしたい
・木曜か金曜の午後なら空いている
・先方の都合を伺いたい

【条件】
・件名も付ける
・長すぎず、3〜5文程度の本文
・へりくだりすぎず、ビジネスとして自然な敬語
```

**ビフォー(自分で書くと)**

> 件名から悩み、「いつもお世話になっております」のあとが続かず、日程変更のお願いが角が立たないか何度も書き直して、15分経過。

**アフター(AIに下書きさせると)**

> 上のプロンプトを送ると、件名・あいさつ・用件・代替日程の提示・結びまで整った下書きが返ってくる。自分は「木曜か金曜」を具体的な日付に直し、相手の名前を入れて確認するだけ。2〜3分で完了。

### トーンを調整する

返ってきた文章が硬すぎる、あるいは軽すぎると感じたら、追加で一言頼むだけで直せます。書き直してもらうより速いです。

**コピペプロンプト②(トーン調整)**

```
今の文章を、もう少し柔らかく、でも軽くなりすぎないトーンに直してください。
親しい取引先に送る想定です。
```

```
今の文章を、より簡潔にしてください。本文は3文以内に。
```

```
今の文章の敬語が過剰なので、自然なビジネス敬語に整えてください。
```

### 言いにくいことを伝えるメール

催促、お断り、お詫び、価格の交渉。気が重いメールほど、AIの下書きが効きます。「角を立てずに伝える言い回し」をAIに考えてもらい、自分は内容の正確さだけ確認します。

**コピペプロンプト③(言いにくいことを伝える)**

```
次の状況で、相手に失礼にならず、でも要件はきちんと伝わるメールの下書きを作ってください。

【状況】先月納品をお願いした資料が、約束の期日を過ぎても届いていない
【関係】継続的に取引のある相手なので、強く言いすぎたくない
【伝えたいこと】状況の確認と、いつ頃いただけそうか教えてほしい

3パターンの言い回しを提案してください。柔らかい順に並べてください。
```

3パターン出させて、自分の状況に一番合うものを選ぶ。この「複数案から選ぶ」やり方は、メール以外でも使える基本テクニックです。

### 返信の下書き — 相手のメールを貼る

相手から来たメールへの返信も、本文を貼り付けて指示すれば下書きが作れます。社内情報や個人情報が含まれる場合の注意は第9章で扱いますが、定型的なやり取りなら大幅に時短できます。

**コピペプロンプト④(返信の下書き)**

```
次は取引先から届いたメールです。これに対する返信の下書きを作ってください。

【こちらの状況・伝えたいこと】
・依頼は引き受けられる
・ただし納期は希望より3日後ろにしてほしい
・見積もりは別途お送りする

【届いたメール】
(ここに相手のメール本文を貼る)

返信は丁寧かつ簡潔に。引き受ける姿勢が伝わるように。
```

### 定型文を「使い回せる型」にする

毎週・毎月送る同じようなメールは、一度AIに「穴埋め式のテンプレート」を作らせておくと、次回から数十秒で書けます。

**コピペプロンプト⑤(テンプレート化)**

```
次のような月次報告メールを、毎月使い回せる穴埋め式テンプレートにしてください。
変わる部分は【 】で囲んで、何を入れるか分かるようにしてください。

(過去に送ったメールの例文を貼る、または内容を箇条書きで説明する)
```

### この章のチェックリスト

- [ ] メールは「箇条書き→AIで文章化」の流れで書いている
- [ ] トーンが合わないときは追加指示で直している(書き直さない)
- [ ] 言いにくいメールは「3パターン提案」で選んでいる
- [ ] 送る前に、宛名・日付・金額・固有名詞を自分で確認している
- [ ] 繰り返し送るメールはテンプレート化した

## 第3章 資料・スライドの下書きをAIに作らせる

### スライドづくりの時間を食うのは「構成」

提案資料やスライドで時間がかかるのは、デザインよりも「どういう順番で、何を、いくつのスライドに分けて話すか」という構成を考える段階です。ここをAIに任せると、白紙の恐怖から解放されます。AIに作らせるのは「中身の骨組み」であって、最終的な見た目は自分で整えます。

### スライドの骨組みを作る

**コピペプロンプト⑥(スライド構成案)**

```
次のテーマで社内向けの提案資料を作ります。スライドの構成案を作ってください。

【テーマ】業務マニュアルを紙からデータに移行する提案
【聞き手】部門の責任者(3〜4名)、時間は15分
【伝えたいこと】今の紙運用の問題、データ化のメリット、移行の手順、必要な準備

各スライドについて、
・スライドのタイトル
・そのスライドで話す要点を箇条書きで3つ程度
を出してください。全体で8〜10枚を目安に。
```

返ってきた構成を見て、順番を入れ替えたり、不要なスライドを削ったりします。AIが出すのはあくまで「たたき台」。これを土台に自分の言葉で肉付けします。

### 各スライドの中身を膨らませる

骨組みができたら、特定のスライドの中身を詳しく書いてもらえます。

**コピペプロンプト⑦(スライド本文の肉付け)**

```
さきほどの構成のうち、「現状の紙運用の問題」のスライドについて、
箇条書きの本文を作ってください。
・項目は4つまで
・1項目は1行(20字以内)で簡潔に
・聞き手が「確かに困っている」と共感できる具体性
```

### 話す原稿(スピーカーノート)も作れる

スライドに書く文字は少なくして、実際に口で話す内容は別に用意したい。そんなときは、スライドごとの「話す原稿」をAIに作らせます。

**コピペプロンプト⑧(スピーカーノート)**

```
このスライドを実際にプレゼンするときの、話す原稿を作ってください。
・スライド1枚につき、話す時間は30〜45秒程度
・原稿口調(です・ます)で、読み上げられる自然な文章
・スライドの箇条書きをそのまま読むのではなく、補足や具体例を交えて
```

### 報告書・企画書の下書き

スライドだけでなく、文章主体の報告書や企画書も同じ要領です。「見出しの構成→各見出しの本文」の二段階で進めます。

**コピペプロンプト⑨(報告書の構成→本文)**

```
次の内容で社内報告書を作ります。まず見出しの構成だけ提案してください。

【報告内容】今期取り組んだ業務改善の結果と、来期の方針
【読み手】上長
【含めたい要素】何をやったか、どんな効果があったか、課題、来期どうするか

A4で1〜2枚に収まる分量を想定。見出し構成が決まったら、次に本文を書きます。
```

構成にOKを出したら「では各見出しの本文を書いてください」と続けます。一気に全部書かせるより、構成を固めてから中身を書くほうが、修正が少なくて済みます。

### ビフォーアフター

**ビフォー** 提案資料を任され、まず何から手をつけるか分からず、過去の似た資料を探して、構成を考えるだけで1時間。

**アフター** プロンプト⑥で構成案を10分で得て、不要なスライドを削り順番を直す。各スライドの中身をプロンプト⑦で膨らませ、自分の言葉に直す。全体の骨組みが30分で固まり、あとは見た目を整えるだけ。

### 注意 — 数字とロジックは必ず自分で

AIが作る構成は流暢ですが、その中身の数字や因果関係は鵜呑みにできません。「効果は○○%」のような数字をAIが勝手に入れていたら、それは事実ではなく作り話です。数字・根拠・固有名詞は、必ず自分の手元の資料で裏を取ってから資料に載せてください。

### この章のチェックリスト

- [ ] スライドは「構成案→中身」の二段階で作っている
- [ ] AIの構成はそのまま使わず、順番・取捨選択を自分でしている
- [ ] 話す内容はスピーカーノートとして別に作らせた
- [ ] 報告書も「見出し構成→本文」で進めている
- [ ] 資料に載せる数字・根拠は自分の資料で裏を取った

## 第4章 表計算・データ整理の時短

### AIは「数式」と「整形」が得意

表計算でつまずくのは、「やりたいことは分かっているのに、どの関数をどう書けばいいか分からない」場面と、「データの形がバラバラで、揃えるのが面倒」な場面です。前者はAIに数式を作らせ、後者はAIに整形のルールを考えさせると、一気に楽になります。

ここで大事な注意があります。**会社の実データ(顧客名・金額・個人情報など)をそのままAIに貼るのは避けてください。** AIには「どういう数式が必要か」「どういう手順で整形するか」を教えてもらい、実データの処理は自分の表計算ソフト上で行います。詳しくは第9章で扱います。

### 数式・関数を教えてもらう

「やりたいこと」を日本語で説明すれば、数式を作ってくれます。実際のデータを見せる必要はありません。

**コピペプロンプト⑩(数式を作ってもらう)**

```
表計算ソフトで使う数式を教えてください。

【やりたいこと】
A列に日付、B列に金額が入っています。
このうち「今月分」だけの金額を合計したいです。

・どの関数を使えばいいか
・実際の数式(セルの位置は例で構いません)
・その数式が何をしているかの簡単な説明
をセットで教えてください。
```

ポイントは「実際の数式」と「説明」をセットで求めること。説明があれば、自分の表に合わせて少し直すときに困りません。数式は必ず自分の表で動かして、結果が正しいか確認してください。

### エラーの原因を聞く

数式がエラーになったとき、その原因をAIに聞けます。エラー表示の文字をそのまま伝えると、よくある原因と直し方を教えてくれます。

**コピペプロンプト⑪(エラーの原因)**

```
表計算ソフトで次の数式を入れたら、エラーになりました。
原因として考えられるものと、直し方を教えてください。

【入れた数式】
(数式をそのまま貼る)
【出たエラー表示】
(エラーの文字をそのまま貼る)
【やりたかったこと】
(日本語で説明)
```

### バラバラなデータを揃える手順を作る

「半角と全角が混ざっている」「日付の書き方がバラバラ」「氏名と部署が一つのセルに入っている」。こうしたデータの揺れを揃える手順も、AIに相談できます。実データは渡さず、「こういう揺れがある」と説明します。

**コピペプロンプト⑫(データ整形の手順)**

```
表計算ソフトで、列のデータを整えたいです。手順を教えてください。

【今の状態】
電話番号の列に、「03-1234-5678」「0312345678」「03(1234)5678」のように
書き方が混在しています。
【揃えたい形】
すべて「03-1234-5678」のようにハイフン区切りに統一したい。
【条件】
プログラミングは使わず、表計算ソフトの機能や関数の範囲でできる方法で。
```

### 集計の考え方を相談する

「このデータから何が言えるか」「どう集計すれば見やすいか」という、分析の入り口もAIに相談できます。ここでもデータの中身ではなく、「列の構成」だけを伝えます。

**コピペプロンプト⑬(集計の方針)**

```
次のような表があります。集計して傾向を見たいのですが、どんな切り口があるか提案してください。

【表の列】
・日付
・商品カテゴリ
・販売数
・金額

実データは見せませんが、どういう集計表やグラフを作ると傾向がつかめるか、
3つほど切り口を提案してください。それぞれ何が分かるかも添えてください。
```

### ビフォーアフター

**ビフォー** 「今月分だけ合計したい」のに関数名が思い出せず、検索して、見つけた数式を自分のケースに合わせられず、30分格闘。

**アフター** プロンプト⑩でやりたいことを日本語で伝え、数式と説明をもらう。自分の表のセル位置に合わせて直し、結果が合っているか確認。5分で完了。

### この章のチェックリスト

- [ ] 実データ(顧客名・金額・個人情報)はAIに貼っていない
- [ ] 数式は「実例＋説明」のセットでもらっている
- [ ] もらった数式は自分の表で動かして検算した
- [ ] データの揺れは「揺れの内容を説明→手順を相談」で揃えた
- [ ] 集計は「列構成だけ伝えて切り口を相談」している

## 第5章 調べ物・要約を速くする

### 「読む量」を減らすのが時短の本質

調べ物に時間がかかるのは、情報を探すこと以上に「大量の文章を読む」ことに時間を取られるからです。AIを使うと、長い文章を要約させたり、要点だけ抜き出させたりして、読む量そのものを減らせます。

ただし大前提があります。**AIが文章の中から「自分の知識で」答えを作ると、事実でない内容(ハルシネーション)が混じることがあります。** 一方、こちらが渡した文章を要約させる使い方は、元の文章が手元にあるぶん、間違いに気づきやすく安全です。だから本章では「自分が持っている資料を要約・整理させる」使い方を中心に紹介します。AIが自分で調べてきたと称する情報は、必ず出典を自分で確認してください。

### 長い文章を要約する

会議資料、マニュアル、長いメールのスレッド。手元にある長文を貼って、要点を出してもらいます。

**コピペプロンプト⑭(要約)**

```
次の文章を要約してください。

【要約の条件】
・全体の要点を、箇条書きで5つ以内に
・専門用語が出てきたら、初めて読む人にも分かるよう一言補足する
・「結局どうすればいいか」が分かるように

【文章】
(ここに長文を貼る)
```

### 目的別に抜き出す

ただ短くするだけでなく、「自分が知りたい観点」だけを抜き出させると、もっと速くなります。

**コピペプロンプト⑮(観点を絞って抽出)**

```
次の資料から、「自分が何をいつまでにやればいいか」だけを抜き出してください。
タスクと期限の一覧にして、表のように整理してください。

【資料】
(ここに資料を貼る)
```

```
次の議論の記録から、「決まったこと」と「まだ決まっていないこと(宿題)」に
分けて整理してください。

(記録を貼る)
```

### 比較・整理して表にする

複数の選択肢や案を比べたいとき、AIに表の形で整理させると一目で分かります。

**コピペプロンプト⑯(比較表を作る)**

```
次の3つの案について、比較表を作ってください。

【比較する観点】コスト、導入の手間、向いている場面
【3つの案】
(各案の説明を貼る、または箇条書きで渡す)

表の形で、観点ごとに各案を並べてください。最後に、どの案がどんな人に向くか一言ずつ添えてください。
```

### 知らない言葉・概念を聞く

専門用語や略語の意味を、文脈に合わせて説明してもらえます。ただし、調べ物として使う場合は、重要な事実は一次情報(公式サイトや原典)で裏を取ってください。AIの説明は「あたりをつける」ために使い、最終確認は人がします。

**コピペプロンプト⑰(用語を分かりやすく)**

```
次の言葉の意味を、専門知識のない人にも分かるように説明してください。
・身近なたとえを一つ入れる
・3〜4文で簡潔に
・もし複数の意味があれば、ビジネスの文脈での意味を優先

【言葉】
(調べたい言葉)
```

### ビフォーアフター

**ビフォー** 20ページの資料を渡され、全部読んで自分のタスクを探すのに40分。

**アフター** プロンプト⑮で「自分のタスクと期限だけ」を抽出させ、表で受け取る。元資料で該当箇所だけ確認。10分で自分のやることが明確に。

### この章のチェックリスト

- [ ] 要約・抽出は「自分が持っている資料」を渡して行っている
- [ ] AIが自分で調べたと称する事実は、出典を確認している
- [ ] 「観点を絞った抽出」で読む量を減らしている
- [ ] 比較は表の形で整理させている
- [ ] 重要な事実は一次情報で裏を取っている

## 第6章 議事録・文字起こしの整形

### 議事録の苦労は「清書」にある

会議中にメモは取れても、それを読める議事録に整えるのに時間がかかります。AIは、雑なメモや文字起こしを、整った議事録の形に直すのが得意です。会議の「内容を作る」のはあなた(メモを取る)、「形に整える」のはAI、という分担です。

なお、文字起こしそのもの(音声を文字にする)については、社内で認められたツール・方法を使ってください。本章では「文字起こしされたテキスト」または「自分が取ったメモ」を整える部分を扱います。会議に機密情報が含まれる場合の注意は第9章を参照してください。

### 雑なメモを議事録にする

**コピペプロンプト⑱(メモ→議事録)**

```
次は会議中に取った雑なメモです。これを読みやすい議事録に整えてください。

【議事録の形式】
・日時、参加者の欄(空欄でいいので枠だけ作る)
・「決定事項」「検討事項」「次回までの宿題(担当と期限)」に分けて整理
・話し言葉は書き言葉に直す
・重複や脱線は省く

【メモ】
(自分のメモを貼る)
```

### 文字起こしの整形

文字起こししたテキストは、「えー」「あの」などの言いよどみや、話が前後する箇所が多く、そのままでは読めません。これを整えます。

**コピペプロンプト⑲(文字起こしを読める形に)**

```
次は会議の文字起こしです。読める文章に整えてください。

【整え方】
・「えー」「あの」「まあ」などの言いよどみを削る
・話が前後している箇所は、内容ごとにまとめ直す
・誰が何を言ったかが分かるよう、発言を整理する
・意味は変えない(勝手に内容を足さない)

【文字起こし】
(テキストを貼る)
```

最後の「意味は変えない(勝手に内容を足さない)」という指示が重要です。これを入れないと、AIが話を盛ったり、言っていないことを補完したりすることがあります。

### 要点だけの「サマリー版」も作る

詳しい議事録とは別に、関係者に共有する短い要約版が必要なときがあります。

**コピペプロンプト⑳(共有用サマリー)**

```
今作った議事録から、関係者に共有するための短い要約を作ってください。
・3〜5行
・「何が決まったか」と「誰が何をいつまでにやるか」が一目で分かるように
・メールやチャットにそのまま貼れる文体で
```

### 必ず原文と照らし合わせる

整形した議事録は、必ず自分のメモや録音と照らし合わせてください。AIは「読みやすく」する過程で、ニュアンスを変えたり、決まっていないことを決まったように書いたりすることがあります。特に「決定事項」と「担当者・期限」は、間違うと後で揉めます。ここは人の目で確認するのが鉄則です。

### ビフォーアフター

**ビフォー** 走り書きのメモを、誰が何を言ったか思い出しながら議事録に清書して1時間。

**アフター** プロンプト⑱でメモを議事録の形に整え、決定事項と宿題が分かれた状態で受け取る。自分のメモと照合して、固有名詞・日付・担当を確認。15分で完了。

### この章のチェックリスト

- [ ] 文字起こしツールは社内で認められた方法を使っている
- [ ] メモは「決定事項・検討事項・宿題」に分けて整形させている
- [ ] 文字起こし整形では「意味を変えない」と必ず指示している
- [ ] 共有用の短いサマリーも別に作っている
- [ ] 整形後、自分のメモ・録音と照合し、固有名詞・日付・担当を確認した

## 第7章 翻訳・英文メール対応

### AI翻訳は「下訳」として使う

英語に苦手意識があると、英文メール一通に何十分もかかります。AIは翻訳と英文作成が得意なので、ここは大きく時短できます。ただし、これも「下訳・草案」として使うのが原則です。送る前に意味を確認し、不安なら日本語に訳し戻して確認します。専門用語や契約に関わる文言は、特に慎重に扱ってください。

### 日本語から英文メールを作る

**コピペプロンプト㉑(英文メール作成)**

```
次の内容で、英文のビジネスメールを作ってください。

【相手】海外の取引先の担当者(面識あり、ビジネスの関係)
【伝えたいこと】
・先日送ってもらった資料のお礼
・1点だけ確認したいことがある(納期について)
・来月、オンラインで打ち合わせができないか打診したい

【条件】
・丁寧だが、堅すぎない自然なビジネス英語
・3段落程度
・件名も付ける
・英文の下に、日本語訳も付けてください
```

最後の「日本語訳も付けて」が大事です。自分で内容を確認できるので、意図と違う英文を送る事故を防げます。

### 受け取った英文を訳す・要点を知る

海外から来た英文メールは、訳すだけでなく「要するに何を求めているか」を一緒に出してもらうと速いです。

**コピペプロンプト㉒(英文メールの理解)**

```
次は海外の取引先から届いた英文メールです。

1. 日本語に訳してください
2. 「相手が何を求めているか」「こちらが返事すべきこと」を箇条書きで整理してください

【メール】
(英文を貼る)
```

### 自分の英文をチェックしてもらう

自分で書いた英文に自信がないとき、添削してもらえます。

**コピペプロンプト㉓(英文の添削)**

```
次は自分で書いた英文メールです。チェックしてください。

・不自然な表現や文法の誤りを直す
・どこをなぜ直したか、簡単に説明する
・元の意味は変えない

【自分の英文】
(英文を貼る)
```

### 表現のニュアンスを相談する

「これは失礼にならないか」「もっと丁寧な言い方はないか」といったニュアンスの相談もできます。

**コピペプロンプト㉔(ニュアンス相談)**

```
英文メールで、相手の提案を断りたいのですが、関係を悪くしたくありません。
角の立たない断り方の英語表現を、3パターン教えてください。
それぞれ、日本語でどんなニュアンスかも添えてください。
```

### 注意 — 機密と専門文言

翻訳でAIを使うときも、社外秘の情報や個人情報を含む文章をそのまま貼るのは避けてください(第9章)。また、契約書・法的文書・技術仕様など、訳の正確さが重大な結果につながる文書は、AIの下訳をたたき台にしつつ、最終的には専門の翻訳者や担当部署に確認してください。

### ビフォーアフター

**ビフォー** 英文メール一通に、辞書と例文を調べながら40分。送ったあとも「変な英語じゃなかったか」と不安。

**アフター** プロンプト㉑で英文と日本語訳をセットで受け取り、意図と合っているか日本語訳で確認。固有名詞と日付だけ直して送付。10分で完了。

### この章のチェックリスト

- [ ] 英文作成は「英文＋日本語訳」をセットでもらっている
- [ ] 受け取った英文は「訳＋求められていることの整理」で把握している
- [ ] 自分の英文は添削してもらい、理由も確認している
- [ ] 機密・個人情報を含む文章はそのまま貼っていない
- [ ] 契約・法務・技術仕様の訳は、最終的に専門部署・専門家に確認している

## 第8章 プロンプトの型テンプレ集

### よいプロンプトに共通する「型」

これまでの章で使ってきたプロンプトには、共通する骨組みがあります。それを覚えておくと、本書に載っていない場面でも自分でプロンプトを組み立てられます。型は次の五つの要素でできています。

1. **役割** — 「あなたは○○です」(任意。立場を指定すると答えが安定する)
2. **やってほしいこと** — 「○○してください」(動詞をはっきり)
3. **材料** — 元になる情報・文章(箇条書きや貼り付け)
4. **条件** — 長さ、形式、トーン、守ってほしいルール
5. **出力の形** — 箇条書き、表、メール形式、など

この五つを意識して書くだけで、出力の質が安定します。逆に、出力がいまいちなときは、この五つのどれかが足りていないことが多いです。

### 万能テンプレート(穴埋め)

```
【役割】あなたは(立場・専門)です。
【依頼】次のことをしてください：(やってほしいこと)
【材料】
(元になる情報・文章をここに)
【条件】
・(長さ・形式)
・(トーン)
・(守ってほしいルール)
【出力の形】(箇条書き／表／メール／など)で出してください。
```

### 場面別・逆引きプロンプト集

そのままコピーして、【 】を自分の状況に置き換えて使ってください。

**▼ 短くしたい**

```
次の文章を、意味を変えずに【半分】の長さに縮めてください。
要点は落とさないでください。
(文章)
```

**▼ 分かりやすくしたい**

```
次の文章を、【中学生】にも分かるように書き直してください。
専門用語は身近な言葉に置き換えてください。
(文章)
```

**▼ 箇条書きを文章に**

```
次の箇条書きを、つながりのある文章にしてください。
【ビジネス文書】の文体で。
(箇条書き)
```

**▼ 文章を箇条書きに**

```
次の文章から要点を抜き出して、箇条書き【5つ以内】にしてください。
(文章)
```

**▼ 案を複数ほしい**

```
(やりたいこと)について、案を【3つ】出してください。
それぞれの長所・短所を一言ずつ添えてください。
```

**▼ たたき台がほしい**

```
(作りたいもの)のたたき台を作ってください。完璧でなくていいので、
あとから直せる「下書き」として、まず全体を作ってください。
```

**▼ 抜け漏れを点検したい**

```
次の(文章・計画・リスト)を見て、抜けている観点や考慮漏れがないか
指摘してください。指摘は箇条書きで。
(対象)
```

**▼ 質問の形で深めたい**

```
(検討したいテーマ)について、私が考えを深められるように、
答えやすい質問を【5つ】投げかけてください。
```

**▼ 言い換えたい**

```
次の表現を、【もっと丁寧な／もっとやわらかい／もっと簡潔な】言い方に
言い換えて、3パターン出してください。
(表現)
```

**▼ チェックリスト化したい**

```
次の作業を、抜け漏れなく進めるためのチェックリストにしてください。
順番に並べて、各項目はチェックボックス形式で。
(作業の説明)
```

### うまくいかないときの直し方

出力が期待と違うとき、最初からやり直す必要はありません。続けて指示すれば直ります。

- **長すぎる →**「もっと短く。○文以内で」
- **硬すぎる/軽すぎる →**「もう少し○○なトーンで」
- **的外れ →**「私が知りたいのは○○の部分です。そこを中心に」
- **抽象的 →**「具体例を一つ入れて」「数字や手順で示して」
- **形式が違う →**「表の形にして」「箇条書きにして」
- **一部だけ直したい →**「全体はそのままで、○○の部分だけ直して」

### 「やり取りを重ねる」のが一番効く

一発で完璧な答えを狙うより、ざっくり出させてから「ここをこう直して」と会話を重ねるほうが、結果的に速くて質も上がります。AIは前のやり取りを覚えているので、少しずつ詰めていけます。これは全章を通じての一番のコツです。

### この章のチェックリスト

- [ ] プロンプトを「役割・依頼・材料・条件・出力の形」で組み立てている
- [ ] うまくいかないときは、最初からでなく追加指示で直している
- [ ] 一発で完璧を狙わず、やり取りを重ねている
- [ ] 逆引き集を手元に置いて、場面に合わせて使っている

## 第9章 情報漏えい・ハルシネーション対策

この章は、便利さの裏側です。読み飛ばさないでください。AIを安全に使い続けられるかどうかは、この章の理解にかかっています。

### その1 — 情報漏えいを防ぐ

#### まず会社のルールを確認する

最初にやるべきは、あなたの会社のルールの確認です。

- そもそもAIツールの業務利用が認められているか
- 認められているなら、どのツールが許可されているか
- 入力してよい情報の範囲はどこまでか

会社が契約している業務用のAIサービスと、個人で使う無料サービスでは、データの扱いが違うことがあります。許可されていないツールに業務データを入れるのは、それ自体が問題になり得ます。判断に迷ったら、入れる前に上長や情報システム部門に確認してください。「確認してから使う」のが、長く安全に使うための一番の近道です。

#### 入れてはいけない情報

会社のルールで許可されている場合でも、次のような情報は、原則としてそのままAIに入れないのが安全です。

- **個人情報** — 氏名、住所、電話番号、メールアドレス、マイナンバーなど
- **顧客・取引先の機密情報** — 契約条件、価格、未公開の取引内容
- **社外秘の社内情報** — 未発表の計画、人事情報、財務の詳細
- **パスワードや認証情報**

#### 「中身を渡さず、形だけ渡す」工夫

第4章でも触れましたが、実データを渡さなくてもAIは助けになります。

- 数式やデータ整形は「列の構成」「揺れの内容」だけ伝え、実データは自分の表で処理する
- メールの下書きは、固有名詞を「A社」「担当者」などに置き換えて作らせ、あとで本物に差し替える
- 文章の要約も、機密部分は伏せたうえで要旨だけを渡す

この「**仮名に置き換える**」「**形だけ渡す**」という習慣をつけると、利便性を保ったまま漏えいのリスクを下げられます。

#### 出力した内容の扱い

AIが作った文章をそのまま社外に出す前に、機密情報がうっかり混ざっていないか確認してください。また、AIとのやり取りの履歴に機密が残ることもあります。会社のルールに従って、履歴の扱いにも気を配ってください。

### その2 — ハルシネーション(もっともらしい嘘)を防ぐ

#### ハルシネーションとは

AIは、知らないことでも「知らない」と言わず、それらしい答えを作ってしまうことがあります。これをハルシネーション(幻覚)と呼びます。存在しない数字、実在しない出典、間違った日付や固有名詞を、まったく自信のある口調で出してきます。たちが悪いのは、文章として自然なので、間違いだと気づきにくいことです。

#### 特に間違えやすいもの

- **数字・統計・割合** — 「○○%」のような数字は、根拠を確認するまで信じない
- **固有名詞・人名・社名・地名** — 実在しないものや、似て非なるものを出すことがある
- **日付・期限・年号**
- **出典・参考資料** — 存在しない資料名を作ることがある
- **法律・制度・手続きの細部** — 古い情報や不正確な情報が混ざる

#### 防ぐための習慣

1. **自分が渡した資料の範囲で答えさせる** — 「この資料の中だけを根拠に答えて」と指定すると、作り話が減ります。要約や抽出が比較的安全なのはこのためです。
2. **数字・固有名詞・日付は必ず裏を取る** — AIの出力をそのまま使わず、一次情報(公式サイト、原典、自社資料)で確認します。
3. **「分からなければ分からないと言って」と頼む** — それでも間違えることはありますが、無理に埋める頻度は下がります。
4. **重要な判断はAIだけで決めない** — 専門的・重大な事柄は、人や専門家の確認を必ず挟みます。

**コピペプロンプト㉕(ハルシネーションを抑える指示)**

```
回答するとき、次を守ってください。
・私が渡した情報の範囲だけを根拠にする
・推測で補う場合は「推測ですが」と明記する
・分からないことは「分かりません」と正直に言う
・数字や固有名詞は、確実なものだけ書く
```

#### 「AIに確認させる」の落とし穴

「これで合っていますか?」とAIに聞いて、「はい、合っています」と返ってきても、それは正しさの保証ではありません。AIは自分の間違いに気づけないことがあります。最終的な正しさの確認は、必ず人がやる。これは本書全体を貫く原則です。

- [ ] 会社のAI利用ルール(可否・許可ツール・入力可能範囲)を確認した
- [ ] 個人情報・機密情報をそのまま入れていない
- [ ] 「仮名に置き換える」「形だけ渡す」を習慣にしている
- [ ] 数字・固有名詞・日付・出典は一次情報で裏を取っている
- [ ] 「合っていますか?」の返事を正しさの保証にしていない
- [ ] 重要な判断はAIだけで決めていない

## 第10章 三日坊主で終わらせない続け方

### 「全部やろう」としない

新しい道具を学ぶとき、一番ありがちな失敗は「全部いっぺんに使いこなそう」とすることです。本書には多くの使い方を載せましたが、最初から全部やる必要はありません。むしろ、一つか二つに絞ってください。

おすすめは、**「自分が一番つらいと感じている作業」を一つだけ選ぶ**ことです。毎朝のメール返信がつらいなら第2章だけ。議事録の清書がつらいなら第6章だけ。一つの作業でAIが効くと実感できれば、自然と他の場面でも使いたくなります。

### 「自分用プロンプト集」を育てる

うまくいったプロンプトは、メモアプリやファイルに保存しておきましょう。自分の仕事に合わせて少しずつ手を入れていくと、「自分専用のプロンプト集」が育ちます。次回からはそれをコピーして【 】を埋めるだけ。これが続けるうえで一番効きます。

保存するときは、

- どんな場面で使うプロンプトか(見出しを付ける)
- 実際に使った文面
- うまくいったときの工夫(メモ)

をセットで残すと、あとで探しやすくなります。

### 完璧主義をやめる

AIの出力が一発で完璧でなくても、がっかりしないでください。七割で十分です。残りは追加指示で直すか、自分で仕上げる。「ゼロから自分で全部やるより速ければ勝ち」くらいの気持ちで使うと、長く続きます。

### 時短した時間を記録する

モチベーションを保つコツは、効果を目に見える形にすることです。「今日のメール、AIで15分短縮できた」と簡単にメモするだけでいい。一週間、一か月とたまると、自分が取り戻した時間の合計が見えてきます。それが次も使おうという気持ちにつながります。

### 周りと共有する(ルールの範囲で)

うまくいったやり方は、同僚と共有すると、自分の理解も深まります。ただし、共有するのは「プロンプトの型」や「使い方のコツ」であって、機密データを含むやり取りそのものではありません。会社のルールの範囲で、知恵を持ち寄ってください。

### 続け方のチェックリスト

- [ ] 最初は「一番つらい作業」一つに絞った
- [ ] うまくいったプロンプトを保存し始めた
- [ ] 一発完璧を求めず、七割で良しとしている
- [ ] 時短できた時間を記録している
- [ ] 共有するのは「型・コツ」だけにしている
