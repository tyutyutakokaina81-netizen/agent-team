# AI業務自動化プロンプト集 Vol.1

実務でそのまま使える30本のプロンプト集です。フリーランス・個人事業・副業・小規模事業者の日々の作業を、汎用AI（ChatGPT / Claude / Gemini など）で少し速く、少し楽にするためのものです。

---

## 使う前に（必ずお読みください）

### 1. 変数の埋め方
各プロンプトの本文には `{{変数}}` の形で穴埋め箇所があります。例えば `{{商品名}}` なら、あなたの実際の商品名に置き換えてからAIに貼り付けてください。`{{……}}` の部分が一つでも残っていると、AIが文脈を補えず精度が落ちます。空欄のままにせず、必ず具体的な内容で埋めてから使ってください。

### 2. AIの出力は必ず人が確認する
AIの文章は「もっともらしいが間違っている」ことがあります。特に **金額・日付・固有名詞・法律や税務に関わる記述・相手に送る前の文面** は、送信・公開・提出の前に必ず自分の目で確認してください。このプロンプト集は「下書きを速く作る道具」であり、「確認を省く道具」ではありません。

### 3. 機密情報の扱いに注意
顧客の個人情報、未公開の契約内容、パスワード、マイナンバーなどの機微な情報は、安易にAIに貼り付けないでください。多くのAIサービスでは入力内容が学習や品質改善に使われる可能性があります。実名・住所・口座番号などは伏せ字（例：`A社`、`◯◯様`）に置き換えてから使うことをおすすめします。会社で利用ルールがある場合はそれに従ってください。

### 4. 税務・法務・専門領域の判断について
経理・契約・労務などに関するプロンプトは「文面づくりの下ごしらえ」を助けるものです。勘定科目の最終判断、契約の法的有効性、税額の確定などは、税理士・社労士・弁護士などの専門家にご確認ください。

# 1. 企画・アイデア出し

## 1. 切り口を広げるブレスト

**用途**：企画やテーマが「一つの方向」で煮詰まったとき。別の角度のアイデアをまとめて出して、選択肢を増やしたいときに使います。

```
あなたはアイデア出しの相棒です。以下のテーマについて、互いに方向性の異なる切り口を10個提案してください。

テーマ：{{テーマ・解決したいこと}}
対象とする相手：{{誰に向けたものか}}
すでに考えている案：{{自分が思いついている案。なければ「なし」}}

条件：
- 10個はできるだけ重複しない角度で
- 各案は「切り口の一言＋具体的にどうするか1〜2文」で
- 突飛すぎるものより、現実に試せるものを優先
- 最後に、特に試す価値が高いと思う3つとその理由を挙げてください
```

**使い方のコツ**：出てきた案をそのまま採用せず、気になった2〜3個を選んで「これをもっと具体化して」と追加で頼むと精度が上がります。

**出力例**：
> 1. **ビフォーアフター型**：依頼前後の変化を写真や数値で見せる紹介ページにする。
> 2. **よくある失敗型**：その分野で初心者がやりがちな失敗を3つ挙げ、回避法を添える。
> …（中略）…
> **特に試す価値が高い3つ**：2（共感を得やすい）、5（…）、8（…）

## 2. サービス・商品のコンセプト整理

**用途**：頭の中にあるサービス内容を、人に説明できる形に整理したいとき。LPや自己紹介の土台になります。

```
以下の情報をもとに、サービスのコンセプトを整理してください。

サービス内容：{{何を提供するか}}
対象顧客：{{どんな人・会社向けか}}
他と違う点（自分で思う強み）：{{あれば。なければ「不明」}}
価格帯：{{だいたいの価格。未定なら「未定」}}

出力してほしい項目：
1. 一言で言うと（30字以内）
2. 誰のどんな困りごとを解決するか
3. 提供価値（3点まで）
4. 想定される懸念や反論と、それへの答え
5. もし強みが弱い場合、補強の方向性を1つ提案

※断定や誇張は避け、現実的な表現にしてください。
```

**使い方のコツ**：「4. 懸念と答え」は営業トークやFAQの素材になります。出力に納得できない箇所は、自分の言葉で書き直してください。

**出力例**：
> 1. **一言**：忙しい個人店向けに、SNS投稿の下書きを毎週まとめて用意する代行です。
> 2. **困りごと**：投稿を続けたいが時間も文章も負担、という店主の手間を減らす。
> …

**用途**：ブログやSNSのネタが続かないとき。一定期間の投稿テーマを一覧で出して、計画的に発信したいときに使います。

出力形式：

- 実際に書けそうな、地に足のついたテーマにしてください

**使い方のコツ**：全部を一度に書こうとせず、まず最初の2〜3回分だけ本文化するのが続けるコツです。

**出力例**：

## 4. ネーミング・タイトル案出し

**用途**：サービス名・商品名・記事タイトル・メール件名など、短い言葉を決めたいとき。複数案を比べて選びたいときに使います。

```
以下について、候補を15個出してください。

決めたいもの：{{サービス名／記事タイトル／メール件名 など}}
内容の説明：{{どんなものか}}
伝えたい雰囲気：{{例：誠実・やわらかい・きびきび}}
避けたい表現：{{例：大げさな言葉、横文字、なくてOK}}

条件：
- 短いもの・説明的なものをバランスよく混ぜる
- 誇張（「最強」「絶対」など）は避ける
- 各候補に「狙い」を一言添える
- 最後におすすめ3つとその理由を
```

**使い方のコツ**：媒体ごとに最適な長さは違います。記事タイトルなら30字前後、メール件名なら20字以内を目安に絞り込みましょう。

**出力例**：
> 1. はじめての確定申告メモ（やさしく・初心者向け）
> 2. 確定申告、つまずきやすい5点（具体的・実用）
> …
> **おすすめ**：2（中身が想像しやすい）、…

# 2. リサーチ・情報整理

## 5. 長文を要点に圧縮する

**用途**：長い記事・資料・議事録などを読む時間がないとき。要点だけ先に把握したいときに使います。

```
以下の文章を要約してください。

【出力してほしいもの】
1. 3行サマリー（全体を一言で）
2. 重要ポイント5点（箇条書き）
3. 自分が行動するなら、次にやるべきこと（あれば）

【元の文章】
{{ここに長文を貼り付け}}

※元の文章にない情報は足さないでください。判断に迷う箇所は「原文では不明」と書いてください。
```

**使い方のコツ**：「元の文章にない情報は足さない」の一文が、AIの作り話（もっともらしい捏造）を減らすのに効きます。重要な資料は要約だけで判断せず原文も確認を。

**出力例**：
> **3行サマリー**：来月から請求書の様式が変わる。対応は経理担当が行う。締切は月末。
> **重要ポイント**：① …／② …

## 6. 比較表をつくる

**用途**：複数の選択肢（ツール・プラン・外注先など）を並べて検討したいとき。判断材料を表にまとめたいときに使います。

```
以下の選択肢を比較する表を作ってください。

比較したいもの：{{選択肢A、B、C…}}
重視する観点：{{例：価格、使いやすさ、サポート、解約のしやすさ}}
私の状況・優先順位：{{例：費用を最優先。一人で使う}}

出力形式：
- 行＝観点、列＝選択肢の表
- 各セルは簡潔に。わからない項目は「要確認」と記載
- 表の下に「あなたの優先順位ならどれが向くか」を1〜2文で
- 最終判断は私が行うので、断定はしないでください
```

**使い方のコツ**：価格や仕様は古くなりがちです。「要確認」と出た項目や金額は、必ず公式情報で裏取りしてください。

**出力例**：
> | 観点 | プランA | プランB |
> |---|---|---|
> | 月額 | 要確認 | 要確認 |
> | 解約 | いつでも可 | 1か月前申請 |
> 一人利用で費用優先なら、解約しやすいAが無難そうです（金額は要確認）。

## 7. 議事録・メモを構造化する

**用途**：会議や打ち合わせの走り書きメモを、後から読める議事録に整えたいときに使います。

```
以下の打ち合わせメモを、読みやすい議事録に整えてください。

【メモ】
{{走り書きのメモを貼り付け。話し言葉のままでOK}}

【出力形式】
- 決定事項
- 宿題（誰が・何を・いつまで）※不明な箇所は「担当未定」「期日未定」と明記
- 検討中・保留の論点
- 次回までの確認事項

※メモにない結論を作らないでください。曖昧な箇所はそのまま「曖昧」と残してください。
```

**使い方のコツ**：「誰が・何を・いつまで」が埋まらない宿題は、会議後すぐ関係者に確認するきっかけになります。

**出力例**：
> **決定事項**：次回までにサンプルを2案用意する。
> **宿題**：サンプル作成（担当：自分／期日：来週水曜）
> **保留**：価格は先方の予算確認後に再検討。

## 8. 質問リストを設計する

**用途**：ヒアリング・初回面談・取材などの前に、聞き漏らしを防ぐ質問リストを用意したいときに使います。

```
以下の目的に向けた質問リストを作ってください。

場面：{{例：新規顧客との初回ヒアリング}}
相手：{{相手の立場}}
私が最終的に知りたいこと：{{ゴール}}
持ち時間：{{例：30分}}

条件：
- 質問は「必ず聞く」「時間が余れば聞く」に分けて
- 相手が答えやすい順番に並べる（重い話は後半に）
- 各質問のねらいを一言添える
- はい/いいえで終わらない、話を引き出す聞き方を中心に
```

**使い方のコツ**：当日はリスト通りに進めず、相手の答えに合わせて深掘りを。リストは「忘れ防止のお守り」として使うのが実践的です。

**出力例**：
> **必ず聞く**
> 1. 今いちばん困っているのはどんな場面ですか？（課題の核を知る）
> 2. これまで試した方法はありますか？（過去の経緯を知る）

## 9. ビジネスメールを書く

**用途**：要件は決まっているが文面を整える時間がないとき。失礼のないメールの下書きを素早く作りたいときに使います。

```
以下の内容でビジネスメールの下書きを作ってください。

宛先の相手：{{例：取引先の担当者}}
関係性・距離感：{{例：初めて／何度かやりとり済み}}
伝えたい用件：{{箇条書きでOK}}
相手にしてほしいこと：{{例：日程の返信、資料の確認}}
希望のトーン：{{例：丁寧だが固すぎない}}

条件：
- 件名も提案する
- 長すぎず、要件が一読で伝わるように
- 過度なへりくだりや定型の連発は避ける
- 日付・金額など事実は私が入れるので {{ }} のままにしてください
```

**使い方のコツ**：送信前に、宛名・日付・金額・添付の有無を必ず目視確認。AIは事実部分を取り違えることがあります。

**出力例**：
> 件名：お見積りの件（{{案件名}}）
> ◯◯様
> お世話になっております。先日ご相談いただいた件について…

## 10. 提案・企画書のたたき台

**用途**：提案書をゼロから書くのがしんどいとき。骨組みと初稿をまとめて出して、そこから直したいときに使います。

```
以下の情報で、提案書のたたき台を作ってください。

提案相手：{{誰に}}
相手の課題・要望：{{わかっている範囲で}}
提案する内容：{{何を提供するか}}
進め方・期間：{{あれば}}
概算費用：{{あれば。なければ「別途」}}

構成：
1. 背景・現状の課題（相手の言葉で）
2. 提案の概要
3. 具体的な進め方
4. 期待できること（誇張せず現実的に）
5. 費用・スケジュール（不明箇所は「要相談」）
6. 次のステップ

※成果を保証する表現は使わないでください。
```

**使い方のコツ**：「期待できること」は控えめな表現のまま使うほうが信頼されます。盛らないことが結果的に通りやすくなります。

**出力例**：
> **1. 背景**：現在、SNS投稿の作成にお時間を取られているとのことでした。
> **2. 提案概要**：週次で投稿案を5本ご用意し…

## 11. ブログ・記事の構成案

**用途**：記事を書く前に、見出しの流れを設計したいとき。書き出しで迷う時間を減らせます。

```
以下のテーマで記事の構成案を作ってください。

テーマ：{{記事のテーマ}}
想定読者：{{誰が・どんな悩みで読むか}}
読者に取ってほしい行動：{{例：問い合わせ、保存、次の記事へ}}
記事の長さの目安：{{例：2000字程度}}

出力：
- タイトル案3つ
- 見出し（H2／H3）の構成と、各見出しで書く内容を1行ずつ
- 導入文の書き出し案（2〜3行）
- 読者の検索意図に沿っているかの自己チェックを最後に一言

※中身のない見出しの水増しはしないでください。
```

**使い方のコツ**：構成が決まれば本文は書きやすくなります。各見出しの「1行メモ」を箇条書きで膨らませていくと効率的です。

**出力例**：
> **タイトル案**：①…／②…／③…
> **構成**：
> H2 そもそも何が問題か → 読者の状況を言語化
> H2 具体的な手順3つ → …

## 12. SNS投稿を複数パターン

**用途**：同じ内容を、媒体やトーン違いで複数パターン出したいとき。テストして反応を比べたいときに使います。

```
以下の内容で、SNS投稿の文案を作ってください。

伝えたいこと：{{投稿の中身}}
媒体：{{例：X／Instagram}}
トーン：{{例：親しみやすい／きちんと}}
してほしい反応：{{例：保存、プロフへの遷移、返信}}

条件：
- パターンを3つ（出だしの切り口を変える）
- 各パターンの「ねらい」を一言添える
- 文字数の目安：{{媒体に合わせて指定。例：140字以内}}
- 過度な煽り（「絶対」「激変」等）は使わない
- ハッシュタグ案を3〜5個
```

**使い方のコツ**：3案の中から1つを選んで投稿し、反応のよかった切り口を次回の起点にすると、無理なく改善が回ります。

**出力例**：
> **パターンA（疑問から入る）**：請求書、毎月どれくらい時間かけていますか？…
> **パターンB（数字から入る）**：…

## 13. 文章を整える（推敲）

**使い方のコツ**：「厳しめでOK」と添えると、当たり障りのない感想ではなく実用的な指摘が返ってきやすくなります。

**出力例**：
> **良い点**：要点が冒頭にある。
> **改善点**：2段落目が抽象的 → 具体例を1つ足すと伝わる。
> **総評**：信頼は伝わるが、次の行動への誘導が弱い。

## 30. 失敗から学びを引き出す

**用途**：うまくいかなかった案件・出来事を、感情的にならず次に活かす形で整理したいときに使います。

```
以下の出来事を、次に活かすために整理してください。

何が起きたか：{{事実を淡々と}}
自分の対応：{{どう動いたか}}
結果：{{どうなったか}}

出力：
1. 事実の整理（責めも擁護もなく）
2. 振り返ると分岐点だった箇所
3. 次に同じ状況が来たときの選択肢（2〜3個）
4. 今すぐできる小さな対策1つ

※自分を過度に責める方向ではなく、再発防止と改善の視点でお願いします。
```

**使い方のコツ**：「今すぐできる小さな対策1つ」だけでも実行すると、同じ失敗の確率が下がります。完璧な再発防止策より、一歩を優先しましょう。

**出力例**：
> **事実の整理**：納期直前に仕様変更が入り、確認が間に合わなかった。
> **分岐点**：仕様を口頭だけで進めた点。
> **次の選択肢**：① 変更は書面で残す ② 締切に余裕を持たせる
> **小さな対策**：依頼受領時に確認事項をメモする欄を作る。

# 巻末：自分用に改造するコツ

ここまでの30本は「そのまま使える土台」です。あなたの仕事に合わせて少し手を加えると、ぐっと使いやすくなります。以下のコツを参考にしてください。

### 1. よく使う前提を「固定」する
事業内容・トーン・対象顧客など、毎回同じ情報は、プロンプトの冒頭に常設の一文として埋め込んでおきましょう。例：「私は◯◯を提供する個人事業主です。文体は丁寧でやわらかめ。以下の指示に従ってください。」これを各プロンプトの先頭に貼るだけで、毎回の入力が減ります。

### 2. 「悪い例」を見せると精度が上がる
望むものだけでなく「こうはしないで」という例を1つ添えると、AIは外しにくくなります。例：「『いつもお世話になっております』のような定型句から始めないでください。」

### 3. 一度で完成させようとしない
最初の出力に「ここをもっと具体的に」「半分の長さに」「3案目だけ深掘りして」と返す“追い対話”が、いちばん質を上げます。プロンプトは会話の最初の一手だと考えてください。

### 4. 自分の「よかった出力」を保存しておく
うまくいった出力や、改造して定着したプロンプトは、メモアプリやドキュメントに「自分専用版」として保存しましょう。これがあなただけのプロンプト資産になります。

### 5. 出力フォーマットを指定する
「表で」「箇条書きで」「200字以内で」「この見出し構成で」と形式を指定すると、そのまま使える出力になりやすくなります。逆に形式を決めないと、毎回ばらつきます。

### 6. 必ず最後は自分の目で
繰り返しになりますが、AIの出力は下書きです。送信・公開・提出の前のひと手間の確認が、いちばんのリスク対策です。便利さと正確さは、あなたの確認で両立します。
