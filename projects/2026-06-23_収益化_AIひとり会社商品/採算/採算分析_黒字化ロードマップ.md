> 3商品を**合算**すれば、各商品が数部ずつでも合計で容易に到達する（§3シナリオ参照）。

### 2-1. 「構造的に赤字になりにくい」ことの数式証明【事実ベースの構造論】

本ラインは **広告費＝¥0・在庫＝¥0（デジタル）** ＝ 1部売るための追加コスト（限界費用 c）がほぼゼロ。

```
1部あたり限界利益 = 手取り単価 p − 限界費用 c
  ここで c ≈ 0（広告ゼロ・在庫ゼロ・複製コストゼロ）
∴ 1部あたり限界利益 = p − 0 = p > 0   （p は常に正：最小でも¥350）

n部売れたときの月間損益 L(n):
  L(n) = (p − c)·n − F        （F = 固定費 ¥5,800）
       = p·n − 5,800           （c≈0 を代入）
```

- **示唆1：限界利益は1部目から必ずプラス**。`p − c = p > 0` なので、**1部でも売れれば限界利益はプラス**。売れば売るほど損は埋まり、損が増えることはない（売っても赤字が悪化しない構造）。
- **示唆2：固定費は既に発生済の sunk（既存ツール）。** 本ラインで追加負担は¥0。よって「この商品を出したせいで赤字になる」経路は存在しない。**最悪でも損益は固定費¥5,800のまま（=出さない場合と同じ）**、上振れだけが乗る。
- **示唆3：黒字化の必要十分条件は `p·n > 5,800` の一点のみ。** 価格 p を上げる／部数 n を増やす／商品点数を積む（後述§4）で n の母数を増やす、の3レバーだけで黒字に到達する。これ以外に赤字要因がない。
- **赤字になる唯一の経路 ＝「新規課金を増やす」こと。** F を¥5,800超に膨らませない限り、構造上このラインは赤字化しない（→§4運用ルール(a)）。

## 4. 黒字を確実にする運用ルール

### (a) 新規課金を増やさない【最重要・赤字化の唯一の経路を塞ぐ】
- §2-1で示した通り、**このラインが赤字になる唯一の経路は固定費 F を¥5,800超に膨らませること**。
- 画像生成AI・有料プラグイン・有料広告・新規SaaS は**追加しない**。必要なら無料枠／既存ツール（Canva・Claude Pro・MS365）で代替。
- 新規課金の検討が出たら「それは固定費 F を上げる＝赤字回避ラインの部数を引き上げる」と必ず換算してから判断する。

### (b) 商品点数を積む（カタログが資産）
- デジタル商品は出品後も消えず、各商品が独立した「24時間営業の窓口」になる。**点数 N が増えるほど月次の母数 n が積み上がる**（§3で反映した累積効果）。
- 制作コストは私(code)の作業＝**追加の現金支出ゼロ**。Vol.2、Vol.3…と横展開しても F は不変＝**点数を積むほど赤字回避ラインから遠ざかる（黒字方向に厚くなる）**。
- 優先順：実績アンカーのある③テンプレ系を厚く → ②プロンプトVol.2へ展開 → ①Kindleでシリーズ化。

### (c) バンドル／値付けの考え方
- **アンカリング**：単品¥980を並べたうえで「3点セット¥1,980」のような束を置くと、束が割安に見え単価が上がる。手取りは束のほうが厚い（1決済で複数商品分）。
- **下限は実績の¥980を錨に**：Vol.1が¥980で成立しているため、同系統は¥980を基準に上下。むやみな安売り（¥300等）は手取りを削るだけで部数が比例増する保証がない＝避ける。
- **チャネル別最適化**：手取り係数が厚いGumroad(0.88)/BOOTH(0.85)を「冷たくてもいいから単価を取りたい商品」に、note(0.72)を「既存読者に温かく届けて導線にする商品」に振り分ける。
- 値上げは部数が安定してから段階的に（¥980→¥1,280→¥1,480）。価格±の影響は§5感度分析参照。

### (d) KDPは自前マーケットの集客がある（冷たい客に届く）
- note/Gumroad/BOOTHは**こちらが集客した客（既存読者＝温かい客）しか来ない**。母数は自分のリーチに上限される。
- **KDPはAmazon自身の検索・関連商品・カテゴリランキングという巨大な自前トラフィックがある**＝こちらの宣伝なしでも「AIひとり会社」を探す**冷たい客（新規・無関係層）に発見されうる**。広告ゼロ運用と相性が良い唯一のチャネル。
- ゆえに①Kindleは「広告費ゼロでも新規流入が見込める集客装置」と位置づけ、§3標準/強気で後半の伸びを①に置いた。**KDP本文末や著者紹介から②③へ送客**すれば、冷たい客→温かい客への変換装置にもなる。

| 全商品 +20%値上げ | ¥17,089 | ＋¥11,289 | 黒字（利益+34%）|

> ⚠️ 価格変更は実際には部数にも影響する（値下げ＝部数増の可能性／値上げ＝部数減の可能性）が、ここでは**価格単独の感度**を見るため部数固定。
> 示唆：**値下げは手取りを直接削るだけで部数増の保証がない**＝広告ゼロ環境では値上げ寄り（単価で取る）が利益効率は良い。ただし新規認知が薄い立ち上げ期は¥980アンカーで部数を作るのが先。

### 5-3. 最悪ケースの確認（赤字耐性）
- 「全商品が最低価格・部数が保守シナリオの半分」でも、**売れた分の限界利益は必ずプラス**＝出さない場合（損益▲¥5,800固定）より必ず良い（§2-1）。
- **本ラインを出すことで損益が悪化するシナリオは構造上存在しない**（新規課金を増やさない限り）。

## 6. 結論（3行）

1. **赤字リスクの有無：構造的にほぼ無い。** 広告ゼロ・在庫ゼロで限界費用≒0、固定費¥5,800は既存ツールで追加負担ゼロ。1部でも売れれば限界利益はプラスで、損益が固定費より悪化する経路は「新規課金を増やす」以外に存在しない（§2-1で数式証明）。
2. **黒字化の最短ルート：実績アンカー¥980の③テンプレ集をnoteで先行販売し、①KindleをAmazon自前集客の入口に据える。** 標準で月次Month2（'26-08）・累計Month3（'26-09）、強気で累計Month2に黒字化。固定費回収は3商品合算で月7〜17部相当。
3. **最も効くレバー：(1)商品点数を積む（カタログ＝資産、追加コスト¥0で母数nを増やす）＞(2)手取りの厚いチャネル選択（Gumroad0.88/BOOTH0.85/KDP70%>note0.72）＞(3)値上げ（広告ゼロ環境では値下げより値上げが利益効率良）。** 逆に最大のリスク要因は固定費を膨らませる新規課金＝これだけは増やさない。
