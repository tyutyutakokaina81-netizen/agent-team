# 高単価B2Bパッケージ設計 — ¥300Kに現実的に届く「海外集客まるごと代行」

> 作成：CEO 兼 CSO（2026-06-23）
> 目的：単価¥30–50K（営業エンジン既存）を **¥60,000〜¥120,000/社/月** に引き上げ、**3〜5社で月商¥300K** に届かせる。
> 土台：`projects/2026-06-23_B2B英語発信代行_営業エンジン/`（提案書・営業文・お試し・実績見本・納品契約・KPI・反論FAQ・アップセルの11点）。
> 制作：AIが全部やる（code）。外部操作（営業送付・契約・投稿）はオーナー/cowork。**ゼロ予算**。
>
> ## 厳守事項（このファイル全体の前提・例外なし）
> - **A4（PII禁止）**：業種カテゴリ・市レベルのみ。実在店名・地区名・町名・住所・人名は一切書かない。
> - **A5（誇張・成果保証NG）**：「来客増・予約増・売上増・順位上昇」を**約束しない**。約束するのは**「海外向けの質の高い英語コンテンツ・導線を計画的に作り続け、運用すること」**だけ。
> - 値段の根拠は**「外注で個別に揃えたときの相場との比較（提供物ベース）」**で説明する。**成果(ROI)ではなく提供物の量と質**で正当化する。
> - 引き上げは押し売り禁止。「相手が困っている／欲しがっている」が見えてから。
> - 法務・税務（インボイス・契約・源泉）は専門家確認。本ファイルは雛形・設計であり助言ではない。

## 0. 結論（最初に）

- **既存¥30–50Kは「発信代行（SNS＋記事）」止まり。生活規模に届かせるには“点”を売っている。**
- 高単価化の本質は**値上げではなく「束ねる」**こと。海外集客に必要な作業は本来バラバラに外注され、事業者は**英語サイト文・SNS・記事・予約導線の英語化・英語問い合わせ対応**を別々に頼む（=面倒で高い）。これを**1社に丸投げできる月額（DFY＝Done-For-You）**にすると、¥30Kの「投稿屋」ではなく**¥60–120Kの「海外集客の英語まわり担当」**になる。
- **¥300Kへの最短**は単価を上げて社数を減らす設計：**¥100K×3社／¥80K×4社（推奨）／¥60K×5社**のいずれか。社数が少ないほどAIの制作キャパに収まりやすく、解約1社のダメージは大きい＝**¥80K×4が損益とリスクのバランス最良**。

## 2. 高単価パッケージ定義（DFY＝完全代行「海外集客まるごと」）

> 既存3プラン（ライト¥30K/スタンダード¥40K/フル¥50K）は**入口・小規模向けとして残す**。
> その上に**高単価3プラン**を積む。差は「英語発信」から**「海外集客の英語まわり全部」**への拡張。

### 束ねる5領域（これが“まるごと”の中身）

```
① 英語サイト文        : 自社サイト/LP/予約ページの英語版テキスト制作・月次更新
② 英語SNS運用         : 英語投稿の制作＋（合意時）投稿代行＋反応の可視化
③ 月次の英語記事       : 海外読者向け紹介記事（料理/施設/文化/周辺）— 資産が貯まる
④ インバウンド導線     : 海外客が「見つけて→読んで→問い合わせる」までの英語動線の設計・文面
⑤ 英語問い合わせ対応   : 英語FAQ・定型返信テンプレ・（合意時）受信した英語問い合わせの翻訳/下書き
```

> ④⑤が¥30–50K帯に**無い**部分。ここを足すと「投稿屋」から「**海外客の入口を英語で整える担当**」に変わる＝高単価の根拠。

|---|---|---|---|---|---|---|---|---|
| **Bronze（海外発信フル＋）** | **¥60,000** | 主要ページ英語版 初期整備＋**月1回更新** | 英語投稿 **月8本** | 英語記事 **月2本** | 問い合わせ導線の英語文面 整備 | 英語FAQ＋返信テンプレ | 月次（発信量・反応の可視化） | 海外客が来ている個人飲食店・小規模宿 |

#### なぜその値段の価値があるか（提供物ベース・成果保証なし）

- **Bronze ¥60K**：既存フル(¥50K)＋**サイト英語文の月次更新と問い合わせ導線**を足した。SNS/記事の本数も上。**「発信」に“見つけてもらう入口（サイト・導線）”が加わった**分の+¥10K。
- **Silver ¥80K**：**サイト/LP/予約の英語化＋月2回更新＋投稿代行＋記事3本＋英語問い合わせの翻訳・下書き**。事業者から見れば「英語担当を外に置いた」状態。後述の外注比較で¥12–20万相当を¥8万に圧縮。
- **Gold ¥100–120K**：上記に**多言語1（中or韓）＋季節キャンペーン＋四半期の方針提案**。多言語は社内スキル（CDO `chinakorea-magazine-monthly`＝中国語6＋韓国語6の月刊制作）が裏付け。「専任の英語/多言語担当を雇うより安く、止まらない」。

> いずれのプランも約束は**「制作物の量と質を計画的に出し続ける／運用する」**こと。**来客・予約・売上・検索順位は約束しない**（各プラン説明文に明記）。

## 3. なぜ事業者が¥10万払うのか — 費用対効果ロジック（誇張なし）

> 論点：**「高い」ではなく「バラバラに揃えると、もっと高い・面倒・止まる」**。ROIではなく**調達コスト比較**で語る（A5順守）。

### ① 外注でバラバラに揃えた場合の相場（提供物ベース・一般的な外注単価レンジ）

> 数値は「一般的な制作外注で各作業を単独発注した場合のレンジ」。**約束ではなく市場相場の例示**として提示する。

| 必要な作業 | 単独外注したときの目安レンジ |
|---|---|
| 英語ネイティブ/プロによる紹介記事 1本 | ¥15,000〜¥40,000/本 |
| 英語SNS投稿の制作（企画込み）月10本 | ¥30,000〜¥60,000/月 |
| 自社サイト/LPの英語化（初期）＋月次更新 | 初期¥50,000〜＋更新¥20,000〜/月 |
| 英語FAQ・問い合わせ返信テンプレ整備 | ¥20,000〜¥50,000（初期） |
| 多言語（中/韓）展開 1言語 | +¥15,000〜/月 |

→ **Silver相当を別々に揃えると、ざっと月¥12〜20万**（初期費は別）。**一括¥80,000は、提供物の量で見て明確に割安**。

### ② 人を雇う場合との比較

- 英語のできる発信担当を**月¥10万で雇うのは現実に不可能**（最低賃金×時間＋社会保険でそれ以上になり、かつ採用・教育・離職リスクがある）。
- **¥80–100Kの月額外注は、雇用より安く・採用不要・離職しない**。これは事業者にとって理解しやすい比較軸。

### ③ 「束ねる」こと自体の価値（面倒の解消）

- 記事屋・SNS屋・サイト屋・翻訳屋を**別々に管理する手間がゼロ**になる。窓口1つ・トンマナ統一・確認1回。
- 「英語まわり、全部こちらで」＝**事業者の意思決定と管理コストを丸ごと外せる**。これが¥30Kの投稿屋には出せない価値。

### ④ 言ってよい／言ってはいけない（A5の線引き）

| 言ってよい（提供物・調達コスト） | 言ってはいけない（成果保証） |
|---|---|
| 「別々に頼むより、提供物の量でみて割安です」 | 「来客が◯%増えます」 |
| 「英語まわりの管理が窓口1つで済みます」 | 「予約が伸びます／売上が上がります」 |
| 「海外の方が読める・問い合わせできる状態を作り続けます」 | 「検索で上位に出ます」 |
| 「発信量・反応（リアクション/保存/コメント）を可視化します」 | 集客結果や売上を当社の発信の因果として断定 |

## 4. 3〜5社を取る営業設計（ファネルのリアルな率）

> 既存「営業エンジン11点」をそのまま使い、**高単価プラン用に提案の“入口”だけ差し替える**。
> 接触手段・PII遵守・しつこくしない原則は `見込み先設計` のまま（公開窓口/紹介/プル中心）。

### ① 狙う業種（高単価が成立する順）

1. **小規模宿（最優先）** — 客単価が高く、海外比率の必要性が明確。¥80–100Kの費用対効果を理解しやすい。サイト・予約導線の英語化ニーズが強い＝Silver/Gold適合。
2. **体験・アクティビティ施設** — 文化背景の英語解説＝当社の編集翻訳が活きる。導線英語化の価値が高い。
3. **物販（通販あり・特産/工芸）** — 商品の英語紹介＋海外向けページ。多言語化（Gold）と相性。
4. **人気の個人飲食店** — 単価はBronze帯が中心だが決裁が早い。社数を埋める即戦力。

> ⚠️ 飲食は**数は取りやすいが単価が伸びにくい**。¥300Kを少社数で作るなら**宿・体験を本命**に置き、飲食はBronzeで補完する。

### ② どこで当たるか（既存 `見込み先設計` 準拠・公開情報のみ）

- 住人としての一次観察（英語発信が無くて惜しい業種カテゴリをメモ／店名は記録しない＝A4）。
- 公開の観光協会・自治体観光ページ・商工会の**公開事業者一覧/特集**。
- 公開イベント・物産展で対面 → 後日 公開フォームで連絡。
- **紹介**（既存読者・知人経由＝温度が高く最も成約しやすい）。
- **プル**（note等で「地域事業者の英語発信を手伝っている」と1本書き、問い合わせ動線を作る）。

### ③ ファネル — 何件当たれば3〜5社か（保守的なリアル率）

> 高単価ほど成約率は下がる。**¥30Kより¥80–100Kの方が決裁が重い**前提で保守的に置く。

```
接触（公開窓口/紹介/プル）
   │  返信率   保守 8% / 標準 15%     ← コールド中心。紹介経由はこの数倍
   ▼
返信・会話
   │  サンプル受領率  50%             ← 無料お試し（架空業種版→相手業種版）を提示
   ▼
サンプル提示
   │  商談化率  40%                   ← 「実物を見て話を聞く」段階
   ▼
商談
   │  成約率   高単価は 25% / 入口プラン 35%
   ▼
成約
```

#### 必要接触数の試算（標準ケース・コールド中心）

- 返信15% × サンプル50% × 商談40% × 成約25% = **接触1件あたり成約 ≒ 0.75%**
- → **4社成約に必要な接触 ≒ 530件**。月の接触上限（既存オペ＝1日新規3件×平日）≒ **月60件**。
- **コールドだけだと到達に時間がかかる（数ヶ月）**。これを縮めるのが下の2レバー：

| レバー | 効果 |
|---|---|
| **紹介を主軸にする** | 紹介経由は返信率・成約率が数倍。**紹介3〜5本で商談2〜3件**は十分射程。コールドの1/5の労力で成約に近い |
| **プル（自分の発信で呼ぶ）** | 問い合わせてきた相手は関心済み＝成約率が跳ねる。note127本の資産＋実績見本で信頼を即提示できる |

> **現実解＝「紹介＋プルで1〜2社を先に取り、その実績見本（実物）でコールドの成約率を底上げ」**。最初の1社が最難関、2社目以降は実物が効いて加速する。

## 5. 納品キャパ — AIで月3〜5社を回せるか（正直に）

> 体制：**code=制作（記事・SNS・サイト文・FAQ・レポート）／cowork=投稿・公開・ネット検証／オーナー=窓口・確認・最終GO**。

### 1社あたりの月次制作量（Silver ¥80K 基準）

| 提供物 | 月量 | AI制作の重さ |
|---|---|---|
| 英語記事 | 3本 | 中（1本=既存note制作と同等。素材があれば速い） |
| 英語SNS投稿 | 10本 | 軽（記事から派生・テンプレ化可能） |
| サイト/LP英語更新 | 月2回 | 軽〜中（初月の整備が重く、以降は更新のみ） |
| 英語FAQ・問い合わせ返信下書き | 随時 | 軽（テンプレ化すれば都度は短時間） |
| 月次レポート | 1本 | 軽（発信量・反応の集計＝定型） |

→ **codeの制作だけなら4社は無理なく回る**（記事は1社3本＝月12本、既存の日次5本ペースより少ない）。

### 律速は「制作」ではなく「窓口・確認・投稿」

| 工程 | 担い手 | ボトルネック度 |
|---|---|---|
| 制作 | code | 低（AIは量に強い） |
| 確認サイクル（社ごとのトンマナ・修正往復） | オーナー | **高**（社数×月次×修正。これが律速） |
| 投稿/公開（媒体ログイン・反映） | cowork/オーナー | 中（社数分のアカウント管理） |
| 英語問い合わせ対応（受信→翻訳→返信） | code制作＋オーナー送信 | 中（件数次第。Silverで「月◯件まで」と上限を切る） |

### 正直な結論

- **3社：余裕で回る。** 制作・確認とも無理なし。
- **4社（推奨上限）：回る。** ただし**確認サイクルの設計（月1回まとめ確認・修正は1往復まで等）を契約で固定**しないと、オーナーの窓口負荷が膨らむ。
- **5社：制作は可能だが、オーナーの確認・投稿・問い合わせ対応が破綻リスク。** 5社やるなら**全社Bronze（問い合わせ対応を軽く）寄せ**＋**投稿代行を減らす**等で負荷を下げるか、coworkの稼働を増やす前提。
