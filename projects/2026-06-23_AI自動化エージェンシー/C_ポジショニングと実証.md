## 0. 要点（先に結論）

- **ポジショニング一言**：「**AIだけで会社を運営し、自社の業務を自動化してきたチームが、あなたの事業の自動化を代行する**」。
- **唯一の差別化＝ドッグフーディング**：競合（量産型AI自動化代行）は"客の業務"だけをいじる。当社は**自分の会社運営そのものをAIで回している**。売り物を自分で毎日使っている、という構造そのものが証拠。
- **実証の見せ方**：機密ゼロで出せる"仕組みの事実"だけを公開する。具体的には ①多役員オーケストレーション ②受託/継続課金パイプライン（jobs.py / retainers.py）③Webツール3つ ④ゼロ予算運用 ⑤毎日のアウトプット（note記事）。**成果額や顧客名は出さず、"動いている仕組み"を見せる**。
- **最初のリードマグネット**：**「あなたの業務、どこからAI化できる？10分セルフ診断シート」**（後述§3。チェックリストより"自分ごと化"が速く、相談への動線が短い）。

## 1. ポジショニング

### 1-1. ポジショニング・ステートメント（確定版）

> **私たちはAIだけで会社を運営し、自社の事務・制作・営業を自動化してきました。**
> **その同じ仕組みを、あなたの事業の自動化に当てはめます。**
> 道具を売るのでも、流行りを代行するのでもなく、**自分たちが毎日使っている運営のしかた**をそのまま持ち込みます。

### 1-2. なぜこれが効くか（競合との差別化）

| 観点 | 量産型AI自動化代行（競合の典型） | 当社 |
|---|---|---|
| 立場 | 客の業務"だけ"をツールでつなぐ | **自分の会社運営そのものをAIで回している** |
| 証拠 | 他社事例のスクショ・実績数字 | **自社の仕組み（公開できる範囲）を見せられる** |
| 動機の一致 | ツールが増えるほど請求が増える側 | **無駄を削る運用を自分で痛い目に遭って作った側** |
| 予算前提 | 高額ツール前提になりがち | **ゼロ予算・無料ツールだけで回した実装を持っている** |
| 継続性 | 納品して終わり | **自社で継続運用している（retainers.pyで毎月回している）** |

> 言い換え：競合は「AI自動化、やります」。当社は「**AI自動化で、自分の会社を回しています。同じことをやります**」。
> この差は **「実演できるかどうか」** に集約される。実演できる＝信頼の最短ルート。

### 1-3. 一言バリエーション（チャネル別に使い分け）

- LP見出し：**「自社をAIで回してきたチームが、あなたの自動化を代行します」**
- SNSプロフィール：**「AIで会社を運営中。その仕組みを中小事業者の業務効率化に転用。ゼロ予算で始められます」**
- 営業文の冒頭：**「私たちは"AI自動化を売る会社"ではなく、"AIで自分の会社を運営している会社"です」**
- 紹介してもらう時の一行：**「自分とこの業務をAIで自動化してて、それを他社にもやってる人たち」**

### 1-4. 言ってはいけないこと（A5順守）

- 「月◯万円稼げる」「◯%効率化を保証」「絶対に売上が上がる」→ **全部NG**。
- 約束していいのは **「あなたの業務のどこをAIに任せられるか、一緒に切り分けて、仕組みにして渡すこと」** まで。
- 成果は"再現性のある仕組み"で語る。"金額の保証"では語らない。

## 2. 実証（ドッグフーディング）の見せ方 — 機密を出さずに信頼を作る

### 2-1. 原則：「成果額・顧客」は隠し、「仕組みの事実」は見せる

公開してよいのは **再現性のある"やり方"** であって、**金額・顧客名・PII ではない**。
"いくら儲けたか"ではなく、"**どう回しているか**"が信頼の素材になる（しかも A4/A5 に抵触しない）。

| 公開してよい（信頼の証拠になる） | 出さない（機密・A4/A5） |
|---|---|
| 役割を分けてAIに会社運営させている構造図 | 顧客名・実在の店名・地区名・住所・人名 |
| 受託/継続課金を1コマンドで管理する仕組みの存在 | 実際の受注額・売上・入金額・利益 |
| 無料ツールだけで運用している事実（ゼロ予算） | 契約内容・請求書・顧客とのやり取り |
| 毎日アウトプットを出し続けている運用 | 「○○社が△△%改善」型の成果保証表現 |

### 2-2. 見せられる"実物"5点（当社の信頼資産）

1. **多役員オーケストレーション**
   - 事実：CMO/CFO/CMO/CPO/CSO/CDO/CAO/COO/CEO の役割を分け、AIが各役を担って会社を運営している。
   - 見せ方：**役割分担の構造図1枚**（人名・顧客なし）。「1人＋AIで"部署"を持つ」という運用の絵。
   - 客への翻訳：「あなたの会社も、経理・制作・営業を"AIの担当"に分けられます」。

2. **受託パイプライン `jobs.py`**
   - 事実：受注→制作→納品→入金を依存ゼロのCLIで管理。ネット不要でも回る。
   - 見せ方：**ターミナルの `list` 出力イメージ（中身はダミー案件）**。「案件が流れる仕組み」を画で見せる。
   - 客への翻訳：「問い合わせ→見積→納品→請求、の流れを、抜け漏れなく回す仕組みを作れます」。

3. **継続課金管理 `retainers.py`**
   - 事実：複数社×月次サイクル×納品状況×継続課金を1コマンドで把握。
   - 見せ方：**"1社あたりの管理の手間を最小化する設計思想"の説明**（顧客名は伏せる）。
   - 客への翻訳：「毎月くり返す業務こそ、AIで定型化して人の手を空けます」。

4. **Webツール3つ（`server.mjs` / `pipeline_server.mjs` / `tavern.html`）**
   - 事実：ダッシュボード・APIサーバー・iPhoneから操作できるパイプラインサーバーを自作・運用。
   - 見せ方：**「スマホから自社の業務を操作している」という運用スタイル**を1枚で。
   - 客への翻訳：「外出先でもスマホから回せる、自分専用の業務ツールを持てます」。

5. **ゼロ予算運用**
   - 事実：固定費 約¥5,800/月・有料ツール無し・無料手段だけで会社を回している。
   - 見せ方：**「高額ツールを売りつけない理由＝自分が使っていないから」**という一貫性。
   - 客への翻訳：「まず無料・低コストでできる範囲から。最初から高い投資は要りません」。

> ※ 上記はすべて **"仕組みが存在する事実"** であり、A4（PII）・A5（誇張/保証）に触れない。
> ※ スクショを出す場合は **必ずダミーデータ**で。実案件・実顧客のデータは映さない（cowork/オーナー作業時の徹底事項）。

### 2-3. ケーススタディの作り方（機密を出さずに"実例感"を出す）

実顧客を晒せない（A4）ので、**ケーススタディは「自社の自動化ストーリー」を1次ケースにする**のが最強かつ唯一クリーン。

**自社ケース（推奨・第1号）：「ゼロ予算で会社の制作・受注・継続課金を自動化した話」**

骨子（Before → 仕組み → After → 転用ポイント）：
- **Before（課題）**：1人で会社を回すと、制作・事務・営業・進行管理が全部"自分"に乗る。手が足りない。
- **仕組み（やったこと）**：役割をAIに分け（§2-2-1）、受注/納品をCLI化（jobs.py）、継続業務を月次テンプレ化（retainers.py）、操作をスマホから（Webツール）、全部無料ツールで。
- **After（変化）**：日次でアウトプットが出続ける／案件が抜け漏れなく流れる／管理の手間が最小化。
  ※ **金額や"◯%改善"は書かない**。書くのは「手が空いた」「抜けが減った」「毎日出せるようになった」等の**定性変化**まで。
- **転用ポイント（客向け）**：「あなたの業種なら、まずここをAIに任せられます」を3つ提示。

**架空業種ケース（営業用・A4安全）**：
- B2B営業エンジンの「お試しサンプル（架空業種で実演）」と同じ思想を流用。
- 例：「ある小さな宿（架空）の予約問い合わせ対応を、AIの一次返信＋人の確認で回す設計」。
- **実在を匂わせない**。業種・市レベルまで。「うちもこうできるのか」を想像させるのが目的。

**テンプレ（ケーススタディ1枚）**：
```
■ 状況（業種・規模・市レベルまで／実名なし）
■ 困っていたこと（人手・抜け漏れ・反復作業）
■ 当社がやること（役割分担・定型化・ツール化）
■ 変化（定性のみ。金額・%保証は書かない）
■ あなたへの転用（同業/近い業種なら、まずここから）
```

## 3. リードマグネット（無料の入口）

### 3-1. 最初に作る1つ（決定）

**「あなたの業務、どこからAI化できる？ 10分セルフ診断シート」**

- 理由：チェックリスト型より **"自分の業務"に当てはめる→気づき→相談したくなる** という動線が短い。
- ゴール：受け取った人が「**うちはここから始められそう**」と自分で言語化できる状態にする。
- 形式：A4・1〜2枚PDF（無料ツールで作れる）／後でWeb診断にも転用可。
- CTA：最後に「**診断結果を持って15分の無料相談へ**」（押し売りしない一文）。

### 3-2. 診断シートの骨子

**Step 0：考え方（1段落）**
- 「AI化は"全部を任せる"ことではなく、"くり返し・抜けやすい・時間を食う"作業から人の手を空けることです」。

**Step 1：あなたの業務を4象限で棚卸し**
- 縦軸＝頻度（毎日くり返す／たまに）、横軸＝定型度（手順が決まっている／毎回考える）。
- **「毎日くり返す × 手順が決まっている」＝最優先のAI化候補**、と気づかせる。

**Step 2：自動化しやすい業務 8チェック**（Yesの数で"AI化の伸びしろ"が見える）
- [ ] 同じ文面のメール・返信を何度も書いている
- [ ] 問い合わせの一次対応に時間を取られている
- [ ] 定型の事務（請求・見積・入力）が反復している
- [ ] SNS・ブログ等の発信が手が回らず止まりがち
- [ ] 顧客リスト・案件の進捗が頭の中／バラバラのメモにある
- [ ] 海外・他言語の対応をしたいができていない
- [ ] 月初・月末に"いつも同じ作業"がまとまって発生する
- [ ] 「これ自分じゃなくてもよくない？」と思う作業がある

**Step 3：3段階の始め方（伸びしろ別）**
- Yes 1–3：**まず1つだけ自動化**（例：定型メールのAI下書き）。無料で今日から。
- Yes 4–6：**業務を"流れ"にする**（問い合わせ→対応→記録の仕組み化）。
- Yes 7–8：**部署ごとAIに分ける**設計へ（当社の多役員方式の転用）。

**Step 4：よくある誤解（安心させる・A5順守）**
- 「高いツールが要る」→ いいえ、**無料・低コストから**始められます（当社もゼロ予算運用）。
- 「全部AIに置き換える」→ いいえ、**人の判断は残し、手を空けるだけ**。
- 「効果は保証される？」→ **保証はしません**。できるのは"仕組みを作って渡す"こと。

**Step 5：次の一歩（CTA）**
- 「チェックの数と、最初に楽にしたい1つをメモして、15分の無料相談へ。診断シートを見ながら一緒に切り分けます」。

### 3-3. 予備（第2弾以降の候補）

- **「中小事業者のためのAI自動化 はじめの一歩チェックリスト」**（より初心者向け・SNS配布向き）。
- **「定型業務 自動化テンプレ集（メール／問い合わせ／月次事務）」**（実用＝当社の jobs.py/retainers.py 思想の簡易版）。

## 4. 権威・インバウンド導線

### 4-1. SEO記事（狙いキーワード）

> 方針：検索意図＝「やり方を知りたい」「失敗したくない」「うちでもできるか」。**実体験（自社で回している）を軸に**書く＝AIっぽい量産記事と差がつく。

| 優先 | 狙いキーワード | 記事角度（当社の強み＝実証で書く） |
|---|---|---|
| ★1 | AI 自動化 代行 中小企業 | 「代行に頼む前に知っておく3つ／自社で自動化した会社の視点」 |
| ★1 | 業務効率化 AI 導入 | 「無料ツールだけで始めるAI業務効率化（ゼロ予算の実例）」 |
| ★2 | AI 自動化 何から | 「"何から自動化するか"の決め方（4象限の棚卸し）」＝診断シート連動 |
| ★2 | 中小企業 DX 進まない | 「DXが進まない本当の理由と、1人＋AIで始める最小の一歩」 |
| ★3 | ChatGPT 業務 活用 事例 | 「実際に会社運営で使っている使い方（役割分担方式）」 |
| ★3 | 定型業務 自動化 ツール | 「高いツールを買う前に。無料でできる定型業務の自動化」 |

- **内部リンク設計**：各記事末尾 → **診断シート（リードマグネット）→ 無料相談**、の一本道。
- **書き手の信頼線**：すべての記事に「**私たち自身がAIで会社を運営しています**」の一文を添える（ドッグフーディングの再提示）。

### 4-2. SNS

- **見せる中身**：①自社の自動化の"裏側"（仕組み・運用スタイル／顧客なし）②よくある業務の自動化Tips ③診断シート配布。
- **トーン**：教える＞売る。「うちはこう回している」を淡々と。**成果保証ワードは使わない（A5）**。
- **運用**：外部発信はオーナー/cowork。code は投稿文・素材を制作（既存 X 投稿運用と同じ分業）。

### 4-3. 紹介（リファラル）

- ゼロ予算で最も効く導線。**「自分の業務をAIで自動化してて、それを他社にもやってる人」**という一行を、満足した相手が言いやすい形にしておく。
- 仕掛け：無料診断→無料相談で"良い体験"を先に渡す。**紹介はお願いより"紹介したくなる体験"**から。

## 5. 海外展開（グローバル版ポジショニング）

英語圏でも "AI automation agency" の需要は大きく、かつ供給は"量産代行"だらけ＝**ドッグフーディングの差別化がそのまま効く**。

- **Global one-liner**：
  > **"We don't just sell AI automation — we run our own company on it. We'll set up yours the same way."**
- **サブコピー**：
  > "A small team operating an AI-run company on a near-zero budget. We bring the same playbook to your business — start free, automate the repetitive, keep the human judgment."
- **角度**：日本側と同一（実演できることが信頼）。+α で「**bilingual / Japan-based**」＝日本の中小事業者の海外発信代行（既存の B2B 英語発信代行）と地続きにできる。
- **A5注意**：英語でも "guarantee" / "X% growth" は使わない。約束は "set up a working system" まで。
