# A. オファー設計と ¥300K 到達設計 ─ 中小事業者向け AI 自動化・業務効率化 導入代行

最終更新: 2026-06-23 / 起案: CEO 兼 CFO

> 【事実】当社は AI のみで運営される多役員フレームワークであり、`tools/jobs.py`（受託パイプライン管理）・
> `tools/retainers.py`（B2B 継続課金管理）・`tools/make_epub.py`（原稿→EPUB 量産）を依存ゼロ・外部ネット不可（A1）の
> 制約下で実装・実働させてきた。本事業はこの「自社で回している自動化」をそのまま外販する設計である。
>
> 【前提・制約】ゼロ予算／制作・コードは code（外部送信不可・A1）／常時稼働サーバ運用なし／
> 営業・納品・客対応の外部操作はオーナーまたは cowork／誇張・成果保証 NG（A5）／PII NG（A4）。
> 数値は「目標・前提」であり成果を保証するものではない。

---

## 0. 要点（結論先出し）

- **オファーは2層**：① 単発構築（¥100–300K）＋ ② 月額運用・改善（¥30–80K）。入口に低リスクの「診断」を置く。
- **¥300K は少数で届く**。最短は **単発¥150K×2件＝¥300K**（その月で達成）。
  継続では **月額¥50K×4社＝¥200K ＋ 単発¥100K×1件＝¥300K**、または **月額¥80K×4社＝¥320K**。
- **売れる理由＝自社が実証**。当社は AI だけで運営し、受託/継続課金/量産の自動化を自前で実装・運用している「生きた実証」。
  「作れます」ではなく「自分たちで毎日回しています」と言える。
- **律速は営業の人手**（A1 で code は外部送信不可）。制作・設計・改善は無限にスケールするが、
  商談・送付・客対応はオーナー/cowork のキャパが上限。ロードマップはこの1点に最適化する。

> サーバ常駐や API 課金を客に背負わせない（ランニングを増やさない＝解約理由を作らない）。

## 2. ¥300K の算数（単価 × 件数）

すべて「届く組み合わせ」を少数で示す。**1〜4件の意思決定で月商¥300K に乗る。**

| パターン | 構成 | 月商 | 必要な人手（外部操作） |
|---|---|---|---|
| **P1 単発即達** | 単発 ¥150K × **2件** | ¥300K | 商談2・納品2。最短だが翌月ゼロベースに戻る（継続なし） |
| **P2 単発＋継続** | 単発 ¥150K × 1 ＋ 月額 ¥50K × 3社 | ¥300K | 単発で当月、月額で翌月以降を底上げ。**最もバランスが良い** |
| **P3 継続主体** | 月額 ¥80K × **4社** | ¥320K | 商談4を一度決めれば毎月再現。**律速に最も優しい（営業を増やさず継続収入）** |
| **P4 混合（推奨）** | 月額 ¥50K × 4社 ＋ 単発 ¥100K × 1件 | ¥300K | 月額¥200K の安定基盤＋単発¥100K の上乗せ。継続を厚く、単発で調整 |
| **P5 診断ファネル例** | 診断 ¥40K × 5 ＋ うち成約 月額¥50K×2 ＋ 単発¥150K×1 | ¥390K | 診断で母数→成約に流す。立ち上げ期の現実的な動き |

**重要な気づき**：単発は「今月の現金」、月額は「来月も残る現金」。
P1（単発2件）は数字上最短だが、毎月ゼロから営業し直す＝律速（営業の人手）を毎月フルに食う。
**P3/P4 のように月額を4社積むと、5ヶ月目以降は新規営業ほぼゼロでも¥300K が再現**する。これが設計の肝。

## 3. なぜこの会社が売れるか（差別化＝生きた実証）

【事実】当社自身が、AI だけで運営される会社として以下を**実装・運用している**：

- **受託の自動化**：`tools/jobs.py` ─ 受注→制作→納品→入金を1ファイル=1案件の台帳で管理（依存ゼロ・ネット不可でも稼働）。
- **継続課金の自動化**：`tools/retainers.py` ─ 社別×月次サイクル×継続課金を1コマンドで把握（請求/PII は .gitignore で分離）。
- **量産・出版の自動化**：`tools/make_epub.py` ─ Markdown 原稿を KDP 入稿可能な EPUB へ自動変換。note 127本量産の実績。
- **多役員オーケストレーション**：CDO/CFO/CMO/CPO/CSO/CAO＋COO/CEO が役割分担し、`_index.md` 台帳・`STATE.md` 永続メモリで自走。

**訴求の核**：競合の多くは「AI 自動化を作れます」と言う。当社は **「自分たちの会社を AI だけで毎日回しています。
そのやり方をそのまま御社へ移植します」** と言える。これは資料ではなく**動いている実物**で示せる唯一性に近い強み。
（※ A5 順守：「世界唯一」等の断定はしない。事実＝自社運用していること、だけを根拠に語る。）

差別化の3点：
1. **実証ベース**：他社は事例集、当社は自社運用ログ・実装コードそのものをデモにできる。
2. **ランニングを増やさない設計**：客に常時サーバ/高額 API を背負わせない（ゼロ予算思想の外販）。解約理由を作らない。
3. **手順書まで渡す自走設計**：属人ブラックボックス化しない＝信頼され、月額の継続率が上がる。

## 4. 6ヶ月ロードマップ（保守＝標準シナリオ）

固定費 ¥5,800/月。黒字 KPI は入金累計 ≥ ¥5,800/月（受託1件で突破）。¥300K は「乗る月」を明示する。
**律速は一貫して「営業の人手」**＝オーナー/cowork が月に回せる商談・送付の本数。制作側（code）は律速にならない。

| 月 | 重点アクション | 想定成果（保守） | 月商目安 |
|---|---|---|---|
| **M1** | 診断パッケージ＋B-1/B-2 の納品テンプレを完成（code）。営業1枚物・実証デモを整備。 | 診断 2–3件 | ¥80–150K |
| **M2** | 診断→構築への接続。単発 B 系を初成約。`retainers.py` で1社目の月額を起票。 | 単発1件＋月額1社 | ¥130–200K |
| **M3** | 構築→月額への転換を徹底（納品時に運用提案をセット）。月額2社目。 | 単発1件＋月額2社 | ¥150–250K |
| **M4** | 月額の安定運用（code が量産・改善を回す）。営業は新規月額に集中。 | 月額3社＋単発1件 | ¥200–280K |
| **M5** | **¥300K 到達想定月**。月額4社（¥50K×4＝¥200K）＋単発¥100K。P4 構成が立つ。 | 月額4社＋単発1件 | **¥300K** |
| **M6** | 月額継続で営業負荷を下げつつ Pro(¥80K) へアップセル。再現性を確認。 | 月額4–5社＋単発 | ¥300–360K |

**律速の明記**：上表は「オーナー/cowork が月に商談 4–6件・納品 2–4件を回せる」前提。
ここが詰まると全シナリオが後ろ倒しになる。したがって **打ち手の優先順位は「営業の手間を減らす仕組み化」**：
- 診断シート/提案書/見積を雛形化（code が作る）＝商談1件あたりの所要を最小化。
- 月額で「営業し直さない収入」を積む＝M5 以降は新規営業ゼロでも¥300K 近辺を維持。
- 納品は手順書込みで自走させる＝サポート工数（＝人手）を増やさない。

## 5. 正直なリスクと対策（A5 順守・誇張しない）

| リスク | 事実 | 対策（価格・差別化で負けない設計） |
|---|---|---|
| **競合急増** | AI 自動化代行は需要が熱い反面、参入も急増中。価格competition に巻き込まれやすい。 | 「実証＝自社運用」を全面に出し、価格勝負を避ける。安値の単発受託合戦に乗らず、**月額の継続価値**で差別化。 |
| **成果が読みにくい/保証できない** | 削減時間や効果は業務依存で、保証は A5 違反かつ実態にも反する。 | 診断で「試算」を提示（保証ではないと明記）。**月額は成果保証でなく"回し続ける運用"を売る**＝期待値の不一致を防ぐ。 |
| **律速＝営業の人手** | code は A1 で外部送信不可。商談・送付・客対応は人に依存し、ここが上限。 | 営業資料・見積・提案を雛形化し1件あたりを軽量化。月額を積んで新規営業依存を減らす（§4）。 |
| **属人化・ブラックボックス不信** | 「AI に任せて中が見えない」は中小事業者の典型的な不安。 | **手順書・プロンプト資産まで渡す自走設計**。客が中身を理解・編集できる＝信頼と継続率を両取り。 |
| **ランニングコスト懸念** | 客側に常時サーバ/高額 API を負わせると解約理由になる。 | ゼロ予算思想を外販：ローカル実行・無料/低額ツール前提。客のランニングを増やさない＝解約しにくい。 |
| **品質ばらつき（量産の宿命）** | 量産は質が落ちると炎上/失注に直結。 | 納品前に `tools/quality_check.py` 相当のチェックを内製運用。トンマナ定義＋チェックリストを標準添付。 |
| **値付けの下振れ** | 安く受けすぎると数で潰れる（人手律速のため件数を増やせない）。 | 単発¥100K 未満は原則受けない。少数高単価＋月額の方が律速に合う（§2 の P3/P4）。 |

**正直な総括**：需要は確かに熱い。だが「誰でも作れる」領域でもあり、差別化しなければ価格で削られる。
当社の勝ち筋は **①自社運用という実証 ②客のランニングを増やさない設計 ③手順書まで渡す自走・継続** の3点で、
**少数の月額契約を積んで律速（営業の人手）を毎月食わない構造**にすること。これが¥300K を「単発の運次第」から
「再現する月商」に変える唯一の道。
