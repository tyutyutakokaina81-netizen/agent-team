# プロジェクト：noteへの執筆・投稿（2026-04-18 開始）

## 目的・背景

- 旧「月30万自動化（4本柱）」は実績が出なかったため停止
- 小さく始めて発信を継続することを優先する
- まずは note への執筆・投稿を主軸として、発信の習慣化と読者獲得から始める

## ゴール

【短期（1ヶ月）】
- 投稿習慣を確立する（最低週1本）
- 自分の発信スタイル・文体を確立する

【中期（3ヶ月）】
- 継続フォロワーを獲得し、有料記事 or マガジン化のテストを行う

【長期】
- noteを軸とした安定的な発信基盤を作る（数値目標は走り出してから決める）

## 関与役職

| 役職 | 担当 |
|------|------|
| CMO  | 記事の企画・タイトル設計・SEO/見出し構成・投稿計画 |
| CPO  | 本文構成テンプレ・連載/マガジン設計・読者導線 |
| CDO  | 執筆ワークフローのプロンプト整備・投稿補助ツール |
| CSO  | 読者反応・コメント対応（投稿後に発生し次第） |
| CFO  | note売上・経費の記録（売上発生後） |

## サブフォルダ構成（採用：役職別＋成果物別ハイブリッド）

```
projects/2026-04-18_note執筆/
├── brief.md          ← 本ファイル
├── CMO/              ← 企画・タイトル案・SEO・投稿カレンダー
├── CPO/              ← 構成テンプレ・連載/マガジン設計
├── CDO/              ← 執筆プロンプト・ツール・ワークフロー
└── posts/            ← 記事本体
    ├── drafts/       ← 下書き（YYYY-MM-DD_タイトル.md）
    └── published/    ← 投稿確定版（投稿後にここに移動）
```

採用理由：企画・構成・ツールは役職で明確に分かれる一方、記事本体は
投稿フローを管理しやすいよう `posts/` に集約した方が運用しやすいため。

## 運用ルール

- 記事の下書きは `posts/drafts/YYYY-MM-DD_タイトル.md` に保存する
- 投稿確定後は `posts/published/` に移動し、投稿日・URLをファイル冒頭に記録する
- 各役職の成果物（テンプレ・プロンプト等）は該当役職フォルダに保存する
- 作成・更新したファイルは必ず該当役職の `_index.md` にも追記する
- **note への実投稿はオーナー承認必須**（外部送信に該当）

## オーナー承認待ち事項（要決定）

以下は走り出す前にオーナーの方針確認が必要：

1. **テーマ／ジャンル**
   - AI活用・Claude Code・自動化の知見を発信するか
   - フリーランスの実務（収支管理・契約・税務など）か
   - まったく別ジャンル（趣味・日常・専門領域）か
   - 複数テーマを並行するか、1テーマに絞るか

2. **投稿頻度の目安**
   - 週1本／週2本／毎日 など

3. **無料／有料の比率**
   - 全部無料スタートか、最初から有料記事を混ぜるか

4. **既存資産の再利用可否**
   - `projects/2026-04-08_月30万自動化/` 内のテンプレ販売知見・SEOライティング知見を
     note記事の素材として転用してよいか
