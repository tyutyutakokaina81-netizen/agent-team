# CPO評価・改善案：¥100入口note「富山 外さない12選」

- **対象（入力・改変禁止）**: `CMO/outputs/2026-07-08_有料note_100円入口_富山外さない12選.md`（公開済 https://note.com/safe_canna441/n/nb90084f378c8 ）
- **目的**: 口座不要で初売上（1件）の確率を最大化する。金額でなく「有料が1件出るか」の入口。
- **原則**: 本ファイルは改善"案"。既存原稿は上書きしない。A5（誇張・断定なし）／A4（番地・個人店名なし）厳守。

## 0. 結論（要点）

- **総合評価：C（辛口・鬼コンサル基準）**。「悪くない・きれい」だが、**"¥100払わないと惜しい"という痛点が作れていない**。無料3件で読者は満足して離脱する構造。
- **まず変えるべき1点＝無料部分の切れ目（クリフハンガー）**。今の「ここから先の9つだ」は"あと9個あります"という**数量の予告**にすぎず、購買トリガーになっていない。ここを「**知らないと損する具体**」に変えるのが最小工数・最大効果。
- 前提：**フォロワー27（STATE.md）＝観客ほぼゼロ**。原稿をAにしても、読む人が居なければ売れない（§5参照）。原稿改善は"確率"、集客は"母数"。両輪。

## 1. 鬼コンサル評価（6軸・C評価前提で辛口）

採点はGS/BCG/マッキンゼー級の「厳しく」。AIは肯定に流れるので意図的に減点方向で見る。

| 軸 | 点(10) | 評価 | 辛口コメント |
|---|---|---|---|
| ① タイトルの引き | 4 | C | 「外さない12選」「保存版」は既視感の塊。ネットに1万本ある型。**"住人"という唯一の武器がタイトルで死んでいる**。「地図検索ワードつき」は機能説明で、感情を動かさない。 |
| ② 無料部分の"続き読みたい"力 | 3 | C− | 最大の欠陥。無料3件（白えび/雨晴/金屋町）が**強すぎて自己完結**。「この3つで来てよかったになる」と書いた瞬間、読者の課題は解決＝離脱。クリフハンガーが「あと9個」という**数量予告**で、欠乏感を作れていない。 |
| ③ 有料の価値 | 5 | C+ | 内容は正確で誠実。ただし残り9件が**"有名どころの列挙"**（富山ブラック/瑞龍寺/ガラス美術館…）＝ガイドブックや検索で無料で出る。¥100の理由が「12個そろう」だけでは弱い。**住人しか知らない付加情報（時間帯/回避/合わせ技）が薄い**。 |
| ④ 価格妥当性 | 7 | B− | ¥100自体は入口として妥当（低摩擦・note最低価格）。ここは大きく崩れていない。問題は価格でなく「払う理由」。 |
| ⑤ ¥980への上げやすさ | 4 | C | 末尾CTAが「行程・時間帯・行き方・宿まで含めた完全版」＝**機能の増量アピール**。¥100を買った人が「9.8倍の価値がある」と感じる**橋（bridge）が無い**。¥100と¥980の差が"量"に見え、"次元"に見えない。 |
| ⑥ 差別化 | 3 | C− | 「住人が選ぶ」が看板なのに、中身が**住人でなくても書ける一般情報**。唯一無二の資産（住んでいる＝生活動線・失敗経験・地元の判断基準）が本文に落ちていない。ここが最大の伸びしろ。 |

**総合：C**。誠実で減点は少ないが、加点（買う理由・独自性・欠乏感）が決定的に不足。「読める記事」であって「買われる商品」になっていない。

### 一番痛い指摘
> **無料部分が"商品のサンプル"でなく"商品そのもの"になっている。**
> 白えび・雨晴・金屋町という"最強の3枚"を無料で出し切り、有料に"二軍"を回している。順序が逆。無料は「地元の判断基準（＝なぜ外さないと言えるのか）」を見せ、有料に「具体の外さない12」を置くべき。

## 2. 具体改善案（A5厳守＝買いたくなるが盛らない）

### 2-1. タイトル改良3案（機能訴求→痛点/独自性訴求へ）

現行：`【¥100・保存版】富山・高岡・氷見、住人が"外さない"12選（地図検索ワードつき）`

- **案A（失敗回避・痛点型）**
  `富山、ガイド通り回って"外した"人へ。高岡住人が友人だけに教える12（¥100）`
  → 「網羅ガイドで失敗した」層の痛みに刺す。住人＝内輪感。

- **案B（判断基準・独自性型）**
  `富山で「どこで食べる？」に住人が5秒で返す答え。外さない12と、選ぶ基準（¥100）`
  → 12個だけでなく"基準"を売る＝住人しか出せない価値を明示。

- **案C（時間・限定リソース型）**
  `富山、半日しか無いならこの順で。高岡住人が絞った"外さない"12（地図ワード付・¥100）`
  → 「時間がない旅行者」の具体シーンに接続。実用の切迫感。

> いずれもA5遵守（「世界一/唯一/最高」不使用、断定回避、住人トーン）。**推奨＝案A**（痛点が最も強い）。

### 2-2. 無料部分の"最後の1行（クリフハンガー）"改良

現行の切れ目（弱い）：
```
この3つだけでも、富山は"来てよかった"になる。
ただ、住人が本当に「これを外すと惜しい」と思うのは、ここから先の9つだ。
■ ここから先が、住人の「外さない」12です
```
問題：①「3つで来てよかった」で満足させて離脱を促す ②残りが"数量（9つ）"の予告で欠乏感ゼロ。

**改善案（欠乏感＋具体の予告＝"知らないと損"に変える）**：
```
この3つは、観光ページにも必ず載っている。誰でも辿り着ける。
問題はその先だ。同じ市場でも「行く時間」で当たり外れが変わるもの、
地元が"わざわざ避ける曜日"があるもの、そして観光客がまず見落とす
「歩いて数分の合わせ技」——住人が友人にだけ言う9つは、ここから。

■ ここから先が、住人の「外さない」12（時間帯・避ける日・合わせ技つき）
```
狙い：
- 無料3件を「誰でも着ける入口」と再定義し、**自己完結感を壊す**（満足で終わらせない）。
- 有料に「時間帯／避ける日／合わせ技」という**無料検索では出ない住人情報**があると予告＝買う理由を先出し。
- ※この予告を出す以上、**有料本文に実際にその情報を足す必要がある**（下の2-3）。予告倒れはA5違反かつ信頼毀損。

### 2-3. 有料部分の見せ方（"列挙"から"住人の判断"へ）

現行は12件の名称＋地図ワードの列挙。**各項目に住人しか出せない1行を足す**だけで価値が跳ねる（誇張でなく事実ベース）。例（A4遵守・個人店名/番地なし）：
- ます寿司：「駅で選ぶなら"酸味強め/厚め"は好みが割れる。**迷ったら小サイズ2種を1個ずつ**が住人の買い方。」
- 富山ブラック：「**単体で頼むと塩辛い。白飯前提**の味。初見はご飯を必ず付ける。」
- 雨晴海岸（無料側に留めるなら別途）：「立山が見えるのは**空気が澄む朝と、冬**。昼や夏の曇りは"ただの海"になりやすい。」
- 環水公園：「**夜のライトは時期で点灯時間が違う**。冬の日没後が静かで良い。」

構造改善：
- 12件を**「食7・歩く5」でなく「まず外さない核4＋余裕があれば8」の2段**に組み替える。読者が"どれから"で迷わない＝実用価値。
- 末尾＋αの「住人メモ」は良い資産。**もっと前・各項目に分散**させ、"住人感"を全編に効かせる。

### 2-4. 末尾CTA文言（¥980への"次元の橋"を架ける）

現行（量の訴求）：
```
この12を「行程」としてつなぎ、時間帯・行き方・宿の選び方まで含めた
完全版が、保存版ガイド（¥980）です。
```
**改善案（点→線＝役割の違いを明示）**：
```
ここまでは「何を外さないか（点）」。
でも実際の旅で迷うのは「どの順で、何時に、どう動くか（線）」です。
――半日/1日/2泊で組んだ"回る順番"、朝いちで行くべき場所、
雨の日の差し替え、宿を取るエリアの選び方までを1本にした保存版が¥980。
"リスト"で足りた人はここで十分。"行程"が要る人だけ、こちらへ。
→ https://note.com/safe_canna441/n/n6e7ca70475c8
```
狙い：¥100＝リスト（点）、¥980＝行程（線）と**役割を分ける**。「¥100で足りた人はそれでいい」と言い切ることで押し売り感を消し（A5）、逆に"線が要る人"の自己選別を促す。

## 3. A/Bテスト設計（1変数のみ・owner/worker実行可能）

### 大原則
- **一度に変えるのは1つだけ**（同時に複数変えると何が効いたか不明＝学びが消耗品にならない）。
- 母数が小さい（フォロワー27）ので、**統計的有意は狙わない。方向性の学習**を目的にする。数字と期限を必ず切る（溝口式：期限なき計画は計画でない）。

### まず試す1つ＝【無料部分の切れ目（クリフハンガー）】
理由：6軸で最下位の②が最大のボトルネック。タイトルより本文内で読者が離脱している可能性が高く、**note編集画面で無料部分だけ差し替え可能＝最小工数**。有料本文の予告倒れを避けるため、2-3の住人1行も同時に入れる（これは"クリフハンガーの裏付け"であり別変数ではない＝予告と実体はセット）。

| 項目 | 内容 |
|---|---|
| 変える点 | 無料部分の最後2ブロック（2-2案）＋見出しに「時間帯・避ける日・合わせ技つき」 |
| 変えない点 | タイトル・価格¥100・有料ラインの位置・ハッシュタグ |
| 成功指標（主） | **購入率＝購入数 ÷ 記事ビュー数**（noteダッシュボードで取得） |
| 成功指標（副） | 有料エリアまでのスクロール到達（noteの"読了/スキ"で代替観測）、スキ数、¥980へのクリック |
| 判定期間 | **14日間**（2026-07-12〜07-26）。ただし**購入1件が出た時点で"入口は機能する"と即判定**（初売上が最優先KPI） |
| 判定ルール | 期間内 購入0かつビュー<50 → **問題は原稿でなく集客（母数）**と確定し、§5へ移行。ビュー≥50で購入0 → タイトル（次の変数）へ。購入≥1 → 成功、型を横展開。 |

## 4. 失敗の構造3問（この評価自体を消耗品にする・溝口式）

- **何が**: 無料部分が商品そのものになり、有料に"買う理由"が無かった（C評価）。
- **なぜ**: 「親切に情報を渡す」＝良い記事の発想で作り、「欠乏を作って対価を得る」＝商品の発想が抜けていた。
- **次どう変える**: 無料＝入口/判断基準、有料＝住人しか出せない具体、という**役割分離**を全商品の型にする。

## 5. A評価で止まるな（思考停止の回避・最重要の1行）

> **たとえ原稿をA評価にしても、フォロワー27＝観客がほぼゼロなら購入は出ない。** 原稿改善は"来た人が買う確率"を上げるだけで、"来る人の数"は1ミリも増やさない。§3の判定で「ビュー<50・購入0」なら、詰まりは原稿でなく**集客（母数）**であることが確定する（STATE.mdの結論と一致）。その時は原稿をいじり続けるのをやめ、**無料の人気記事から本記事への内部導線／note内交流／KDPでの客の"借用"**へ資源を移すこと。原稿A化と集客は"どちらか"でなく"順番と両輪"。**この¥100入口の役割は「客が来た時に確実に1件へ変える"変換器"を先に仕込んでおく」こと**であり、それ自体が集客の代わりにはならない。
