# クラウドソーシング受託 — サービスメニュー・料金・受注判断基準（CFO/CSO）

最終更新: 2026-06-23 ／ 担当: CFO（採算・黒字維持）×CSO（受注判断・顧客対話）

---

## 0. この資料の前提と免責（必読）

- **目的**：クラウドソーシング（ランサーズ／クラウドワークス／ココナラ等）で受託する際の、
  ブレない「メニュー＋料金＋受注判断」を1枚に固める。場当たりの安請け合いを防ぎ、**黒字を維持する**ための社内基準。
- **黒字の定義（最低ライン）**：固定費 **¥5,800/月**（Claude Pro ¥3,000＋Canva ¥1,500＋M365 ¥1,300）を
  粗利で確実に超えること。【事実：CLAUDE.md ランニングコスト】
- **A4（PII禁止）**：本資料・見積・納品物に、実在の個人名・住所・電話・地区/町名・取引先名は書かない（業種・市レベルまで）。
- **A5（誇張・成果保証禁止）**：「必ず売れる」「順位1位保証」「来客増を約束」等は言わない。
  約束するのは **アウトプットの質と納期** であって、クライアント側の成果（売上・順位・反響）は保証しない。
- **料金の扱い**：すべて**市場の一般的な相場帯の「目安」**。断定ではない。
  プラットフォーム・時期・難易度で変動するため、出品時は**最新の相場をオーナーが [要確認]** すること。
  → 表中の価格は「この帯を下回ったら受けない」という**自衛ライン**として使う。

## 1. サービスメニュー（8種・単位／想定価格帯／標準納期／スコープ）

## 2. 時給換算の考え方（割に合うかの判定軸）

> 出典思想：Vol.1「案件別時給」＝`CMO/outputs/2026-06-03_note記事_自分の時給を計算する話.md`。
> **「単価」ではなく「時間あたりの手取り＝実効時給」で受注を判断する。**

### 2-1. 実効時給の式
```
実効時給 ＝ (請求額 − プラットフォーム手数料) ÷ かかる総時間

・総時間 ＝ 提案/見積 + ヒアリング + 調査 + 制作 + 修正 + 納品 + 事務 の合計
  （実作業だけでなく、やり取り全部を含めるのがVol.1の肝）
・手数料 ＝ 受注額のおおむね15〜22%[要確認]
```

### 2-2. 社内の最低時給ライン（自衛）
| ライン | 実効時給 | 判断 |
|---|---|---|
| 目標 | **¥2,500/時 以上** | 積極的に受ける |
| 許容 | ¥1,500〜¥2,500/時 | 受けてよい（リピート見込み・実績作りなら可） |
| 警告 | ¥1,000〜¥1,500/時 | 原則辞退。初回お試し1件までに限定 |
| 即辞退 | **¥1,000/時 未満** | 受けない（≒バイト以下。Vol.1の「コンビニより低い」の再来を防ぐ） |

### 2-3. 試算例（記事執筆3,000字）
- 請求 ¥6,000 ／ 手数料20%＝▲¥1,200 ／ 手取り ¥4,800
- 総時間：構成1.0h＋執筆2.5h＋推敲1.0h＋やり取り0.5h＝**5.0h**
- 実効時給 ＝ ¥4,800 ÷ 5.0h ＝ **¥960/時** → ❌ 即辞退ラインに接触
- → **対策**：(a) 文字単価を¥2.0以上に上げる、(b) 構成テンプレ・AI下書きで総時間を3.5hに圧縮 →
  ¥4,800÷3.5h＝**¥1,371/時**（許容下限へ改善）。**「単価UP×時短」の両輪**でラインを超える設計にする。

> 教訓：低文字単価（¥1.0前後）の記事は、手数料と修正対応を入れると簡単に時給割れする。
> **安い記事を量で取る戦略は赤字化しやすい** → メニューの下限ガードで防ぐ。

## 3. 受ける／受けない 判断チェックリスト（CSO）

提案を出す前に全項目を確認。**1つでも「即辞退」に該当したら出さない。**

### 受けてよい（GO）
- [ ] 想定実効時給が **¥1,500/時 以上**（第2章）
- [ ] スコープ（成果物・分量・回数）が**明文化**できる／こちらから定義して合意できる
- [ ] 修正回数の上限に合意できる（標準2回／追加は有料）
- [ ] 納期に**バッファ込みで間に合う**（既存の発信業務を圧迫しない）
- [ ] 依頼内容が自社の得意領域（執筆・英訳・編集・資料・プロンプト）に重なる
- [ ] PIIを**匿名化／市レベル**で扱える

### 受けない（NO GO・1つでも該当で辞退）
- [ ] **安すぎ**：実効時給 ¥1,000/時 未満になる単価（量でカバーは赤字）
- [ ] **無限修正**：「納得いくまで」「何回でも無料修正」が条件。回数上限を拒否される
- [ ] **スコープ不明**：「いい感じに」「お任せで」だけで、分量・用途・締切が決まらない／決めさせてくれない
- [ ] **PII過多**：実名・住所・連絡先・機微情報を含み、匿名化できない（A4違反リスク）
- [ ] **成果保証の要求**：「上位表示保証」「売上◯倍」「来客増」を約束させられる（A5違反）
- [ ] **無償テスト過大**：1記事まるごと等、実質納品レベルの無料テストを要求される
- [ ] **連絡が地雷**：即レス常時要求・深夜連絡前提・人格否定/値切り交渉が攻撃的
- [ ] **権利・責任が不当**：著作権全譲渡なのに低単価／誤情報の最終責任を全部こちらに押し付け
- [ ] **規約グレー**：プラットフォーム外への誘導強要・本人確認回避など

> 迷ったら「**実効時給**」と「**スコープが文章で固定できるか**」の2点に戻る。両方クリアできなければ辞退。

## 4. 黒字ライン早見（月¥5,800 を超えるのに必要な受注の目安）

> 前提【試算】：プラットフォーム手数料20%控除後の手取りで計算。固定費¥5,800/月を**手取りで**超えれば黒字。
> （実際は所得税・住民税等が別途かかるが、ここは「事業として赤字にしない」＝固定費回収の最低ライン。）

| サービス | 1案件の請求(目安) | 手数料20%後の手取り | **固定費¥5,800回収に必要な本数/月** |
|---|---|---|---|
| 記事執筆 3,000字 | ¥6,000 | ¥4,800 | **約1.3本 → 2本で黒字** |
| 記事執筆（文字単価¥1.0の薄利） | ¥3,000 | ¥2,400 | 約2.5本 → 3本（時給割れ注意・非推奨） |
| 和文英訳 1,000字 | ¥9,000 | ¥7,200 | **約0.8本 → 1本で黒字** |
| 編集翻訳 1本 | ¥12,000 | ¥9,600 | **1本で黒字**（最効率帯） |
| SEO構成 1本 | ¥5,000 | ¥4,000 | 約1.5本 → 2本 |
| 資料作成 10p | ¥15,000 | ¥12,000 | **1本で黒字** |
| 商品説明文 1点 | ¥4,000 | ¥3,200 | 約1.8本 → 2本 |
| 議事録整形 30分 | ¥3,000 | ¥2,400 | 約2.5本 → 3本 |
| プロンプト設計 1件 | ¥6,000 | ¥4,800 | 約1.3本 → 2本 |

### 早見の結論
- **損益分岐の最短ルート**：英訳/編集翻訳/資料作成は **月1本** で固定費を回収できる。
- **記事執筆は月2本**（適正単価なら）で黒字。**薄利単価だと月3本必要＝割に合わない**。
- **安全運用の目安**：月の手取りで **¥10,000以上**（固定費の約1.7倍）を確保すると、
  手数料率の変動・1件のトラブル返金があっても赤字に落ちない。
  → 例：**編集翻訳1本＋記事執筆1本** で達成可能。

## 5. 値上げ・リピート単価UPの考え方

### 5-1. 値上げの判断トリガー（どれか1つで検討）
- 直近3〜5案件で**実効時給が目標¥2,500/時を安定して超えている**（＝余力がある＝上げても通る）
- 依頼が**断れないほど来ている**（供給が需要に追いつかない＝価格シグナル）
- 高評価・リピート指名が付いた（実績が単価を裏付ける。**ただし誇張表記はしない／A5**）

### 5-2. 上げ方（角を立てない）
1. **新規から先に上げる**：既存はしばらく据え置き、新規見積から新単価を適用。
2. **段階値上げ**：一気に倍ではなく **+10〜20%** ずつ。理由は「品質維持／納期厳守のため受注数を絞る」。
3. **メニュー再設計で実質値上げ**：下限プランを廃止し、標準プランを底上げ（安いプランを“見せない”）。

### 5-3. リピート単価UP（同じ客から、より良く稼ぐ）
- **単発→継続（保守/定例）契約へ**：月◯本セットの定額化（例：英訳 月4本パック）。
  単価は少し下げても**実効時給は上がる**（営業・ヒアリング時間が毎回ゼロになるため）。← これが本命。
- **アップセル**：記事執筆 → SEO構成＋英訳＋商品説明文を束ねる（1案件の単価を上げる）。
- **クロスセル**：和文英訳の客に「海外読者向けの編集翻訳」を提案（高単価メニューへ誘導）。
- **値引きより“付加価値”**：安くするのではなく、納期短縮オプション・追加修正券などで満足度を上げ、指名リピートにする。
- **リピート割の設計**：割引は「**継続前提でこちらの実効時給が上がるとき**」だけ出す（営業コスト削減分を還元）。
  単発の値切りには割引で応じない（一度下げると戻せない）。

## 6. 運用メモ（更新ルール）
- 価格帯はすべて目安。プラットフォーム手数料率・相場は変動するため、**出品/見積前にオーナーが最新値を [要確認]**。
- 本資料更新時は `CFO/_index.md` および `CSO/_index.md`（該当役職台帳）に追記、`context/STATE.md` に反映。
- A4/A5 を破る依頼は、単価が高くても受けない（規則 > 売上）。
