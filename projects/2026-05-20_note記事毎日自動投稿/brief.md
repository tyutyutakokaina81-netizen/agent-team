## 3. サブシステム別 設計詳細

### 3.1 文体ガイド（assets/style_guide.md）

過去のnote投稿から抽出する要素：

- 一人称・語尾・改行リズム
- 出だしのパターン（問いかけ／日常風景／引用）
- 段落の長さ（短い段落を多用 / 長文1ブロック）
- 絵文字・記号の使用頻度
- タイトルのつけ方

→ **オーナーから過去記事URLを3〜5本いただく** ことで自動分析可能。

### 3.2 サムネ品質改善

オーナーフィードバック：**「ここの画像はいまひとつ」**

原因仮説：
- A. 画像生成エンジンの選定が悪い（DALL-E 3よりMidjourney v6の方が写真品質は高い）
- B. プロンプトの方向性が記事内容と合っていない
- C. 過去サムネのスタイル（おそらくよりカジュアル／親しみやすい）と乖離

**対応**:
1. オーナーから「過去のお気に入りサムネ画像3〜5枚」または「過去記事URL」を提供してもらう
2. 画像をスタイル分析（色味・構図・テキスト有無・タッチ）
3. プロンプトテンプレートを書き換え
4. 画像生成エンジンを選定（候補比較表は §6 を参照）

### 3.3 認証情報管理（GitHub Secrets）

```
NOTE_EMAIL              # noteログイン用メール
NOTE_PASSWORD           # noteログイン用パスワード
NOTE_SESSION_COOKIE     # 事前取得したセッションcookie（推奨）
ANTHROPIC_API_KEY       # 記事生成用
IMAGE_API_KEY           # 画像生成API用（OpenAI / Stability AI 等）
```

**重要**: パスワード直接ログインは2FA・CAPTCHAでブロックされる可能性大。  
推奨は「ローカルで1回手動ログイン → セッションcookieをエクスポート → Secretsに保存」方式。  
cookieは1〜2週間で失効するので、月次でリフレッシュ運用を組む。

### 3.4 失敗時のリカバリ

- note側UI変更でセレクタが壊れる → スクリーンショット保存＋GitHub Issues自動起票
- 画像生成API失敗 → 過去サムネ流用（assets/thumbnail_reference/からランダム選択）
- 記事生成失敗 → スキップ（次の日に再試行）

## 4. 段階導入ロードマップ

一気にやらず、4フェーズで段階的に。

| Phase | 内容 | 必要なオーナー入力 | 完了基準 |
|-------|------|------------------|---------|
| **P1** | 文体ガイド + 記事生成スクリプト（手動実行）| 過去記事URL 3〜5本 | `python generate_article.py` で本文が出る |
| **P2** | 画像生成スクリプト（手動実行）| 過去サムネ画像 3〜5枚 | `python generate_thumbnail.py` で画像が出る |
| **P3** | note自動投稿スクリプト（手動実行）| noteログイン情報 or セッションcookie | `python post_to_note.py` で公開できる |
| **P4** | GitHub Actions化（毎朝cron）| GitHub Secrets設定 | 1週間連続で自動投稿成功 |

各フェーズの完了をオーナー確認してから次へ進む。

## 6. 画像生成エンジン候補比較

| エンジン | 写実度 | API成熟度 | 月額目安 | コメント |
|---------|-------|----------|---------|---------|
| DALL-E 3 (OpenAI) | ★★★★ | 高 | ¥0.04/枚 | API簡単。日本のローカル景観の精度はやや弱い |
| Midjourney | ★★★★★ | 中（Discord経由 or 非公式API） | ¥1.5K/月+ | 写真品質は最強だが自動化が面倒 |
| Stable Diffusion XL (Stability AI) | ★★★ | 高 | ¥0.02/枚 | 安いがプロンプト調整が必要 |
| Imagen 3 (Google) | ★★★★ | 中（Vertex AI経由） | ¥0.03/枚 | 日本の風景の理解度は高めの印象 |
| Flux.1 (Black Forest Labs) | ★★★★ | 高 | ¥0.05/枚 | 新興。プロンプト追従性が高い |

**推奨**: P2 はまず DALL-E 3 で立ち上げ、品質に不満ならFlux/Imagenに切替。

## 7. 法務・規約リスク（CFO確認推奨）

- note.com 利用規約：自動化ツールでの投稿は明示禁止ではないが、グレー。アカウントBANリスクあり
- 画像生成AIの商用利用ライセンス：エンジンごとに異なる
- 著作権：実在の場所・人物が映る場合は要注意

**対応**: 投稿頻度を「毎日1本」に抑え、過剰な自動化感を出さない。CFOに規約レビュー依頼を別途起票する。
