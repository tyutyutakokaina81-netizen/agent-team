# フリーペーパー 号テンプレート＆品質チェックリスト（毎号同じ型で出す）

> CPO（プロダクト）作成。狙い＝**フリーペーパーを「毎号同じ型で安定して出せる」状態にする**。
> 誰が書いても、どの号でも、創刊号（research/15）と同じ品質・同じ構成・同じ規則順守で量産できる「型」を1ファイルにまとめる。
> 前提：**無料・ゼロ予算**（デジタルのみ／無料ツールのみ／印刷は広告が付くまでしない）。本命＝海外読者への発信（North Star）。
> 基準形＝`research/15_フリーペーパー創刊号_食特集.md`（13品＋日英・禁止語0件・[要確認]21）。媒体方針＝`research/13`。
> PIIは市レベルまで（地区名・町名・住所・実在店名は書かない）。誇張・テンプレ・適当はNG（A5）。

このファイルは4部構成：
1. **誌面構成テンプレ**（1号まるごとの型：表紙→ごあいさつ→特集→各品/各所の固定フォーマット→送客）
2. **執筆テンプレ**（1記事＝1ブロックの空欄付きフォーム＝誰でも同じ品質で書ける）
3. **無料配布の仕組み**（Web/PDF/QRのみ・ゼロ予算）
4. **品質チェックリスト**（A1〜A7／禁止語ゼロ／PII市レベル／[要確認]の扱い）

---

## 1. 誌面構成テンプレ（1号の型）

毎号、下の順番・下の要素で組む。号テーマだけ差し替えれば、構成は固定（＝安定供給の核）。

### 1-0. 号の基本スペック（毎号共通）

| 項目 | 既定値 | 補足 |
|---|---|---|
| 言語 | 日本語＋English 併記 | 各ブロックに日本語読み物→English を必ず両方 |
| 視点 | 住人の一人称（高岡・氷見・富山に住む側） | 観光ガイド調NG（A・住人視点MUST） |
| 1号の本編数 | **8〜13ブロック**（創刊号は13品） | 息切れ回避。最低8、上限13を目安に |
| 体裁 | A4・8〜12ページ相当・デジタルファースト | PDF＋Web。印刷は広告が付いてから |
| 刊行 | 季刊（年4回）で創刊 → 反応次第で隔月 | 無理に月刊にしない |
| 送客 | 各ブロック末尾から note 本編・英語ガイドへ QR | 深掘りは誌面に詰め込まず note へ |
| 写真 | オーナー手配（A3・写真風で統一） | code は本文に `[写真]` placeholder を置くだけ |

### 1-1. 号の並び順（固定テンプレ）

```
① 表紙           号テーマ＋写真＋媒体名＋号数
② 創刊/巻頭のごあいさつ   日本語の短い前書き → English（A note from the editors）
③ 特集ブロック群       本編8〜13個。各ブロックは下記「固定フォーマット」で統一
④ （任意）実用まとめ面    旬カレンダー等。号テーマに応じて入れる/省く
⑤ 広告枠            広告が付いたら挿入（無いうちは省略＝ゼロ予算）
⑥ 奥付＋送客          媒体方針・[要確認]の注記・QR・次号予告
```

### 1-2. 各ブロックの固定フォーマット（最重要・毎ブロック同じ型）

特集の1品・1所・1テーマは、**必ず**次の4パートをこの順で持つ（research/15 と完全一致）。

```
## N. <見出し> — <一言サブ（その回の固有の角度）>

<日本語の読み物：住人視点・誌面向けに圧縮。3〜5段落。>

### English

<English：毎回違う書き出し（A6・同パターン2連続NG）。3〜4段落。>

### 実用情報 ｜ Good to know
- **旬 / Season**：…
- **特徴 / What**：…
- **食べ方（使い方/見方） / How**：…（日本語）。<English one line>.
- **どこで / Where**：…市レベルまで（Fish shops … across Toyama 等）。
- **持ち帰り（注意/補足）/ Take home**：…未確証は[要確認 / to verify]。
```

ポイント（毎号守る）：
- **見出しは「名称 — その回固有の角度」**。号内で角度が重複しないようにする（A5「同じ結論の連発」回避）。
- 日本語パートは**観察→事実→住人の感情**の流れ。「Xには、Yがある」連発・「世界唯一」断定はしない（A5）。
- English は**情報の英訳ではなく、英語として読める文章**。書き出しの型を毎回変える（A6）。
- 実用情報は**5行固定**（旬／特徴／食べ方／どこで／持ち帰り）。テーマが食でない号は項目名を読み替える（下表）。

### 1-3. テーマ別の実用情報ラベル読み替え（食以外の号でも型を崩さない）

| 号テーマ | 旬→ | 特徴→ | 食べ方→ | どこで→ | 持ち帰り→ |
|---|---|---|---|---|---|
| 食（基準） | 旬 / Season | 特徴 / What | 食べ方 / How | どこで / Where | 持ち帰り / Take home |
| 場所・文化 | 季節 / Best season | 何 / What | 見どころ / How to enjoy | 場所（市） / Where | アクセスの目安 / Getting there |
| 暮らし・行事 | 時期 / When | 何 / What | 楽しみ方 / How | どこで（市） / Where | 持ち帰り・補足 / Note |

→ **5行・日英・[要確認]運用は不変**。ラベルだけ替える＝どの号でも同じ「型」で出せる。

## 2. 執筆テンプレ（1記事＝1ブロック・空欄フォーム）

下のフォームをコピーし、空欄`<…>`を埋めるだけで、創刊号と同品質の1ブロックが完成する。
**判断に迷う数値・店名・通販可否・統計は空欄に埋めず `[要確認 / to verify]` と書く**（後述4-4）。

```markdown
## N. <食材/場所/テーマ名> — <その回だけの角度（10〜20字。号内で重複させない）>

<段落1：住人の日常の入り口。「観光名物ではなく、ふだんの暮らしにこうある」という一次観察から始める。1〜3文。>

<段落2：事実・背景。サイズ/深さ/旬/歴史/理由など。検証可能な事実のみ。不確実な数値は本文に断定で書かず、実用情報側で[要確認]にする。>

<段落3：食べ方・使い方・見方。具体的に。家庭でどうしているか。>

<段落4（任意）：住人の感情・気づきで締める。教訓の押し付け・誇張・「世界唯一」NG。>

### English

<Para 1：毎回違う書き出しで。前ブロックと同じ型で始めない（A6）。>

<Para 2：fact / background（英語として自然に）。>

<Para 3：how to eat / use / see it。>

<Para 4（任意）：a quiet closing line。"If you make things..." 型の反復で締めない。>

### 実用情報 ｜ Good to know
- **旬 / Season**：<和文>（<EN>）。確証なければ[要確認 / to verify]。
- **特徴 / What**：<和文一行>。
- **食べ方 / How**：<和文>。<EN one line>.
- **どこで / Where**：<市レベルまで>。<EN, e.g. ... across Toyama>.
- **持ち帰り / Take home**：<和文>。通販可否・店名は確証なければ[要確認 / to verify]。
```

### 2-1. 書き手向けミニ指示（このフォームの使い方）

- **1ブロック＝完結**。他ブロックを読まなくても成立させる。
- **段落数の目安**：日本語3〜5段落／English 3〜4段落／実用5行。長すぎたら削る（深掘りは note へQR）。
- **角度の重複チェック**：書き終えたら号内の他ブロックと見出しサブを並べ、結論・角度がかぶっていないか確認（A5）。
- **English は和文の直訳にしない**。書き出しの型（情景/会話/行動/問い…）を毎回変える（A6）。
- **PII**：地区名・町名・住所・実在店名は書かない。産地は「富山市・射水市」など市レベルまで（A4）。
- **誇張語の禁止**（4-2の禁止語リスト参照）。数値の断定が必要なら[要確認]へ逃がす。

## 3. 無料配布の仕組み（Web／PDF／QR のみ・ゼロ予算）

【前提】お金はかけない（ゼロ予算・確定方針）。**印刷は広告が付くまでしない**。配布は全てデジタル。

### 3-1. 出力形態（無料ツールのみ）

| 形態 | 中身 | 作り方（ゼロ予算） | 配布 |
|---|---|---|---|
| Web版 | 号の全文（日英・各ブロック） | Markdown → 既存の Pages（`tavern.html`/GitHub Pages）系で1ページ公開 | URL を共有 |
| PDF版 | 同じ内容を1ファイルに | 無料ツールで Markdown/HTML → PDF 書き出し（オーナー/cowork端末） | DLリンク／QR |
| QRカード | Web版URLへ飛ぶQR | 無料QR生成（URL→QR画像）。**印刷はせず画面/既存物に表示** | 下記「設置」 |

> code（私）は A1 で外部公開・QR設置・PDF配布まで到達不可。**Web/PDF/QRの生成データまでを内製し、公開操作はオーナー/coworkが実行**。

### 3-2. 設置（PIIを避け、施設タイプ・市レベルで記載）

- Web版URL＋QRを、**宿泊施設のフロント・カフェ・観光案内所・駅・土産店**などに「QRだけ」置く（印刷最小・紙のQRカードのみ）。
- インバウンド経路：宿泊施設、観光案内、SNS（X / Reddit / Instagram）、note 送客。
- SNSで号の公開を告知し、Web版へ誘導。各ブロック末尾の note QR で深掘りへ送る。

### 3-3. 紙版（広告が付いてからのみ・原則オフ）

- 広告収益が付くまで**紙は刷らない**（ゼロ予算厳守）。
- 広告が付いたら、印刷原価を広告で賄える範囲で小部数のみ。印刷・設置はオーナー/cowork。

### 3-4. ゼロ予算チェック（配布前に確認）

- [ ] 有料サービス・有料素材・有料フォント・有料ツールを使っていない
- [ ] 写真はオーナー手配（A3・写真風）。本文には `[写真]` placeholder のみ
- [ ] 印刷を前提にしていない（デジタル＝Web/PDF/QRで完結）
- [ ] 公開・QR設置・SNS告知は「オーナー/cowork実行」の手順として明記してある

## 4. 品質チェックリスト（配布前に全項目クリア必須）

号を出す前に、**全ブロック分**で下を通す。1つでもNGがあれば配布しない。

### 4-1. 絶対制約 A1〜A7

- [ ] **A1**：外部ネット前提の記述がない（code が取得/投稿した体で書いていない）。公開はオーナー/cowork手順として記載
- [ ] **A2/A3**：画像は生成していない。サムネ/写真はオーナー手配（写真風）。本文は `[写真]` placeholder のみ
- [ ] **A4（PII）**：地区名・町名・住所・実在店名・個人名がゼロ。産地・場所は**市レベルまで**
- [ ] **A5（テンプレ/誇張）**：「Xには、Yがある」連発なし／号内で結論・角度の重複なし／誇張断定なし
- [ ] **A6（英語反復）**：English の書き出しが直前ブロックと同型でない／"If you make things..." 型の反復なし
- [ ] **A7**：ファイルを commit する前提で作業（保存先・台帳更新は別途。※本テンプレ運用時の注記）

### 4-2. 禁止語ゼロ（誇張・断定）

本文・English・実用情報のどこにも下記を**使わない**（創刊号は禁止語0件が基準）：

- 「世界唯一」「世界一」「日本一」「日本初」「最高級」「最高」「絶対」「必ず」「唯一無二」
- 英語："the best", "world-famous", "only ... in the world", "must", "never fail", "guaranteed"
- 比較最上級の断定（数値・出典なしの「一番」「No.1」）

→ 強調したい場合は **事実＋住人の主観**に置き換える（例：「日本一おいしい」→「うちでは年に数回の特別な魚」）。
→ 統計的に上位という話は**断定せず**「全国でトップクラスにあがる年があると言われる[要確認]」の形に（research/15 の昆布の書き方）。

### 4-3. 構成・型の遵守

- [ ] 各ブロックに **日本語読み物 → English → 実用情報** が揃っている（欠けなし）
- [ ] 実用情報が **5行**（旬／特徴／食べ方／どこで／持ち帰り、テーマ別ラベル可）
- [ ] 各ブロックが **住人の一人称**で、観光ガイド調になっていない
- [ ] 見出しが「名称 — 固有の角度」で、号内で角度が重複していない
- [ ] 巻頭ごあいさつに日本語＋English の両方がある
- [ ] 奥付に媒体方針・[要確認]注記・送客QR・（任意）次号予告がある

### 4-4. [要確認]の扱い（誤情報で信用を壊さない）

- [ ] 確証のない **数値・旬・通販可否・店名・統計** は本文で断定せず `[要確認 / to verify]` を付けた
- [ ] [要確認] は**実用情報側に寄せる**（本文は断定を避け、事実のみで成立させる）
- [ ] 号末に「**各[要確認]はオーナーが事実検証してから配布する**」旨を明記
- [ ] 配布版（Web/PDF）公開時には、検証済みに更新するか、未検証なら[要確認]表記を残す（嘘で出さない）

> 無料でも誤情報は信用を壊す。**[要確認]＝配布前ゲート**。検証はオーナー（外部web）が担当（A1で code は検証不可）。

### 4-5. 最終ゲート（配布OK判定）

- [ ] 4-1〜4-4 すべてクリア
- [ ] 3-4 ゼロ予算チェッククリア
- [ ] 公開手順（Web/PDF/QR設置）がオーナー/cowork向けに書いてある
→ 全てチェックで **配布可**。1つでも未達なら差し戻し。

## 付録：1号を出すまでの最短手順（運用フロー）

1. **号テーマ決定**（CEO/CMO）。食／場所文化／暮らし行事 から1つ。
2. **ブロック数を決める**（8〜13）。既存 note・マガジン在庫から流用元を割り当て（書き下ろし最小）。
3. **各ブロックを §2 の執筆フォームで作成**（誰が書いても同じ型）。
4. **§1 の並び順で1号に組む**（表紙→ごあいさつ→特集→送客）。
5. **§4 の品質チェックリストを全ブロックで通す**。[要確認]を一覧化。
6. **§3 で Web/PDF/QR データ化**（code が生成→公開はオーナー/cowork）。
7. オーナーが[要確認]を検証 → 配布。

> このファイル単体で、**号テーマだけ替えれば毎号同じ品質で量産できる**。基準サンプルは research/15。
```
