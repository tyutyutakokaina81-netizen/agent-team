# 新収益手段ブレスト — CMO×CSO 計6案

作成: 2026-06-23 ／ 担当: CMO（3案）＋ CSO（3案）

## 前提（この6案が守るルール）

- **ゼロ予算**：無料ツール・無料枠のみ。広告費・印刷費・有料SaaSは使わない。
- **AIが制作・並べる**：企画・本文・素材生成・台帳整理は code（私）が全部やる。
- **外部操作はオーナー or cowork**：投稿・出品・送付・アカウント認証は人間/cowork側。
- **「借りた集客」重視**：自前でフォロワーをゼロから作る案は避け、既存プラットフォームの
  検索・発見・配信エンジンに相乗りする。
- **North Star と非競合**：高岡・氷見・富山の海外発信（無料）を妨げない。むしろ流用できる資産を活かす。
- **除外済み（重複しない）**：Kindle出版／Gumroadデジタル商品／B2B英語発信代行／クラウドソーシング受託／Notionテンプレ／プロンプト集。
- **A4/A5順守**：PII・誇張・成果保証なし。規約違反・スパム手法は最初から除外。

> ⚠️ スコアは「売れる確度／速さ／上限／手間」を 高中低で**正直に**付ける。
> 「速さ＝着金までの速さ」「上限＝月あたり到達しうる規模感」「手間＝オーナー/cowork側の継続負荷」。
> どれも成果を保証するものではない（A5）。

# CMO 案（コンテンツ・集客起点）

## CMO-1. ニュースレター（無料）→ 後付けスポンサー枠

**①概要**
既存の高岡・氷見・富山 英語記事を、無料ニュースレター（Beehiiv/Substack 無料枠）に
「週1ダイジェスト」として束ねて配信。本文は既存資産の再編集なので新規執筆コスト最小。
購読者が一定数たまった段階で、誌面下部に**地域事業者の1枠スポンサー**（B2B英語発信代行の入口と合流）を後付けする。読者課金はしない＝無料で海外読者最大化（North Star一致）。

**②なぜ集客が借りられるか**
Substack/Beehiiv は「おすすめ（recommendation）」「Notes/network」機能で**他ニュースレターから購読者が流れてくる**配信網を持つ。検索やSNS共有も乗る。ゼロから自前リスト構築しなくても、プラットフォームの発見導線に相乗りできる。

**③ゼロ予算で可能か**
可能。両サービスとも無料枠で配信・購読者管理・recommendation が使える。決済（後のスポンサー請求）はオーナーの既存手段でよい。

**④オーナー/coworkの最小操作**
- 初回：アカウント作成1回（オーナー）。
- 週次：code が組んだダイジェスト原稿を cowork が貼り付け→送信（コピペ1回/週）。
- スポンサー枠は「来たら拾う」（向こうから問い合わせ or 営業エンジン経由）。

**⑤実現性スコア**
- 売れる確度：**低**（スポンサー収益化は購読者規模に依存し時間がかかる）
- 速さ：**低**（着金までが遠い。まず読者を貯める段階）
- 上限：**中**（軌道に乗れば月数万〜の枠が複数）
- 手間：**低**（既存記事の再編集＋週1コピペ）

**⑥最初の一歩**
code が既存EN記事から「創刊号＋第2号」のダイジェスト2本を組む → cowork が Beehiiv 無料アカウントで配信テスト。収益化はせず、まず North Star の配信チャネルとして立ち上げる。

## CMO-2. Pinterest → 既存note/ブログへの送客（アフィリエイト/将来スポンサーの土台）

**①概要**
高岡・氷見・富山の「食・風景・旅」を、Pinterest の縦長ピン（複数枚/記事）として量産し、
リンク先を既存の note 記事・英語記事に貼る。Pinterest は**検索エンジン的**で、ピンが
長期間（半年〜）流入を生む“ストック型”。集めたトラフィックを、将来の有料マガジン/スポンサー/物販リンクのどれにでも繋げられる「送客パイプ」を先に作る。

**②なぜ集客が借りられるか**
Pinterest は Google 同様の検索＆レコメンドで、フォロワー0でも「保存・発見」経由で
**海外ユーザー（特に旅行・料理クラスタ）からの流入**が見込める。日本の食・風景は海外保存率が高いジャンル。SNSのような毎日投稿の鮮度競争ではなくストック資産になる。

**③ゼロ予算で可能か**
可能。Pinterest は無料。ピン画像は A2/A3 制約上 code は生成できないが、cowork の無料画像生成 or オーナーの実写を使えばよい（テキスト主体の縦長ピンなら無料Canva枠でも可）。

**④オーナー/coworkの最小操作**
- code がピンの「タイトル＋説明文（英語SEO）＋リンク＋画像指示」をCSVで用意。
- cowork が画像を当てて一括予約投稿（無料スケジューラ or 手動）。週まとめてでよい。

**⑤実現性スコア**
- 売れる確度：**低〜中**（直接の収益ではなく送客。収益化は接続先次第）
- 速さ：**低**（検索インデックスに乗るまで数週〜数ヶ月）
- 上限：**中**（旅行ジャンルは保存が伸びれば月数万PV規模も）
- 手間：**低**（ストック型。一度作れば放置で流入）

**⑥最初の一歩**
code が反応の良い既存記事 上位5本ぶんの英語ピン（各3枚＝15ピン）の文面CSVを作る → cowork が画像を当てて投稿し、2週間後に Pinterest アナリティクスで流入を確認。

## CMO-3. faceless ショート動画（YouTube Shorts / TikTok）AI台本 → 概要欄送客

**①概要**
高岡・氷見・富山の「30秒・顔出しなし」ショートを、既存記事を素材に量産。
構成：実写写真/オーナー素材のスライド＋字幕＋AIナレーション台本（code が台本・字幕・英訳を作る）。
概要欄/プロフィールに「記事を読む」「（将来）マガジン」リンク。**動画プラットフォームの
発見アルゴリズムに乗せて海外視聴者を集める**チャネル。収益は (a) 送客、(b) 一定再生で
プラットフォーム配分（来たら拾う）。

**②なぜ集客が借りられるか**
Shorts/TikTok は**フォロワー0でもおすすめ欄で拡散する**数少ないチャネル。
「日本・地方・食・海」は海外で強い視聴ジャンル。アルゴリズム配信＝借りた集客の典型。

**③ゼロ予算で可能か**
可能（台本・字幕・翻訳は code、編集は無料アプリ/CapCut無料枠）。
ただし A2 で動画/画像生成は code 不可 → 編集と書き出しは cowork/オーナー操作が必須。

**④オーナー/coworkの最小操作**
- code が「台本＋字幕テキスト＋使用写真の指示＋英語/日本語キャプション」を1本ずつ納品。
- cowork/オーナーが無料アプリでスライド化→書き出し→投稿（1本あたり数分の編集）。

**⑤実現性スコア**
- 売れる確度：**低**（直接収益化は不確実・配分は規模依存）
- 速さ：**中**（バズれば早い／読めない）
- 上限：**中〜高**（当たれば到達は大きいが不安定）
- 手間：**中**（動画書き出し・投稿の人手が毎本かかる）

**⑥最初の一歩**
code が既存の強い記事（雨晴・白えび等）から30秒台本3本＋字幕を作る → cowork が1本だけ
試作・投稿し、48時間の表示回数で「このチャネルに投資する価値があるか」を判断（撤退ライン込み）。

# CSO 案（営業・取引起点）

## CSO-1. Fiverr / Upwork の「productized gig」＝固定パッケージ販売

**①概要**
受託（個別見積）ではなく、**値段・納品物・納期を固定したパッケージ**を Fiverr に「商品」として並べる。例：「日本の地方の食・旅コンテンツ 英語ライティング（3記事パック）」「日本語→英語の自然な翻訳・ローカライズ（◯語まで）」など、当社が既に量産できる作業を**棚売り**にする。受託（クラウドソーシング受託）が“応募して取りに行く”のに対し、これは“並べて向こうから注文が来る”ので運用が別物。

**②なぜ集客が借りられるか**
Fiverr/Upwork は**買い手が検索して来るマーケットプレイス**。出品＝そのプラットフォームの
検索・カテゴリ・レコメンドに載る。自前で客を探さなくても、ギグが露出する。

**③ゼロ予算で可能か**
可能。出品無料、手数料は売れた時のみ。サンプル成果物は既存資産を流用、紹介文・FAQ・
パッケージ定義は code が作る。

**④オーナー/coworkの最小操作**
- 出品作成1回（オーナー、gig文面は code が用意）。
- 注文が来たら：code が成果物制作 → cowork/オーナーが納品ボタン操作のみ。

**⑤実現性スコア**
- 売れる確度：**中**（明確な需要があるカテゴリ。ただし初期はレビュー0で埋もれやすい）
- 速さ：**中**（出品即着金ではないが、受注すれば数日で着金）
- 上限：**中**（単価×件数。レビューが付くと伸びる）
- 手間：**低〜中**（制作は code。出品管理・納品操作は軽い）

**⑥最初の一歩**
code が「最も尖った1ギグ」（例：日本の地方食コンテンツの英訳/執筆パック）の
タイトル・説明・3段階パッケージ・FAQ・サンプルを完成稿で用意 → オーナーが Fiverr に出品。

## CSO-2. 自社デジタル商品を「他者に売ってもらう」＝アフィリエイト/リセラー提供（紹介の逆）

**①概要**
当社の既存デジタル商品（Gumroad商品・将来のマガジン等）に**アフィリエイト報酬を設定**し、
旅行ブロガー・日本好き発信者・ニュースレター運営者に「あなたが紹介して売れたら◯%」と
**売ってもらう側を増やす**。自分で集客するのではなく、他人の集客を成果報酬で借りる。

**②なぜ集客が借りられるか**
これは「集客の外注」そのもの。Gumroad は**アフィリエイト機能が標準装備（無料）**で、
リンク発行・成果計測・報酬分配を自動でやる。紹介者は自分のオーディエンスに当社商品を
流してくれる＝**他人のフォロワーを借りる**。成果が出た時だけ報酬＝ゼロリスク。

**③ゼロ予算で可能か**
可能。Gumroad アフィリエイトは無料。報酬は売上から自動配分なので持ち出しゼロ。
紹介者向けの「アフィリエイト募集ページ＋紹介用素材（画像・文面）」は code が作る。

**④オーナー/coworkの最小操作**
- Gumroad 管理画面でアフィリエイト％を設定（1回）。
- 候補の発信者に「紹介しませんか」打診（文面は code・送付はオーナー/cowork、数通）。

**⑤実現性スコア**
- 売れる確度：**中**（商品力と紹介者の質に依存。ハマれば自走）
- 速さ：**低〜中**（紹介者を見つけ口説く時間が要る）
- 上限：**中〜高**（良い紹介者が複数付けば自前集客の上限を超える）
- 手間：**低**（仕組みは自動。あとは紹介者開拓だけ）

**⑥最初の一歩**
code が既存 Gumroad 商品にアフィリエイト報酬率の推奨と「紹介者募集1枚（素材リンク付き）」
を用意 → オーナーが Gumroad で％設定 → 相性の良い発信者3名へ打診文を送る。

## CSO-3. 「Done-for-you データ/リスト・ミニ調査」パッケージ（productized research）

**①概要**
当社が AI で速く作れる**情報整形物**を固定商品にする。例：「日本の地方◯◯（食・観光・移住）に
関する英語まとめ調査レポート」「特定テーマの英語FAQ/比較表」など、ライティングとは別軸の
“調べて整える”成果物。販路は (a) Gumroad のテンプレ/レポート枠（※Notion/プロンプト集とは別物の
調査・リスト系）、(b) Fiverr の research gig。受託の個別対応と違い、汎用テーマを**作り置き販売**できる。

**②なぜ集客が借りられるか**
Gumroad/Fiverr の検索・カテゴリに「research」「report」「list」需要がある。
作り置きをマーケットに置けば、プラットフォームの発見導線で買い手が来る。
さらに既存の note/Pinterest 流入を商品リンクに繋げれば送客も乗る。

**③ゼロ予算で可能か**
可能。調査・整形・PDF化は code（無料）。出品無料、手数料は売れた時のみ。
ただし「外部ネットでの一次調査」が必要な部分は A1 で code 不可 → cowork の検証 or
公開済みの自社既存リサーチ資産を再編集する形に限定する（誇張・出典不明の数字は出さない＝A5）。

**④オーナー/coworkの最小操作**
- 出品1回（文面は code）。
- 一次情報の事実確認が要る商品は cowork に検証発注（既存の ops/ キューで運用）。

**⑤実現性スコア**
- 売れる確度：**低〜中**（テーマ選定が当たれば売れるが、汎用調査は競合多）
- 速さ：**中**（出品後、需要があれば数日で着金）
- 上限：**低〜中**（単価が上げにくい・差別化が要る）
- 手間：**中**（事実検証を cowork に頼る分、回転に人手が挟まる）

**⑥最初の一歩**
code が「自社の強み（高岡・氷見・富山＝日本の地方リアル）に直結する1テーマ」の
レポート見本（目次＋サンプル章）を作り、需要を測るために Gumroad に1点だけ薄く出品して反応を見る。
