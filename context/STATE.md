## 「実行して」プロトコル（最重要）

オーナーが **「実行して」**（または「実行」「今日の分」）とだけ言ったら、以下を**承認不要で自動実行**する：

1. このファイル（context/STATE.md）と `CMO/_index.md` を読む
2. **デフォルト動作 = 今日の note 記事を1本作成**（CMO役職）
   - テーマ未指定なら CMO が企画から決める（context/ にネタがあれば優先）
   - 本文はそのまま貼れる完成稿。海外読者にも届くよう英語要約を併記
   - サムネ：実写は生成不可 → 「実写生成プロンプト」＋「デザインPNG」を作る
3. 保存先：`CMO/outputs/YYYY-MM-DD_note記事_*.md`（＋サムネ素材）
4. **必ず** `CMO/_index.md` の成果物ログと **この STATE.md の「最新状態」「次にやること」を更新**
5. **無確認で自動 commit & push**（オーナー承認済み・下記ポリシー参照）
6. 完了報告（要点→詳細→次アクション）

## 環境の絶対制約（毎回思い出すこと・loopしない）

- Web/リモートの**使い捨てコンテナ**。セッションごとに新規clone、終了後破棄。
- **永続するのはリポジトリ内のコミット済みファイルだけ**。私に他の記憶領域は無い。
  - 前のセッションで作った/承認した成果物も、commit&pushされていなければ消えている。**復元不可**。
- **外部ネットワークは全面ブロック**（2026-05-28 実測：note.com・Google等すべて HTTP 403）。
  - → note への**接続・閲覧・投稿・過去記事/過去サムネの確認は不可**。「ログインしてるよ」は
    オーナーの端末側の話で、私の隔離コンテナとはセッション非共有。**何度言われても私からは到達不能**。
  - → フリー素材写真の**取得も不可**（同じく403）。
- **実写画像の生成も不可**（画像生成AIが無い）。
- **サムネ方針（オーナー承認済み）**：オーナーは「実写」を要望し、**Pillow製のデザインモックはNG（却下済み）**。
  - 実写サムネを得る唯一の道：(1) オーナーが画像生成AIに「実写生成プロンプト」を貼って作る、
    または (2) **オーナーが見本/実物サムネ画像をチャットにアップロード**→私が保存し型を踏襲。
  - 参照したい過去サムネがある場合は、**オーナーにアップロードを依頼する**（私からは取りに行けない）。
