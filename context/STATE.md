## ★連絡体制・相互応援型（オーナー指示 2026-06-07・最重要）

**今回の最大の課題＝2エージェントが別ブランチで並行作業し大量重複・齟齬が発生**
（役割分担doc 2種／ops実装 2種／cowork が18記事を別ブランチで執筆＝code は未把握だった）。
原因は「差分を溜めて後でまとめようとした」こと。再発防止の連絡体制：

1. **canonical = `main`**。両者とも**作業前に必ず `git pull`→作業後すぐ `commit & push`**（差分を溜めない）。
2. **着手前に宣言**：`ops/inbox` に「これからやること」を1行投函してから着手（二重作業の事前回避）。
3. **相互応援型**：どちらも相手に依頼可。役割の重心は固定＝
   - **code**：git・整理・台帳・執筆・判断・dedup（決定的でトークン効率が良い）
   - **cowork**：ブラウザ/ネット実行・大量バッチ・並列（code に不可能 or トークン高コスト）
4. **トークン相互協力**：大量生成・並列・スクレイピングは **cowork に寄せて code のトークンを節約**。どちらかが詰まれば相手が引き取る。
5. **齟齬チェック定例化**：統合時は `git diff main...<branch>` を必ず確認してからマージ。
6. **共有領域は単独編集しない（2026-06-07・オーナー指示）**：`CMO/outputs/` の記事・公開スケジュールの変更は、まず `ops/inbox` で予告→cowork が pull→反映。code も単独で記事を動かさず共有手順に乗せる。
