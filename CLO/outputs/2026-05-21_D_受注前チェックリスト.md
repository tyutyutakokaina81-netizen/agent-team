# 柱D 受注前リーガルチェックリスト

作成日：2026-05-21  
作成：CLO  
用途：柱D（エクセル入力・Webスクレイピング自動受注パイプライン）の `02_evaluate.py`
      評価フェーズに組み込む法務観点チェックリスト

> **【注記】** スクレイピング・データ入力業務の受注可否を一般的な法務観点から
> 判定するチェックリストです。判断に迷う場合は弁護士に確認してください。

---

## 1. 受注可否の判定フロー

```
案件情報を受信
   ↓
①プラットフォーム規約チェック（既存 risk_and_feasibility.md 準拠）
   ↓ OK
②対象サイト規約・robots.txt チェック
   ↓ OK
③個人情報の有無
   ↓ OK / または別契約で対応可
④不正アクセス・ログイン要否
   ↓ NG/Yes → 辞退
   ↓ OK
⑤発注目的の確認（違法目的でないか）
   ↓ OK
⑥データベース著作権の侵害可能性
   ↓ OK
⑦アクセス頻度・規模の妥当性
   ↓ OK
   ↓
受注可（業務委託契約 or クラウドソーシング取引）
```

---

## 2. 各チェック項目の詳細

### ①プラットフォーム規約（クラウドワークス・ランサーズ）

- [ ] 自動応募ではないか（応募文生成は OK、送信は人手）
- [ ] AI使用がプロフィール・応募文で明示されているか
- [ ] スクレイピング案件はクライアントの依頼として受注（プラットフォーム自体への
      スクレイピングではない）

### ②対象サイト規約・robots.txt

- [ ] 対象サイトの利用規約を確認したか
   → 「スクレイピング禁止」「自動収集禁止」の記載があれば辞退
- [ ] robots.txt を確認したか
   → 取得対象のディレクトリで Disallow されていないか
- [ ] 規約・robots.txt が不明確な場合：対象サイト運営者に問い合わせるか辞退

### ③個人情報の有無

| 取得項目 | 個人情報該当性 |
|---------|--------------|
| 店舗名のみ | 該当しない（事業情報） |
| 法人名のみ | 該当しない |
| 個人事業主の屋号 | 微妙（本名と紐付くと該当） |
| 個人名 | 該当 |
| メールアドレス（個人用） | 該当 |
| 電話番号（個人用） | 該当 |
| 公開されている代表者名 | 該当（個人情報保護法上） |

- [ ] 取得対象に個人情報が含まれるか
   → 含まれる場合：第三者提供制限・利用目的の特定が必要
   → 大量・組織的な収集の場合は要弁護士相談

### ④不正アクセス・ログイン要否

- [ ] ログイン不要のサイト → OK
- [ ] 公開URL からのみ取得 → OK
- [ ] ID/パスワードの突破要求 → **NG（不正アクセス禁止法違反）**
- [ ] パラメータ改竄・URL推測でアクセス制限を回避 → **NG**
- [ ] 会員制サイトのスクレイピング → 会員規約次第。多くは禁止

### ⑤発注目的の確認

クライアントへ受注前にヒアリングする項目：

- [ ] データの利用目的は何か
- [ ] 営業リスト作成の場合、無差別営業に使用しないか
- [ ] 個人情報を含む場合、利用目的が個人情報保護法上適法か
- [ ] 名簿販売・第三者提供の予定はないか
- [ ] 競合誹謗中傷・嫌がらせ目的でないか

**目的が不明確、または違法性の疑いがある場合は辞退**

### ⑥データベース著作権

- [ ] 対象サイトが「編集著作物」と評価される構造か
   - 例：ぐるなび、食べログ、タウンページ、HotPepper Beauty 等の
     体系的な編集データベース → 大量取得は侵害リスク高
   - 例：個別の公表サイト・公式サイトの公開情報 → 侵害リスク低
- [ ] 対象データの量・選別がデータベースの本質的部分を構成するか
- [ ] 大量取得・全件複製の場合は要弁護士相談

### ⑦アクセス頻度・規模の妥当性

- [ ] 1秒間に1リクエスト以下に制限する
- [ ] 大量件数（数万件以上）の場合は分散実行
- [ ] User-Agent を明示し、責任の所在を明らかにする
- [ ] 対象サイトに過度な負荷を与えるおそれがある場合は事前合意取得

## 3. 自動評価スクリプトへの組み込み案

`02_evaluate.py` に以下のフラグ判定を追加：

```python
# 法務リスク判定フラグ
LEGAL_FLAGS = {
    "platform_rule_ok": True,          # プラットフォーム規約OK
    "target_terms_checked": False,     # 対象サイト規約未確認 → 要対応
    "robots_txt_checked": False,       # robots.txt未確認 → 要対応
    "contains_personal_info": False,   # 個人情報含む？
    "requires_login_bypass": False,    # ログイン突破要求？ → True なら NG
    "purpose_clarified": False,        # 発注目的確認済？
    "database_copyright_risk": False,  # データベース著作権リスク
    "access_rate_safe": True,          # アクセス頻度妥当？
}

def is_acceptable(flags):
    if flags["requires_login_bypass"]:
        return False, "不正アクセス禁止法違反のおそれ → 辞退"
    if not flags["target_terms_checked"]:
        return False, "対象サイト規約の確認が必要 → 保留"
    if flags["contains_personal_info"] and not flags["purpose_clarified"]:
        return False, "個人情報の利用目的を確認 → 保留"
    if flags["database_copyright_risk"]:
        return False, "データベース著作権リスク → 要相談"
    return True, "受注可"
```

実装は CDO に依頼する。

## 4. 受注辞退時のテンプレ文面

```
ご依頼ありがとうございます。

恐れ入りますが、本案件は以下の理由により当方では
お引き受けが難しい状況です。

【理由】
- 対象サイトの利用規約上、自動収集が制限されているため
- 個人情報が含まれており、利用目的の特定が困難なため
- その他、法令遵守の観点から確認事項が多いため

別の方法（公式API利用、サイト運営者への直接問い合わせ等）の
ご検討をおすすめいたします。

何卒ご理解のほどよろしくお願いいたします。
```
