# SNS運用代行 ステマ規制チェックリスト

作成日：2026-05-21  
作成：CLO  
用途：柱B（SNS運用代行）の運用フローに組み込むチェックリスト  
根拠：景品表示法 第5条第3号、内閣府告示第19号（2023年10月1日施行）

> **【注記】** ステマ規制に関する一般的な実務観点のチェックリストです。
> 個別ケースの判断は弁護士に確認することを推奨します。

---

## 1. ステマ規制の基本

### 規制対象（景表法のステマ告示）

> 事業者が自己の供給する商品・サービスについて行う表示であって、
> 一般消費者が事業者の表示であることを判別することが困難であるもの

平たく言うと：
- **事業者が背後にいる広告なのに**
- **第三者の自発的な感想のように見せている**
- **= ステマ違反**

### 違反すると
- 景品表示法違反として **措置命令・課徴金** の対象（広告主＝クライアント側）
- エージェンシー（弊社）も措置命令の対象となり得る
- レピュテーション毀損

## 2. ケース別判定

### ケース A：クライアント自身のSNSアカウントを運用代行する（**柱Bの主たる業務**）

| ケース | ステマ規制 | 対応 |
|--------|----------|------|
| クライアント自身のアカウントで自社サービスを宣伝 | **対象外**（広告主＝アカウント運営者で一致） | 表示不要 |
| クライアント自身のアカウントで「お客様の声」を装って投稿 | **違反リスク有** | お客様の声は実在の顧客の同意取得・「お客様の声」と明示 |
| 自社サービス紹介で「絶対に効く／確実に売上倍増」等 | **景表法（優良誤認）違反リスク** | 効果効能の断定表現を避ける |

### ケース B：第三者・インフルエンサーに依頼する案件を弊社が受託する

| ケース | ステマ規制 | 対応 |
|--------|----------|------|
| クライアントから「インフルエンサーに紹介してもらいたい」依頼 | **対象（要表示）** | 必ず #PR / #広告 / #提供 を明示 |
| 商品提供のみで紹介依頼なし（インフルエンサーの自発的紹介） | 対象外 | ただし「商品提供を受けた」と明示することを推奨 |
| 弊社内でアカウントを複数運用しクライアント商品を紹介 | **対象** | 各投稿に #PR を明示 |

### ケース C：クライアントの「お客様の声」「レビュー」をSNSで紹介する

| ケース | ステマ規制 | 対応 |
|--------|----------|------|
| 実在の顧客からの感想を、本人同意のうえ紹介 | 対象外（事業者表示として明示すればOK） | 「お客様からの声」と明示 |
| 架空のお客様の声を作成 | **完全に違反** | 絶対にやらない |
| 良いレビューだけ抜粋して並べる（不利な情報を意図的に隠す） | 不当表示リスク | 適切な範囲で紹介 |

## 3. 受注時チェックリスト（CSO・CMO 用）

### 受注前の必須確認

- [ ] クライアントのSNSアカウントは **クライアント自身の名義** か？
- [ ] 投稿主体は誰になるか（クライアント／弊社／第三者）？
- [ ] 第三者紹介・インフルエンサー起用の予定はあるか？
- [ ] クライアント業種は規制業種（薬機法／医療法／弁護士法）に該当するか？
- [ ] 過去の「お客様の声」投稿は実在顧客の同意を得ているか？

### 規制業種の追加チェック

| 業種 | 適用法令 | 注意点 |
|------|---------|--------|
| 飲食 | 景表法、食品表示法 | 「日本一」「最高」等の最上級表現NG、原産地・アレルゲン表示 |
| 美容院・整体・エステ | 景表法、薬機法 | 「効く」「治る」「痩せる」等の医療効果表現NG |
| 美容医療・クリニック | 医療広告ガイドライン、医療法 | 治療内容・効果・副作用・費用の併記義務 |
| 弁護士・税理士・社労士 | 各士業法、業務独占範囲 | 弁護士法72条（非弁行為）注意 |
| 健康食品 | 薬機法、健康増進法 | 効果効能の断定表現は不可 |
| 投資・金融 | 金商法 | 元本保証・確実な利益表現NG |

## 4. 投稿前チェックリスト（CMO 用）

各投稿を予約投稿する前に以下を確認する：

### 基本チェック
- [ ] 投稿内容に断定的な効果効能表現はないか？
- [ ] 数値（売上◯倍／フォロワー◯倍）に根拠データがあるか？
- [ ] 画像・写真は権利クリアか？
- [ ] ハッシュタグに業種規制違反語が含まれていないか？

### ステマ規制チェック
- [ ] **クライアント自身のアカウントから自社サービス宣伝** → 表示不要、ただし優良誤認・有利誤認に注意
- [ ] **インフルエンサー起用のキャンペーン投稿** → `#PR` `#広告` `#提供` を投稿本文の冒頭または明確な位置に表示
- [ ] **タイアップ案件** → 「○○とのタイアップ投稿」「広告」と明示
- [ ] **商品提供のみ受けて自発的に紹介する場合** → 「商品提供を受けて」と明示（推奨）

### 「お客様の声」紹介時のチェック
- [ ] 投稿元の顧客から書面（メール含む）で **転載・編集の同意** を得ているか？
- [ ] 個人名・写真を出す場合、本人同意を取っているか？
- [ ] 「お客様の声」「ご利用者様の声」など出典を明示しているか？

## 5. 違反時の対応フロー

### 弊社が違反を発見した場合
1. CMO が即時投稿削除
2. CLO にエスカレーション
3. クライアントに通知・原因究明
4. 必要に応じて訂正投稿
5. 再発防止策を契約書・運用フローに反映

### 消費者庁・第三者から指摘を受けた場合
1. 即時 CLO 経由で弁護士へ相談
2. 該当投稿を直ちに非公開化
3. 事実関係調査
4. 措置命令対応

## 6. 営業段階での対応

### 営業提案文の表現修正（CSO/CMO 連携）

`proposal_templates.md` 内の以下の表現は **景表法上のリスク** があるため修正済み／要修正：

| 修正前 | 修正後 |
|--------|--------|
| 「問い合わせが2〜3倍になるケースが多い」 | 「効果には個人差があります。過去事例として◯◯社では問い合わせが増加したケースがあります（※具体実績がある場合のみ）」 |
| 「集客力が高まる」 | 「集客力向上を目指します」（断定を避ける） |
| 「初月半額の25,000円でお試し可能」 | 「初回お試し価格 25,000円（次月以降は通常価格 50,000円）」 |

### 契約書での明示
SNS運用代行契約書テンプレに以下を盛り込み済み：
- 投稿主体の明確化（広告主＝クライアント自身）
- ステマ規制違反となる依頼は受託しない旨
- 第三者紹介案件では弊社が#PR表示を行う旨
