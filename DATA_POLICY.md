# データ・公開ポリシー

> このリポジトリは **PRIVATE** です。あらゆる成果物は外部公開を意図していません。

## 基本方針

1. **リポジトリは GitHub 上で `private` 設定であることを必須とする**
2. **README その他で「公開製品」「オープンソース」「販促」のような表現を禁止**
3. **第三者への共有・引用・転載は一切禁止**
4. **個人情報（実名・住所・口座・電話・メール）は git に乗せない**
5. **取引先・顧客の実名・連絡先は git に乗せない**
6. **戦略・価格・収益の実数値は git に乗せない（プレースホルダのみ可）**

## コミットしてよいもの

| 種別 | 可否 | 備考 |
|------|------|------|
| 役職定義・運用ルール（CLAUDE.md / company.md） | ✅ | 抽象的な役割記述 |
| テンプレート設計（構造・スキーマ） | ✅ | 雛形のみ |
| 自動化スクリプト（.mjs / .sh） | ✅ | コードのみ |
| 戦略・市場調査（一般情報ベース） | ✅ | 出典明記、固有名詞は注意 |
| プレースホルダ入りの請求書テンプレ | ✅ | `[氏名]` `[住所]` 等のみ |

## コミットしてはいけないもの

| 種別 | 理由 |
|------|------|
| オーナー本人の実名・住所・電話 | 個人情報 |
| 銀行口座番号・カード番号 | 金融情報 |
| 顧客・取引先の実名・連絡先 | 第三者個人情報 |
| 実請求書 PDF・契約書 PDF | 取引情報 |
| 実際の売上・入金額 | 経営情報 |
| API キー・トークン・パスワード | 認証情報 |
| note・X 等のアカウント情報 | アカウント情報 |
| 集客記事の「公開即用」.txt | 販売前提の資産 |

## コミット前チェックリスト

```
□ README に「公開」「オープンソース」「販促」の語を入れていないか
□ ファイル内に実名・住所・電話・メールアドレスが含まれていないか
□ 実取引額・実顧客名・実案件名が含まれていないか
□ API キー・パスワード・トークンが含まれていないか
□ センシティブ PDF はサブフォルダ（_invoices/ 等）に置いてあるか
```

## 違反時の対応

センシティブ情報を誤ってコミットしてしまった場合：

```
1. 直ちに git reset --soft HEAD~1 で取り消し
2. ファイルを正しい場所（_invoices/ 等）に移動
3. .gitignore に再現防止のパターン追加
4. commit を作り直す（push 前なら）
5. push 後に発覚した場合：
   - リポジトリのオーナーに即連絡
   - git filter-repo で履歴から消去
   - 漏洩した認証情報は即ローテーション
```

## このポリシーの維持

- このファイルは git に乗せる（公開禁止のルール自体は OK）
- 違反パターンを発見次第、`.gitignore` と本ファイルを更新する
- セキュリティに迷ったら **乗せない** を選ぶ（不可逆性の原則）
